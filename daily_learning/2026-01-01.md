## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**สมการการอนุรักษ์มวล (Continuity Equation)**

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**คำอธิบายพจน์:**
- $\rho$ (rho): **ความหนาแน่น** (density) - มวลต่อหน่วยปริมาตร [kg/m³]
- $t$: **เวลา** (time) [s]
- $\mathbf{U}$: **สนามความเร็ว** (velocity field) - เวกเตอร์ความเร็ว [m/s]
- $\nabla \cdot$: **Divergence** (การแตกตัว) - ตัวดำเนินการเวกเตอร์ที่วัดการไหลออกจากจุดหนึ่ง

สำหรับ **การไหลแบบไม่อัดตัว (incompressible flow)** ที่ $\rho = \text{constant}$:

$$
\nabla \cdot \mathbf{U} = 0
$$

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**สมการการอนุรักษ์โมเมนตัม (Momentum Equation)**

$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**คำอธิบายพจน์:**
- $p$: **ความดัน** (pressure) [Pa]
- $\boldsymbol{\tau}$ (tau): **เทนเซอร์ความเค้น** (stress tensor) - แรงเฉือนภายในของไหล
- $\mathbf{g}$: **ความเร่งเนื่องจากแรงโน้มถ่วง** (gravitational acceleration) [m/s²]
- $\nabla p$: **Gradient ของความดัน** (pressure gradient) - แรงผลักดันจากความดัน
- $\nabla \cdot \boldsymbol{\tau}$: **Divergence ของเทนเซอร์ความเค้น** (viscous term) - แรงต้านความหนืด

สำหรับ **ของไหลนิวตัน (Newtonian fluid)** แบบไม่อัดตัว:

$$
\frac{\partial \mathbf{U}}{\partial t} + (\mathbf{U} \cdot \nabla) \mathbf{U} = -\frac{1}{\rho} \nabla p + \nu \nabla^2 \mathbf{U} + \mathbf{g}
$$

โดยที่ $\nu = \mu/\rho$ คือ **ความหนืดจลน์** (kinematic viscosity) [m²/s]

---

### 1.3 Conservation of Energy

**สมการการอนุรักษ์พลังงาน (Energy Equation)**

$$
\frac{\partial (\rho e)}{\partial t} + \nabla \cdot (\rho e \mathbf{U}) = -\nabla \cdot \mathbf{q} + \boldsymbol{\tau} : \nabla \mathbf{U} + \dot{q}_v
$$

**คำอธิบายพจน์:**
- $e$: **พลังงานภายในเฉพาะ** (specific internal energy) [J/kg]
- $\mathbf{q}$: **ฟลักซ์ความร้อน** (heat flux) [W/m²]
- $\dot{q}_v$: **แหล่งกำเนิดความร้อน** (volumetric heat source) [W/m³]
- $\boldsymbol{\tau} : \nabla \mathbf{U}$: **การกระจายพลังงานเนื่องจากความเค้น** (viscous dissipation)

ในรูปแบบ **เอนทาลปี (enthalpy form)**:

$$
\frac{\partial (\rho h)}{\partial t} + \nabla \cdot (\rho h \mathbf{U}) = \frac{Dp}{Dt} + \nabla \cdot (k \nabla T) + \Phi_v
$$

โดยที่ $h$ คือ **เอนทาลปีเฉพาะ** (specific enthalpy) และ $k$ คือ **สัมประสิทธิภาพการนำความร้อน** (thermal conductivity)

---

### 1.4 Equation of State

**สมการสถานะ (Equation of State)**

สำหรับ **ก๊าซอุดมคติ (Ideal Gas)**:

$$
p = \rho R T
$$

**คำอธิบายพจน์:**
- $R$: **ค่าคงที่ก๊าซเฉพาะ** (specific gas constant) [J/(kg·K)]
- $T$: **อุณหภูมิ** (temperature) [K]

สำหรับ **ของไหลไม่อัดตัว (Incompressible Fluid)**:

$$
\rho = \text{constant}
$$

---

### 1.5 Summary of Governing Equations in Vector Form

| สมการ | รูปเวกเตอร์ทั่วไป | การไหลไม่อัดตัว |
|--------|---------------------|-------------------|
| **มวล** | $\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0$ | $\nabla \cdot \mathbf{U} = 0$ |
| **โมเมนตัม** | $\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}$ | $\frac{\partial \mathbf{U}}{\partial t} + (\mathbf{U} \cdot \nabla) \mathbf{U} = -\frac{1}{\rho}\nabla p + \nu \nabla^2 \mathbf{U} + \mathbf{g}$ |
| **พลังงาน** | $\frac{\partial (\rho e)}{\partial t} + \nabla \cdot (\rho e \mathbf{U}) = -\nabla \cdot \mathbf{q} + \boldsymbol{\tau} : \nabla \mathbf{U}$ | $\frac{\partial T}{\partial t} + \mathbf{U} \cdot \nabla T = \alpha \nabla^2 T$ |

โดยที่ $\alpha = k/(\rho c_p)$ คือ **สัมประสิทธิภาพการแพร่ความร้อน** (thermal diffusivity)

## 3. Code Walkthrough

### 3.1 UEqn.H

ไฟล์ `UEqn.H` ใช้สร้างสมการโมเมนตัมแบบไม่เชิงเส้น (momentum equation) สำหรับแก้สมการ Navier-Stokes ใน OpenFOAM

```cpp
// สร้างเทนเซอร์ความเค้นหนืด (viscous stress tensor)
tmp<fvVectorMatrix> tUEqn
(
    fvm::div(phi, U)      // พจน์เนื่องจากการพาความเร็ว (convection)
  + fvm::laplacian(nu, U) // พจน์เนื่องจากความหนืด (viscous diffusion)
  + fvc::div(phi, T)      // พจน์ความปั่นป่วน (turbulence)
);

fvVectorMatrix& UEqn = tUEqn.ref();

// แก้สมการโมเมนตัมแบบไม่เชิงเส้น
if (pimple.momentumPredictor())
{
    solve(UEqn == -fvc::grad(p) + rho*g); // เพิ่ม gradient ของความดันและแรงโน้มถ่วง
}
```

**คำอธิบาย:**
- `fvm::div(phi, U)`: พจน์การพา (convection term) - ใช้ fvm (finite volume method) เพื่อสร้างเมทริกซ์
- `fvm::laplacian(nu, U)`: พจน์การแพร่ (diffusion term) - คำนวณ Laplacian ของความเร็ว
- `fvc::grad(p)`: Gradient ของความดัน - ใช้ fvc (finite volume calculus) สำหรับคำนวณโดยตรง
- `solve()`: แก้ระบบสมการเชิงเส้น Ax = b

---

### 3.2 pEqn.H

ไฟล์ `pEqn.H` ใช้สร้างและแก้สมการความดัน (pressure equation) เพื่อบังคับให้ได้รักษาสมการต่อเนื่อง (continuity equation) สำหรับการไหลแบบไม่อัดตัว

```cpp
// สร้างสมการความดันจากสมการโมเมนตัมและการต่อเนื่อง
volScalarField rAU(1.0/UEqn.A());  // ผกผันของเมทริกซ์สัมประสิทธิ์
volVectorField HbyA(constrainHbyA(rAU*UEqn.H(), U, p));  // เทอร์ม H ที่ถ่วงด้วย A

surfaceScalarField phiHbyA
(
    "phiHbyA",
    fvc::flux(HbyA)
);

// แก้ไข flux ที่ขอบเขตเพื่อให้สอดคล้องกับเงื่อนไขขอบเขต
adjustPhi(phiHbyA, U, p);

// สร้างเมทริกซ์สมการความดัน
tmp<fvScalarMatrix> tpEqn
(
    fvm::laplacian(rAU, p) == fvc::div(phiHbyA)
);

// แก้สมการความดัน
tpEqn.ref().setReference(pRefCell, pRefValue);
tpEqn.solve();

// คำนวณ flux ใหม่จากความดันที่ได้
phi = phiHbyA - tpEqn.ref().flux();

// แก้ไขความเร็วเพื่อให้สอดคล้องกับ flux ใหม่
U = HbyA - rAU*fvc::grad(p);
U.correctBoundaryConditions();
```

**คำอธิบาย:**
- `rAU(1.0/UEqn.A())`: คำนวณ inverse ของเมทริกซ์สัมประสิทธิ์จากสมการโมเมนตัม
- `HbyA`: เทอร์มที่เกิดจากการถ่วงเวกเตอร์ H ด้วย inverse ของ A
- `fvc::flux()`: คำนวณ flux ผ่านพื้นผิวเซลล์จากเวกเตอร์ความเร็ว
- `adjustPhi()`: ปรับ flux เพื่อรักษาสมการต่อเนื่องที่ขอบเขต
- `fvm::laplacian(rAU, p)`: สร้างเมทริกซ์ Laplacian ของความดัน
- `setReference()`: กำหนดค่าอ้างอิงสำหรับความดันเพื่อแก้ปัญหา singular matrix
- `phi = phiHbyA - tpEqn.ref().flux()`: อัปเดต flux จากความดันที่คำนวณได้
- `U.correctBoundaryConditions()`: แก้ไขค่าความเร็วที่ขอบเขตให้สอดคล้อง

### 3.3 createFields.H

ไฟล์ `createFields.H` ใช้สร้างและเริ่มต้นฟิลด์ต่างๆ (fields) ที่จำเป็นสำหรับการแก้สมการใน OpenFOAM รวมถึงความเร็ว ความดัน และคุณสมบัติทางกายภาพ

```cpp
// สร้างฟิลด์ความเร็ว (velocity field)
Info<< "Reading field U\n" << endl;
volVectorField U
(
    IOobject
    (
        "U",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้างฟิลด์ความดัน (pressure field)
Info<< "Reading field p\n" << endl;
volScalarField p
(
    IOobject
    (
        "p",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้างฟิลด์ความหนืดจลน์ (kinematic viscosity)
singlePhaseTransportModel laminarTransport(U, phi);

// สร้างโมเดลความปั่นป่วน (turbulence model)
autoPtr<incompressible::turbulenceModel> turbulence
(
    incompressible::turbulenceModel::New(U, phi, laminarTransport)
);

// สร้างฟิลด์ flux ผ่านพื้นผิวเซลล์
surfaceScalarField phi
(
    IOobject
    (
        "phi",
        runTime.timeName(),
        mesh,
        IOobject::READ_IF_PRESENT,
        IOobject::AUTO_WRITE
    ),
    fvc::flux(U)
);
```

**คำอธิบาย:**
- `volVectorField U`: สร้างฟิลด์เวกเตอร์ความเร็วบนจุดศูนย์กลางเซลล์ อ่านค่าเริ่มต้นจากไฟล์ `0/U`
- `volScalarField p`: สร้างฟิลด์สเกลาร์ความดันบนจุดศูนย์กลางเซลล์ อ่านค่าเริ่มต้นจากไฟล์ `0/p`
- `singlePhaseTransportModel`: สร้างโมเดลการขนส่ง (transport model) สำหรับคำนวณความหนืด
- `incompressible::turbulenceModel::New()`: สร้างโมเดลความปั่นป่วน (เช่น k-ε, k-ω SST) ตามที่ระบุใน `constant/turbulenceProperties`
- `surfaceScalarField phi`: สร้างฟิลด์ flux ผ่านพื้นผิวเซลล์ คำนวณจากความเร็วด้วย `fvc::flux(U)`
- `IOobject`: คลาสที่กำหนดวิธีการอ่าน/เขียนฟิลด์ พารามิเตอร์คือชื่อ ไดเรกทอรี และโหมดการอ่าน/เขียน
- `MUST_READ`: บังคับให้อ่านค่าจากไฟล์ หากไม่พบจะเกิด error
- `AUTO_WRITE`: เขียนฟิลด์อัตโนมัติเมื่อถึงเวลาบันทึกผลลัพธ์

## 4. Dictionary Analysis & Configuration

### 4.1 `system/fvSchemes`

Controls spatial and temporal discretization schemes for the finite volume method.

```cpp
// Temporal discretization
ddtSchemes
{
    default         Euler;           // First-order implicit
    // backward        2;              // Second-order implicit
    // CrankNicolson   0.9;            // Blended Euler/second-order
}

// Gradient calculation
gradSchemes
{
    default         Gauss linear;    // Central differencing
    // leastSquares   1;               // Least squares reconstruction
}

// Divergence terms (convection)
divSchemes
{
    default         none;
    div(phi,U)      Gauss upwind;    // First-order upwind (stable)
    // div(phi,U)      Gauss linear;    // Central differencing (accurate but unstable)
    // div(phi,U)      Gauss limitedLinearV 1;  // TVD scheme (recommended)
    div(phi,k)      Gauss upwind;
    div(phi,epsilon) Gauss upwind;
}

// Laplacian terms (diffusion)
laplacianSchemes
{
    default         Gauss linear corrected;  // Non-orthogonal correction
    // Gauss linear uncorrected;            // For orthogonal meshes only
}

// Interpolation to cell faces
interpolationSchemes
{
    default         linear;
}

// Surface normal gradient
snGradSchemes
{
    default         corrected;       // Non-orthogonal correction
}
```

**Key Points:**
- **Upwind schemes** are stable but numerically diffusive (first-order accurate)
- **Linear schemes** are second-order accurate but can cause oscillations
- **TVD schemes** (limitedLinear, vanLeer) combine accuracy with stability
- **Corrected** schemes add terms for non-orthogonal mesh corrections

---

### 4.2 `system/fvSolution`

Controls linear solver settings and algorithm parameters.

```cpp
solvers
{
    p
    {
        solver          GAMG;          // Geometric-Algebraic Multi-Grid
        tolerance       1e-06;         // Absolute convergence tolerance
        relTol          0.01;          // Relative convergence tolerance
        smoother        GaussSeidel;
        nPreSweeps      0;
        nPostSweeps     2;
        cacheAgglomeration on;
        agglomerator    faceAreaPair;
        mergeLevels     1;
    }

    pFinal
    {
        $p;             // Inherit from p
        relTol          0;            // Tighten tolerance for final iteration
    }

    U
    {
        solver          smoothSolver;  // Iterative solver with smoother
        smoother        GaussSeidel;
        tolerance       1e-05;
        relTol          0.1;
    }

    "(k|epsilon|omega)"
    {
        solver          PBiCGStab;     // Stabilized Bi-Conjugate Gradient
        preconditioner  DILU;          // Diagonal Incomplete LU
        tolerance       1e-05;
        relTol          0.1;
    }
}

PIMPLE
{
    nCorrectors      3;               // Number of pressure correctors
    nNonOrthogonalCorrectors 0;       // For highly non-orthogonal meshes
    nAlphaCorr       1;               // Volume fraction correctors (multiphase)
    nOuterCorrectors 1;               // Outer loop iterations (transient)
    
    pRefCell         0;               // Reference cell for pressure
    pRefValue        0;               // Reference pressure value [Pa]
    
    residualControl
    {
        p               1e-4;         // Residual control for convergence
        U               1e-4;
        "(k|epsilon)"   1e-3;
    }
}

relaxationFactors
{
    fields
    {
        p               0.3;          // Under-relaxation for pressure
        rho             0.05;         // For compressible flows
    }
    equations
    {
        U               0.7;          // Under-relaxation for momentum
        "(k|epsilon)"   0.8;          // Turbulence equations
    }
}
```

**Key Points:**
- **GAMG** is fastest for pressure Poisson equations on large meshes
- **PBiCGStab** is preferred for asymmetric systems (convection-dominated)
- **Under-relaxation** prevents divergence in steady-state/iterative solvers
- **Residual control** allows early exit when convergence is achieved
- **nCorrectors** controls PISO loop iterations (typically 2-4)

bool massConservationCheck::execute()
{
    const scalar currentMass = calculateTotalMass();
    const scalar massError = mag(currentMass - initialMass_) / initialMass_;
    
    Info<< type() << " " << name() << " output:" << nl
        << "  Initial mass: " << initialMass_ << " kg" << nl
        << "  Current mass: " << currentMass << " kg" << nl
        << "  Relative error: " << massError*100 << " %" << nl;
        
    if (massError > 0.01)  // 1% threshold
    {
        WarningInFunction
            << "Mass conservation error exceeds 1%!" << endl;
    }
    
    return true;
}

scalar massConservationCheck::calculateTotalMass() const
{
    if (rhoField_.valid())
    {
        // Compressible: integrate rho over domain
        return fvc::domainIntegrate(rhoField_()).value();
    }
    else
    {
        // Incompressible: rho = 1, return volume
        return fvc::domainIntegrate(volScalarField::Internal(
            dimensionedScalar("1", dimless, 1.0)
        )).value();
    }
}

### Check 1: Discretization Scheme Selection

**Question:** For a steady-state simulation of high-Reynolds number flow (Re ≈ 10⁶) past a bluff body, which convection scheme would you choose and why?

<details>
<summary>Answer</summary>

**Recommended scheme:** `Gauss limitedLinearV 1`

**Reasoning:**
1. **High Reynolds number** → convection-dominated flow, diffusion is negligible
2. **Bluff body** → flow separation, recirculation zones, potential instabilities
3. **Steady-state** → need stable convergence

**Why not other schemes:**
- `Gauss upwind`: Too diffusive, will smear vortices and wake structures
- `Gauss linear`: Unconditionally unstable for high Re, will cause oscillations/divergence
- `Gauss limitedLinearV 1`: TVD scheme that limits gradients to prevent oscillations while maintaining second-order accuracy in smooth regions

**Configuration:**
```cpp
divSchemes
{
    div(phi,U)      Gauss limitedLinearV 1;
}
```
</details>

### Check 2: Pressure Reference Cell

**Question:** Why is `pRefCell` and `pRefValue` needed in incompressible solvers? What happens if you don't set it?

<details>
<summary>Answer</summary>

**Explanation:**

In incompressible flow, the pressure Poisson equation derived from the continuity equation is:

$$
\nabla^2 p = \frac{\rho}{\Delta t} \nabla \cdot \mathbf{U}^*
$$

This equation has a **singular matrix** because:
1. Pure Neumann boundary conditions (gradient specified everywhere)
2. Pressure appears only as gradients (∇p), so p + constant is also a solution
3. The matrix has a nullspace (eigenvector of all ones)

**What `pRefCell` does:**
- Removes the singularity by fixing pressure at one cell
- Makes the system solvable by providing a reference level
- The solver adjusts all pressures relative to this point

**If not set:**
- **GAMG solver**: May converge slowly or fail with "matrix is singular" error
- **Other solvers**: Will likely fail to converge or give arbitrary pressure levels

**Best practice:**
```cpp
PIMPLE
{
    pRefCell        0;      // Usually first cell (index 0)
    pRefValue       0;      // Reference pressure [Pa]
}
```

For incompressible flows, only pressure *differences* matter physically, so the reference value is arbitrary (often set to 0 for gauge pressure).
</details>

### Check 3: Under-Relaxation Factors

**Question:** In a steady-state RANS simulation, the residuals are oscillating and not converging. The current settings are `p: 0.3`, `U: 0.7`. What should you try?

<details>
<summary>Answer</summary>

**Diagnosis:** Oscillating residuals typically indicate instability from:
1. Too aggressive under-relaxation factors
2. Strong coupling between pressure and velocity
3. Poor initial conditions

**Solutions (in order of preference):**

**Option 1: Reduce under-relaxation**
```cpp
relaxationFactors
{
    fields
    {
        p               0.2;    // Reduce from 0.3
    }
    equations
    {
        U               0.5;    // Reduce from 0.7
    }
}
```

**Option 2: Use relaxation for final corrector only**
```cpp
relaxationFactors
{
    fields
    {
        p               0.3;
    }
    equations
    {
        U               0.7;
    }
}
PIMPLE
{
    nCorrectors      3;
    consistent       yes;    // Use consistent SIMPLE algorithm
}
```

**Option 3: Add residual control**
```cpp
PIMPLE
{
    residualControl
    {
        p               1e-4;
        U               1e-4;
    }
}
```

**Option 4: Check mesh quality**
```bash
checkMesh -allGeometry -allTopology
```
Look for:
- Non-orthogonality > 70°
- Skewness > 4
- Aspect ratio > 1000

**Rule of thumb:** Start with conservative values (p: 0.2-0.3, U: 0.5-0.6), then gradually increase as solution stabilizes.
</details>

### Check 4: PISO vs PIMPLE

**Question:** What is the difference between PISO and PIMPLE algorithms? When should you use each?

<details>
<summary>Answer</summary>

**PISO (Pressure Implicit with Splitting of Operators):**
- Designed for **transient** simulations
- Performs multiple pressure corrections per time step
- No under-relaxation needed (time step provides stability)
- Algorithm:
  1. Solve momentum equation
  2. Solve pressure equation
  3. Correct velocity and pressure
  4. Repeat steps 2-3 for `nCorrectors` iterations
  5. Advance to next time step

**PIMPLE (PISO + SIMPLE):**
- Merged algorithm combining PISO and SIMPLE
- Can handle both **steady-state** and **transient** simulations
- Uses under-relaxation for steady-state cases
- Adds outer corrector loop (`nOuterCorrectors`)
- Algorithm:
  1. Solve momentum equation
  2. Solve pressure equation
  3. Correct velocity and pressure
  4. Repeat steps 2-3 for `nCorrectors` iterations
  5. Repeat steps 1-4 for `nOuterCorrectors` iterations (if > 1)
  6. Advance to next time step (if transient)

**When to use:**

| Scenario | Algorithm | Settings |
|----------|-----------|----------|
| Transient, small time steps | PISO | `nOuterCorrectors 1` |
| Transient, large time steps | PIMPLE | `nOuterCorrectors 2-4` |
| Steady-state | PIMPLE | `nOuterCorrectors > 1`, add under-relaxation |
| Pseudo-transient (steady) | PIMPLE | `nOuterCorrectors 1`, localEuler ddtScheme |

**Configuration examples:**

```cpp
// Transient PISO
PIMPLE
{
    nCorrectors          2;
    nNonOrthogonalCorrectors 0;
    nOuterCorrectors     1;  // Pure PISO
}

// Steady-state SIMPLE
PIMPLE
{
    nCorrectors          2;
    nOuterCorrectors     10;  // Multiple outer iterations
    residualControl      // Convergence criteria
    {
        p               1e-4;
        U               1e-4;
    }
}
```
</details>

### Check 5: fvMatrix Assembly

**Question:** In the momentum equation `fvm::div(phi, U)`, what is the difference between `fvm::div()` and `fvc::div()`? When would you use each?

<details>
<summary>Answer</summary>

**`fvm` (finite volume method):**
- Creates **implicit** matrix coefficients
- Contributes to the matrix **A** and source **b** in Ax = b
- Used when the variable is being solved for
- More stable but requires matrix assembly

**`fvc` (finite volume calculus):**
- Performs **explicit** calculation
- Returns a field (volVectorField, volScalarField, etc.)
- Used when the variable is known from previous iteration
- Faster but less stable (CFL-limited)

**Examples:**

```cpp
// Implicit: U is unknown, add to matrix
tmp<fvVectorMatrix> UEqn
(
    fvm::div(phi, U)           // Implicit convection
  + fvm::laplacian(nu, U)      // Implicit diffusion
);

// Explicit: T is known from previous iteration
tmp<fvVectorMatrix> UEqn
(
    fvm::div(phi, U)
  + fvc::div(phi, T)           // Explicit turbulence term
);

// Explicit: calculate gradient for source term
volVectorField gradP = fvc::grad(p);  // Explicit pressure gradient
solve(UEqn == -gradP);                 // Add to RHS
```

**When to use:**

| Term | Type | Reason |
|------|------|--------|
| Convection of U | `fvm::div(phi, U)` | U is unknown, implicit is stable |
| Diffusion of U | `fvm::laplacian(nu, U)` | U is unknown, implicit is stable |
| Pressure gradient | `fvc::grad(p)` | p is from previous iteration |
| Turbulence source | `fvc::div(phi, T)` | T is frozen in momentum eq |
| Time derivative | `fvm::ddt(U)` | U is unknown |

**Performance trade-off:**
- Implicit: Larger matrix, slower assembly, more iterations per solve
- Explicit: Smaller matrix, faster assembly, may need smaller time steps

**Best practice:** Use implicit (`fvm`) for terms involving the unknown variable, explicit (`fvc`) for known source terms.
</details>
