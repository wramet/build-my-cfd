## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**สมการการอนุรักษ์มวล (Conservation of Mass)**

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**คำอธิบายพจน์:**
- $\rho$ (rho) = **ความหนาแน่น (density)** - มวลต่อหน่วยปริมาตร [kg/m³]
- $\mathbf{U}$ = **สนามความเร็ว (velocity field)** - เวกเตอร์ความเร็ว [m/s]
- $\nabla \cdot$ = **ไดเวอร์เจนซ์ (divergence)** - ตัวดำเนินการเชิงอนุพันธ์
- $t$ = **เวลา (time)** [s]

สำหรับการไหลแบบ **ไม่อัดตัว (incompressible flow)**: $\nabla \cdot \mathbf{U} = 0$

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**สมการการอนุรักษ์โมเมนตัม (Conservation of Momentum)**

$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**คำอธิบายพจน์:**
- $p$ = **ความดัน (pressure)** [Pa]
- $\boldsymbol{\tau}$ (tau) = **เทนเซอร์ความเค้น (stress tensor)** - แรงเฉือน/ความหนืด
- $\mathbf{g}$ = **ความเร่งเนื่องจากแรงโน้มถ่วง (gravitational acceleration)** [m/s²]

**สำหรับของไหลนิวตัน (Newtonian Fluid):**

$$
\boldsymbol{\tau} = \mu \left[ \nabla \mathbf{U} + (\nabla \mathbf{U})^T \right] - \frac{2}{3}\mu (\nabla \cdot \mathbf{U})\mathbf{I}
$$

- $\mu$ (mu) = **ความหนืดพลศาสตร์ (dynamic viscosity)** [Pa·s]
- $\mathbf{I}$ = **เทนเซอร์เอกลักษณ์ (identity tensor)**

---

### 1.3 Conservation of Energy

**สมการการอนุรักษ์พลังงาน (Conservation of Energy)**

$$
\frac{\partial (\rho e)}{\partial t} + \nabla \cdot (\rho e \mathbf{U}) = -\nabla \cdot \mathbf{q} + \boldsymbol{\tau} : \nabla \mathbf{U} + S_h
$$

**คำอธิบายพจน์:**
- $e$ = **พลังงานภายในต่อหน่วยมวล (internal energy per mass)** [J/kg]
- $\mathbf{q}$ = **ฟลักซ์ความร้อน (heat flux)** [W/m²]
- $S_h$ = **แหล่งกำเนิดความร้อน (heat source)** [W/m³]

**กฎของฟูริเยร์ (Fourier's Law):**
$$
\mathbf{q} = -k \nabla T
$$
- $k$ = **ความนำความร้อน (thermal conductivity)** [W/(m·K)]
- $T$ = **อุณหภูมิ (temperature)** [K]

---

### 1.4 Equation of State

**สมการสถานะ (Equation of State)**

สำหรับ **แก๊สอุดมคติ (Ideal Gas)**:
$$
p = \rho R T
$$
- $R$ = **ค่าคงที่ของแก๊สเฉพาะ (specific gas constant)** [J/(kg·K)]

สำหรับ **ของไหลไม่อัดตัว (Incompressible Fluid)**:
$$
\rho = \text{constant}
$$

---

### 1.5 Transport Equations (General Form)

**สมการเชิงอนุพันธ์ย่อยทั่วไป (General Transport Equation)**

$$
\frac{\partial (\rho \phi)}{\partial t} + \nabla \cdot (\rho \phi \mathbf{U}) = \nabla \cdot (\Gamma_\phi \nabla \phi) + S_\phi
$$

**คำอธิบายพจน์:**
- $\phi$ (phi) = **ตัวแปรขนส่ง (transported variable)** - เช่น อุณหภูมิ, ความเข้มของสาร
- $\Gamma_\phi$ (Gamma) = **สัมประสิทธิ์การแพร่ (diffusion coefficient)**
- $S_\phi$ = **เทอมแหล่งกำเนิด/ดูดซับ (source/sink term)**

**องค์ประกอบของสมการ:**
1. **เทอมไม่คงที่ (Unsteady term)**: $\frac{\partial (\rho \phi)}{\partial t}$
2. **เทอมการพาแบบเบ็ดเสร็จ (Convection term)**: $\nabla \cdot (\rho \phi \mathbf{U})$
3. **เทอมการแพร่ (Diffusion term)**: $\nabla \cdot (\Gamma_\phi \nabla \phi)$
4. **เทอมแหล่งกำเนิด (Source term)**: $S_\phi$

## 3. Code Walkthrough

### 3.1 simpleFoam Solver - Steady-State Incompressible Flow

**ไฟล์หลัก:** `$FOAM_SOLVERS/incompressible/simpleFoam/simpleFoam.C`

**สมการที่แก้:**
- $\nabla \cdot \mathbf{U} = 0$ (Continuity)
- $\nabla \cdot (\mathbf{U} \mathbf{U}) - \nabla \cdot (\nu \nabla \mathbf{U}) = -\nabla p$ (Momentum)

---

#### Code Structure:

```cpp
#include "fvCFD.H"  // รวมไลบรารีพื้นฐาน Finite Volume
#include "singlePhaseTransportModel.H"  // โมเดลความหนืด
#include "turbulentTransportModel.H"    // โมเดลความปั่นป่วน

int main(int argc, char *argv[])
{
    // 1. สร้าง mesh และอ่านข้อมูลเริ่มต้น
    #include "setRootCaseLists.H"
    #include "createTime.H"
    #include "createMesh.H"
    
    // 2. สร้างฟิลด์ตัวแปร (p, U)
    volScalarField p(pPtr());  // ความดัน
    volVectorField U(URef());  // ความเร็ว
    
    // 3. สร้าง surface field สำหรับ flux
    surfaceScalarField phi
    (
        IOobject("phi", runTime.timeName(), mesh, ...),
        fvc::flux(U)  // φ = U·S_f
    );
    
    // 4. ตั้งค่าโมเดลความหนืดและความปั่นป่วน
    singlePhaseTransportModel laminarTransport(U, phi);
    autoPtr<incompressible::turbulenceModel> turbulence
    (
        incompressible::turbulenceModel::New(U, phi, laminarTransport)
    );
    
    // 5. Simple Loop
    while (simple.loop())
    {
        // 5.1 แก้สมการโมเมนตัม
        tmp<fvVectorMatrix> tUEqn
        (
            fvm::div(phi, U)           // เทอมการพา: ∇·(UU)
          + fvm::laplacian(nu, U)      // เทอมการแพร่: ∇·(ν∇U)
          + fvc::div(-phi, U)          // Explicit convection
        );
        
        tUEqn().relax();  // Under-relaxation
        
        // 5.2 แก้สมการความดัน (Poisson equation)
        // ∇·(∇p) = ∇·(U*)/Δt
        fvScalarMatrix pEqn
        (
            fvm::laplacian(p) == fvc::div(tUEqn())
        );
        
        pEqqn.solve();
        
        // 5.3 แก้ไข flux และความเร็ว
        phi -= pEqqn.flux();
        U.correctBoundaryConditions();
    }
    
    return 0;
}
```

---

#### คำอธิบายสำคัญ:

| ฟังก์ชัน | ความหมาย |
|-----------|----------|
| `fvm::div(phi, U)` | **Implicit convection** - ใช้เมทริกซ์ A สำหรับการแก้สมการ |
| `fvc::flux(U)` | **Calculate flux** - คำนวณ φ = U·S_f แบบ explicit |
| `fvm::laplacian(nu, U)` | **Diffusion term** - ∇·(ν∇U) |
| `tUEqn().relax()` | **Under-relaxation** - ป้องกันการหลุดของการวนซ้ำ |
| `pEqqn.solve()` | **Solve pressure** - แก้สมการ Poisson สำหรับ p |

---

#### ขั้นตอน SIMPLE Algorithm:

1. **Guess pressure field** ($p^*$)
2. **Solve momentum equation** → ได้ $\mathbf{U}^*$
3. **Solve pressure correction** ($\nabla^2 p' = \frac{\rho}{\Delta t} \nabla \cdot \mathbf{U}^*$)
4. **Correct velocity** → $\mathbf{U}^{n+1} = \mathbf{U}^* - \frac{\Delta t}{\rho} \nabla p'$
5. **Repeat** จนกว่าจะลู่เข้า (converge)

### 4.1 fvSchemes - Discretization Schemes

**ไฟล์:** `system/fvSchemes`

```cpp
ddtSchemes
{
    default         Euler;              // First-order implicit
    // backward     Second-order implicit (more accurate)
}

gradSchemes
{
    default         Gauss linear;       // Central differencing
    // Gauss upwind  First-order upwind (stable, diffusive)
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss upwind;       // Convection: upwind for stability
    div(phi,k)      Gauss upwind;       // Turbulence: upwind
    div(phi,epsilon) Gauss upwind;
    div((nuEff*dev2(T(grad(U))))) Gauss linear;  // Stress tensor: central
}

laplacianSchemes
{
    default         Gauss linear corrected;  // Diffusion: central with non-orthogonal correction
}

interpolationSchemes
{
    default         linear;
}

snGradSchemes
{
    default         corrected;          // Surface normal gradient with correction
}
```

**คำอธิบาย:**
- **Gauss linear**: การประมาณค่าเชิงเส้น (linear interpolation) ระหว่างเซลล์ - แม่นยำแต่อาจไม่เสถียร
- **Gauss upwind**: ใช้ค่าจากจุดข้างบนตามกระแส (upwind) - เสถียรแต่เพิ่มความหนืดเชิงตัวเลข (numerical diffusion)
- **corrected**: แก้ไขเอฟเฟกต์จาก mesh ที่ไม่ตั้งฉาก (non-orthogonal)

### 4.2 fvSolution - Linear Solvers & Algorithms

**ไฟล์:** `system/fvSolution`

```cpp
solvers
{
    p
    {
        solver          GAMG;              // Geometric-Algebraic Multi-Grid
        tolerance       1e-06;             // Convergence tolerance
        relTol          0.1;               // Relative tolerance (per iteration)
        smoother        GaussSeidel;       // Smoother for coarse levels
    }

    pFinal
    {
        $p;                                // Inherit from p
        relTol          0;                 // Final solve: strict tolerance
    }

    U
    {
        solver          smoothSolver;      // Iterative solver
        smoother        GaussSeidel;
        tolerance       1e-05;
        relTol          0.1;
    }

    "(k|epsilon|omega)"
    {
        solver          PBiCGStab;         // Preconditioned Bi-Conjugate Gradient Stabilized
        preconditioner  DILU;              // Diagonal Incomplete LU
        tolerance       1e-05;
        relTol          0.1;
    }
}

SIMPLE
{
    nNonOrthogonalCorrectors 0;            // Iterations for non-orthogonal mesh
    consistent      yes;                   // Use consistent flux approach

    residualControl
    {
        p               1e-4;              // Convergence criteria
        U               1e-4;
        "(k|epsilon)"   1e-4;
    }

    relaxFactors
    {
        fields
        {
            p               0.3;           // Pressure relaxation
            rho             1;             // Density (compressible)
        }
        equations
        {
            U               0.7;           // Momentum relaxation
            "(k|epsilon)"   0.8;           // Turbulence relaxation
        }
    }
}
```

**คำอธิบาย:**
- **GAMG**: เหมาะสำหรับสมการ Poisson (pressure) - ลดขนาดเมทริกซ์อย่างรวดเร็ว
- **PBiCGStab**: เหมาะสำหรับสมการที่ไม่สมมาตร (convection-dominated)
- **relaxation factors**: ป้องกันการ oscillate ของการวนซ้ำ (0 < factor < 1)

### Task 2: Implement a Custom Source Term

**เป้าหมาย:** เพิ่ม source term สำหรับสมการโมเมนตัมที่แทนด้วย:
$$
S_U = -\alpha \mathbf{U} \quad \text{(Damping force)}
$$

**วิธีการ:**

1. สร้าง `fvOptions` file:
```cpp
// constant/fvOptions

momentumDamping
{
    type            scalarExplicitSource;  // หรือ codedSource
    active          on;

    scalarExplicitSourceCoeffs
    {
        selectionMode   all;               // Apply to all cells

        volumeMode      absolute;          // หรือ specific

        // Source values per field
        U               (-10 -10 -10);     // Damping coefficient [1/s]
    }
}
```

2. หรือใช้ `codedSource` สำหรับ source term ที่ซับซ้อน:
```cpp
myMomentumSource
{
    type            coded;
    active          on;

    codedSourceCoeffs
    {
        selectionMode   all;
        fieldNames      (U);
        code
        #{
            const vectorField& C = mesh_.C();
            scalarField& source = eqn.source();

            // Apply damping based on position
            forAll(C, i)
            {
                if (C[i].x() > 0.5)
                {
                    source[i] -= 10.0 * U[i][0];  // Damp Ux
                }
```

### Task 3: Post-Processing with Python

**เป้าหมาย:** สร้าง script สำหรับ plot กราฟ residual จาก log file

**ไฟล์:** `plot_residuals.py`

```python
import re
import matplotlib.pyplot as plt

def parse_residuals(log_file):
    """Extract residuals from OpenFOAM log file"""
    
    time_steps = []
    residuals = {'U': [], 'p': [], 'k': [], 'epsilon': []}
    
    # Pattern: "time: Ux = value, Uy = value, ..."
    # Or: "Solving for U, Initial residual = 0.001, Final residual = 1e-5"
    
    pattern = r"Solving for (\w+), Initial residual = ([\d.e-]+)"
    
    with open(log_file, 'r') as f:
        for line in f:
            match = re.search(pattern, line)
            if match:
                var = match.group(1)
                residual = float(match.group(2))
                
                if var in residuals:
                    residuals[var].append(residual)
    
    return residuals

def plot_residuals(residuals):
    """Plot convergence history"""
    
    plt.figure(figsize=(10, 6))
    
    for var, data in residuals.items():
        if data:
            plt.semilogy(data, label=var, marker='o', markersize=2)
    
    plt.xlabel('Iteration')
    plt.ylabel('Initial Residual')
    plt.title('OpenFOAM Solver Convergence')
    plt.legend()
    plt.grid(True, which='both', alpha=0.3)
    plt.savefig('residuals.png', dpi=150)
    plt.show()

# Usage
if __name__ == "__main__":
    residuals = parse_residuals('log.simpleFoam')
    plot_residuals(residuals)
```

### Check 1: Finite Volume Discretization

**คำถาม:** จงอธิบายความแตกต่างระหว่าง `fvm::div(phi, U)` และ `fvc::div(phi, U)` และเมื่อไหร่ควรใช้แบบไหน

<details>
<summary>📖 คำตอบ</summary>

**คำตอบ:**

| ตัวดำเนินการ | ประเภท | การใช้งาน |
|------------|--------|----------|
| `fvm::div(phi, U)` | **Implicit** (Finite Volume Matrix) | สร้างเมทริกซ์ A ในสมการ Aψ = b<br>ใช้เมื่อต้องการ implicit discretization<br>เสถียรกว่าแต่ต้องแก้ระบบสมการเชิงเส้น |
| `fvc::div(phi, U)` | **Explicit** (Finite Volume Calculus) | คำนวณค่าและคืนผลลัพธ์เป็น field<br>ใช้เมื่อค่าเป็นที่รู้จักจาก time step ก่อนหน้า<br>เร็วแต่มีข้อจำกัดเรื่องเสถียรภาพ (CFL condition) |

**ตัวอย่างการใช้งาน:**
```cpp
// Implicit convection (สำหรับ steady-state หรือ implicit transient)
fvVectorMatrix UEqn
(
    fvm::ddt(U) + fvm::div(phi, U) - fvm::laplacian(nu, U)
);

// Explicit convection (สำหรับ transient ที่ต้องการความแม่นยำ)
tmp<volVectorField> tUconv = fvc::div(phi, U);
fvVectorMatrix UEqn
(
    fvm::ddt(U) + tUconv - fvm::laplacian(nu, U)
);
```

**กฎพื้นฐาน:**
- ใช้ `fvm` สำหรับเทอมที่มีความสำคัญต่อเสถียรภาพ (convection, diffusion)
- ใช้ `fvc` สำหรับ source terms หรือเมื่อต้องการค่าเพื่อ post-processing
</details>

### Check 2: Pressure-Velocity Coupling

**คำถาม:** จงอธิบายขั้นตอนของ SIMPLE algorithm และบทบาทของ pressure correction equation

<details>
<summary>📖 คำตอบ</summary>

**คำตอบ:**

**SIMPLE (Semi-Implicit Method for Pressure-Linked Equations)** Algorithm:

1. **Guess pressure field** ($p^*$) - เริ่มต้นด้วยค่าความดันเดามา

2. **Solve momentum equation** → ได้ velocity field ($\mathbf{U}^*$):
   $$
   \nabla \cdot (\mathbf{U}^* \mathbf{U}^*) - \nabla \cdot (\nu \nabla \mathbf{U}^*) = -\nabla p^*
   $$

3. **Solve pressure correction equation** (Poisson equation):
   $$
   \nabla^2 p' = \frac{\rho}{\Delta t} \nabla \cdot \mathbf{U}^*
   $$
   โดยที่ $p'$ คือ pressure correction และ $p^{new} = p^* + p'$

4. **Correct velocity and pressure**:
   $$
   \mathbf{U}^{new} = \mathbf{U}^* - \frac{\Delta t}{\rho} \nabla p'
   $$
   $$
   p^{new} = p^* + \alpha_U p' \quad (\alpha_U = \text{under-relaxation factor})
   $$

5. **Repeat** จนกว่าจะลู่เข้า (residuals < tolerance)

**Pressure Correction Equation คืออะไร?**
- เป็นสมการ Poisson ที่มาจากการบังคับให้ velocity field สอดคล้องกับ continuity equation ($\nabla \cdot \mathbf{U} = 0$)
- ถ้า divergence ของ $\mathbf{U}^*$ ไม่เป็นศูนย์ แปลว่า mass ไม่ถูกอนุรักษ์ จึงต้องแก้ไข pressure เพื่อให้ velocity ปรับตัว

**ใน OpenFOAM:**
```cpp
// Momentum equation (with guessed pressure)
tmp<fvVectorMatrix> tUEqn
(
    fvm::div(phi, U) - fvm::laplacian(nu, U)
 == -fvc::grad(p)
);

// Pressure correction
fvScalarMatrix pEqn
(
    fvm::laplacian(p) == fvc::div(tUEqn())  // ∇²p = ∇·U*
);

pEqn.solve();

// Correct flux
phi -= pEqn.flux();  // φ = φ - ∇p
```
</details>

### Check 3: Boundary Conditions

**คำถาม:** จงเปรียบเทียบ `fixedValue`, `zeroGradient`, และ `mixed` boundary conditions และให้ตัวอย่างการใช้งาน

<details>
<summary>📖 คำตอบ</summary>

**คำตอบ:**

| Boundary Condition | สมการ | ความหมาย | ตัวอย่างการใช้งาน |
|-------------------|--------|----------|------------------|
| **fixedValue** | $\phi = \phi_0$ | กำหนดค่าคงที่ที่ boundary (Dirichlet) | - Velocity inlet: $\mathbf{U} = (10, 0, 0)$ m/s<br>- Temperature wall: $T = 300$ K<br>- Pressure outlet: $p = 0$ Pa (gauge) |
| **zeroGradient** | $\nabla \phi \cdot \mathbf{n} = 0$ | ค่าที่ boundary เท่ากับค่าใน cell ข้างๆ (Neumann = 0) | - Pressure at wall: $\nabla p \cdot \mathbf{n} = 0$<br>- Velocity at symmetry plane<br>- Temperature at adiabatic wall |
| **mixed** (Robin) | $\alpha \phi + \beta (\nabla \phi \cdot \mathbf{n}) = \gamma$ | ผสมระหว่าง Dirichlet และ Neumann | - Convection wall: $-k \nabla T \cdot \mathbf{n} = h(T - T_\infty)$<br>- Outlet ที่มี backflow |

**ตัวอย่างใน OpenFOAM:**

```cpp
// 0/U
inlet
{
    type            fixedValue;
    value           uniform (10 0 0);  // มิติ: [m/s]
}

walls
{
    type            noSlip;             // เทียบเท่า fixedValue (0 0 0)
}

outlet
{
    type            zeroGradient;       // ∇U·n = 0
}

symmetry
{
    typesymmetry;
    // อัตโนมัติ: U_normal = 0, ∇U_tangent·n = 0
}
```

```cpp
// 0/T
walls
{
    type            externalWallHeatFlux;
    mode            coefficient;        // Mixed BC
    
    h               uniform 10;         // Heat transfer coeff [W/m²K]
    Ta              uniform 300;        // Ambient temp [K]
    // สมการ: -k∇T·n = h(T - Ta)
}
```

**คำอธิบายเพิ่มเติม:**
- **fixedValue**: ใช้เมื่อรู้ค่าที่ boundary แน่นอน (เช่น inlet conditions)
- **zeroGradient**: ใช้เมื่อไม่มีการแลกเปลี่ยนที่ boundary (เช่น symmetry, fully developed flow)
- **mixed**: ใช้เมื่อมีการแลกเปลี่ยนอย่างสมดุล (เช่น convection, radiation)

**สูตรคำนวณค่าที่ boundary:**
```cpp
// fixedValue: บังคับค่า
patchField[i] = fixedValue_;

// zeroGradient: ค่าเท่ากับ cell ข้างๆ
patchField[i] = internalField[i];

// mixed: ผสมกัน
patchField[i] = (gamma_ - beta_ * gradPhi) / alpha_;
```
</details>

### Check 4: Under-Relaxation

**คำถาม:** จงอธิบายแนวคิดของ under-relaxation และผลกระทบต่อความเสถียรของการวนซ้ำ

<details>
<summary>📖 คำตอบ</summary>

**คำตอบ:**

**Under-Relaxation คืออะไร?**
- เทคนิคการลดการเปลี่ยนแปลงของค่าตัวแปรระหว่าง iteration เพื่อป้องกันการหลุด (divergence)
- สมการการอัปเดต:
  $$
  \phi^{n+1} = \phi^n + \alpha_\phi (\phi^* - \phi^n)
  $$
  โดยที่:
  - $\phi^n$ = ค่าเดิม
  - $\phi^*$ = ค่าใหม่จากการแก้สมการ
  - $\alpha_\phi$ = under-relaxation factor ($0 < \alpha_\phi \leq 1$)

**ค่าที่นิยมใช้:**

| ตัวแปร | Under-Relaxation Factor | คำอธิบาย |
|---------|------------------------|----------|
| Pressure ($p$) | 0.2 - 0.3 | ต่ำมากเพราะ pressure มีผลต่อ velocity โดยตรง |
| Velocity ($\mathbf{U}$) | 0.5 - 0.7 | ปานกลาง เพื่อให้ลู่เข้าเร็วพอ |
| Turbulence ($k$, $\epsilon$, $\omega$) | 0.7 - 0.9 | สูง เพราะไม่มีผลรุนแรงต่อ convergence |
| Density ($\rho$) | 1.0 (ไม่ใช้) | สำหรับ incompressible flow |

**ใน fvSolution:**
```cpp
relaxFactors
{
    fields
    {
        p               0.3;    // Pressure: ลดการเปลี่ยนแปลง 70%
    }
    equations
    {
        U               0.7;    // Momentum: ลดการเปลี่ยนแปลง 30%
        k               0.8;    // Turbulence: ลดการเปลี่ยนแปลง 20%
    }
}
```

**ผลกระทบ:**
- **$\alpha$ ต่ำเกินไป**: ลู่เข้าช้ามาก (slow convergence) แต่เสถียร
- **$\alpha$ สูงเกินไป**: ลู่เข้าเร็วแต่อาจ diverge ได้
- **ค่าที่เหมาะสม**: ต้องทดลอง (trial and error) ขึ้นกับปัญหา

**ตัวอย่าง:**
```cpp
// ถ้าไม่ใช้ under-relaxation (α = 1.0)
p_new = 1000 Pa  // อาจทำให้ velocity เปลี่ยนมากเกินไป

// ถ้าใช้ under-relaxation (α = 0.3)
p_new = 100 + 0.3 * (1000 - 100) = 370 Pa  // ปลอดภัยกว่า
```

**เทคนิคการปรับ:**
1. เริ่มต้นด้วย $\alpha$ ต่ำๆ (เช่น 0.2)
2. ดูว่า iteration ลู่เข้าหรือไม่
3. ถ้าเสถียร ให้เพิ่ม $\alpha$ ขึ้นเรื่อยๆ จนถึงจุดที่เร็วที่สุดที่ยังเสถียร
</details>

### Check 5: Mesh Quality & Non-Orthogonality

**คำถาม:** จงอธิบายผลของ non-orthogonal mesh ต่อความแม่นยำและวิธีแก้ไข

<details>
<summary>📖 คำตอบ</summary>

**คำตอบ:**

**Non-Orthogonal Mesh คืออะไร?**
- Mesh ที่เส้นเชื่อมระหว่าง cell centers ไม่ตั้งฉากกับ face
- เกิดบ่อยใน complex geometries (เช่น ท่งเหมือน, airfoils)

**ผลกระทบ:**
- **ความแม่นยำลดลง**: การประมาณค่า gradient มี error เพิ่มขึ้น
- **การแก้สมการซับซ้อน**: ต้องมีการแก้ไข (correction) หลายครั้ง

**วิธีวัด:**
```bash
checkMesh -allGeometry -allTopology
```

ผลลัพธ์:
```
Mesh non-orthogonality Max: 70.5 average: 15.2
*Non-orthogonality > 70 degrees*: 12 cells
```

**การแก้ไขใน fvSchemes:**
```cpp
laplacianSchemes
{
    default         Gauss linear corrected;  // ใช้ corrected
    // หรือ
    default         Gauss linear limited 0.5;  // จำกัด correction
}

snGradSchemes
{
    default         corrected;  // Surface normal gradient ก็ต้อง corrected
}
```

**การแก้ไขใน fvSolution:**
```cpp
SIMPLE
{
    nNonOrthogonalCorrectors 2;  // แก้สมการ 2 ครั้งสำหรับ non-orthogonal mesh
    // ค่าปกติ: 0 (orthogonal mesh)
    // ค่าแนะนำ: 1-3 (mildly non-orthogonal)
    // ค่าสูง: 4+ (highly non-orthogonal)
}
```

**ขั้นตอนการแก้สมการ:**
```cpp
// รอบที่ 1: แก้สมการโดยไม่มี correction
solve(fvm::laplacian(nu, U));

// รอบที่ 2: แก้สมการพร้อม correction
for (int i=0; i<nNonOrthogonalCorrectors; i++)
{
    solve(fvm::laplacian(nu, U));
}
```

**เคล็ดลับ:**
- ถ้า non-orthogonality > 70 degrees: ควรสร้าง mesh ใหม่
- ถ้า 50-70 degrees: ใช้ `nNonOrthogonalCorrectors = 2-3`
- ถ้า < 50 degrees: ใช้ `nNonOrthogonalCorrectors = 0-1`

**ตัวอย่างการสร้าง mesh:**
```cpp

(

    {
        type            patch;
        faces           ((0 1 2 3));
    }
    // ... กำหนด blocks ให้ orthogonal มากที่สุด
);
```

**สรุป:**
- **Orthogonal mesh**: แม่นยำที่สุด, เร็วที่สุด
- **Non-orthogonal mesh**: ต้อง trade-off ระหว่างความง่ายในการสร้าง mesh กับความแม่นยำ
- **การแก้ไข**: ใช้ `corrected` schemes และ `nNonOrthogonalCorrectors`
