## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**Vector Form:**
$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**Term-by-Term Explanation (Thai):**
- $\frac{\partial \rho}{\partial t}$: อัตราการเปลี่ยนแปลงของความหนาแน่นของไหลเทียบกับเวลา (Unsteady term - เอกภาพเวลา)
- $\nabla \cdot (\rho \mathbf{U})$: การไหลเข้า-ออกของมวล (Mass flux divergence - การแพร่กระจายของการไหลของมวล)
- $= 0$: กฎการอนุรักษ์มวล (Mass conservation - มวลไม่ถูกสร้างหรือทำลาย)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์มวล (Conservation of Mass)** ซึ่งระบุว่ามวลของไหลไม่สามารถถูกสร้างขึ้นหรือสูญหายไปได้ ในปริมาตรควบคุมใดๆ อัตราการเปลี่ยนแปลงของมวลต้องสมดุลกับการไหลเข้าและการไหลออก

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**Vector Form:**
$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**Term-by-Term Explanation (Thai):**
- $\frac{\partial (\rho \mathbf{U})}{\partial t}$: อัตราการเปลี่ยนแปลงของโมเมนตัมเทียบกับเวลา (Unsteady term - เอกภาพเวลาของโมเมนตัม)
- $\nabla \cdot (\rho \mathbf{U} \mathbf{U})$: การพาแบบ Convective ของโมเมนตัม (Convective term - การพาโมเมนตัม)
- $-\nabla p$: แรงดันความดัน (Pressure gradient force - แรงที่เกิดจากความแตกต่างของความดัน)
- $\nabla \cdot \boldsymbol{\tau}$: แรงเฉือน (Viscous stress term - แรงเฉือนจากความหนืด)
- $\rho \mathbf{g}$: แรงโน้มถ่วง (Body force - แรงภายนอก เช่น แรงโน้มถ่วง)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์โมเมนตัม (Conservation of Momentum)** ซึ่งเป็นการประยุกต์ใช้กฎข้อที่สองของนิวตันสำหรับของไหล โดยระบุว่าการเปลี่ยนแปลงของโมเมนตัมในปริมาตรควบคุมเท่ากับผลรวมของแรงทั้งหมดที่กระทำต่อของไหลนั้น ได้แก่ แรงดัน แรงเฉือน และแรงภายนอก

---

### 1.3 Constitutive Relation (Newtonian Fluid)

**Stress Tensor for Newtonian Fluid:**
$$
\boldsymbol{\tau} = \mu \left[ \nabla \mathbf{U} + (\nabla \mathbf{U})^T \right] - \frac{2}{3} \mu (\nabla \cdot \mathbf{U}) \mathbf{I}
$$

**Term-by-Term Explanation (Thai):**
- $\mu$: ความหนืดของไหล (Dynamic viscosity - ความหนืดพลวัต)
- $\nabla \mathbf{U}$: เทนเซอร์อัตราการเฉือน (Velocity gradient tensor - การไล่ระดับความเร็ว)
- $(\nabla \mathbf{U})^T$: เทนเซอร์ทรานสโพส (Transpose of velocity gradient)
- $\mathbf{I}$: เทนเซอร์เอกลักษณ์ (Identity tensor)

**Physical Meaning:**
สมการนี้เชื่อมโยงความสัมพันธ์ระหว่าง **ความเครียด (Stress)** และ **อัตราการเฉือน (Strain rate)** สำหรับของไหลแบบนิวตัน (Newtonian Fluid) ซึ่งแรงเฉือนแปรผันโดยตรงกับอัตราการเฉือน

---

### 1.4 Energy Equation (Optional for Incompressible Flow)

**Vector Form:**
$$
\frac{\partial (\rho h)}{\partial t} + \nabla \cdot (\rho \mathbf{U} h) = \frac{Dp}{Dt} + \nabla \cdot (k \nabla T) + \boldsymbol{\tau} : \nabla \mathbf{U} + S_h
$$

**Term-by-Term Explanation (Thai):**
- $h$: เอนทาลปี (Enthalpy - พลังงานความร้อน)
- $k$: ความนำความร้อน (Thermal conductivity - สัมประสิทธิ์การนำความร้อน)
- $T$: อุณหภูมิ (Temperature)
- $\boldsymbol{\tau} : \nabla \mathbf{U}$: การกระจายของพลังงานเนื่องจากความหนืด (Viscous dissipation)
- $S_h$: แหล่งกำเนิดความร้อน (Heat source)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์พลังงาน (Conservation of Energy)** ซึ่งระบุว่าการเปลี่ยนแปลงของพลังงานในระบบเท่ากับผลรวมของการถ่ายเทความร้อน งานที่เกิดจากแรงดัน และแหล่งกำเนิดพลังงานอื่นๆ

## 2. OpenFOAM Class Hierarchy & Implementation

### 2.1 Core Field Classes

OpenFOAM uses template-based field classes to represent physical quantities (scalar, vector, tensor) over the computational mesh.

```
GeometricField<Type, PatchField, GeoMesh>
│
├── fvFieldMatrix (used in finite volume solvers)
│   └── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
│
├── DimensionedField<Type, GeoMesh>
│   └── $FOAM_SRC/OpenFOAM/fields/DimensionedFields/
│
└── tmp<T> (smart pointer for memory management)
    └── $FOAM_SRC/OpenFOAM/memory/tmp/
```

**Key Template Instantiations:**
- `volScalarField` = GeometricField<scalar, fvPatchField, volMesh>
- `volVectorField` = GeometricField<vector, fvPatchField, volMesh>
- `surfaceScalarField` = GeometricField<scalar, fvsPatchField, surfaceMesh>

**Memory Layout:**
```
volScalarField p
│
├── [Internal Field]  (cell values)
│   └── List<scalar>  → [p0, p1, p2, ..., pN]
│       Size: N cells
│       Location: $FOAM_SRC/OpenFOAM/fields/GeometricFields/
│
└── [Boundary Field]  (patch values)
    └── PtrList<fvPatchScalarField>
        ├── patch0 (inlet)   → FixedValueFvPatchScalarField
        ├── patch1 (outlet)  → ZeroGradientFvPatchScalarField
        ├── patch2 (walls)   → NoSlipFvPatchField<vector>
        └── ...
        Size: Σ (patch face counts)
        Location: $FOAM_SRC/finiteVolume/fields/fvPatchFields/
```

---

### 2.2 Matrix Classes (Linear System Solvers)

OpenFOAM represents discretized equations as sparse linear systems using the matrix classes.

```
lduMatrix (Lower-Diagonal-Upper matrix)
│   └── $FOAM_SRC/OpenFOAM/matrices/lduMatrix/
│
└── fvMatrix<Type> : public lduMatrix
    └── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
```

**Inheritance Hierarchy:**
```
lduMatrix
│
├── lduAddressing (mesh connectivity)
│   ├── lowerAddr()  → owner cell indices
│   ├── upperAddr()  → neighbor cell indices
│   └── losortAddr() → sorting for symmetric matrices
│
└── fvMatrix<Type>
    ├── coefficients (source terms, boundary contributions)
    ├── psi (reference to solution field)
    └── dimensions (dimensional consistency checking)
```

**Memory Layout (Sparse Matrix Storage):**
```
fvMatrix<scalar> matrix
│
├── Diagonal
│   └── scalarField diag  → [D0, D1, D2, ..., DN]
│       Size: N cells
│
├── Lower (sub-diagonal)
│   └── scalarField lower → [L0, L1, L2, ..., LM]
│       Size: M faces (where M = number of internal faces)
│
├── Upper (super-diagonal)
│   └── scalarField upper → [U0, U1, U2, ..., UM]
│       Size: M faces
│
└── Source
    └── Field<Type> source → [S0, S1, S2, ..., SN]
        Size: N cells
```

**Source Files:**
- `lduMatrix`: `$FOAM_SRC/OpenFOAM/matrices/lduMatrix/lduMatrix.H`
- `fvMatrix`: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/fvMatrix/fvMatrix.H`

### 2.3 Mesh Classes

```
polyMesh
│   ├── points_  (vertex coordinates)
│   ├── faces_   (face definitions)
│   ├── cells_   (cell-to-face addressing)
│   └── owner_ / neighbor_ (face-to-cell addressing)
│
└── fvMesh : public polyMesh
    ├── Sf (surface area vectors)
```

**Memory Layout:**
```
fvMesh mesh
│
├── points_
│   └── pointField → List<point>
│       Size: N vertices

│
├── faces_
│   └── faceList → List<face>
│       Size: M faces

│
├── owner_
│   └── labelList → [c0, c1, c2, ..., cM]
│       Size: M faces
│       Meaning: owner cell index for each face
│
└── neighbor_
    └── labelList → [n0, n1, n2, ..., nM]

```

**Source Files:**
- `polyMesh`: `$FOAM_SRC/OpenFOAM/meshes/polyMesh/polyMesh.H`

### 2.4 Boundary Condition Classes

Boundary conditions are implemented using polymorphic patch field classes.

```
fvPatchField<Type>
│   ├── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
│   │
│   ├── fixedValueFvPatchField
│   │   └── Dirichlet BC (value specified)
│   │
│   ├── zeroGradientFvPatchField
│   │   └── Neumann BC (∂φ/∂n = 0)
│   │
│   ├── fixedGradientFvPatchField
│   │   └── Neumann BC (gradient specified)
│   │
│   └── mixedFvPatchField
│       └── Robin BC (αφ + β∂φ/∂n = γ)
```

**Virtual Function Interface:**
```cpp
class fvPatchField<Type>
{
    // Update coefficients (called during matrix assembly)
    virtual void updateCoeffs() = 0;
    
    // Evaluate boundary value
    virtual tmp<Field<Type>> valueInternalCoeffs(
        const tmp<scalarField>&
    ) const = 0;
    
    // Gradient contribution
    virtual tmp<Field<Type>> gradientInternalCoeffs(
        const tmp<scalarField>&
    ) const = 0;
};
```

**Source Files:**
- Base class: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/fvPatchField/fvPatchField.H`
- Derived classes: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/*/*FvPatchField.H`

## 4. Dictionary Analysis & Configuration

### 4.1 `fvSchemes` - Discretization Schemes

The `fvSchemes` dictionary controls how the governing equations are discretized. Key sections relevant to the momentum equation:

**Example `system/fvSchemes`:**
```cpp
ddtSchemes
{
    default         Euler;              // First-order time integration
    // alternative: backward;          // Second-order implicit
}

gradSchemes
{
    default         Gauss linear;       // Central differencing for gradients
    grad(p)         Gauss linear;       // Pressure gradient
    grad(U)         Gauss linear;       // Velocity gradient
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwindV grad(U);  // Convective term - upwind
    div(phi,k)      Gauss upwind;        // Turbulence kinetic energy
    div(phi,epsilon) Gauss upwind;       // Dissipation rate
    div((nuEff*dev2(T(grad(U))))) Gauss linear;  // Viscous stress term
}

laplacianSchemes
{
    default         Gauss linear corrected;  // Diffusion terms with correction
}

interpolationSchemes
{
    default         linear;
}

snGradSchemes
{
    default         corrected;          // Surface normal gradient
}
```

**Physical Mapping:**
- `div(phi,U)` → $\nabla \cdot (\rho \mathbf{U} \mathbf{U})$ (Convective term)
- `laplacian(nuEff,U)` → $\nabla \cdot (\mu \nabla \mathbf{U})$ (Viscous term)
- `grad(p)` → $\nabla p$ (Pressure gradient)

### 4.2 `fvSolution` - Linear Solver Settings

The `fvSolution` dictionary controls how the discretized linear systems are solved:

**Example `system/fvSolution`:**
```cpp
solvers
{
    p
    {
        solver          GAMG;              // Geometric-Algebraic Multi-Grid
        tolerance       1e-06;             // Convergence tolerance
        relTol          0.01;              // Relative tolerance
        smoother        GaussSeidel;       // Smoother for GAMG
    }

    pFinal
    {
        $p;                                // Inherit from p
        relTol          0;                 // Tight tolerance for final iteration
    }

    U
    {
        solver          smoothSolver;      // Iterative solver
        smoother        GaussSeidel;
        tolerance       1e-05;
        relTol          0.1;
    }
}

SIMPLE
{
    nNonOrthogonalCorrectors 0;            // Non-orthogonal mesh correction
    consistent      yes;                   // Consistent SIMPLE algorithm
}

relaxationFactors
{
    fields
    {
        p               0.3;               // Pressure relaxation
        U               0.7;               // Momentum relaxation
    }
}
```

**Key Concepts:**
- **GAMG**: Fast solver for pressure Poisson equation (elliptic)
- **smoothSolver**: Good for momentum equations (hyperbolic-dominated)
- **Under-relaxation**: Prevents divergence in segregated algorithms

## 5. Hands-on: Practical Tasks & Coding

### Task 1: Add Coriolis Force to Momentum Equation

**Objective:** Modify the momentum equation to include Coriolis force for rotating reference frames.

**Physical Background:**
$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g} - 2\rho \boldsymbol{\Omega} \times \mathbf{U}
$$

**Solution:**

Create a custom solver `myRotatingSimpleFoam` by modifying `UEqn.H`:

```cpp
// File: myRotatingSimpleFoam/UEqn.H

// Momentum equation
tmp<fvVectorMatrix> tUEqn
(
    fvm::ddt(U)
  + fvm::div(phi, U)
  + MRF.DDt(U)                    // Moving Reference Frame (Coriolis + Centrifugal)
  + turbulence->divDevReff(U)
 ==
    fvOptions(U)
);
fvVectorMatrix& UEqn = tUEqn.ref();

UEqn.relax();

fvOptions.constrain(UEqn);

solve(UEqn == -fvc::grad(p));

fvOptions.correct(U);
```

**Key Changes:**
1. Add `MRF.DDt(U)` term (includes Coriolis and centrifugal forces)
2. Requires `MRFProperties` dictionary in `constant/`:

```cpp
// constant/MRFProperties
MRF
{
    cellZone    rotor;
    active      yes;

    origin      (0 0 0);
    axis        (0 0 1);      // Rotation axis (z-direction)
    omega       104.7;        // Angular velocity [rad/s] (~1000 RPM)
}
```

### Task 2: Implement Custom Boundary Condition

**Objective:** Create a time-varying velocity inlet boundary condition.

**Physical Background:**
$$
U_{inlet}(t) = U_0 + A \sin(2\pi f t)
$$

**Solution:**

Create `timeVaryingFixedValueFvPatchVectorField.C`:

```cpp
// File: customBC/timeVaryingFixedValueFvPatchVectorField.C

#include "timeVaryingFixedValueFvPatchVectorField.H"
#include "addToRunTimeSelectionTable.H"
#include "volFields.H"

// * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * //

Foam::timeVaryingFixedValueFvPatchVectorField::
timeVaryingFixedValueFvPatchVectorField
(
    const fvPatch& p,
    const DimensionedField<vector, volMesh>& iF
)
:
    fixedValueFvPatchVectorField(p, iF),
    U0_(Zero),
    amplitude_(Zero),
    frequency_(0.0)
{}


void Foam::timeVaryingFixedValueFvPatchVectorField::updateCoeffs()
{
    if (this->updated())
    {
        return;
    }

    const scalar t = this->db().time().timeOutputValue();
    const scalar omega = 2.0 * constant::mathematical::pi * frequency_;

    // U(t) = U0 + A * sin(omega * t)
    vectorField::operator=(U0_ + amplitude_ * Foam::sin(omega * t));

    fixedValueFvPatchVectorField::updateCoeffs();
}


// * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * //

namespace Foam
{
    makePatchTypeField
    (
        fvPatchVectorField,
        timeVaryingFixedValueFvPatchVectorField
    );
}
```

**Usage in `0/U`:**
```cpp
inlet
{
    type            timeVaryingFixedValue;
    U0              (10 0 0);      // Mean velocity [m/s]
    amplitude       (2 0 0);       // Oscillation amplitude [m/s]
    frequency       0.5;           // Frequency [Hz]
}
```

### Task 3: Add Porous Media Source Term

**Objective:** Implement momentum sink term for porous media flow.

**Physical Background:**
$$
S_{porous} = -(\mu \mathbf{D} + \frac{1}{2}\rho |\mathbf{U}| \mathbf{F}) \cdot \mathbf{U}
$$

**Solution:**

Modify `UEqn.H` to include porous resistance:

```cpp
// File: myPorousSimpleFoam/UEqn.H

// Momentum equation
tmp<fvVectorMatrix> tUEqn
(
    fvm::ddt(U)
  + fvm::div(phi, U)
  + turbulence->divDevReff(U)
 ==
    fvOptions(U)                    // Includes porous media via fvOptions
);
fvVectorMatrix& UEqn = tUEqn.ref();

UEqn.relax();

// Add explicit porous source term
const volVectorField& porositySource = 
    mesh.lookupObject<volVectorField>("porositySource");

UEqn -= porositySource;

fvOptions.constrain(UEqn);

solve(UEqn == -fvc::grad(p));

fvOptions.correct(U);
```

**Define `constant/fvOptions`:**
```cpp
porosity1
{
    type            explicitPorositySource;
    active          yes;

    explicitPorositySourceCoeffs
    {
        selectionMode   cellZone;
        cellZone        porousZone;

        type            DarcyForchheimer;

        D   (5e7 -1000 -1000);      // Viscous resistance [1/m²]
        F   (0 0 0);                // Inertial resistance [1/m]
        
        coordinateSystem
        {
            type    cartesian;
            origin  (0 0 0);
            coordinateRotation
            {
                type    axesRotation;
                e1      (1 0 0);
                e2      (0 1 0);
            }
        }
    }
}
```

### Question 1: Mass Conservation
**ในสมการต่อเนื่อง (Continuity Equation) ถ้า $\nabla \cdot (\rho \mathbf{U}) > 0$ แปลว่าอะไร?**

<details>
<summary>คำตอบ</summary>

**Answer:** แปลว่ามีการไหลออกจากปริมาตรควบคุมมากกว่าการไหลเข้า (Mass outflow > Mass inflow) ซึ่งทำให้ความหนาแน่นลดลงในจุดนั้น (ถ้า steady-state แล้ว $\nabla \cdot (\rho \mathbf{U}) = 0$)

**Physical Interpretation:**
- $\nabla \cdot (\rho \mathbf{U}) > 0$ → Source (แหล่งกำเนิดมวล)
- $\nabla \cdot (\rho \mathbf{U}) < 0$ → Sink (แหล่งดูดซับมวล)
- $\nabla \cdot (\rho \mathbf{U}) = 0$ → Incompressible flow

### Question 2: Pressure-Velocity Coupling
**ทำไมใน OpenFOAM ต้องมีการแก้สมการความดัน (Pressure Equation) แยกจากสมการโมเมนตัม?**

<details>
<summary>คำตอบ</summary>

**Answer:** เพราะไม่มีสมการชัดเจนสำหรับความดันในสมการโมเมนตัม (ไม่มี term $\frac{\partial p}{\partial t}$) ความดันถูกกำหนดโดยอ้อมผ่านเงื่อนไขการอนุรักษ์มวล

**Algorithm (SIMPLE/PISO/PIMPLE):**
1. Solve Momentum Equation → คำนวณ $\mathbf{U}^*$ (intermediate velocity)
2. Solve Pressure Poisson Equation → $\nabla^2 p = \frac{\rho}{\Delta t} \nabla \cdot \mathbf{U}^*$
3. Correct Velocity → $\mathbf{U}^{n+1} = \mathbf{U}^* - \frac{\Delta t}{\rho} \nabla p$

**Code Location:**
- `pEqn.H` in solvers

</details>

### Question 3: Boundary Condition Types
**`fixedValue` และ `zeroGradient` ต่างกันอย่างไร และใช้เมื่อไร?**

<details>
<summary>คำตอบ</summary>

**Answer:**

| Boundary Condition | Mathematical Type | Physical Meaning | Use Case |
|-------------------|-------------------|------------------|----------|
| `fixedValue` | Dirichlet: $\phi = \phi_{wall}$ | กำหนดค่าโดยตรง | Inlet velocity, Wall temperature |
| `zeroGradient` | Neumann: $\frac{\partial \phi}{\partial n} = 0$ | ไม่มีการไหลผ่านขอบ | Outlet pressure, Symmetry plane |

**Example:**
```cpp
inlet
{
    type            fixedValue;
    value           uniform (10 0 0);  // Dirichlet
}

outlet
{
    type            zeroGradient;      // Neumann (∂U/∂n = 0)
}
```

</details>

### Question 4: Under-Relaxation
**ทำไมต้องมี Under-Relaxation Factors ใน `fvSolution`?**

<details>
<summary>คำตอบ</summary>

**Answer:** เพื่อป้องกันการแตกตัว (divergence) ของการคำนวณใน segregated algorithm (แก้ทีละตัวแปร)

**Mathematical Form:**
$$
\phi_{new} = \alpha \phi_{calculated} + (1 - \alpha) \phi_{old}
$$

เมื่อ $\alpha$ คือ relaxation factor (0 < $\alpha$ ≤ 1)

**Typical Values:**
```cpp
relaxationFactors
{
    p               0.3;    // Pressure: อ่อนไหวมาก (แก้ยาก)
    U               0.7;    // Velocity: ปานกลาง
    k               0.7;    // Turbulence: ปานกลาง
}
```

**Physical Reason:**
- Pressure equation มีความเป็น elliptic สูง (global coupling)
- ถ้า $\alpha$ สูงเกินไป → oscillate และ diverge
- ถ้า $\alpha$ ต่ำเกินไป → converge ช้ามาก

</details>

### Question 5: Matrix Assembly
**ใน `fvMatrix` คำว่า "Diagonal", "Lower", และ "Upper" หมายถึงอะไร?**

<details>
<summary>คำตอบ</summary>

**Answer:** เป็นการเก็บ sparse matrix แบบ LDU (Lower-Diagonal-Upper) format

**Matrix Structure:**
$$
\begin{bmatrix}
D_0 & U_0 & 0 & \cdots \\
L_0 & D_1 & U_1 & \cdots \\
0 & L_1 & D_2 & \cdots \\
\vdots & \vdots & \vdots & \ddots
\end{bmatrix}
\begin{bmatrix}
\phi_0 \\
\phi_1 \\
\phi_2 \\
\vdots
\end{bmatrix}
=
\begin{bmatrix}
S_0 \\
S_1 \\
S_2 \\
\vdots
\end{bmatrix}
$$

**Physical Mapping:**
- **Diagonal ($D_i$)**: Contribution from cell $i$ itself (coefficients from $\phi_P$)
- **Lower ($L_{ij}$)**: Contribution from neighbor cell $j$ where $j < i$ (owner)
- **Upper ($U_{ij}$)**: Contribution from neighbor cell $j$ where $j > i$ (neighbor)

**Memory Efficiency:**
- Full matrix: $N^2$ entries (dense)
- LDU format: $N + 2M$ entries (sparse, $M$ = number of faces)
- Example: 1M cells → ~3M entries (vs 1 trillion for dense!)

**Code Location:**
- `$FOAM_SRC/OpenFOAM/matrices/lduMatrix/lduMatrix.H`

</details>
