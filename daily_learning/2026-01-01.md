## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**สมการการอนุรักษ์มวล (Continuity Equation)**

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**คำอธิบายพจน์:**
- $\rho$ (rho) = **ความหนาแน่น** (density) [kg/m³]
- $\mathbf{U}$ = **สนามความเร็ว** (velocity field) [m/s]
- $\nabla \cdot$ = **ตัวดำเนินการไดเวอร์เจนซ์** (divergence operator)
- $t$ = **เวลา** (time) [s]

สำหรับการไหลแบบ **ไม่อัดตัว (incompressible flow)**: $\nabla \cdot \mathbf{U} = 0$

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**สมการการอนุรักษ์โมเมนตัม (Momentum Equation)**

$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**คำอธิบายพจน์:**
- $p$ = **ความดัน** (pressure) [Pa]
- $\boldsymbol{\tau}$ (tau) = **เทนเซอร์ความเค้น** (stress tensor) [Pa]
- $\mathbf{g}$ = **ความเร่งเนื่องจากแรงโน้มถ่วง** (gravitational acceleration) [m/s²]
- $\nabla p$ = **การไล่ระดับความดัน** (pressure gradient)
- $\rho \mathbf{U} \mathbf{U}$ = **flux ของโมเมนตัม** (momentum flux)

สำหรับ **ของไหลนิวตัน (Newtonian fluid)**:

$$
\boldsymbol{\tau} = \mu \left[ \nabla \mathbf{U} + (\nabla \mathbf{U})^T \right] - \frac{2}{3}\mu (\nabla \cdot \mathbf{U})\mathbf{I}
$$

- $\mu$ (mu) = **ความหนืดพลศาสตร์** (dynamic viscosity) [Pa·s]
- $\mathbf{I}$ = **เทนเซอร์เอกลักษณ์** (identity tensor)

---

### 1.3 Conservation of Energy

**สมการการอนุรักษ์พลังงาน (Energy Equation)**

$$
\frac{\partial (\rho e)}{\partial t} + \nabla \cdot (\rho e \mathbf{U}) = -\nabla \cdot \mathbf{q} + \boldsymbol{\tau} : \nabla \mathbf{U} + S_h
$$

**คำอธิบายพจน์:**
- $e$ = **พลังงานภายในต่อหน่วยมวล** (specific internal energy) [J/kg]
- $\mathbf{q}$ = **flux ของความร้อน** (heat flux) [W/m²]
- $S_h$ = **แหล่งกำเนิดความร้อน** (heat source) [W/m³]
- $\boldsymbol{\tau} : \nabla \mathbf{U}$ = **การกระจายของพลังงานเนื่องจากความเค้น** (viscous dissipation)

ตาม **กฎของฟูริเยร์ (Fourier's Law)**:

$$
\mathbf{q} = -k \nabla T
$$

- $k$ = **สัมประสิทธิภาพการนำความร้อน** (thermal conductivity) [W/(m·K)]
- $T$ = **อุณหภูมิ** (temperature) [K]

---

### 1.4 Equation of State

**สมการสถานะ (Equation of State)**

สำหรับ **ก๊าซอุดมคติ (Ideal Gas)**:

$$
p = \rho R T
$$

- $R$ = **ค่าคงที่ของก๊าซเฉพาะ** (specific gas constant) [J/(kg·K)]

สำหรับ **ของไหลที่ไม่อัดตัว (Incompressible Fluid)**:

$$
\rho = \text{constant}
$$

---

### 1.5 Transport Equations (General Form)

**สมการขนส่งทั่วไป (General Transport Equation)**

$$
\frac{\partial (\rho \phi)}{\partial t} + \nabla \cdot (\rho \phi \mathbf{U}) = \nabla \cdot (\Gamma_\phi \nabla \phi) + S_\phi
$$

**คำอธิบายพจน์:**
- $\phi$ (phi) = **ปริมาณที่ถูกขนส่ง** (transported quantity) - สเกลาร์ใดๆ
- $\Gamma_\phi$ (Gamma) = **สัมประสิทธิภาพการแพร่** (diffusion coefficient)
- $S_\phi$ = **แหล่งกำเนิดหรือรับ** (source/sink term)

**องค์ประกอบของสมการ:**
1. **Unsteady term** $\frac{\partial (\rho \phi)}{\partial t}$ = การเปลี่ยนแปลงตามเวลา
2. **Convection term** $\nabla \cdot (\rho \phi \mathbf{U})$ = การพาความร้อน/โมเมนตัม
3. **Diffusion term** $\nabla \cdot (\Gamma_\phi \nabla \phi)$ = การแพร่
4. **Source term** $S_\phi$ = แหล่งกำเนิดหรือการรับ

---

## 2. OpenFOAM Class Hierarchy & Implementation

### 2.1 Core Field Classes

OpenFOAM represents governing equations through field classes that store and manipulate data at mesh points/cells.

```
GeometricField (Templated abstract base)
│
├── DimensionedField
│   └── Field (1D array of values)
│
├── volScalarField  (e.g., p, T, rho)
├── volVectorField  (e.g., U)
├── volSymmTensorField (e.g., tau)
│
└── surfaceScalarField (e.g., phi - flux)
```

## 3. Code Walkthrough

### 3.1 UEqn.H

ไฟล์ `UEqn.H` ใช้สำหรับสร้างสมการโมเมนตัม (Momentum Equation) ใน OpenFOAM solver ต่างๆ เช่น `simpleFoam`, `pimpleFoam` โดยใช้ finite volume method

**โครงสร้างหลักของ UEqn.H:**

```cpp
// สร้างสมการโมเมนตัมแบบไม่รวมแรงดัน (momentum equation without pressure gradient)
tmp<fvVectorMatrix> UEqn
(
    fvm::ddt(U)                     // Unsteady term: ∂U/∂t
  + fvm::div(phi, U)                // Convection term: ∇·(UU)
  + turbulence->divDevReff(U)       // Turbulent diffusion: ∇·(τ_eff)
 ==
    fvOptions(U)                    // Source terms (optional)
);

// แก้ไข boundary conditions และ source terms
UEqn.relax();

// เพิ่มแรงดันเข้าไปในสมการ (pressure gradient term)
if (pimple.momentumPredictor())
{
    solve(UEqn == -fvc::grad(p));   // -∇p
}
```

**คำอธิบายพจน์สำคัญ:**

| Operator | ความหมาย | ประเภท |
|----------|----------|--------|
| `fvm::ddt(U)` | อนุพันธ์เวลาของความเร็ว | Implicit |
| `fvm::div(phi, U)` | Divergence ของ flux ความเร็ว | Implicit |
| `turbulence->divDevReff(U)` | Divergence ของเทนเซอร์ความเค้นที่มีผลจากความปั่น | Implicit |
| `fvc::grad(p)` | Gradient ของความดัน | Explicit |
| `fvOptions(U)` | Source terms จาก OpenFOAM framework | Explicit/Implicit |

**ความแตกต่างระหว่าง `fvm::` และ `fvc::`:**
- **`fvm::`** (finite volume method) = สร้างเมทริกซ์สำหรับการแก้ปัญหาแบบ **implicit** (ค่าที่ boundary และ internal field รวมอยู่ในเมทริกซ์)
- **`fvc::`** (finite volume calculus) = คำนวณค่าโดยตรงแบบ **explicit** (ใช้ค่าจาก time step ก่อนหน้า)

**การทำงานร่วมกับ Pressure-Velocity Coupling:**

สำหรับ incompressible flow สมการโมเมนตัมและความต่อเนื่องจะถูกแก้ไขพร้อมกันผ่านขั้นตอน:

1. **Predictor step:** แก้สมการโมเมนตัมโดยประมาณความดัน
2. **Pressure correction:** แก้สมการความดัน (Poisson equation) เพื่อให้ได้ velocity field ที่ satisfy continuity
3. **Corrector step:** แก้ไข velocity field ด้วยความดันใหม่

**ตัวอย่างจาก `simpleFoam`:**

```cpp
// Momentum predictor
tmp<fvVectorMatrix> UEqn
(
    fvm::div(phi, U)
  + turbulence->divDevReff(U)
 ==
    fvOptions(U)
);

UEqn.relax();

fvOptions.constrain(UEqn);

solve(UEqn == -fvc::grad(p));

fvOptions.correct(U);
```

### 3.2 pEqn.H

ไฟล์ `pEqn.H` ใช้สำหรับสร้างและแก้สมการความดัน (Pressure Equation) ซึ่งเป็นส่วนสำคัญของ pressure-velocity coupling ใน OpenFOAM solvers เช่น `simpleFoam`, `pimpleFoam`

**โครงสร้างหลักของ pEqn.H (สำหรับ incompressible flow):**

```cpp
// สร้างสมการความดันจากสมการโมเมนตัมและความต่อเนื่อง
volScalarField rAU(1.0/UEqn.A());         // Reciprocal of diagonal coefficients
volVectorField HbyA(constrainHbyA(rAU*UEqn.H())); // Explicit flux term
surfaceScalarField phiHbyA
(
    "phiHbyA",
    fvc::flux(HbyA)
  + fvc::interpolate(rAU)*fvc::ddtCorr(U, phi)
);

// ปรับ pressure boundary conditions
adjustPhi(phiHbyA, U, p);

// สร้างสมการ Poisson สำหรับความดัน
while (simple.correctNonOrthogonal())
{
    fvScalarMatrix pEqn
    (
        fvm::laplacian(rAU, p) == fvc::div(phiHbyA)
    );

    pEqn.setReference(pRefCell, pRefValue);
    pEqn.solve();

    if (simple.finalNonOrthogonalIter())
    {
        phi = phiHbyA - pEqn.flux();
    }
}

// แก้ไข velocity field ด้วยความดันใหม่
U = HbyA - rAU*fvc::grad(p);
U.correctBoundaryConditions();
```

**คำอธิบายพจน์สำคัญ:**

| Operator/Variable | ความหมาย |
|-------------------|----------|
| `UEqn.A()` | ส่วนประกอบ diagonal ของเมทริกซ์โมเมนตัม |
| `UEqn.H()` | ส่วนประกอบ off-diagonal (source term) |
| `rAU` | ค่าผกผันของสัมประสิทธิ์เมทริกซ์ (1/A) |
| `phiHbyA` | Flux ที่คำนวณจาก velocity โดยประมาณ |
| `fvm::laplacian(rAU, p)` | Diffusion term ของสมการความดัน |
| `fvc::div(phiHbyA)` | Divergence ของ flux (source term) |
| `pEqn.solve()` | แก้ระบบสมการเชิงเส้นสำหรับความดัน |
| `pEqn.flux()` | คำนวณ pressure flux correction |

**ขั้นตอนการแก้สมการความดัน (Pressure Correction):**

1. **Construct HbyA:** คำนวณ velocity field โดยประมาณจากสมการโมเมนตัม
2. **Build pressure equation:** สร้างสมการ Poisson สำหรับความดันจาก continuity equation
3. **Solve pressure:** แก้สมการความดันโดยใช้ linear solver
4. **Correct flux:** ปรับ mass flux ให้สอดคล้องกับความดันใหม่
5. **Correct velocity:** แก้ไข velocity field ด้วย gradient ของความดัน

**ตัวอย่างจาก `simpleFoam` (steady-state incompressible):**

```cpp
// Pressure equation
volScalarField rAU(1.0/UEqn.A());
volVectorField HbyA(constrainHbyA(rAU*UEqn.H()));
surfaceScalarField phiHbyA("phiHbyA", fvc::flux(HbyA));

adjustPhi(phiHbyA, U, p);

// Non-orthogonal correction loop
while (simple.correctNonOrthogonal())
{
    fvScalarMatrix pEqn
    (
        fvm::laplacian(rAU, p) == fvc::div(phiHbyA)
    );

    pEqn.setReference(pRefCell, pRefValue);
    pEqn.solve();

    if (simple.finalNonOrthogonalIter())
    {
        phi = phiHbyA - pEqn.flux();
    }
}

// Velocity correction
U = HbyA - rAU*fvc::grad(p);
U.correctBoundaryConditions();
```

### 3.3 createFields.H

ไฟล์ `createFields.H` ใช้สำหรับสร้างและเริ่มต้นฟิลด์ต่างๆ (fields) ที่จำเป็นสำหรับการแก้สมการใน OpenFOAM solver โดยทำหน้าที่อ่านค่าจากไฟล์คอนฟิกูเรชันและสร้าง objects ต่างๆ ขึ้นมาในหน่วยความจำ

**โครงสร้างหลักของ createFields.H:**

```cpp
// สร้าง mesh object จากไฟล์ case directory
fvMesh mesh
(
    Foam::IOobject
    (
        fvMesh::defaultRegion,
        runTime.timeName(),
        runTime,
        IOobject::MUST_READ
    )
);

// สร้าง pressure field และอ่านค่าเริ่มต้นจาก 0/ หรือ 0.org/
volScalarField p
(
    IOobject
    (
        "p",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้าง velocity field
volVectorField U
(
    IOobject
    (
        "U",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้าง mass flux field (surface scalar field)
surfaceScalarField phi
(
    IOobject
    (
        "phi",
        runTime.timeName(),
        mesh,
        IOobject::READ_IF_PRESENT,
        IOobject::AUTO_WRITE
    ),
    fvc::flux(U)
);

// สร้าง transport properties object (ความหนืด)
singlePhaseTransportModel laminarTransport(U, phi);

// สร้าง turbulence model
autoPtr<incompressible::turbulenceModel> turbulence
(
    incompressible::turbulenceModel::New(U, phi, laminarTransport)
);
```

**คำอธิบายพจน์สำคัญ:**

| Class/Object | ความหมาย |
|--------------|----------|
| `fvMesh` | Finite volume mesh - โครงสร้างเมชสำหรับ FVM |
| `volScalarField` | ฟิลด์สเกลาร์ที่จุดศูนย์กลางเซลล์ (เช่น ความดัน) |
| `volVectorField` | ฟิลด์เวกเตอร์ที่จุดศูนย์กลางเซลล์ (เช่น ความเร็ว) |
| `surfaceScalarField` | ฟิลด์สเกลาร์ที่ผิวเซลล์ (เช่น mass flux) |
| `IOobject` | คลาสสำหรับจัดการการอ่าน/เขียนไฟล์ |
| `MUST_READ` | ต้องอ่านค่าจากไฟล์ (error ถ้าไม่มี) |
| `AUTO_WRITE` | เขียนค่าลงไฟล์อัตโนมัติเมื่อสิ้นสุด time step |
| `fvc::flux(U)` | คำนวณ mass flux จาก velocity field |
| `singlePhaseTransportModel` | โมเดลความหนืดสำหรับ single-phase flow |
| `turbulenceModel` | โมเดลความปั่น (k-ε, k-ω SST, etc.) |

**การทำงานของ IOobject flags:**

- **`MUST_READ`**: บังคับให้อ่านค่าจากไฟล์ใน time directory (เช่น `0/`, `0.01/`) ถ้าไม่พบไฟล์จะเกิด error
- **`READ_IF_PRESENT`**: อ่านค่าถ้าไฟล์มีอยู่ ถ้าไม่มีจะใช้ค่าเริ่มต้นหรือสร้างฟิลด์ใหม่
- **`AUTO_WRITE`**: เขียนค่าฟิลด์ลงไฟล์อัตโนมัติเมื่อ solver เสร็จสิ้นแต่ละ time step
- **`NO_WRITE`**: ไม่เขียนค่าฟิลด์ (ใช้สำหรับ temporary fields)

**ตัวอย่างการอ่านค่าจากไฟล์คอนฟิกูเรชัน:**

```cpp
// อ่านค่าความหนืดจาก transportProperties
Info<< "Reading transportProperties\n" << endl;
IOdictionary transportProperties
(
    IOobject
    (
        "transportProperties",
        runTime.constant(),
        mesh,
        IOobject::MUST_READ_IF_MODIFIED,
        IOobject::NO_WRITE
    )
);

dimensionedScalar nu
(
    "nu",
    dimViscosity,
    transportProperties
);
```

**ขั้นตอนการสร้างฟิลด์:**

1. **สร้าง mesh**: อ่านเมชจาก `constant/polyMesh/`
2. **สร้าง pressure field**: อ่านค่าเริ่มต้นจาก `0/p`
3. **สร้าง velocity field**: อ่านค่าเริ่มต้นจาก `0/U`
4. **สร้าง flux field**: คำนวณจาก velocity field
5. **สร้าง physical models**: อ่านค่าความหนืดและ turbulence model

## 4. Dictionary Analysis & Configuration

### 4.1 fvSchemes Analysis

ไฟล์ `system/fvSchemes` ใช้กำหนดรูปแบบการ discretization ของสมการใน OpenFOAM ซึ่งมีผลต่อความแม่นยำและเสถียรภาพของการคำนวณ

**โครงสร้างหลักของ fvSchemes:**

```
ddtSchemes
{
    default         Euler;
}

gradSchemes
{
    default         Gauss linear;
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwindV grad(U);
}

laplacianSchemes
{
    default         Gauss linear corrected;
}
```

**คำอธิบายแต่ละส่วน:**

#### 4.1.1 ddtSchemes (Time Derivative Schemes)

**สมการอนุพันธ์เวลา (Temporal Discretization)**

รูปแบบ: `ddt(phi)` หรือ `ddt(rho, phi)`

| Scheme | ความหมาย | คำอธิบาย |
|--------|----------|-----------|
| `Euler` | Implicit Euler | อันดับที่ 1 (First-order) เสถียรแต่ความแม่นยำต่ำ |
| `backward` | Backward differentiation | อันดับที่ 2 (Second-order) เสถียรสำหรับ unsteady flow |
| `CrankNicolson` | Crank-Nicolson | อันดับที่ 2 มีความแม่นยำสูงแต่อาจไม่เสถียร |
| `localEuler` | Local time stepping | ใช้สำหรับ steady-state acceleration |

**ตัวอย่างการใช้งาน:**
```
ddtSchemes
{
    default         backward;           // Second-order implicit
    ddt(rho,U)      Euler;              // First-order for momentum
}
```

#### 4.1.2 gradSchemes (Gradient Schemes)

**สมการ gradient (Spatial Gradient Discretization)**

รูปแบบ: `grad(phi)` หรือ `grad(U)`

| Scheme | ความหมาย | คำอธิบาย |
|--------|----------|-----------|
| `Gauss linear` | Linear interpolation | อันดับที่ 2 ใช้สำหรับ mesh ที่มีคุณภาพดี |
| `Gauss linearUpwind` | Upwind-biased | ลด oscillation สำหรับ convection-dominated flow |
| `leastSquares` | Least squares | อันดับที่ 2 เหมาะกับ non-orthogonal mesh |
| `fourth` | Fourth-order | ความแม่นยำสูงแต่ใช้เวลาคำนวณนาน |

**ตัวอย่างการใช้งาน:**
```
gradSchemes
{
    default         Gauss linear;
    grad(p)         Gauss linear;        // Pressure gradient
    grad(U)         cellLimited Gauss linear 1; // Limited velocity gradient
}
```

#### 4.1.3 divSchemes (Divergence Schemes)

**สมการ divergence (Convection Term Discretization)**

รูปแบบ: `div(phi, U)` หรือ `div(phi, T)`

| Scheme | ความหมาย | คำอธิบาย |
|--------|----------|-----------|
| `Gauss upwind` | Upwind scheme | อันดับที่ 1 เสถียรมากแต่มี numerical diffusion |
| `Gauss linearUpwind` | Linear upwind | อันดับที่ 2 ลด diffusion แต่อาจไม่เสถียร |
| `Gauss linearUpwindV` | Linear upwind with gradient | อันดับที่ 2 ใช้ gradient จาก `gradSchemes` |
| `Gauss QUICK` | QUICK scheme | อันดับที่ 3 ความแม่นยำสูงสำหรับ structured mesh |
| `Gauss limitedLinearV` | Limited linear | อันดับที่ 2 มี limiter ป้องกัน oscillation |
| `Gauss vanLeer` | van Leer scheme | TVD scheme ลด oscillation ใน flow ที่มี shock |

**ตัวอย่างการใช้งาน:**
```
divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwindV grad(U);  // Momentum convection
    div(phi,k)      Gauss limitedLinearV 1;       // Turbulent kinetic energy
    div(phi,epsilon) Gauss limitedLinearV 1;      // Dissipation rate
    div((nuEff*dev2(T(grad(U))))) Gauss linear;   // Diffusion term
}
```

**ข้อควรระวัง:**
- ใช้ `upwind` สำหรับการเริ่มต้น (initialization) เพื่อความเสถียร
- เปลี่ยนเป็น `linearUpwindV` หรือ `limitedLinearV` เมื่อ solution ลู่เข้า
- หลีกเลี่ยง `central differencing` (Gauss linear) สำหรับ convection term ที่มี Reynolds number สูง

#### 4.1.4 laplacianSchemes (Laplacian Schemes)

**สมการ Laplacian (Diffusion Term Discretization)**

รูปแบบ: `laplacian(nu, U)` หรือ `laplacian(D, T)`

| Scheme | ความหมาย | คำอธิบาย |
|--------|----------|-----------|
| `Gauss linear` | Linear interpolation | อันดับที่ 2 ใช้สำหรับ orthogonal mesh |
| `Gauss linear corrected` | Corrected linear | อันดับที่ 2 มี non-orthogonal correction |
| `Gauss linear limited` | Limited linear | อันดับที่ 2 มี limiter ป้องกัน oscillation |
| `Gauss fourth` | Fourth-order | ความแม่นยำสูงแต่ใช้เวลานาน |

**ตัวอย่างการใช้งาน:**
```
laplacianSchemes
{
    default         Gauss linear corrected;
    laplacian(nu,U) Gauss linear uncorrected;  // Momentum diffusion
    laplacian((1|A(U)),p) Gauss linear corrected; // Pressure equation
    laplacian(D,k) Gauss linear corrected;     // Turbulent diffusion
}
```

**คำอธิบาย interpolation schemes:**
- `uncorrected`: ไม่มี non-orthogonal correction (เร็วแต่ความแม่นยำต่ำสำหรับ non-orthogonal mesh)
- `corrected`: มี non-orthogonal correction (แม่นยำแต่ช้ากว่า)
- `limited`: มี limiter ป้องกัน unbounded values

**ตัวอย่างการตั้งค่าสำหรับ case ทั่วไป:**

```
// สำหรับ steady-state incompressible flow (เช่น simpleFoam)
ddtSchemes
{
    default         steadyState;
}

gradSchemes
{
    default         Gauss linear;
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwindV grad(U);
    div(phi,k)      Gauss limitedLinearV 1;
    div(phi,epsilon) Gauss limitedLinearV 1;
}

laplacianSchemes
{
    default         Gauss linear corrected;
}
```

```
// สำหรับ transient incompressible flow (เช่น pimpleFoam)
ddtSchemes
{
    default         backward;
}

gradSchemes
{
    default         Gauss linear;
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwindV grad(U);
    div(phi,k)      Gauss limitedLinearV 1;
    div(phi,epsilon) Gauss limitedLinearV 1;
}

laplacianSchemes
{
    default         Gauss linear corrected;
}
```

**ข้อแนะนำ:**
1. เริ่มต้นด้วย scheme ที่เสถียรที่สุด (Euler, upwind)
2. ค่อยๆ เปลี่ยนเป็น scheme ที่มีความแม่นยำสูงขึ้นเมื่อ solution ลู่เข้า
3. ใช้ `corrected` สำหรับ mesh ที่มี non-orthogonality สูง (> 60 องศา)
4. ตรวจสอบ Courant number (Co) และ Peclet number (Pe) ก่อนเลือก scheme

### 4.2 fvSolution Analysis

ไฟล์ `system/fvSolution` ใช้กำหนดวิธีการแก้สมการ (solver algorithms), relaxation factors, และค่าควบคุมอื่นๆ ที่มีผลต่อความเร็วและเสถียรภาพของการคำนวณ

**โครงสร้างหลักของ fvSolution:**

```
solvers
{
    p
    {
        solver          GAMG;
        tolerance       1e-06;
        relTol          0.01;
    }

    pFinal
    {
        $p;
        relTol          0;
    }

    U
    {
        solver          smoothSolver;
        smoother        symGaussSeidel;
        tolerance       1e-05;
        relTol          0.1;
    }
}

PIMPLE
{
    nCorrectors     2;
    nNonOrthogonalCorrectors 0;
    nAlphaCorr      1;
    nAlphaSubCycles 2;
    cAlpha          1;
}

relaxationFactors
{
    fields
    {
        p               0.3;
        rho             1;
    }
    equations
    {
        U               0.3;
        "(U|k|epsilon)" 0.3;
    }
}

potentialFlow
{
    nNonOrthogonalCorrectors 10;
}
```

**คำอธิบายแต่ละส่วน:**

#### 4.2.1 solvers (Linear Solvers)

**ตัวแก้สมการเชิงเส้น (Linear Equation Solvers)**

รูปแบบ: `solver`, `tolerance`, `relTol`

| Solver | ความหมาย | คำอธิบาย |
|--------|----------|-----------|
| `GAMG` | Geometric-Algebraic Multi-Grid | เหมาะสำหรับสมการ Poisson (ความดัน) บน mesh ขนาดใหญ่ |
| `PCG` | Preconditioned Conjugate Gradient | สำหรับ symmetric matrix (ความดัน, incompressible) |
| `PBiCGStab` | Preconditioned Bi-Conjugate Gradient Stabilized | สำหรับ non-symmetric matrix (ความเร็ว, compressible) |
| `smoothSolver` | Smooth solver | ใช้ smoother เช่น Gauss-Seidel สำหรับการลู่เข้าเบื้องต้น |

**Preconditioners:**

| Preconditioner | ความหมาย | คำอธิบาย |
|----------------|----------|-----------|
| `DIC` | Diagonal Incomplete Cholesky | สำหรับ symmetric matrix |
| `DILU` | Diagonal Incomplete LU | สำหรับ non-symmetric matrix |
| `GAMG` | Multi-grid preconditioner | เร็วสำหรับ large-scale problems |

**Tolerance Parameters:**

- **`tolerance`**: ค่าความคลาดเคลื่อนสัมบูรณ์ (absolute tolerance) - เช่น `1e-06`
- **`relTol`**: ค่าความคลาดเคลื่อนสัมพัทธ์ (relative tolerance) - เช่น `0.01` (1%)

**ตัวอย่างการใช้งาน:**
```
solvers
{
    p
    {
        solver          GAMG;
        tolerance       1e-06;
        relTol          0.01;
        smoother        GaussSeidel;
        nPreSweeps      0;
        nPostSweeps     2;
        cacheAgglomeration on;
        agglomerator    faceAreaPair;
        mergeLevels     1;
    }

    U
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-05;
        relTol          0.1;
    }
}
```

#### 4.2.2 PIMPLE (PISO + SIMPLE)

**อัลกอริทึม Pressure-Velocity Coupling**

รูปแบบ: `nCorrectors`, `nNonOrthogonalCorrectors`, `nOuterCorrectors`

| Parameter | ความหมาย | คำอธิบาย |
|-----------|----------|-----------|
| `nCorrectors` | จำนวน pressure correctors | สำหรับ PISO loop (มาตรฐาน: 2-3) |
| `nNonOrthogonalCorrectors` | จำนวน non-orthogonal corrections | สำหรับ mesh ที่ไม่ orthogonal (มาตรฐาน: 0-1) |
| `nOuterCorrectors` | จำนวน outer correctors | สำหรับ SIMPLE loop (steady-state) |
| `nAlphaCorr` | จำนวน volume fraction correctors | สำหรับ multiphase flow |
| `nAlphaSubCycles` | จำนวน sub-cycles สำหรับ volume fraction | เพื่อเสถียรภาพของ interface |

**ตัวอย่างการใช้งาน:**
```
PIMPLE
{
    nCorrectors                 2;
    nNonOrthogonalCorrectors    1;
    nOuterCorrectors            50;     // สำหรับ transient ที่ต้องการความแม่นยำสูง
    nAlphaCorr                  1;
    nAlphaSubCycles             2;
    cAlpha                      1;
    
    // Residual control
    residualControl
    {
        p               1e-4;
        U               1e-3;
        "(k|epsilon|omega)" 1e-3;
    }
}
```

**คำอธิบาย:**
- **PISO (Pressure Implicit with Splitting of Operators)**: ใช้สำหรับ transient flow แก้ pressure-velocity coupling ซ้ำๆ ในแต่ละ time step
- **SIMPLE (Semi-Implicit Method for Pressure-Linked Equations)**: ใช้สำหรับ steady-state flow แก้ pressure-velocity coupling ซ้ำๆ จนกว่าจะลู่เข้า
- **PIMPLE**: การรวมกันของ PISO และ SIMPLE สำหรับ transient ที่ต้องการความแม่นยำสูง

#### 4.2.3 relaxationFactors

**ตัวปรับ relaxation factors**

รูปแบบ: `fields`, `equations`

| Parameter | ความหมาย | คำอธิบาย |
|-----------|----------|-----------|
| `fields` | Relaxation สำหรับ fields | ใช้กับ p, rho, etc. |
| `equations` | Relaxation สำหรับ equations | ใช้กับ U, k, epsilon, etc. |

**ค่าแนะนำ:**

| Variable | Relaxation Factor | คำอธิบาย |
|----------|------------------|-----------|
| `p` | 0.2 - 0.5 | ความดัน (ต่ำ = เสถียรกว่า) |
| `U` | 0.3 - 0.7 | ความเร็ว |
| `k` | 0.3 - 0.7 | Turbulent kinetic energy |
| `epsilon` | 0.3 - 0.7 | Dissipation rate |
| `rho` | 0.05 - 0.1 | ความหนาแน่น (compressible) |

**ตัวอย่างการใช้งาน:**
```
relaxationFactors
{
    fields
    {
        p               0.3;
        rho             0.05;
    }
    equations
    {
        U               0.3;
        k               0.5;
        epsilon         0.5;
    }
}
```

**ข้อแนะนำ:**
- เริ่มต้นด้วย relaxation factor ต่ำ (0.2 - 0.3) สำหรับ case ที่ยาก
- เพิ่มค่า relaxation factors ค่อยๆ เมื่อ solution ลู่เข้า
- ใช้ `final` keyword สำหรับ final iteration (เช่น `pFinal` ใช้ `relTol 0`)

#### 4.2.4 potentialFlow

**การคำนวณ potential flow เบื้องต้น**

รูปแบบ: `nNonOrthogonalCorrectors`

**ตัวอย่างการใช้งาน:**
```
potentialFlow
{
    nNonOrthogonalCorrectors 10;
}
```

**คำอธิบาย:**
- ใช้สำหรับสร้างค่าเริ่มต้นของความดันและความเร็วด้วย potential flow theory
- เหมาะสำหรับ case ที่มีความซับซ้อนสูงเพื่อให้การลู่เข้าดีขึ้น

**ตัวอย่างการตั้งค่าสำหรับ case ทั่วไป:**

```
// สำหรับ steady-state incompressible flow (เช่น simpleFoam)
solvers
{
    p
    {
        solver          GAMG;
        tolerance       1e-06;
        relTol          0.01;
    }
    
    U
    {
        solver          smoothSolver;
        smoother        symGaussSeidel;
        tolerance       1e-05;
        relTol          0.1;
    }
}

SIMPLE
{
    nNonOrthogonalCorrectors 0;
    
    residualControl
    {
        p               1e-5;
        U               1e-5;
        "(k|epsilon)"   1e-4;
    }
}

relaxationFactors
{
    fields
    {
        p               0.3;
    }
    equations
    {
        U               0.5;
        "(k|epsilon)"   0.5;
    }
}
```

```
// สำหรับ transient incompressible flow (เช่น pimpleFoam)
solvers
{
    p
    {
        solver          GAMG;
        tolerance       1e-06;
        relTol          0.01;
    }
    
    pFinal
    {
        $p;
        relTol          0;
    }
    
    U
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-05;
        relTol          0.1;
    }
}

PIMPLE
{
    nCorrectors                 2;
    nNonOrthogonalCorrectors    1;
    nOuterCorrectors            1;
}

relaxationFactors
{
    fields
    {
        p               0.3;
    }
    equations
    {
        U               0.5;
    }
}
```

**ข้อแนะนำ:**
1. เริ่มต้นด้วย tolerance หลวมๆ (เช่น `1e-4`) และ relaxation factors ต่ำๆ
2. ค่อยๆ ทำให้ tolerance แน่นขึ้น (เช่น `1e-6`) เมื่อ solution ลู่เข้า
3. ใช้ `GAMG` สำหรับ pressure equation บน mesh ขนาดใหญ่ (> 100k cells)
4. ใช้ `residualControl` เพื่อหยุดการคำนวณเมื่อ residuals ต่ำกว่าค่าที่กำหนด
5. ตรวจสอบ convergence ด้วยการ plot residuals และ monitor ค่าที่จุดสนใจ
