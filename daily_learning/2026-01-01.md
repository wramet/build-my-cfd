## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**สมการการอนุรักษ์มวล (Continuity Equation)**

$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**คำอธิบายพจน์:**
- $\rho$ (rho) = **ความหนาแน่น** (density) [kg/m³]
- $\mathbf{U}$ = **สนามความเร็ว** (velocity field) [m/s]
- $\nabla \cdot$ = **ตัวดำเนินการไดเวอร์เจนซ์** (divergence operator)
- $t$ = **เวลา** (time) [s]

สำหรับการไหลแบบ **ไม่อัดตัว (incompressible flow)**: $\nabla \cdot \mathbf{U} = 0$

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**สมการการอนุรักษ์โมเมนตัม (Momentum Equation)**

$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**คำอธิบายพจน์:**
- $p$ = **ความดัน** (pressure) [Pa]
- $\boldsymbol{\tau}$ (tau) = **เทนเซอร์ความเค้น** (stress tensor) [Pa]
- $\mathbf{g}$ = **ความเร่งเนื่องจากแรงโน้มถ่วง** (gravitational acceleration) [m/s²]
- $\nabla p$ = **การไล่ระดับความดัน** (pressure gradient)
- $\rho \mathbf{U} \mathbf{U}$ = **flux ของโมเมนตัม** (momentum flux)

สำหรับ **ของไหลนิวตัน (Newtonian fluid)**:

$$
\boldsymbol{\tau} = \mu \left[ \nabla \mathbf{U} + (\nabla \mathbf{U})^T \right] - \frac{2}{3}\mu (\nabla \cdot \mathbf{U})\mathbf{I}
$$

- $\mu$ (mu) = **ความหนืดพลศาสตร์** (dynamic viscosity) [Pa·s]
- $\mathbf{I}$ = **เทนเซอร์เอกลักษณ์** (identity tensor)

---

### 1.3 Conservation of Energy

**สมการการอนุรักษ์พลังงาน (Energy Equation)**

$$
\frac{\partial (\rho e)}{\partial t} + \nabla \cdot (\rho e \mathbf{U}) = -\nabla \cdot \mathbf{q} + \boldsymbol{\tau} : \nabla \mathbf{U} + S_h
$$

**คำอธิบายพจน์:**
- $e$ = **พลังงานภายในต่อหน่วยมวล** (specific internal energy) [J/kg]
- $\mathbf{q}$ = **flux ของความร้อน** (heat flux) [W/m²]
- $S_h$ = **แหล่งกำเนิดความร้อน** (heat source) [W/m³]
- $\boldsymbol{\tau} : \nabla \mathbf{U}$ = **การกระจายของพลังงานเนื่องจากความเค้น** (viscous dissipation)

ตาม **กฎของฟูริเยร์ (Fourier's Law)**:

$$
\mathbf{q} = -k \nabla T
$$

- $k$ = **สัมประสิทธิภาพการนำความร้อน** (thermal conductivity) [W/(m·K)]
- $T$ = **อุณหภูมิ** (temperature) [K]

---

### 1.4 Equation of State

**สมการสถานะ (Equation of State)**

สำหรับ **ก๊าซอุดมคติ (Ideal Gas)**:

$$
p = \rho R T
$$

- $R$ = **ค่าคงที่ของก๊าซเฉพาะ** (specific gas constant) [J/(kg·K)]

สำหรับ **ของไหลที่ไม่อัดตัว (Incompressible Fluid)**:

$$
\rho = \text{constant}
$$

---

### 1.5 Transport Equations (General Form)

**สมการขนส่งทั่วไป (General Transport Equation)**

$$
\frac{\partial (\rho \phi)}{\partial t} + \nabla \cdot (\rho \phi \mathbf{U}) = \nabla \cdot (\Gamma_\phi \nabla \phi) + S_\phi
$$

**คำอธิบายพจน์:**
- $\phi$ (phi) = **ปริมาณที่ถูกขนส่ง** (transported quantity) - สเกลาร์ใดๆ
- $\Gamma_\phi$ (Gamma) = **สัมประสิทธิภาพการแพร่** (diffusion coefficient)
- $S_\phi$ = **แหล่งกำเนิดหรือรับ** (source/sink term)

**องค์ประกอบของสมการ:**
1. **Unsteady term** $\frac{\partial (\rho \phi)}{\partial t}$ = การเปลี่ยนแปลงตามเวลา
2. **Convection term** $\nabla \cdot (\rho \phi \mathbf{U})$ = การพาความร้อน/โมเมนตัม
3. **Diffusion term** $\nabla \cdot (\Gamma_\phi \nabla \phi)$ = การแพร่
4. **Source term** $S_\phi$ = แหล่งกำเนิดหรือการรับ

## 2. OpenFOAM Class Hierarchy & Implementation

### 2.1 Core Field Classes

OpenFOAM represents governing equations through field classes that store and manipulate data at mesh points/cells.

```
GeometricField (Templated abstract base)
│
├── DimensionedField
│   └── Field (1D array of values)
│
├── volScalarField  (e.g., p, T, rho)
├── volVectorField  (e.g., U)
├── volSymmTensorField (e.g., tau)
│
└── surfaceScalarField (e.g., phi - flux)
```

**Key Files:**
- `$FOAM_SRC/OpenFOAM/fields/GeometricField/GeometricField.C`
- `$FOAM_SRC/OpenFOAM/fields/volFields/volFields.H`
- `$FOAM_SRC/OpenFOAM/fields/Fields/Field/Field.C`

---

The finite volume discretization relies on mesh and scheme classes.

```
polyMesh (Top-level mesh)
│
├── fvMesh (Finite volume mesh)
│   │
│   ├── fvSchemes (Discretization schemes)
│   │   ├── ddtSchemes  (Time derivatives)

│   │   └── laplacianSchemes (Diffusion schemes)
│   │
│   └── solution (Solver controls)
```

## 3. Code Walkthrough

### 3.1 UEqn.H

ไฟล์ `UEqn.H` ใช้สำหรับสร้างสมการโมเมนตัม (Momentum Equation) ใน OpenFOAM solver ต่างๆ เช่น `simpleFoam`, `pimpleFoam` โดยใช้ finite volume method

**โครงสร้างหลักของ UEqn.H:**

```cpp
// สร้างสมการโมเมนตัมแบบไม่รวมแรงดัน (momentum equation without pressure gradient)
tmp<fvVectorMatrix> UEqn
(
    fvm::ddt(U)                     // Unsteady term: ∂U/∂t
  + fvm::div(phi, U)                // Convection term: ∇·(UU)
  + turbulence->divDevReff(U)       // Turbulent diffusion: ∇·(τ_eff)
 ==
    fvOptions(U)                    // Source terms (optional)
);

// แก้ไข boundary conditions และ source terms
UEqn.relax();

// เพิ่มแรงดันเข้าไปในสมการ (pressure gradient term)
if (pimple.momentumPredictor())
{
    solve(UEqn == -fvc::grad(p));   // -∇p
}
```

**คำอธิบายพจน์สำคัญ:**

| Operator | ความหมาย | ประเภท |
|----------|----------|--------|
| `fvm::ddt(U)` | อนุพันธ์เวลาของความเร็ว | Implicit |
| `fvm::div(phi, U)` | Divergence ของ flux ความเร็ว | Implicit |
| `turbulence->divDevReff(U)` | Divergence ของเทนเซอร์ความเค้นที่มีผลจากความปั่น | Implicit |
| `fvc::grad(p)` | Gradient ของความดัน | Explicit |
| `fvOptions(U)` | Source terms จาก OpenFOAM framework | Explicit/Implicit |

**ความแตกต่างระหว่าง `fvm::` และ `fvc::`:**
- **`fvm::`** (finite volume method) = สร้างเมทริกซ์สำหรับการแก้ปัญหาแบบ **implicit** (ค่าที่ boundary และ internal field รวมอยู่ในเมทริกซ์)
- **`fvc::`** (finite volume calculus) = คำนวณค่าโดยตรงแบบ **explicit** (ใช้ค่าจาก time step ก่อนหน้า)

**การทำงานร่วมกับ Pressure-Velocity Coupling:**

สำหรับ incompressible flow สมการโมเมนตัมและความต่อเนื่องจะถูกแก้ไขพร้อมกันผ่านขั้นตอน:

1. **Predictor step:** แก้สมการโมเมนตัมโดยประมาณความดัน
2. **Pressure correction:** แก้สมการความดัน (Poisson equation) เพื่อให้ได้ velocity field ที่ satisfy continuity
3. **Corrector step:** แก้ไข velocity field ด้วยความดันใหม่

**ตัวอย่างจาก `simpleFoam`:**

```cpp
// Momentum predictor
tmp<fvVectorMatrix> UEqn
(
    fvm::div(phi, U)
  + turbulence->divDevReff(U)
 ==
    fvOptions(U)
);

UEqn.relax();

fvOptions.constrain(UEqn);

solve(UEqn == -fvc::grad(p));

fvOptions.correct(U);
```

### 3.2 pEqn.H

ไฟล์ `pEqn.H` ใช้สำหรับสร้างและแก้สมการความดัน (Pressure Equation) ซึ่งเป็นส่วนสำคัญของ pressure-velocity coupling ใน OpenFOAM solvers เช่น `simpleFoam`, `pimpleFoam`

**โครงสร้างหลักของ pEqn.H (สำหรับ incompressible flow):**

```cpp
// สร้างสมการความดันจากสมการโมเมนตัมและความต่อเนื่อง
volScalarField rAU(1.0/UEqn.A());         // Reciprocal of diagonal coefficients
volVectorField HbyA(constrainHbyA(rAU*UEqn.H())); // Explicit flux term
surfaceScalarField phiHbyA
(
    "phiHbyA",
    fvc::flux(HbyA)
  + fvc::interpolate(rAU)*fvc::ddtCorr(U, phi)
);

// ปรับ pressure boundary conditions
adjustPhi(phiHbyA, U, p);

// สร้างสมการ Poisson สำหรับความดัน
while (simple.correctNonOrthogonal())
{
    fvScalarMatrix pEqn
    (
        fvm::laplacian(rAU, p) == fvc::div(phiHbyA)
    );

    pEqn.setReference(pRefCell, pRefValue);
    pEqn.solve();

    if (simple.finalNonOrthogonalIter())
    {
        phi = phiHbyA - pEqn.flux();
    }
}

// แก้ไข velocity field ด้วยความดันใหม่
U = HbyA - rAU*fvc::grad(p);
U.correctBoundaryConditions();
```

**คำอธิบายพจน์สำคัญ:**

| Operator/Variable | ความหมาย |
|-------------------|----------|
| `UEqn.A()` | ส่วนประกอบ diagonal ของเมทริกซ์โมเมนตัม |
| `UEqn.H()` | ส่วนประกอบ off-diagonal (source term) |
| `rAU` | ค่าผกผันของสัมประสิทธิ์เมทริกซ์ (1/A) |
| `phiHbyA` | Flux ที่คำนวณจาก velocity โดยประมาณ |
| `fvm::laplacian(rAU, p)` | Diffusion term ของสมการความดัน |
| `fvc::div(phiHbyA)` | Divergence ของ flux (source term) |
| `pEqn.solve()` | แก้ระบบสมการเชิงเส้นสำหรับความดัน |
| `pEqn.flux()` | คำนวณ pressure flux correction |

**ขั้นตอนการแก้สมการความดัน (Pressure Correction):**

1. **Construct HbyA:** คำนวณ velocity field โดยประมาณจากสมการโมเมนตัม
2. **Build pressure equation:** สร้างสมการ Poisson สำหรับความดันจาก continuity equation
3. **Solve pressure:** แก้สมการความดันโดยใช้ linear solver
4. **Correct flux:** ปรับ mass flux ให้สอดคล้องกับความดันใหม่
5. **Correct velocity:** แก้ไข velocity field ด้วย gradient ของความดัน

**ตัวอย่างจาก `simpleFoam` (steady-state incompressible):**

```cpp
// Pressure equation
volScalarField rAU(1.0/UEqn.A());
volVectorField HbyA(constrainHbyA(rAU*UEqn.H()));
surfaceScalarField phiHbyA("phiHbyA", fvc::flux(HbyA));

adjustPhi(phiHbyA, U, p);

// Non-orthogonal correction loop
while (simple.correctNonOrthogonal())
{
    fvScalarMatrix pEqn
    (
        fvm::laplacian(rAU, p) == fvc::div(phiHbyA)
    );

    pEqn.setReference(pRefCell, pRefValue);
    pEqn.solve();

    if (simple.finalNonOrthogonalIter())
    {
        phi = phiHbyA - pEqn.flux();
    }
}

// Velocity correction
U = HbyA - rAU*fvc::grad(p);
U.correctBoundaryConditions();
```

### 3.3 createFields.H

ไฟล์ `createFields.H` ใช้สำหรับสร้างและเริ่มต้นฟิลด์ต่างๆ (fields) ที่จำเป็นสำหรับการแก้สมการใน OpenFOAM solver โดยทำหน้าที่อ่านค่าจากไฟล์คอนฟิกูเรชันและสร้าง objects ต่างๆ ขึ้นมาในหน่วยความจำ

**โครงสร้างหลักของ createFields.H:**

```cpp
// สร้าง mesh object จากไฟล์ case directory
fvMesh mesh
(
    Foam::IOobject
    (
        fvMesh::defaultRegion,
        runTime.timeName(),
        runTime,
        IOobject::MUST_READ
    )
);

// สร้าง pressure field และอ่านค่าเริ่มต้นจาก 0/ หรือ 0.org/
volScalarField p
(
    IOobject
    (
        "p",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้าง velocity field
volVectorField U
(
    IOobject
    (
        "U",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);

// สร้าง mass flux field (surface scalar field)
surfaceScalarField phi
(
    IOobject
    (
        "phi",
        runTime.timeName(),
        mesh,
        IOobject::READ_IF_PRESENT,
        IOobject::AUTO_WRITE
    ),
    fvc::flux(U)
);

// สร้าง transport properties object (ความหนืด)
singlePhaseTransportModel laminarTransport(U, phi);

// สร้าง turbulence model
autoPtr<incompressible::turbulenceModel> turbulence
(
    incompressible::turbulenceModel::New(U, phi, laminarTransport)
);
```

**คำอธิบายพจน์สำคัญ:**

| Class/Object | ความหมาย |
|--------------|----------|
| `fvMesh` | Finite volume mesh - โครงสร้างเมชสำหรับ FVM |
| `volScalarField` | ฟิลด์สเกลาร์ที่จุดศูนย์กลางเซลล์ (เช่น ความดัน) |
| `volVectorField` | ฟิลด์เวกเตอร์ที่จุดศูนย์กลางเซลล์ (เช่น ความเร็ว) |
| `surfaceScalarField` | ฟิลด์สเกลาร์ที่ผิวเซลล์ (เช่น mass flux) |
| `IOobject` | คลาสสำหรับจัดการการอ่าน/เขียนไฟล์ |
| `MUST_READ` | ต้องอ่านค่าจากไฟล์ (error ถ้าไม่มี) |
| `AUTO_WRITE` | เขียนค่าลงไฟล์อัตโนมัติเมื่อสิ้นสุด time step |
| `fvc::flux(U)` | คำนวณ mass flux จาก velocity field |
| `singlePhaseTransportModel` | โมเดลความหนืดสำหรับ single-phase flow |
| `turbulenceModel` | โมเดลความปั่น (k-ε, k-ω SST, etc.) |

**การทำงานของ IOobject flags:**

- **`MUST_READ`**: บังคับให้อ่านค่าจากไฟล์ใน time directory (เช่น `0/`, `0.01/`) ถ้าไม่พบไฟล์จะเกิด error
- **`READ_IF_PRESENT`**: อ่านค่าถ้าไฟล์มีอยู่ ถ้าไม่มีจะใช้ค่าเริ่มต้นหรือสร้างฟิลด์ใหม่
- **`AUTO_WRITE`**: เขียนค่าฟิลด์ลงไฟล์อัตโนมัติเมื่อ solver เสร็จสิ้นแต่ละ time step
- **`NO_WRITE`**: ไม่เขียนค่าฟิลด์ (ใช้สำหรับ temporary fields)

**ตัวอย่างการอ่านค่าจากไฟล์คอนฟิกูเรชัน:**

```cpp
// อ่านค่าความหนืดจาก transportProperties
Info<< "Reading transportProperties\n" << endl;
IOdictionary transportProperties
(
    IOobject
    (
        "transportProperties",
        runTime.constant(),
        mesh,
        IOobject::MUST_READ_IF_MODIFIED,
        IOobject::NO_WRITE
    )
);

dimensionedScalar nu
(
    "nu",
    dimViscosity,
    transportProperties
);
```

**ขั้นตอนการสร้างฟิลด์:**

1. **สร้าง mesh**: อ่านเมชจาก `constant/polyMesh/`
2. **สร้าง pressure field**: อ่านค่าเริ่มต้นจาก `0/p`
3. **สร้าง velocity field**: อ่านค่าเริ่มต้นจาก `0/U`
4. **สร้าง flux field**: คำนวณจาก velocity field
5. **สร้าง physical models**: อ่านค่าความหนืดและ turbulence model
