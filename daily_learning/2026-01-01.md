## Part 1: Governing Equations - Theory & Implementation

### 1.1 Continuity Equation (Mass Conservation)

**Mathematical Form:**
```
∂ρ/∂t + ∇·(ρU) = 0
```

สำหรับ incompressible flow (ρ = constant):
```
∇·U = 0
```

**OpenFOAM Implementation - Source Code Analysis:**

**Location:** `$FOAM_SRC/finiteVolume/cfdTools/general/adjustPhi/adjustPhi.C`

```cpp
// Continuity ถูก enforce ผ่าน pressure equation

**Pressure Correction Equation (PISO):**
```cpp
// จาก: $FOAM_SRC/finiteVolume/fvMatrices/fvScalarMatrix.C
// ∇·(1/A ∇p) = ∇·(U*/dt)  หรือ  ∇²p = ∇·U*/dt

// Discrete form:
// Σ_f (S_f/A_P) · (∇p)_f = Σ_f (S_f · U*_f)/dt

// โดยที่:
// - S_f = face area vector
// - A_P = central coefficient จาก momentum matrix
// - U* = predicted velocity จาก momentum equation
```

### 1.2 Momentum Equation (Navier-Stokes)

**Mathematical Form:**
```
∂(ρU)/∂t + ∇·(ρUU) = -∇p + ∇·(μ∇U) + ρg
```

**OpenFOAM Implementation - Source Code Analysis:**

**Location:** `$FOAM_APP/solvers/incompressible/simpleFoam/UEqn.H`

```cpp
// SIMPLE algorithm - momentum predictor
tmp<fvVectorMatrix> tUEqn
(
    // 1. Unsteady term: ∂U/∂t
    // Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmDdt/fvmDdt.C
    // สร้าง diagonal coefficient: V_cell/dt
    fvm::ddt(U)              

    // 2. Convection term: ∇·(UU)
    // Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmDiv/fvmDiv.C
    // ใช้ Gauss theorem: ∫_V ∇·(UU) dV = ∮_S (UU)·dS
    // Discretization: Σ_f (U_f · S_f) U_f
    // U_f ถูก interpolate จาก cell centers ถึง face centers
    + fvm::div(phi, U)         

    // 3. Diffusion term: ∇·(ν∇U)
    // Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmLaplacian/fvmLaplacian.C
    // ใช้ Gauss theorem: ∫_V ∇·(ν∇U) dV = ∮_S ν(∇U)·dS
    // Discretization: Σ_f ν_f (∇U)_f · S_f
    // สร้าง coefficients: diagonal (A_P), upper, lower
    + fvm::laplacian(nu, U)    

    // 4. Source terms (optional)
    // Source: $FOAM_SRC/fvOptions/interpolation/interpolationCellPoint.C
    ==                          
    fvOptions(U)             
);

// 5. Relaxation (สำคัญสำหรับ convergence!)
// Source: $FOAM_SRC/finiteVolume/fvMatrices/fvMatrix.C
UEqn.relax();

// 6. Solve momentum (optional predictor)
// ใน SIMPLE: ไม่ solve UEqn โดยตรง แต่ใช้ H/A
// ใน PISO: solve momentum predictor ก่อน
// solve(UEqn == -fvc::grad(p));
```

**Convection Term Discretization Detail:**

```cpp
// Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmDiv/fvmDiv.C
// การ discretize div(phi, U):

// 1. Interpolate U จาก cell center ถึง face center
//    U_f = interpolation(U_P, U_N)
//    schemes: upwind, linear, linearUpwind, QUICK, etc.

// 2. คำนวณ flux ผ่าน face
//    F_f = phi_f · U_f
//    โดยที่ phi_f = U_f · S_f (face area vector)

// 3. Assemble ลง matrix
//    diag_coeff += Σ_outgoing_faces F_f
//    source     += Σ_incoming_faces F_f * U_N

// Upwind scheme example:
if (phi_f > 0)  // flow from P to N
    U_f = U_P;   // use upstream value
else
    U_f = U_N;
```

**Diffusion Term Discretization Detail:**

```cpp
// Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmLaplacian/fvmLaplacian.C
// การ discretize laplacian(nu, U):

// 1. คำนวณ gradient ที่ face center
//    (∇U)_f = (U_N - U_P) / d_PN  (linear interpolation)
//    โดยที่ d_PN = distance between cell centers

// 2. คำนวณ non-orthogonal correction (ถ้า mesh ไม่ orthogonal)
//    correction = (∇U)_f · (S_f - d_PN * e_PN)
//    โดยที่ e_PN = unit vector from P to N

// 3. Assemble ลง matrix
//    diag_coeff += Σ_f nu_f * |S_f| / d_PN
//    upper       -= nu_f * |S_f| / d_PN
//    lower       -= nu_f * |S_f| / d_PN
```

## Part 2: GeometricField Class Hierarchy - DEEP DIVE

### 2.1 Class Hierarchy Overview

```
GeometricField<Type, PatchField, GeoMesh>
├── DimensionedField<Type, GeoMesh>      // Physical dimensions + mesh
│   ├── Field<Type>                      // Raw data container
│   │   └── List<Type>                   // 1D array (internal storage)
│   └── dimensionSet                     // [mass length time temperature]
├── PtrList<PatchField<Type>>            // Boundary conditions
│   ├── [0] fixedValueFvPatchField       // Patch 0 BC
│   ├── [1] zeroGradientFvPatchField     // Patch 1 BC
│   └── ...
└── GeometricBoundaryField               // Boundary field management
```

### 2.3 Source Code Analysis - GeometricField.H

**Location:** `$FOAM_SRC/OpenFOAM/fields/GeometricField/GeometricField.H`

```cpp
template<class Type, template<class> class PatchField, class GeoMesh>
class GeometricField
:
    public DimensionedField<Type, GeoMesh>,
    public List<Field<Type>, GeoMesh>
{
    // ========== INHERITED MEMBERS ==========
    // จาก DimensionedField:
    //   - dimensionSet dimensions_;        // Physical units
    //   - GeoMesh& mesh_;                  // Mesh reference
    //   - word name_;                      // Field name
    //   - IOobject io_;                    // I/O handling
    
    // จาก List<Field<Type>>:
    //   - Field<Type> internalField_;      // Cell center values
    //   - จริงๆ คือ List<Type> แต่ wrapper ด้วย Field
    
    // ========== OWN MEMBERS ==========
protected:
    // Boundary field - pointer list ของ patch fields
    // PtrList คือ smart pointer array ของ OpenFOAM
    PtrList<PatchField<Type>> boundaryField_;
    
    // Boundary field manager
    GeometricBoundaryField boundary_;


public:
    // ========== CONSTRUCTORS ==========
    
    // Constructor 1: Read from file (MUST_READ)
    GeometricField
    (
        const IOobject&,                    // path, case, read/write options
        const GeoMesh&                      // mesh reference
    );
    
    // Constructor 2: Create new field (internalField + boundaryField)
    GeometricField
    (
        const IOobject&,
        const GeoMesh&,
        const dimensionSet&,                // physical dimensions
        const Field<Type>&                  // initial internal values
    );
    
    // Constructor 3: Create with calculated boundary
    GeometricField
    (
        const IOobject&,
        const GeoMesh&,
        const dimensioned<Type&,            // value + dimensions
        const bool calcBoundary = true      // calculate BC?
    );
    
    
    // ========== ACCESS METHODS ==========
    
    // Internal field access
    const Field<Type>& internalField() const
    {
        return *this;  // Returns List<Field>>::operator[]
    }
    
    Field<Type>& internalField()
    {
        return *this;
    }
    
    // Boundary field access
    const GeometricBoundaryField& boundaryField() const
    {
        return boundary_;
    }
    
    // ========== OPERATORS ==========
    
    // Field operations (element-wise)
    void operator=(const GeometricField<Type, PatchField, GeoMesh>&);
    void operator+=(const GeometricField<Type, PatchField, GeoMesh>&);
    void operator-=(const GeometricField<Type, PatchField, GeoMesh>&);
    
    // Scalar operations
    void operator=(const Type&);
    void operator*=(const scalar);
    void operator/=(const scalar);
    
    
    // ========== BOUNDARY CONDITION UPDATE ==========
    
    // Update boundary field coefficients
    // ถูกเรียกก่อน solving matrix equation
    void updateCoeffs()
    {
        // Loop ทุก patch และ update BC
        forAll(boundaryField_, patchi)
        {
            boundaryField_[patchi].updateCoeffs();
        }
    }
    
    
    // ========== GEOMETRIC OPERATIONS ==========
    
    // Gradient: ∇p
    // Source: $FOAM_SRC/finiteVolume/fvc/fvcGrad.C
    tmp<GeometricField<typename outerProduct<vector, Type>::type, 
                      fvPatchField, volMesh>> 
    grad() const;
    
    // Divergence: ∇·U
    // Source: $FOAM_SRC/finiteVolume/fvc/fvcDiv.C
    tmp<GeometricField<typename innerProduct<vector, Type>::type, 
                      fvPatchField, volMesh>> 
    div() const;
    
    // Laplacian: ∇²p
    // Source: $FOAM_SRC/finiteVolume/fvc/fvcLaplacian.C
    tmp<fvScalarMatrix> laplacian(const dimensionedScalar&) const;
};
```

**Memory Allocation Process:**

```cpp
// Source: $FOAM_SRC/OpenFOAM/fields/GeometricField/GeometricField.C

// Constructor: Create from IOobject
template<class Type, template<class> class PatchField, class GeoMesh>
GeometricField<Type, PatchField, GeoMesh>::GeometricField
(
    const IOobject& io,
    const GeoMesh& mesh
)
:
    // 1. Initialize base class (DimensionedField)
    DimensionedField<Type, GeoMesh>(io, mesh, dimless),
    
    // 2. Initialize internal field (List<Field<>>)
    List<Field<Type>, GeoMesh>(mesh.size()),  // Allocate nCells elements
    
    // 3. Initialize boundary field
    boundaryField_(mesh.boundary().size())    // Allocate nPatches pointers
{
    // 4. Read internal field from file
    // File format: (cell0Value cell1Value ... cellNValue)
    // อยู่ใน: 0/p หรือ time directories
    this->readField(io.name(), "value");
    
    // 5. Create boundary fields
    // อ่าน BC types จาก: 0/p boundary field
    // แล้ว instantiate appropriate PatchField classes
    forAll(boundaryField_, patchi)
    {
        boundaryField_.set
        (
            patchi,
            PatchField<Type>::New
            (
                mesh.boundary()[patchi],    // fvPatch
                *this                        // parent field
            )
        );
    }
    
    // 6. Read boundary values from file
    // File format: 
    //   boundaryField
    //   {
    //       inlet
    //       {
    //           type            fixedValue;
    //           value           uniform 100000;
    //       }
    //       outlet
    //       {
    //           type            zeroGradient;
    //       }
    //   }
    this->readBoundaryField(io.name());
}

// Memory allocation detail:
// 1. internalField_ ใช้ List::allocate() → new Type[nCells]
// 2. boundaryField_ ใช้ PtrList::set() → new PatchField<Type>[nPatchFaces]
// 3. ทุกอย่างเป็น contiguous memory สำหรับ performance
```

**Memory Access Pattern:**

```cpp
// GOOD: Sequential access (cache-friendly)
const scalarField& p = p.internalField();
forAll(p, i)
{
    scalar val = p[i];  // Sequential memory access
    // Do something...
}

// BAD: Random access (cache misses)
for (label i = 0; i < nCells; i += 10)  // Stride 10
{
    scalar val = p[i];
}

// OpenFOAM iterator (cache-friendly too)
forAllConstIter(typename GeometricField<Type, PatchField, GeoMesh>, p, iter)
{
    Type val = *iter;  // Sequential iteration
}
```

### 3.3 Dictionary Parsing Workflow

```cpp
// LINE 32-38: Read viscosity
dimensionedScalar nu
(
    transportProperties.lookup("nu")
);

// ========== DICTIONARY PARSING WORKFLOW ==========
// Source: $FOAM_SRC/OpenFOAM/db/dictionary/dictionary.C

// 1. File format (constant/transportProperties):
/*-------------------------------------------
FoamFile
{
    version     2.0;
    format      ascii;
    class       dictionary;
    location    "constant";
    object      transportProperties;
}
// * * * * * * * * * //

transportModel  Newtonian;

nu              [0 2 -1 0 0 0 0]  1.5e-05;
-------------------------------------------*/

// 2. Parsing process:
//
// Step 1: Read file into memory
//   → ITstream สร้าง token list จาก file content
//   → Tokens: [transportModel, Newtonian, ;, nu, [, 0, 2, -1, ..., ], 1.5e-05, ;]
//
// Step 2: Build dictionary (hash table)
//   → dictionary::add(entry*)
//   → IDLList<entry> ใช้ double-linked list เก็บ entries
//   → HashTable<entry*, word> ใช้ hash table lookup
//
// Step 3: Lookup "nu"
//   → dictionary::lookup("nu")
//   → HashTable lookup: hash("nu") → index → entry*
//   → Return ITstream pointing to entry data
//
// Step 4: Parse dimensionedScalar
//   → dimensionedScalar constructor receives ITstream
//   → Read: [0 2 -1 0 0 0 0]  → dimensionSet
//   → Read: 1.5e-05           → scalar value
//   → Store: name_="nu", value_=1.5e-05, dimensions_=[0 2 -1 0 0 0 0]

// 3. Dictionary internal structure:
//
// class dictionary
// {
//     // Storage: double-linked list of entries

// ========== GAUSS THEOREM IMPLEMENTATION ==========
//
// "Gauss" หมายถึง ใช้ Gauss divergence theorem:
//
// ∫_V ∇·φ dV = ∮_S φ·dS
//
// Discrete form:
// Σ_cells ∇·φ |_P * V_P = Σ_faces φ_f · S_f
//
// ดังนั้น:
// ∇·φ |_P = (1/V_P) * Σ_faces φ_f · S_f
//
// โดยที่:
// - V_P = cell volume
// - S_f = face area vector
// - φ_f = face value (interpolated from cell centers)

// ========== INTERPOLATION SCHEMES ==========
//
// upwind (First-order):
//   φ_f = φ_P  if (U·S_f > 0)  // flow from P to N
//   φ_f = φ_N  if (U·S_f < 0)  // flow from N to P
//
//   Pros: Stable, bounded
//   Cons: Diffusive (first-order accurate)
//
// linear (Second-order central):
//   φ_f = f * φ_P + (1-f) * φ_N
//   where f = distance(N, face) / distance(N, P)
//
//   Pros: Second-order accurate
//   Cons: Unstable for high Reynolds number (oscillations)
//
// linearUpwind (Second-order upwind):
//   φ_f = φ_P + (∇φ)_P · (r_f - r_P)
//   where (∇φ)_P = limited gradient
//
//   Pros: Second-order, more stable than linear
//   Cons: More expensive (need gradient calculation)
//
// QUICK (Third-order):
//   φ_f = quadratic interpolation using P, N, and upwind cell
//
//   Pros: Third-order accurate
//   Cons: Expensive, can be unbounded

// ========== GRADIENT SCHEMES ==========
//
// linear (Gauss theorem):
//   ∇φ |_P = (1/V_P) * Σ_faces φ_f * S_f
//
//   where φ_f = (φ_P + φ_N) / 2  (linear interpolation)
//
// leastSquares:
//   Fit linear function to neighbor cell values
//   Minimize Σ (φ_i - (φ_P + ∇φ·r_i))²
//
//   Pros: More accurate on non-orthogonal meshes
//   Cons: More expensive
//
// fourth:
//   Fourth-order accurate (rarely used)
//   Requires wider stencil

//
// Gauss linear corrected:
//   ∇·(Γ∇φ) |_P = Σ_faces Γ_f * (∇φ)_f · S_f
//

//   "uncorrected": Ignore non-orthogonality (faster, less accurate)

## Part 5: SIMPLE Algorithm - Pseudocode and Implementation

### 5.1 SIMPLE Algorithm Overview

**SIMPLE** = **S**emi-**I**mplicit **M**ethod for **P**ressure-**L**inked **E**quations

**Purpose:** Couple pressure and velocity in incompressible flow

**Key Idea:**
1. Solve momentum equation with guessed pressure
2. Solve pressure correction equation
3. Correct velocity and pressure
4. Repeat until convergence

### 5.2 SIMPLE Algorithm Pseudocode

```cpp
// ========== SIMPLE ALGORITHM PSEUDOCODE ==========
// Source: $FOAM_APP/solvers/incompressible/simpleFoam/simpleFoam.C

// Initialize fields
volScalarField p(...);      // Pressure
volVectorField U(...);      // Velocity
surfaceScalarField phi(...); // Flux

// Main time loop
while (runTime.loop())
{
    // ========== OUTER ITERATION LOOP ==========
    // For steady-state, iterate until convergence
    // For transient, usually 1 iteration per time step
    
    int outerCorr = 0;
    scalar maxResidual = 1.0;
    
    while (maxResidual > convergenceTolerance && outerCorr < maxOuterIter)
    {
        outerCorr++;
        
        // ========== STEP 1: MOMENTUM PREDICTOR ==========
        // Solve momentum equation with current pressure
        // A(U) * U = H(U) - ∇p
        
        tmp<fvVectorMatrix> tUEqn
        (
            fvm::ddt(U)              // Unsteady (if transient)
          + fvm::div(phi, U)         // Convection
          + fvm::laplacian(nu, U)    // Diffusion
          ==
            fvOptions(U)             // Source terms
        );
        
        // Relax momentum equation (CRITICAL for stability!)
        // UEqn.relax() means:
        //   A_P *= 1/alpha
        //   source += (1-alpha)/alpha * A_P * U_old
        UEqn.relax();
        
        // Calculate H/A and HbyA
        // H = source - off-diagonal contributions
        // A = diagonal coefficients
        // HbyA = H / A
        volVectorField HbyA(constrainHbyA(tUEqn.H(), U, p));
        
        // Surface field of H/A
        surfaceScalarField phiHbyA
        (
            "phiHbyA",
            fvc::flux(HbyA)
          + fvc::interpolate(rAU)*fvc::ddtCorr(U, phi)
        );
        
        // Adjust flux for mass conservation
        // Source: $FOAM_SRC/finiteVolume/cfdTools/general/adjustPhi/adjustPhi.C
        adjustPhi(phiHbyA, U, p);
        
        
        // ========== STEP 2: PRESSURE CORRECTION ==========
        // Solve pressure equation: ∇·(rAU ∇p) = ∇·(phiHbyA)
        // Discrete form: Σ_f (S_f/A_P) · (∇p)_f = Σ_f (phiHbyA)_f
        
        // Non-orthogonal correction loop
        for (int nonOrth = 0; nonOrth <= nNonOrthogonalCorrectors; nonOrth++)
        {
            // Create pressure equation
            // laplacian(rAU, p) == div(phiHbyA)
            fvScalarMatrix pEqn
            (
                fvm::laplacian(rAU, p)
              == fvc::div(phiHbyA)
            );
            
            // Set reference pressure (fix singularity)
            pEqn.setReference(pRefCell, pRefValue);
            
            // Solve pressure equation
            pEqn.solve();
            
            // Correct flux (only after final non-orthogonal iteration)
            if (nonOrth == nNonOrthogonalCorrectors)
            {
                phi = phiHbyA - pEqn.flux();
            }
        }
        
        
        // ========== STEP 3: VELOCITY CORRECTION ==========
        // U = HbyA - rAU * ∇p
        
        // Explicit gradient (using fvc, not fvm)
        U = HbyA - rAU*fvc::grad(p);
        
        // Correct boundary conditions
        U.correctBoundaryConditions();
        
        
        // ========== STEP 4: PRESSURE RELAXATION ==========
        // p = alpha * p_new + (1-alpha) * p_old
        p.relax();
        
        
        // ========== STEP 5: CHECK CONVERGENCE ==========
        // Calculate residuals
        scalar pRes = pEqn.initialResidual();
        scalar URes = UEqn.initialResidual();
        maxResidual = max(pRes, URes);
        
        Info<< "Outer iteration " << outerCorr
            << ": pRes = " << pRes
            << ", URes = " << URes << endl;
    }
    
    
    // ========== STEP 6: UPDATE TURBULENCE ==========
    // Solve turbulence equations (k, epsilon, omega, etc.)
    turbulence->correct();
    
    
    // ========== STEP 7: WRITE OUTPUT ==========
    runTime.write();
}

### 5.3 Relaxation Factors - Why They Matter

```cpp
// ========== RELAXATION MECHANISM ==========
// Source: $FOAM_SRC/finiteVolume/fvMatrices/fvMatrix.C

// Under-relaxation formula:
// φ_new = α * φ_calculated + (1-α) * φ_old
//
// where:
//   α = relaxation factor (0 < α ≤ 1)
//   α = 1.0: no relaxation (use calculated value directly)
//   α = 0.3: strong relaxation (70% old, 30% new)
//
// Why needed?
//   - SIMPLE algorithm assumes pressure is "correct" when solving momentum
//   - This is NOT true in early iterations
//   - Without relaxation, solution oscillates and diverges
//   - Relaxation damps the oscillations

// Implementation in matrix form:
// Original equation: A_P * φ_P + Σ_N A_N * φ_N = b_P
//
// After relaxation:
//   (A_P/α) * φ_P + Σ_N A_N * φ_N = b_P + (1-α)/α * A_P * φ_P_old
//
// Source code:
template<class Type>
void fvMatrix<Type>::relax(const scalar alpha)
{
    // Scale diagonal
    diag() /= alpha;
    
    // Add source term from old value
    source() += (1.0 - alpha)/alpha * diag() * psi().internalField();
}

// Typical relaxation factors:
//   p:    0.2 - 0.3  (pressure needs strong relaxation)
//   U:    0.5 - 0.7  (velocity moderate)
//   k, ε: 0.5 - 0.8  (turbulence moderate)
```

### Q1: อธิบายความแตกต่างระหว่าง fvc และ fvm พร้อมตัวอย่าง code

**Answer:**
```cpp
// fvc (finite volume calculus): Explicit operators
// - คำนวณค่าจาก field ที่มีอยู่ (known values)
// - ไม่สร้าง matrix coefficients
// - ใช้สำหรับ terms ที่ไม่ต้องการ implicit solving
// - Return: tmp<GeometricField<...>>

// Examples:
volScalarField gradP = fvc::grad(p);              // ∇p
volScalarField divU = fvc::div(phi, U);           // ∇·(U)
surfaceScalarField flux = fvc::flux(U);           // U·S_f
volScalarField laplacianP = fvc::laplacian(nu, p); // ∇·(ν∇p)

// Source: $FOAM_SRC/finiteVolume/fvc/fvcGrad.C
// Implementation:
// 1. Interpolate values to face centers
// 2. Apply Gauss theorem: ∫_V ∇φ dV = ∮_S φ dS
// 3. Return field at cell centers


// fvm (finite volume method): Implicit operators
// - สร้าง matrix coefficients สำหรับ solving
// - Return: tmp<fvMatrix<...>>
// - ใช้สำหรับ terms ที่ต้องการ implicit solving

// Examples:
tmp<fvVectorMatrix> tUEqn
(
    fvm::ddt(U)              // ∂U/∂t → diagonal coefficient
  + fvm::div(phi, U)         // ∇·(UU) → upper + lower coefficients
  + fvm::laplacian(nu, U)    // ∇·(ν∇U) → upper + lower coefficients
);

// Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmDiv/fvmDiv.C
// Implementation:
// 1. Discretize term using finite volume method
// 2. Add coefficients to matrix:
//    - diag[owner]   += outgoing flux
//    - upper[face]   -= flux coefficient
//    - lower[face]   -= flux coefficient
// 3. Return matrix for solving

// KEY DIFFERENCE:
// fvc::div(phi, U)  → returns volVectorField (explicit)
// fvm::div(phi, U)  → returns fvVectorMatrix (implicit)

### Q3: ขั้นตอน PISO algorithm ทำอย่างไร

**Answer:**
1. Solve momentum predictor: `solve(UEqn == -grad(p))`
2. Loop `nCorrectors` times:
   a. Calculate `HbyA = UEqn.H() / UEqn.A()`
   b. Solve pressure: `laplacian(rAU, p) == div(phiHbyA)`
   c. Correct flux: `phi -= pEqn.flux()`
   d. Correct velocity: `U = HbyA - rAU * grad(p)`
3. Update turbulence model

### Q4: แตกต่างระหว่าง volScalarField และ surfaceScalarField

**Answer:**
- `volScalarField`: Values at cell centers (GeometricField<scalar, fvPatchField, volMesh>)
  - ใช้สำหรับ pressure, temperature, etc.
  - จัดเก็บที่ cell centers
  
- `surfaceScalarField`: Values at face centers (GeometricField<scalar, fvsPatchField, surfaceMesh>)
  - ใช้สำหรับ flux (phi = U·Sf)
  - จัดเก็บที่ face centers
  - จำเป็นสำหรับ finite volume method

### Q5: อธิบายการทำงานของ fvm::ddt(U)

**Answer:**
```cpp
// Euler scheme (first-order)
fvm::ddt(U) = (U^n - U^(n-1)) / dt

// backward scheme (second-order)
fvm::ddt(U) = (1.5*U^n - 2*U^(n-1) + 0.5*U^(n-2)) / dt

// Source: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvmDdt/fvmDdt.C
// สร้าง diagonal coefficients ใน matrix:
// diag_coeff = V_cell / dt
// source_term = V_cell * U^(n-1) / dt
```
