## 1. Theory: Core Equations & Physics

### 1.1 Conservation of Mass (Continuity Equation)

**Vector Form:**
$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**Term-by-Term Explanation (Thai):**
- $\frac{\partial \rho}{\partial t}$: อัตราการเปลี่ยนแปลงของความหนาแน่นของไหลเทียบกับเวลา (Unsteady term - เอกภาพเวลา)
- $\nabla \cdot (\rho \mathbf{U})$: การไหลเข้า-ออกของมวล (Mass flux divergence - การแพร่กระจายของการไหลของมวล)
- $= 0$: กฎการอนุรักษ์มวล (Mass conservation - มวลไม่ถูกสร้างหรือทำลาย)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์มวล (Conservation of Mass)** ซึ่งระบุว่ามวลของไหลไม่สามารถถูกสร้างขึ้นหรือสูญหายไปได้ ในปริมาตรควบคุมใดๆ อัตราการเปลี่ยนแปลงของมวลต้องสมดุลกับการไหลเข้าและการไหลออก

---

### 1.2 Conservation of Momentum (Navier-Stokes Equation)

**Vector Form:**
$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot \boldsymbol{\tau} + \rho \mathbf{g}
$$

**Term-by-Term Explanation (Thai):**
- $\frac{\partial (\rho \mathbf{U})}{\partial t}$: อัตราการเปลี่ยนแปลงของโมเมนตัมเทียบกับเวลา (Unsteady term - เอกภาพเวลาของโมเมนตัม)
- $\nabla \cdot (\rho \mathbf{U} \mathbf{U})$: การพาแบบ Convective ของโมเมนตัม (Convective term - การพาโมเมนตัม)
- $-\nabla p$: แรงดันความดัน (Pressure gradient force - แรงที่เกิดจากความแตกต่างของความดัน)
- $\nabla \cdot \boldsymbol{\tau}$: แรงเฉือน (Viscous stress term - แรงเฉือนจากความหนืด)
- $\rho \mathbf{g}$: แรงโน้มถ่วง (Body force - แรงภายนอก เช่น แรงโน้มถ่วง)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์โมเมนตัม (Conservation of Momentum)** ซึ่งเป็นการประยุกต์ใช้กฎข้อที่สองของนิวตันสำหรับของไหล โดยระบุว่าการเปลี่ยนแปลงของโมเมนตัมในปริมาตรควบคุมเท่ากับผลรวมของแรงทั้งหมดที่กระทำต่อของไหลนั้น ได้แก่ แรงดัน แรงเฉือน และแรงภายนอก

---

### 1.3 Constitutive Relation (Newtonian Fluid)

**Stress Tensor for Newtonian Fluid:**
$$
\boldsymbol{\tau} = \mu \left[ \nabla \mathbf{U} + (\nabla \mathbf{U})^T \right] - \frac{2}{3} \mu (\nabla \cdot \mathbf{U}) \mathbf{I}
$$

**Term-by-Term Explanation (Thai):**
- $\mu$: ความหนืดของไหล (Dynamic viscosity - ความหนืดพลวัต)
- $\nabla \mathbf{U}$: เทนเซอร์อัตราการเฉือน (Velocity gradient tensor - การไล่ระดับความเร็ว)
- $(\nabla \mathbf{U})^T$: เทนเซอร์ทรานสโพส (Transpose of velocity gradient)
- $\mathbf{I}$: เทนเซอร์เอกลักษณ์ (Identity tensor)

**Physical Meaning:**
สมการนี้เชื่อมโยงความสัมพันธ์ระหว่าง **ความเครียด (Stress)** และ **อัตราการเฉือน (Strain rate)** สำหรับของไหลแบบนิวตัน (Newtonian Fluid) ซึ่งแรงเฉือนแปรผันโดยตรงกับอัตราการเฉือน

---

### 1.4 Energy Equation (Optional for Incompressible Flow)

**Vector Form:**
$$
\frac{\partial (\rho h)}{\partial t} + \nabla \cdot (\rho \mathbf{U} h) = \frac{Dp}{Dt} + \nabla \cdot (k \nabla T) + \boldsymbol{\tau} : \nabla \mathbf{U} + S_h
$$

**Term-by-Term Explanation (Thai):**
- $h$: เอนทาลปี (Enthalpy - พลังงานความร้อน)
- $k$: ความนำความร้อน (Thermal conductivity - สัมประสิทธิ์การนำความร้อน)
- $T$: อุณหภูมิ (Temperature)
- $\boldsymbol{\tau} : \nabla \mathbf{U}$: การกระจายของพลังงานเนื่องจากความหนืด (Viscous dissipation)
- $S_h$: แหล่งกำเนิดความร้อน (Heat source)

**Physical Meaning:**
สมการนี้แสดงถึง **กฎการอนุรักษ์พลังงาน (Conservation of Energy)** ซึ่งระบุว่าการเปลี่ยนแปลงของพลังงานในระบบเท่ากับผลรวมของการถ่ายเทความร้อน งานที่เกิดจากแรงดัน และแหล่งกำเนิดพลังงานอื่นๆ

## 2. OpenFOAM Class Hierarchy & Implementation

### 2.1 Core Field Classes

OpenFOAM uses template-based field classes to represent physical quantities (scalar, vector, tensor) over the computational mesh.

```
GeometricField<Type, PatchField, GeoMesh>
│
├── fvFieldMatrix (used in finite volume solvers)
│   └── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
│
├── DimensionedField<Type, GeoMesh>
│   └── $FOAM_SRC/OpenFOAM/fields/DimensionedFields/
│
└── tmp<T> (smart pointer for memory management)
    └── $FOAM_SRC/OpenFOAM/memory/tmp/
```

**Key Template Instantiations:**
- `volScalarField` = GeometricField<scalar, fvPatchField, volMesh>
- `volVectorField` = GeometricField<vector, fvPatchField, volMesh>
- `surfaceScalarField` = GeometricField<scalar, fvsPatchField, surfaceMesh>

**Memory Layout:**
```
volScalarField p
│
├── [Internal Field]  (cell values)
│   └── List<scalar>  → [p0, p1, p2, ..., pN]
│       Size: N cells
│       Location: $FOAM_SRC/OpenFOAM/fields/GeometricFields/
│
└── [Boundary Field]  (patch values)
    └── PtrList<fvPatchScalarField>
        ├── patch0 (inlet)   → FixedValueFvPatchScalarField
        ├── patch1 (outlet)  → ZeroGradientFvPatchScalarField
        ├── patch2 (walls)   → NoSlipFvPatchField<vector>
        └── ...
        Size: Σ (patch face counts)
        Location: $FOAM_SRC/finiteVolume/fields/fvPatchFields/
```

---

### 2.2 Matrix Classes (Linear System Solvers)

OpenFOAM represents discretized equations as sparse linear systems using the matrix classes.

```
lduMatrix (Lower-Diagonal-Upper matrix)
│   └── $FOAM_SRC/OpenFOAM/matrices/lduMatrix/
│
└── fvMatrix<Type> : public lduMatrix
    └── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
```

**Inheritance Hierarchy:**
```
lduMatrix
│
├── lduAddressing (mesh connectivity)
│   ├── lowerAddr()  → owner cell indices
│   ├── upperAddr()  → neighbor cell indices
│   └── losortAddr() → sorting for symmetric matrices
│
└── fvMatrix<Type>
    ├── coefficients (source terms, boundary contributions)
    ├── psi (reference to solution field)
    └── dimensions (dimensional consistency checking)
```

**Memory Layout (Sparse Matrix Storage):**
```
fvMatrix<scalar> matrix
│
├── Diagonal
│   └── scalarField diag  → [D0, D1, D2, ..., DN]
│       Size: N cells
│
├── Lower (sub-diagonal)
│   └── scalarField lower → [L0, L1, L2, ..., LM]
│       Size: M faces (where M = number of internal faces)
│
├── Upper (super-diagonal)
│   └── scalarField upper → [U0, U1, U2, ..., UM]
│       Size: M faces
│
└── Source
    └── Field<Type> source → [S0, S1, S2, ..., SN]
        Size: N cells
```

**Source Files:**
- `lduMatrix`: `$FOAM_SRC/OpenFOAM/matrices/lduMatrix/lduMatrix.H`
- `fvMatrix`: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/fvMatrix/fvMatrix.H`

---

### 2.3 Mesh Classes

The mesh represents the computational domain and its connectivity.

```
polyMesh
│   ├── points_  (vertex coordinates)
│   ├── faces_   (face definitions)
│   ├── cells_   (cell-to-face addressing)
│   └── owner_ / neighbor_ (face-to-cell addressing)
│
└── fvMesh : public polyMesh
    ├── Sf (surface area vectors)
    ├── magSf (face magnitudes)
    └── C (cell centers)
```

**Memory Layout:**
```
fvMesh mesh
│
├── points_
│   └── pointField → List<point>
│       Size: N vertices
│       Data: [(x0,y0,z0), (x1,y1,z1), ...]
│
├── faces_
│   └── faceList → List<face>
│       Size: M faces
│       Data: [[v0, v1, v2], [v3, v4], ...]
│
├── owner_
│   └── labelList → [c0, c1, c2, ..., cM]
│       Size: M faces
│       Meaning: owner cell index for each face
│
└── neighbor_
    └── labelList → [n0, n1, n2, ..., nM]
        Size: M internal faces
        Meaning: neighbor cell index for internal faces
```

**Source Files:**
- `polyMesh`: `$FOAM_SRC/OpenFOAM/meshes/polyMesh/polyMesh.H`
- `fvMesh`: `$FOAM_SRC/finiteVolume/fvMesh/fvMesh.H`

---

### 2.4 Boundary Condition Classes

Boundary conditions are implemented using polymorphic patch field classes.

```
fvPatchField<Type>
│   ├── $FOAM_SRC/finiteVolume/fields/fvPatchFields/
│   │
│   ├── fixedValueFvPatchField
│   │   └── Dirichlet BC (value specified)
│   │
│   ├── zeroGradientFvPatchField
│   │   └── Neumann BC (∂φ/∂n = 0)
│   │
│   ├── fixedGradientFvPatchField
│   │   └── Neumann BC (gradient specified)
│   │
│   └── mixedFvPatchField
│       └── Robin BC (αφ + β∂φ/∂n = γ)
```

**Virtual Function Interface:**
```cpp
class fvPatchField<Type>
{
    // Update coefficients (called during matrix assembly)
    virtual void updateCoeffs() = 0;
    
    // Evaluate boundary value
    virtual tmp<Field<Type>> valueInternalCoeffs(
        const tmp<scalarField>&
    ) const = 0;
    
    // Gradient contribution
    virtual tmp<Field<Type>> gradientInternalCoeffs(
        const tmp<scalarField>&
    ) const = 0;
};
```

**Source Files:**
- Base class: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/fvPatchField/fvPatchField.H`
- Derived classes: `$FOAM_SRC/finiteVolume/fields/fvPatchFields/*/*FvPatchField.H`

### 2.5 Time Class

The `Time` class controls time-stepping and run-time selection.

```
Time
│   ├── timeValue_ (current simulation time)
│   ├── timeIndex_ (integer time step counter)
│   ├── deltaT_ (time step size)
│   └── outputTime_ (control for writing results)
│
└── $FOAM_SRC/OpenFOAM/db/Time/
```

**Source Files:**
- `$FOAM_SRC/OpenFOAM/db/Time/Time.H`
