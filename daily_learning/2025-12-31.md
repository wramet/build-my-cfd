## 1. ทฤษฎีพื้นฐาน (รวดเร็ว)

### 1.1 Continuity Equation (Incompressible)
$$
\nabla \cdot \vec{U} = 0
$$

ใน OpenFOAM: `fvc::div(phi)` หรือ `fvm::div(phi, U)`

### 1.2 Navier-Stokes Equation
$$
\frac{\partial \vec{U}}{\partial t} + \nabla \cdot (\vec{U}\vec{U}) = -\frac{1}{\rho}\nabla p + \nu \nabla^2 \vec{U} + \vec{g}
$$

ใน OpenFOAM (`UEqn.H`):
```cpp
fvVectorMatrix UEqn
(
    fvm::ddt(U)                 // ∂U/∂t
  + fvm::div(phi, U)            // ∇·(UU) [implicit]
  - fvm::laplacian(nu, U)       // -ν∇²U
 ==
  - fvc::grad(p)                // -∇p/ρ [explicit]
);
```

### 2.2 Common Typedefs

```cpp
// In volFieldsFwd.H
typedef GeometricField<scalar, fvPatchField, volMesh> volScalarField;
```

**IOobject Analysis:**
```cpp
class IOobject
{
    word name_;           // "p", "U", "T"
    fileName instance_;   // Time directory: "0", "0.1"
    objectRegistry& db_;  // Pointer to mesh (contains field registry)
    
    enum readOption
    {
        MUST_READ,        // Error if not found
        READ_IF_PRESENT,  // Optional
        NO_READ           // Don't read from disk
    };
    
    enum writeOption
    {
        AUTO_WRITE,       // Write at runTime.writeInterval
        NO_WRITE          // Never write
    };
};
```

**Tip:** ทุกครั้งที่สร้าง `GeometricField`, มันจะ register ตัวเองใน `objectRegistry` (accessed via `mesh.lookupObject<volScalarField>("p")`)

## 4. 📖 Dictionary Deep Dive

### 4.1 fvSchemes (Discretization Methods)

**File:** `system/fvSchemes`

```cpp
ddtSchemes
{
    default         Euler;  // ∂/∂t ~ (φ^{n+1} - φ^n) / Δt
}

gradSchemes
{
    default         Gauss linear;  // ∇φ using Gauss theorem + linear interpolation
    grad(p)         Gauss linear;
    grad(U)         cellLimited Gauss linear 1;  // Limited to prevent unboundedness
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwind grad(U);  // Upwind + linear (2nd order)
    div((nuEff*dev2(T(grad(U)))))  Gauss linear;  // Viscous stress term
}

laplacianSchemes
{
    default         Gauss linear corrected;  // ∇²φ with non-orthogonal correction
}
```

**Code Implementation (`fvSchemes.C`):**
```cpp
// When you call fvm::div(phi, U)...
const word& schemeName = mesh.divScheme("div(phi,U)");  // "Gauss linearUpwind grad(U)"

// Parse scheme
ITstream& schemeStream = mesh.divScheme("div(phi,U)");
word method = schemeStream;  // "Gauss"

if (method == "Gauss")
{
    // Read interpolation scheme
    tmp<surfaceInterpolationScheme<Type>> tinterp =
        surfaceInterpolationScheme<Type>::New(mesh, phi, schemeStream);
    
    return fvm::GaussDiv(phi, vf, tinterp);
}
```

**Goal:** เขียน utility `printMaxU` ที่อ่าน velocity field และ print ค่า max

```cpp
#include "fvCFD.H"

int main(int argc, char *argv[])
{
    #include "setRootCase.H"
    #include "createTime.H"
    #include "createMesh.H"

    // Read latest time
    instantList timeDirs = timeSelector::select0(runTime, args);
    runTime.setTime(timeDirs[timeDirs.size()-1], timeDirs.size()-1);

    // Read velocity field
    volVectorField U
    (
        IOobject
        (
            "U",
            runTime.timeName(),
            mesh,
            IOobject::MUST_READ,
            IOobject::NO_WRITE
        ),
        mesh
    );

    // Calculate magnitude
    volScalarField magU = mag(U);

    // Find max
    scalar maxVel = gMax(magU.internalField());
    label maxCellI = findMax(magU.internalField());

    Info<< "Max velocity: " << maxVel << " m/s" << nl
        << "At cell: " << maxCellI << nl
        << "Position: " << mesh.C()[maxCellI] << endl;

    return 0;
}
```

### Task 2: Custom Boundary Condition - Time-Varying Inlet

**Goal:** สร้าง BC ที่ค่า velocity เปลี่ยนตามเวลา `U(t) = U0 * sin(2πt/T)`

**File:** `src/finiteVolume/fields/fvPatchFields/derived/timeVaryingInlet/timeVaryingInletFvPatchVectorField.H`

```cpp
#ifndef timeVaryingInletFvPatchVectorField_H
#define timeVaryingInletFvPatchVectorField_H

#include "fixedValueFvPatchFields.H"

class timeVaryingInletFvPatchVectorField
:
    public fixedValueFvPatchVectorField
{
    // Private data
    vector U0_;         // Base velocity
    scalar period_;     // Oscillation period

public:
    TypeName("timeVaryingInlet");

    // Constructor
    timeVaryingInletFvPatchVectorField
    (
        const fvPatch&,
        const DimensionedField<vector, volMesh>&,
        const dictionary&
    );

    // Update coefficients
    virtual void updateCoeffs();
};

#endif
```

**Implementation (`timeVaryingInletFvPatchVectorField.C`):**

```cpp
#include "timeVaryingInletFvPatchVectorField.H"
#include "addToRunTimeSelectionTable.H"
#include "fvPatchFieldMapper.H"
#include "volFields.H"
#include "mathematicalConstants.H"

using namespace Foam::constant::mathematical;

// Constructor
timeVaryingInletFvPatchVectorField::timeVaryingInletFvPatchVectorField
(
    const fvPatch& p,
    const DimensionedField<vector, volMesh>& iF,
    const dictionary& dict
)
:
    fixedValueFvPatchVectorField(p, iF, dict),
    U0_(dict.lookup("U0")),
    period_(readScalar(dict.lookup("period")))
{
    if (dict.found("value"))
    {
        fvPatchField<vector>::operator=(Field<vector>("value", dict, p.size()));
    }
    else
    {
        fvPatchField<vector>::operator=(U0_);
    }
}

void timeVaryingInletFvPatchVectorField::updateCoeffs()
{
    if (updated())
    {
        return;
    }

    // Get current time
    scalar t = this->db().time().value();

    // Calculate velocity: U = U0 * sin(2πt/T)
    vector Ucurrent = U0_ * Foam::sin(2.0*pi*t/period_);

    // Set all face values
    fvPatchField<vector>::operator==(Ucurrent);

    fixedValueFvPatchVectorField::updateCoeffs();
}

// Register the patch type
makePatchTypeField
(
    fvPatchVectorField,
    timeVaryingInletFvPatchVectorField
);
```

**Usage in `0/U`:**

```cpp
inlet
{
    type            timeVaryingInlet;
    U0              (1 0 0);      // Base velocity [m/s]
    period          2.0;          // Period [s]
}
```

## 6. 🧠 Concept Check

**ข้อ 1:** ถ้า `volScalarField p` มี `nCells = 1000` cells และ `nPatches = 3` (ขนาด 50, 30, 20), หน่วยความจำที่ใช้คือเท่าไร?

<details>
<summary>เฉลย</summary>
`1000 + 50 + 30 + 20 = 1100 scalars` = `1100 * 8 bytes = 8.8 KB` (สำหรับ double precision)
</details>

**ข้อ 2:** ใน `fvm::div(phi, U)`, ทำไมต้องส่ง `phi` เข้าไป? นำไปใช้อย่างไร?

<details>
<summary>เฉลย</summary>
`phi` คือ **Face Flux** ($\vec{U}_f \cdot \vec{S}_f$) ซึ่งกำหนดทิศทางการไหล สำหรับ **Upwind Scheme** จะใช้ `phi` เพื่อตัดสินใจว่าจะเอาค่า $U$ จาก Cell ไหน (ถ้า `phi > 0` ใช้ owner, ถ้า `phi < 0` ใช้ neighbour)
</details>

**ข้อ 3:** ใน `createFields.H`, ทำไม `phi` ถึงเป็น `surfaceScalarField` ไม่ใช่ `volScalarField`?

<details>
<summary>เฉลย</summary>
เพราะ `phi` คือ Flux **ที่หน้า Face** ($\phi_f$) ไม่ใช่ที่ Cell Center จึงใช้ `surfaceScalarField` (ขนาด `nFaces`) แทน `volScalarField` (ขนาด `nCells`)
</details>

**ข้อ 4:** ถ้าลบ `correctBoundaryConditions()` ออกจาก solver จะเกิดอะไรขึ้น?

<details>
<summary>เฉลย</summary>
Boundary values จะไม่ถูก update ตามเงื่อนไข (เช่น fixedValue, zeroGradient) ทำให้ผลลัพธ์ผิดพลาด โดยเฉพาะ BC ที่ขึ้นกับเวลาหรือ field อื่นๆ (coupled BC)
</details>

**ข้อ 5:** ใน Task 2, ทำไมต้อง inherit จาก `fixedValueFvPatchVectorField` ไม่ใช่ `fvPatchVectorField` โดยตรง?

<details>
<summary>เฉลย</summary>
`fixedValueFvPatchVectorField` มี logic สำหรับ "กำหนดค่าตายตัว" อยู่แล้ว (implement `evaluate()`, `updateCoeffs()` skeleton) เราแค่ override `updateCoeffs()` เพื่อคำนวณค่าใหม่ตามเวลา ถ้า inherit จาก `fvPatchVectorField` ต้องเขียน logic ทั้งหมดเอง
</details>
