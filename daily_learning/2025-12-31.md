```cpp
#include "singlePhaseTransportModel.H"

    #include "createTime.H"              // Create runTime object
    #include "createMesh.H"              // Create fvMesh object

    // ⭐ SIMPLE ALGORITHM: Semi-Implicit Method for Pressure-Linked Equations
    // 1. Solve momentum equation (U) with current pressure field
    // 2. Solve pressure equation to enforce continuity (mass conservation)
    // 3. Correct velocity field using new pressure
    // 4. Under-relax to ensure stability
    // * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * //

        Info << "Time = " << runTime.timeName() << nl << endl;

        {

            #include "pEqn.H"    // ⭐ PRESSURE EQUATION & CORRECTION

        Info << "ExecutionTime = " << runTime.elapsedCpuTime() << " s"
             << "  ClockTime = " << runTime.elapsedClockTime() << " s"

    }

    Info << "End\n" << endl;

```

**Key Takeaways**:
- `while (simple.loop())` → SIMPLE algorithm iteration loop
- `UEqn.H` → สร้าง momentum equation matrix (ไม่ solve ทันที)

- `turbulence->correct()` → update turbulence viscosity (nut, k, epsilon, omega)

## 🔥 Source Code Deep Dive #2: UEqn.H - Momentum Equation Assembly

**Location**: `$FOAM_SOLVERS/incompressible/simpleFoam/UEqn.H`

```cpp
// ⭐ MOMENTUM EQUATION: ∂(ρU)/∂t + ∇·(φU) - ∇·(μ∇U) = -∇p
// For steady-state incompressible: ∇·(φU) - ∇·(μ∇U) = -∇p

// --- Solve momentum equation (predictor step)

// ⭐ ASSEMBLE IMPLICIT TERMS (stored in fvMatrix)
tmp<fvVectorMatrix> UEqn
(
    // ⭐ CONVECTION TERM: ∇·(φU)
    // fvm::div(φ, U) → IMPLICIT assembly: [A][U] = [b]
    // Uses upwind scheme by default (bounded)
    fvm::div(phi, U)

    // ⭐ DIFFUSION TERM: -∇·(μ∇U)
    // fvm::laplacian(nuEff, U) → IMPLICIT assembly
    // nuEff = nu (laminar) + nut (turbulent)
    + fvm::laplacian(nuEff, U)
);

// ⭐ ADD EXPLICIT TERMS (right-hand side source)
UEqn().relax();  // Under-relax momentum equation: UEqn = (1-alpha)*UEqn + alpha*UEqn_new

// ⭐ ADD PRESSURE GRADIENT: -∇p
// fvc::grad(p) → EXPLICIT calculation (not in matrix)
// This creates the pressure-velocity coupling!
if (simple.momentumPredictor())
{
    solve(UEqn() == -fvc::grad(p));  // ⭐ SOLVE: [A][U] = [b - ∇p]
}
```

**Line-by-Line Analysis**:

| Line | Code | Meaning | Matrix Position |
|------|------|---------|-----------------|
| 1 | `fvm::div(phi, U)` | Convection: ∇·(φU) | **Diagonal + Off-diagonal** (implicit) |
| 2 | `fvm::laplacian(nuEff, U)` | Diffusion: ∇·(μ∇U) | **Diagonal + Off-diagonal** (implicit) |
| 3 | `UEqn().relax()` | Under-relaxation | Modifies diagonal coefficients |
| 4 | `fvc::grad(p)` | Pressure gradient | **RHS source** (explicit) |

**Matrix Form**:
```
[A]{U} = {b} - ∇p

where:
[A] = convection_matrix + diffusion_matrix + under-relaxation_diagonal
{b} = boundary_conditions + source_terms
```

## 🔥 Source Code Deep Dive #3: pEqn.H - Pressure Equation & Correction

**Location**: `$FOAM_SOLVERS/incompressible/simpleFoam/pEqn.H`

```cpp
// ⭐ PRESSURE EQUATION: ∇²p = ∇·(U*/H)
// Derived from momentum equation + continuity equation
// H = A⁻¹ · (b - ∇p) → explicit momentum flux

// ⭐ CALCULATE EXPLICIT MOMENTUM FLUX
// UEqn.H() → returns [A]⁻¹ (inverse diagonal)
// UEqn.H() & U → H/A (cell-centered "pseudo-velocity")
volScalarField rUA = 1.0/UEqn().A();  // Reciprocal of diagonal coefficients

// ⭐ CALCULATE CELL-CENTERED VELOCITY WITHOUT PRESSURE GRADIENT
U = rUA * UEqn().H();  // U = H/A (explicit, uncorrected)
UEqn.clear();  // ⭐ FREE MEMORY: matrix no longer needed!

// ⭐ CALCULATE FLUX AT CELL FACES
phi = fvc::interpolate(U, "interpolate(U)") & mesh.Sf();
adjustPhi(phi, U, p);  // ⭐ FIX MASS FLUX FOR BOUNDARIES (to ensure continuity)

// ⭐ NON-ORTHOGONAL CORRECTION LOOP
// For non-orthogonal meshes, pressure equation is solved multiple times
// with explicit non-orthogonal correction terms
for (int nonOrth=0; nonOrth<=simple.nNonOrthCorr(); nonOrth++)
{
    // ⭐ PRESSURE EQUATION: ∇·(rUA ∇p) = ∇·(φ*)
    // fvm::laplacian(rUA, p) → IMPLICIT: [A_p]{p} = {b}
    // fvc::div(phi) → EXPLICIT source: mass imbalance
    fvScalarMatrix pEqn
    (
        fvm::laplacian(rUA, p)  // ⭐ IMPLICIT: [L(p)] = [b]
        == fvc::div(phi)        // ⭐ EXPLICIT: ∇·(φ*) = mass imbalance
    );

    // ⭐ REFERENCE PRESSURE CELL (fix pressure value to avoid singularity)
    pEqn.setReference(pRefCell, pRefValue);

    // ⭐ SOLVE PRESSURE EQUATION
    pEqn.solve();

    // ⭐ NON-ORTHOGONAL CORRECTION (if not last iteration)
    if (nonOrth == simple.nNonOrthCorr())
    {
        // ⭐ CORRECT FLUX USING PRESSURE GRADIENT
        // φ = φ* - (rUA)·∇p·|Sf|
        phi -= pEqn.flux();  // Explicit correction
    }
}

// ⭐ UNDER-RELAX PRESSURE
p.relax();

// ⭐ CORRECT VELOCITY USING NEW PRESSURE GRADIENT
// U = U* - rUA·∇p
U += rUA * fvc::grad(p);  // U = H/A - (1/A)·∇p

// ⭐ CONTINUITY CHECK
#include "continuityErrs.H"  // Report mass imbalance: sum(φ·Sf)
```

**Algorithm Flow**:
```
1. UEqn.H() & U → Calculate explicit velocity U* (without pressure)
2. rUA = 1.0/A() → Get inverse diagonal coefficients
3. interpolate(U) & Sf → Calculate face fluxes φ*
4. Solve: ∇·(rUA·∇p) = ∇·(φ*) → Find pressure field
5. φ = φ* - rUA·∇p·|Sf| → Correct fluxes
6. U = U* - rUA·∇p → Correct velocities
7. Check continuity → Verify ∑(φ·Sf) ≈ 0
```

## 🔥 Source Code Deep Dive #4: fvm:: vs fvc:: Operators

**Location**: `$FOAM_SRC/finiteVolume/finiteVolume/fvm/fvm.H` และ `$FOAM_SRC/finiteVolume/finiteVolume/fvc/fvc.H`

### 4.1 `fvm::` - Finite Volume Method (Implicit Matrix Assembly)

```cpp
// ⭐ FILE: $FOAM_SRC/finiteVolume/finiteVolume/fvm/fvm.C
namespace fvm
{
    // ⭐ CONVECTION: ∇·(φU)
    // Returns: tmp<fvMatrix<Type>>
    template<class Type>
    tmp<fvMatrix<Type>> div
    (
        const surfaceScalarField& phi,   // Mass flux: φ = ρU·Sf (at faces)
        const GeometricField<Type, fvPatchField, volMesh>& vf  // Field: U (at cells)
    )
    {
        // ⭐ MATRIX ASSEMBLY ALGORITHM:
        // 1. Interpolate U from cell centers to face centers
        // 2. Apply upwind/central differencing/TVD schemes
        // 3. Assemble coefficients:
        //    - Owner cell: += φ_f (positive flux)
        //    - Neighbor cell: -= φ_f (negative flux)
        // 4. Add to diagonal and off-diagonal of fvMatrix

        // Matrix equation: [A]·[U] = [b]
        // where A contains convection coefficients

        return tmp<fvMatrix<Type>>
        (
            new fvMatrix<Type>
            (
                vf,                     // Field being discretized
                phi.dimensions()*vf.dimensions()  // Dimensions
            )
        );
    }

    // ⭐ LAPLACIAN: ∇·(Γ∇U)
    template<class Type>
    tmp<fvMatrix<Type>> laplacian
    (
        const GeometricField<Type, fvPatchField, volMesh>& gamma,  // Diffusion coefficient: Γ
        const GeometricField<Type, fvPatchField, volMesh>& vf       // Field: U
    )
    {
        // ⭐ MATRIX ASSEMBLY ALGORITHM:
        // 1. Calculate face gradient: ∇U_f = (U_N - U_P) / |d_NP|
        // 2. Interpolate gamma to faces: Γ_f
        // 3. Assemble coefficients:
        //    - Owner cell diagonal: += Γ_f·|Sf|/|d_NP|
        //    - Neighbor cell off-diagonal: -= Γ_f·|Sf|/|d_NP|
        // 4. Add boundary contributions

        // Matrix equation: [A]·[U] = [b]
        // where A contains diffusion coefficients

        return tmp<fvMatrix<Type>>
        (
            new fvMatrix<Type>(vf, gamma.dimensions()*vf.dimensions())
        );
    }
}
```

**Key Point**: `fvm::` operators **create matrices** that can be solved!
- `fvm::div(phi, U)` → Returns `fvMatrix<vector>` (for U equation)
- `fvm::laplacian(nu, U)` → Returns `fvMatrix<vector>` (for U equation)
- `fvm::laplacian(rUA, p)` → Returns `fvMatrix<scalar>` (for p equation)

### 4.2 `fvc::` - Finite Volume Calculus (Explicit Calculation)

```cpp
// ⭐ FILE: $FOAM_SRC/finiteVolume/finiteVolume/fvc/fvc.C
namespace fvc
{
    // ⭐ GRADIENT: ∇p (explicit)
    template<class Type>
    tmp<GeometricField<typename outerProduct<vector, Type>::type, fvPatchField, volMesh>>
    grad
    (
        const GeometricField<Type, fvPatchField, volMesh>& vf  // Input field: p
    )
    {
        // ⭐ EXPLICIT GRADIENT CALCULATION:
        // 1. Calculate face values: p_f = interpolate(p)
        // 2. Calculate gradient using Gauss theorem:
        //    ∇p = (1/V)∑(p_f·Sf)
        // 3. Return field (NOT a matrix!)

        // Result: volVectorField (∇p at cell centers)
        // Can be used as RHS source term

        return tmp<GeometricField<typename outerProduct<vector, Type>::type, fvPatchField, volMesh>>
        (
            new GeometricField<typename outerProduct<vector, Type>::type, fvPatchField, volMesh>
            (
                vf.name() + "_grad",
                vf.dimensions()/dimLength,
                mesh,
                calculatedFvPatchField<typename outerProduct<vector, Type>::type>::typeName
            )
        );
    }

    // ⭐ DIVERGENCE: ∇·φ (explicit)
    tmp<volScalarField> div
    (
        const surfaceScalarField& phi  // Flux at faces
    )
    {
        // ⭐ EXPLICIT DIVERGENCE CALCULATION:
        // ∇·φ = (1/V)∑(φ·Sf)
        // Used for continuity check!

        return tmp<volScalarField>
        (
            new volScalarField
            (
                "div(phi)",
                phi.dimensions()/dimLength,
                mesh
            )
        );
    }

    // ⭐ INTERPOLATION: cell → face
    template<class Type>
    tmp<GeometricField<Type, fvsPatchField, surfaceMesh>>
    interpolate
    (
        const GeometricField<Type, fvPatchField, volMesh>& vf,  // Cell-centered field
        const word& name  // Interpolation scheme: "linear", "upwind", etc.
    )
    {
        // ⭐ INTERPOLATE FROM CELL CENTERS TO FACE CENTERS:
        // Linear: φ_f = (φ_P + φ_N)/2
        // Upwind: φ_f = φ_P (if φ > 0) or φ_N (if φ < 0)

        // Result: surfaceGeometricField (at faces)
        return surfaceInterpolation::interpolate(vf, name);
    }
}
```

**Key Point**: `fvc::` operators **calculate fields directly** (no matrix solve)!
- `fvc::grad(p)` → Returns `volVectorField` (∇p at cells)
- `fvc::div(phi)` → Returns `volScalarField` (∇·φ at cells)
- `fvc::interpolate(U)` → Returns `surfaceVectorField` (U at faces)

---

### 4.3 Comparison Table

| Aspect | `fvm::` (Implicit) | `fvc::` (Explicit) |
|--------|-------------------|-------------------|
| **Return Type** | `fvMatrix<Type>` | `GeometricField<Type, ...>` |
| **Usage** | Build equation matrix | Calculate intermediate values |
| **Example** | `fvm::div(phi, U)` | `fvc::grad(p)` |
| **Storage** | Coefficients: diagonal, off-diagonal, source | Field values at cells/faces |
| **Solver** | Required (`matrix.solve()`) | Not needed (direct calculation) |
| **Stability** | More stable (bounded) | Less stable (may oscillate) |
| **CPU Cost** | Higher (matrix assembly + solve) | Lower (direct calculation) |
| **Memory** | Higher (store matrix) | Lower (store field only) |

## 🔥 Source Code Deep Dive #5: Matrix Assembly & Solver

### 5.1 `fvMatrix` Class

**Location**: `$FOAM_SRC/finiteVolume/finiteVolume/fvMatrix/fvMatrix.H`

```cpp
template<class Type>
class fvMatrix
    : public refCount,
      public lduMatrix  // ⭐ Lower-Upper Decomposition Matrix base class
{
private:
    // ⭐ FIELD BEING DISCRETIZED
    GeometricField<Type, fvPatchField, volMesh>& psi_;

    // ⭐ DIMENSIONS (for dimensional checking)
    dimensions dimensions_;

    // ⭐ SOURCE TERM (RHS vector)
    Field<Type> source_;

    // ⭐ BOUNDARY CONDITIONS
    FieldField<Field, Type> internalCoeffs_;   // Coefficients for boundary cells
    FieldField<Field, Type> boundaryCoeffs_;   // Source contributions from boundaries

    // lduMatrix members (inherited):
    // - Diagonal coefficients: Field<scalar> diag_
    // - Off-diagonal coefficients: Field<scalar> upper_, lower_
    // - Addressing: owner_, neighbor_ (cell connectivity)

public:
    // ⭐ SOLVE THE MATRIX EQUATION
    tmp<Field<Type>> solve();

    // ⭐ RETURN FLUX FOR PRESSURE CORRECTION
    tmp<surfaceScalarField> flux() const;

    // ⭐ ACCESS OPERATORS
    const Field<Type>& source() const { return source_; }
    scalarVectorProduct H() const;  // ⭐ RETURNS H/A (used in pressure equation)
    const scalarField& diag() const { return lduMatrix::diag(); }
};
```

**Matrix Structure** (LDU format):
```
For each cell i:
  A[i][i]·ψ_i + Σ(A[i][j]·ψ_j) = b_i

where:
  - A[i][i] → diag_[i] (diagonal coefficient)
  - A[i][j] → upper_[face_ij] or lower_[face_ij] (off-diagonal)
  - b_i → source_[i] (RHS source term)
  - ψ_i → field value at cell i (unknown)
```

**Memory Layout**:
```
Cell 0: diag[0]·ψ_0 + upper[face_01]·ψ_1 + upper[face_02]·ψ_2 = source[0]
Cell 1: lower[face_01]·ψ_0 + diag[1]·ψ_1 + upper[face_13]·ψ_3 = source[1]
Cell 2: lower[face_02]·ψ_0 + diag[2]·ψ_2 + ... = source[2]
...
```

### 5.2 `lduMatrix` Class (Base Class)

**Location**: `$FOAM_SRC/ matrices/lduMatrix/lduMatrix.H`

```cpp
class lduMatrix
{
    // ⭐ LDU MATRIX STORAGE
    // Only stores non-zero coefficients (sparse matrix)

    scalarField diag_;    // Diagonal coefficients [nCells]
    scalarField upper_;   // Upper triangular coefficients [nInternalFaces]
    scalarField lower_;   // Lower triangular coefficients [nInternalFaces]

    // ⭐ ADDRESSING (connectivity)
    const lduAddressing& lduAddr_;

    // lduAddressing contains:
    // - owner_: owner cell for each face [nInternalFaces]
    // - neighbor_: neighbor cell for each face [nInternalFaces]
    // - losortAddr_: sorting for efficient access

public:
    // ⭐ SOLVER INTERFACE
    solverPerformance solve
    (
        Field<scalar>& psi,           // Field to solve (output)
        const Field<scalar>& source,  // RHS source term
        const scalarField& T,         // Diagonal scaling (optional)
        const direction cmpt=0        // Component (for vectors)
    ) const;

    // ⭐ MATRIX-VECTOR MULTIPLICATION
    void Amul
    (
        Field<scalar>& Ax,
        const Field<scalar>& x
    ) const;  // Ax = A·x
};
```

**Key Concept**: LDU = **Lower-Diagonal-Upper** (sparse matrix format)
- Only stores non-zero coefficients (face connectivity)
- Efficient for unstructured meshes
- Enables iterative solvers (CG, GAMG, BiCGStab)

### 5.3 Solver Hierarchy

```
lduSolver
  |
  +--> iterativeSolver
         |
         +--> smoothSolver  (Jacobi, Sor, DILU)
         |
         +--> conjugateGradientSolver  (CG)
         |
         +--> GAMGSolver  (Geometric-Algebraic Multi-Grid)
         |
         +--> BiCGStabSolver  (Bi-Conjugate Gradient Stabilized)
```

**Solver Selection** (in `fvSolution` dictionary):
```cpp

    {
```

```
│  - lower_: [nFaces] lower coefficients                               │
│  - source_: [nCells] RHS source term                                │
└─────────────────────────────────────────────────────────────────────┘
                            ↓ ↓ ↓
┌─────────────────────────────────────────────────────────────────────┐
│                    SOLVER (lduMatrix::solve)                         │
├─────────────────────────────────────────────────────────────────────┤
│  solve(fvMatrix, field):                                             │
│  1. Preconditioning (optional):                                     │
│     - Jacobi: M = diag(A)⁻¹                                         │
│     - DILU: Diagonal Incomplete LU                                  │
│                                                                      │
│  2. Iterative solver:                                                │
│     - GAMG: Coarse grid correction + smoothsolver                   │
│     - CG: Conjugate Gradient (symmetric matrices only)              │
│     - BiCGStab: Bi-Conjugate Gradient Stabilized                    │
│                                                                      │
│  3. Convergence check:                                              │
│     - residual = ||b - A·x|| / ||b||                                │
│     - if (residual < tolerance) break;                              │
│                                                                      │
│  4. Return solution field (updated U, p, etc.)                      │
└─────────────────────────────────────────────────────────────────────┘
                            ↓ ↓ ↓
┌─────────────────────────────────────────────────────────────────────┐
│                       FIELD UPDATE (Correction)                      │
├─────────────────────────────────────────────────────────────────────┤
│  U = H/A - rUA·∇p                                                     │
│  p = p_new                                                           │
│  φ = φ* - rUA·∇p·|Sf|                                                 │
│  φ.correctBoundaryConditions();  // Fix boundary fluxes             │
└─────────────────────────────────────────────────────────────────────┘
                            ↓ ↓ ↓
┌─────────────────────────────────────────────────────────────────────┐
│                      OUTPUT (Time Directory)                         │
├─────────────────────────────────────────────────────────────────────┤
│  1/                                                                  │
│  ├ U (volVectorField)                                               │
│  ├ p (volScalarField)                                               │
│  └ phi (surfaceScalarField - optional)                              │
│                                                                      │
│  Format: binary (default) or ASCII                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## ✅ Concept Check

### Q1: Matrix Assembly

**ใน UEqn.H มีคำสั่ง:
```cpp
tmp<fvVectorMatrix> UEqn
(
    fvm::div(phi, U) + fvm::laplacian(nuEff, U)
);
```

**คำถาม**: `fvm::div(phi, U)` สร้างอะไร? และเก็บข้อมูลอะไรบ้าง?

**Answer**:
- `fvm::div(phi, U)` → สร้าง `fvMatrix<vector>` (matrix equation)
- เก็บ:
  - `diag_`: diagonal coefficients [nCells]
  - `upper_`, `lower_`: off-diagonal coefficients [nFaces]
  - `source_`: RHS source terms [nCells]
  - `internalCoeffs_`, `boundaryCoeffs_`: boundary contributions

---

### Q2: Implicit vs Explicit

**คำถาม**: แยกความแตกต่างระหว่าง:
- `fvm::laplacian(rUA, p)` ใน pEqn.H
- `fvc::grad(p)` ใน UEqn.H

**Answer**:
| Aspect | `fvm::laplacian(rUA, p)` | `fvc::grad(p)` |
|--------|--------------------------|----------------|
| Type | Implicit matrix assembly | Explicit field calculation |
| Returns | `fvMatrix<scalar>` | `volVectorField` |
| Used for | Building pressure equation | Pressure gradient source term |
| Requires solver | Yes (`pEqn.solve()`) | No (direct calculation) |
| Storage | Matrix coefficients | Field values (∇p at cells) |

---

### Q3: SIMPLE Algorithm Steps

**คำถาม**: ใน pEqn.H มีขั้นตอน:
```cpp
U = rUA * UEqn().H();
phi = fvc::interpolate(U) & mesh.Sf();
// ... solve pressure equation ...
U += rUA * fvc::grad(p);
```

**อธิบาย**: แต่ละบรรทัดทำอะไร?

**Answer**:
1. `U = rUA * UEqn().H()` → Calculate explicit velocity U* without pressure gradient
   - `UEqn().H()` → Returns H (explicit momentum flux excluding pressure)
   - `rUA = 1.0/A()` → Inverse diagonal coefficients
   - Result: U* = H/A

2. `phi = fvc::interpolate(U) & mesh.Sf()` → Calculate face fluxes
   - Interpolate U from cells to faces
   - Dot product with face surface vectors
   - Result: φ* = U·Sf (mass flux at faces)

3. Solve pressure equation → Find pressure field p that satisfies continuity

4. `U += rUA * fvc::grad(p)` → Correct velocity using pressure gradient
   - `fvc::grad(p)` → ∇p at cells
   - `rUA * ∇p` → (1/A)·∇p
   - Result: U = U* - (1/A)·∇p (final corrected velocity)

---

### Q4: Memory and Storage

**คำถาม**: ใน fvMatrix และ lduMatrix เก็บข้อมูลอย่างไร? ทำไมไม่เก็บ full matrix?

**Answer**:
- **LDU format** (Lower-Diagonal-Upper):
  - เก็บเฉพาะ non-zero coefficients
  - `diag_`: [nCells] → diagonal elements
  - `upper_`, `lower_`: [nInternalFaces] → face connectivity
  - **ไม่เก็บ zero elements** → ประหยัด memory!

- **ทำไมไม่เก็บ full matrix?**
  - Full matrix: [nCells × nCells] → เก็บทุก elements (รวม zeros)
  - LDU matrix: [nCells + 2×nFaces] → เก็บเฉพาะ non-zeros
  - สำหรับ 1 million cells:
    - Full: 10¹² elements (impossible!)
    - LDU: ~7×10⁶ elements (feasible!)

- **Sparse matrix advantages**:
  - Memory efficient
  - Fast matrix-vector multiplication
  - Enables iterative solvers (CG, GAMG)
