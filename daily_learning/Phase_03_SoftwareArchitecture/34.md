# Day 34: Dynamic Configuration — Factory + JSON Integration

## Part 1: Pattern Identification

### The Integration Challenge

We've built two powerful systems:
1. **Self-registering factory** (Day 32) — Plugins register automatically
2. **JSON configuration** (Day 33) — Load settings from files

**The Goal:** Combine them so JSON config drives factory creation:

```json
// config.json
{
  "solver": {
    "type": "GaussSeidel",
    "tolerance": 1e-6,
    "maxIterations": 1000
  }
}
```

```cpp
// Load from JSON and create via factory
auto config = loadConfig("config.json");
auto solver = Factory::instance().create(
    config["solver"]["type"],
    /* parameters from JSON */
);
```

### Dynamic Configuration Pattern

```mermaid
flowchart LR
    A[JSON Config File] --> B[Parse with nlohmann/json]
    B --> C[Extract Type Name]
    C --> D[Factory::create]
    D --> E[Return Plugin Instance]
    E --> F[Configure with JSON Params]
```

### Why This Matters

**Benefits:**
- **Zero recompilation**: Add solvers without rebuilding
- **User-extensible**: Users can add custom solvers via plugins
- **Runtime flexibility**: Change behavior via config file
- **Testability**: Easy to test different configurations

**Real-world examples:**
- OpenFOAM's `fvSchemes` and `fvSolution` dictionaries
- VTK's pipeline configuration
- Game engine component systems

## Part 2: Theory — Parameter Injection

### The Constructor Problem

Different solvers need different parameters:

```cpp
class JacobiSolver {
    int maxIterations_;
    double tolerance_;

public:
    JacobiSolver(int maxIter, double tol)
        : maxIterations_(maxIter), tolerance_(tol) {}
};

class PCGSolver {
    int maxIterations_;
    double tolerance_;
    std::string preconditioner_;

public:
    PCGSolver(int maxIter, double tol, const std::string& prec)
        : maxIterations_(maxIter), tolerance_(tol), preconditioner_(prec) {}
};
```

**Problem:** Factory can't know which parameters each solver needs!

### Solution: JSON as Universal Parameter Container

```cpp
// All solvers accept JSON and extract what they need
class LinearSolver {
public:
    virtual void configure(const json& params) = 0;
    virtual bool solve() = 0;
};

class JacobiSolver : public LinearSolver {
    int maxIterations_;
    double tolerance_;

public:
    void configure(const json& params) override {
        maxIterations_ = params.value("maxIterations", 1000);
        tolerance_ = params.value("tolerance", 1e-6);
    }
};
```

### Two-Phase Initialization

**Phase 1: Creation** — Factory creates object
**Phase 2: Configuration** — JSON configures object

```cpp
auto solver = factory.create("GaussSeidel");  // Phase 1
solver->configure(json_config["solver"]);     // Phase 2
```

## Part 3: C++ Mechanics — Integrated Implementation

### Enhanced Factory with JSON Support

```cpp
// SolverFactory.H
#pragma once
#include "LinearSolver.H"
#include <nlohmann/json.hpp>
#include <memory>
#include <string>
#include <unordered_map>

using json = nlohmann::json;

class SolverFactory {
private:
    using Creator = std::function<std::unique_ptr<LinearSolver>()>;

    std::unordered_map<std::string, Creator> registry_;

    SolverFactory() = default;

public:
    static SolverFactory& instance() {
        static SolverFactory inst;
        return inst;
    }

    void registerSolver(const std::string& name, Creator creator) {
        registry_[name] = creator;
    }

    std::unique_ptr<LinearSolver> create(const std::string& name) {
        auto it = registry_.find(name);
        if (it != registry_.end()) {
            return it->second();
        }
        return nullptr;
    }

    // Create and configure from JSON
    std::unique_ptr<LinearSolver> createFromConfig(const json& config) {
        std::string type = config["type"];
        auto solver = create(type);

        if (solver) {
            solver->configure(config);
        }

        return solver;
    }
};
```

### Configurable Solver Interface

```cpp
// LinearSolver.H
#pragma once
#include <nlohmann/json.hpp>
#include <vector>

using json = nlohmann::json;

class LinearSolver {
protected:
    int maxIterations_;
    double tolerance_;
    bool configured_ = false;

public:
    LinearSolver() : maxIterations_(1000), tolerance_(1e-6) {}

    virtual ~LinearSolver() = default;

    // Configure from JSON
    virtual void configure(const json& params) {
        if (params.contains("maxIterations")) {
            maxIterations_ = params["maxIterations"];
        }
        if (params.contains("tolerance")) {
            tolerance_ = params["tolerance"];
        }
        configured_ = true;
    }

    virtual bool solve(const std::vector<double>& matrix,
                       std::vector<double>& solution,
                       const std::vector<double>& rhs) = 0;

    virtual std::string name() const = 0;

protected:
    void validateConfigured() const {
        if (!configured_) {
            throw std::runtime_error("Solver not configured");
        }
    }
};
```

### Concrete Solvers

```cpp
// JacobiSolver.C
#include "LinearSolver.H"
#include "SolverRegistrar.H"

class JacobiSolver : public LinearSolver {
public:
    void configure(const json& params) override {
        LinearSolver::configure(params);
        // Jacobi-specific parameters could go here
    }

    bool solve(const std::vector<double>& matrix,
               std::vector<double>& solution,
               const std::vector<double>& rhs) override {
        validateConfigured();

        int n = solution.size();
        std::vector<double> x_new(n);

        for (int iter = 0; iter < maxIterations_; ++iter) {
            for (int i = 0; i < n; ++i) {
                double sum = rhs_[i];
                for (int j = 0; j < n; ++j) {
                    if (i != j) {
                        sum -= matrix[i * n + j] * solution[j];
                    }
                }
                x_new[i] = sum / matrix[i * n + i];
            }

            // Check convergence
            double error = 0.0;
            for (int i = 0; i < n; ++i) {
                error += std::abs(x_new[i] - solution[i]);
                solution[i] = x_new[i];
            }

            if (error < tolerance_) {
                return true;
            }
        }
        return false;
    }

    std::string name() const override { return "Jacobi"; }
};

REGISTER_SOLVER(JacobiSolver, "jacobi")
```

```cpp
// GaussSeidelSolver.C
#include "LinearSolver.H"
#include "SolverRegistrar.H"

class GaussSeidelSolver : public LinearSolver {
public:
    bool solve(const std::vector<double>& matrix,
               std::vector<double>& solution,
               const std::vector<double>& rhs) override {
        validateConfigured();

        int n = solution.size();

        for (int iter = 0; iter < maxIterations_; ++iter) {
            double maxError = 0.0;

            for (int i = 0; i < n; ++i) {
                double oldValue = solution[i];
                double sum = rhs[i];

                for (int j = 0; j < i; ++j) {
                    sum -= matrix[i * n + j] * solution[j];
                }
                for (int j = i + 1; j < n; ++j) {
                    sum -= matrix[i * n + j] * solution[j];
                }

                solution[i] = sum / matrix[i * n + i];
                maxError = std::max(maxError, std::abs(solution[i] - oldValue));
            }

            if (maxError < tolerance_) {
                return true;
            }
        }
        return false;
    }

    std::string name() const override { return "Gauss-Seidel"; }
};

REGISTER_SOLVER(GaussSeidelSolver, "gauss_seidel")
```

```cpp
// PCGSolver.C
#include "LinearSolver.H"
#include "SolverRegistrar.H"

class PCGSolver : public LinearSolver {
    std::string preconditioner_;

public:
    void configure(const json& params) override {
        LinearSolver::configure(params);
        preconditioner_ = params.value("preconditioner", std::string("none"));
    }

    bool solve(const std::vector<double>& matrix,
               std::vector<double>& solution,
               const std::vector<double>& rhs) override {
        validateConfigured();

        int n = solution.size();
        std::vector<double> residual(n), p(n), Ap(n);
        std::vector<double> z(n);  // For preconditioner

        // r = b - Ax
        for (int i = 0; i < n; ++i) {
            residual[i] = rhs[i];
            for (int j = 0; j < n; ++j) {
                residual[i] -= matrix[i * n + j] * solution[j];
            }
        }

        // Precondition: z = M^(-1) * r
        applyPreconditioner(matrix, residual, z);

        // p = z
        p = z;

        double rho_old = dot(z, residual);

        for (int iter = 0; iter < maxIterations_; ++iter) {
            // Ap = A * p
            for (int i = 0; i < n; ++i) {
                Ap[i] = 0;
                for (int j = 0; j < n; ++j) {
                    Ap[i] += matrix[i * n + j] * p[j];
                }
            }

            double alpha = rho_old / dot(p, Ap);

            // x = x + alpha * p
            for (int i = 0; i < n; ++i) {
                solution[i] += alpha * p[i];
            }

            // r = r - alpha * Ap
            double rho_new = 0;
            for (int i = 0; i < n; ++i) {
                residual[i] -= alpha * Ap[i];
                rho_new += residual[i] * residual[i];
            }

            if (std::sqrt(rho_new) < tolerance_) {
                return true;
            }

            // Precondition: z = M^(-1) * r
            applyPreconditioner(matrix, residual, z);

            double beta = rho_new / rho_old;
            rho_old = rho_new;

            // p = z + beta * p
            for (int i = 0; i < n; ++i) {
                p[i] = z[i] + beta * p[i];
            }
        }

        return false;
    }

    std::string name() const override {
        return "PCG(" + preconditioner_ + ")";
    }

private:
    double dot(const std::vector<double>& a, const std::vector<double>& b) {
        double result = 0;
        for (size_t i = 0; i < a.size(); ++i) {
            result += a[i] * b[i];
        }
        return result;
    }

    void applyPreconditioner(const std::vector<double>& matrix,
                             const std::vector<double>& r,
                             std::vector<double>& z) {
        if (preconditioner_ == "diagonal") {
            for (size_t i = 0; i < r.size(); ++i) {
                z[i] = r[i] / matrix[i * (i + 1)];
            }
        } else {
            // No preconditioning
            z = r;
        }
    }
};

REGISTER_SOLVER(PCGSolver, "pcg")
```

## Part 4: Implementation Exercise

### Complete Solver Manager

```cpp
// SolverManager.H
#pragma once
#include "LinearSolver.H"
#include "SolverFactory.H"
#include <nlohmann/json.hpp>
#include <vector>
#include <string>
#include <fstream>

using json = nlohmann::json;

class SolverManager {
private:
    std::vector<std::unique_ptr<LinearSolver>> solvers_;
    std::vector<json> configs_;

public:
    // Load configuration from JSON file
    void loadConfig(const std::string& filename) {
        std::ifstream f(filename);
        json config = json::parse(f);

        // Clear existing
        solvers_.clear();
        configs_.clear();

        // Load solvers from config
        if (config.contains("solvers")) {
            for (const auto& solverConfig : config["solvers"]) {
                auto solver = SolverFactory::instance().createFromConfig(solverConfig);
                if (solver) {
                    solvers_.push_back(std::move(solver));
                    configs_.push_back(solverConfig);
                }
            }
        }
    }

    // Test all configured solvers
    void runBenchmark(const std::vector<double>& matrix,
                     std::vector<double>& solution,
                     const std::vector<double>& rhs) {
        std::cout << "=== Solver Benchmark ===" << std::endl;
        std::cout << "Matrix size: " << solution.size() << " x " << solution.size() << std::endl;

        for (size_t i = 0; i < solvers_.size(); ++i) {
            std::cout << "\n--- Solver " << (i + 1) << " ---" << std::endl;
            std::cout << "Type: " << solvers_[i]->name() << std::endl;
            std::cout << "Config: " << configs_[i].dump() << std::endl;

            // Reset solution
            std::fill(solution.begin(), solution.end(), 0.0);

            auto start = std::chrono::high_resolution_clock::now();
            bool converged = solvers_[i]->solve(matrix, solution, rhs);
            auto end = std::chrono::high_resolution_clock::now();

            std::chrono::duration<double> elapsed = end - start;

            std::cout << "Converged: " << (converged ? "Yes" : "No") << std::endl;
            std::cout << "Time: " << elapsed.count() << " seconds" << std::endl;
        }
    }

    size_t solverCount() const { return solvers_.size(); }
};
```

```cpp
std::vector<double> createTridiagonalMatrix(int n) {
    std::vector<double> matrix(n * n, 0.0);
```

## Part 5: Trade-offs

### Configuration vs. Hard-coding

| Aspect | Hard-coded | Config-driven |
|--------|------------|---------------|
| **Flexibility** | Low — recompile to change | High — edit JSON |
| **Performance** | Slightly faster (compile-time opt) | Minimal runtime overhead |
| **Complexity** | Simpler | More complex infrastructure |
| **User-friendliness** | Developer-only | User-configurable |

### Error Handling Strategies

```cpp
// Strategy 1: Fail fast
void configure(const json& params) {
    if (!params.contains("tolerance")) {
        throw std::runtime_error("Missing 'tolerance'");
    }
    tolerance_ = params["tolerance"];
}

// Strategy 2: Use defaults
void configure(const json& params) {
    tolerance_ = params.value("tolerance", 1e-6);
}

// Strategy 3: Validate after configuration
void configure(const json& params) {
    tolerance_ = params.value("tolerance", 1e-6);
    if (tolerance_ <= 0) {
        throw std::runtime_error("Tolerance must be positive");
    }
}
```

### Best Practices

1. **Schema validation**: Validate JSON structure early
2. **Type safety**: Use proper type conversions
3. **Default values**: Provide sensible defaults
4. **Clear error messages**: Tell users what's wrong
5. **Logging**: Log configuration decisions

### main() That Loads JSON and Runs Both Solvers

```cpp
// main_cfd.C — load two-solver CFD config and run sequentially
#include "SolverManager.H"
#include <iostream>
#include <iomanip>
#include <vector>
#include <cmath>

// Build a diagonally-dominant tridiagonal system (represents 1D Poisson)
static void buildSystem(int n,
                        std::vector<double>& A,
                        std::vector<double>& x,
                        std::vector<double>& b) {
    A.assign(n * n, 0.0);
    x.assign(n, 0.0);
    b.assign(n, 1.0);
    for (int i = 0; i < n; ++i) {
        A[i * n + i] = 2.0;
        if (i > 0)     A[i * n + (i - 1)] = -1.0;
        if (i < n - 1) A[i * n + (i + 1)] = -1.0;
    }
}

static double residualNorm(const std::vector<double>& A,
                           const std::vector<double>& x,
                           const std::vector<double>& b) {
    int n = static_cast<int>(x.size());
    double norm = 0.0;
    for (int i = 0; i < n; ++i) {
        double r = b[i];
        for (int j = 0; j < n; ++j) r -= A[i * n + j] * x[j];
        norm += r * r;
    }
    return std::sqrt(norm);
}

int main(int argc, char* argv[]) {
    const char* configFile = (argc > 1) ? argv[1] : "cfd_run.json";
    const int n = 50;

    std::vector<double> A, x, b;
    buildSystem(n, A, x, b);

    SolverManager manager;
    manager.loadConfig(configFile);

    std::cout << "CFD Run: " << configFile << "\n";
    std::cout << "System size: " << n << "x" << n << "\n\n";

    // Run each solver in sequence; each starts from the previous solution
    for (size_t idx = 0; idx < manager.solverCount(); ++idx) {
        auto& solver = manager.solverAt(idx);
        std::cout << "Pass " << (idx + 1) << ": " << solver.name() << "\n";
        bool ok = solver.solve(A, x, b);
        double res = residualNorm(A, x, b);
        std::cout << "  Converged: " << (ok ? "yes" : "no")
                  << "  |r| = " << std::scientific << std::setprecision(3)
                  << res << "\n\n";
    }

    return 0;
}
```

### Quick Reference: Factory + JSON Pattern

| Component | Responsibility | Key Method |
|-----------|---------------|------------|
| `LinearSolver` base class | Defines interface + shared fields | `configure(json)`, `solve(A, x, b)` |
| `SolverFactory` singleton | Owns registry, creates by string key | `createFromConfig(json)` |
| `REGISTER_SOLVER` macro | Links plugin class to string key at startup | Static initializer |
| `SolverManager` | Loads JSON file, holds solver vector | `loadConfig(path)`, `runBenchmark()` |
| JSON config file | Drives runtime behavior, no recompile | `"type"` key selects solver |

**Pattern summary:** The JSON `"type"` field is the only coupling between config file and C++ code. Everything else (parameters, field names, tolerances) flows through the generic `configure(json)` interface. Adding a new solver requires only: a new `.C` file with the implementation plus a `REGISTER_SOLVER` call — no changes to `SolverFactory`, `SolverManager`, or `main`.

### Connection to Day 35: Object Registry

Day 35 builds the Object Registry pattern, which is the natural next step after Factory + JSON. The factory creates objects by name; the registry **stores and retrieves** those objects by name throughout the simulation lifetime. In OpenFOAM's `objectRegistry`, every field (`volScalarField`, `volVectorField`) is registered at construction and looked up by name during boundary condition evaluation and I/O. The combination of Day 34's factory (creation) and Day 35's registry (lifetime management) mirrors exactly how OpenFOAM's `runTime` object manages the simulation state: fields are factory-constructed from `fvSchemes`/`fvSolution` dictionaries, then stored in the `objectRegistry` for the duration of the run.
