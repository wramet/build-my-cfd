# Day 33: Configuration I/O — JSON with nlohmann/json

## Part 1: Pattern Identification

### The Configuration Problem

CFD solvers require extensive configuration:
- Discretization schemes (upwind, central, TVD)
- Linear solver settings (solver type, tolerance, max iterations)
- Time step controls (CFL, fixed Δt, adjustable)
- Boundary conditions (types, values, profiles)
- Material properties (density, viscosity, etc.)

**The Challenge:** How to load and manage this configuration flexibly?

```cpp
// Bad: Hardcoded settings
double tolerance = 1e-6;
int maxIterations = 1000;
std::string solverType = "GaussSeidel";

// Better: Load from configuration file
// solver_config.json:
// {
//   "solver": {
//     "type": "GaussSeidel",
//     "tolerance": 1e-6,
//     "maxIterations": 1000
//   }
// }
```

### Why JSON?

| Format | Human-readable | Parsing | Type-safe | Hierarchical |
|---------|---------------|---------|-----------|--------------|
| **JSON** | ✅ Yes | ✅ Easy | ✅ Yes | ✅ Yes |
| XML | ✅ Yes | ❌ Verbose | ✅ Yes | ✅ Yes |
| INI | ✅ Yes | ✅ Easy | ❌ No | ❌ Flat |
| Binary | ❌ No | ✅ Fast | ❌ No | ❌ No |

JSON provides the best balance for configuration files.

### nlohmann/json Library

The de facto standard JSON library for modern C++:

```cpp
#include <nlohmann/json.hpp>

using json = nlohmann::json;

// Parse JSON from file
std::ifstream f("config.json");
json config = json::parse(f);

// Access values
double tolerance = config["solver"]["tolerance"];
std::string type = config["solver"]["type"];
```

### C++ Type Mapping

```cpp
json j;

j["string"] = "text";           // → std::string
j["number"] = 3.14;             // → double
j["integer"] = 42;              // → int
j["boolean"] = true;            // → bool
j["array"] = {1, 2, 3};         // → std::vector
j["object"] = {{"key", "val"}}; // → std::map
```

## Part 3: C++ Mechanics — nlohmann/json Usage

### Basic Operations

```cpp
#include <nlohmann/json.hpp>
#include <fstream>
#include <iostream>

using json = nlohmann::json;

int main() {
    // 1. Create JSON object
    json config;

    config["solver"] = {
        {"type", "GaussSeidel"},
        {"tolerance", 1e-6},
        {"maxIterations", 1000}
    };

    config["time"] = {
        {"scheme", "Euler"},
        {"maxDeltaT", 0.01}
    };

    // 2. Serialize to string
    std::string config_str = config.dump(4);  // 4-space indent
    std::cout << config_str << std::endl;

    // 3. Write to file
    std::ofstream out("config.json");
    out << config.dump(4);

    // 4. Read from file
    std::ifstream in("config.json");
    json loaded = json::parse(in);

    // 5. Access values
    std::string solver_type = loaded["solver"]["type"];
    double tolerance = loaded["solver"]["tolerance"];

    // 6. Type checking
    if (loaded.contains("solver")) {
        if (loaded["solver"]["type"].is_string()) {
            std::cout << "Solver type: " << loaded["solver"]["type"] << std::endl;
        }
    }

    // 7. Default values
    double tol = loaded.value("solver/tolerance", 1e-6);

    return 0;
}
```

### Custom Type Serialization

```cpp
struct BoundaryCondition {
    std::string name;
    std::string type;
    double value;
};

// to_json for serialization
void to_json(json& j, const BoundaryCondition& bc) {
    j = {
        {"name", bc.name},
        {"type", bc.type},
        {"value", bc.value}
    };
}

// from_json for deserialization
void from_json(const json& j, BoundaryCondition& bc) {
    j.at("name").get_to(bc.name);
    j.at("type").get_to(bc.type);
    j.at("value").get_to(bc.value);
}

// Usage:
BoundaryCondition bc{"inlet", "fixedValue", 1.0};

// Serialize
json j = bc;
std::cout << j.dump() << std::endl;
// Output: {"name":"inlet","type":"fixedValue","value":1.0}

// Deserialize
auto bc2 = j.get<BoundaryCondition>();
```

### Error Handling

```cpp
try {
    std::ifstream f("config.json");
    json config = json::parse(f);

    // Validate required fields
    if (!config.contains("solver")) {
        throw std::runtime_error("Missing 'solver' section");
    }

    // Access with type checking
    if (!config["solver"]["type"].is_string()) {
        throw std::runtime_error("'solver.type' must be a string");
    }

} catch (const json::parse_error& e) {
    std::cerr << "Parse error: " << e.what() << std::endl;
} catch (const json::type_error& e) {
    std::cerr << "Type error: " << e.what() << std::endl;
} catch (const std::exception& e) {
    std::cerr << "Error: " << e.what() << std::endl;
}
```

### Step 3: Configuration Loader with Validation

```cpp
// ConfigLoader.H
#pragma once
#include "Config.H"
#include "ConfigIO.H"
#include <fstream>
#include <stdexcept>
#include <iostream>

class ConfigLoader {
public:
    static SolverSettings load(const std::string& filename) {
        std::ifstream f(filename);

        if (!f.is_open()) {
            throw std::runtime_error("Cannot open config file: " + filename);
        }

        json j;
        try {
            j = json::parse(f);
        } catch (const json::parse_error& e) {
            throw std::runtime_error("JSON parse error: " + std::string(e.what()));
        }

        SolverSettings settings = j.get<SolverSettings>();

        // Validate
        validate(settings);

        return settings;
    }

    static void save(const std::string& filename, const SolverSettings& settings) {
        json j = settings;

        std::ofstream out(filename);
        if (!out.is_open()) {
            throw std::runtime_error("Cannot write config file: " + filename);
        }

        out << j.dump(4);  // 4-space indent
    }

private:
    static void validate(const SolverSettings& settings) {
        // Validate solver config
        if (settings.solver.type.empty()) {
            throw std::runtime_error("Solver type cannot be empty");
        }

        if (settings.solver.tolerance <= 0) {
            throw std::runtime_error("Solver tolerance must be positive");
        }

        if (settings.solver.maxIterations <= 0) {
            throw std::runtime_error("Max iterations must be positive");
        }

        // Validate time config
        if (settings.time.maxDeltaT <= 0) {
            throw std::runtime_error("maxDeltaT must be positive");
        }

        if (settings.time.cflNumber <= 0) {
            throw std::runtime_error("CFL number must be positive");
        }

        // Validate boundaries
        if (settings.boundaries.empty()) {
            throw std::runtime_error("At least one boundary condition required");
        }

        // Validate output
        if (settings.output.writeTimes.empty()) {
            throw std::runtime_error("At least one output time required");
        }
    }

    static void print(const SolverSettings& settings) {
        std::cout << "=== Solver Settings ===" << std::endl;
        std::cout << "Solver: " << settings.solver.type << std::endl;
        std::cout << "Tolerance: " << settings.solver.tolerance << std::endl;
        std::cout << "Max iterations: " << settings.solver.maxIterations << std::endl;

        std::cout << "\nTime: " << settings.time.scheme << std::endl;
        std::cout << "Max Δt: " << settings.time.maxDeltaT << std::endl;
        std::cout << "CFL: " << settings.time.cflNumber << std::endl;

        std::cout << "\nBoundaries:" << std::endl;
        for (const auto& bc : settings.boundaries) {
            std::cout << "  " << bc.name << ": " << bc.type
                      << " = " << bc.value << std::endl;
        }

        std::cout << "\nOutput:" << std::endl;
        std::cout << "  Times: ";
        for (double t : settings.output.writeTimes) {
            std::cout << t << " ";
        }
        std::cout << std::endl;
    }
};
```

### Step 5: Using the Configuration

```cpp
// main.C
#include "ConfigLoader.H"
#include "SolverFactory.H"
#include "LinearSolver.H"
#include <iostream>

int main() {
    try {
        // Load configuration
        auto settings = ConfigLoader::load("solver_config.json");

        // Print configuration
        ConfigLoader::print(settings);

        // Create solver using factory
        const int n = 100;
        std::vector<double> matrix(n * n, 0.0);
        std::vector<double> solution(n, 0.0);
        std::vector<double> rhs(n, 1.0);

        // Assemble matrix (simplified tridiagonal)
        for (int i = 0; i < n; ++i) {
            matrix[i * n + i] = 2.0;
            if (i > 0) matrix[i * n + (i - 1)] = -1.0;
            if (i < n - 1) matrix[i * n + (i + 1)] = -1.0;
        }

        auto solver = SolverFactory::instance().create(
            settings.solver.type,
            matrix,
            solution,
            rhs
        );

        if (!solver) {
            std::cerr << "Unknown solver type: " << settings.solver.type << std::endl;
            return 1;
        }

        std::cout << "\n=== Solving ===" << std::endl;
        std::cout << "Solver: " << solver->name() << std::endl;

        auto start = std::chrono::high_resolution_clock::now();
        bool converged = solver->solve(
            settings.solver.tolerance,
            settings.solver.maxIterations
        );
        auto end = std::chrono::high_resolution_clock::now();

        std::chrono::duration<double> elapsed = end - start;

        std::cout << "Converged: " << (converged ? "Yes" : "No") << std::endl;
        std::cout << "Time: " << elapsed.count() << " seconds" << std::endl;

    } catch (const std::exception& e) {
        std::cerr << "Error: " << e.what() << std::endl;
        return 1;
    }

    return 0;
}
```

## Part 5: Trade-offs

### JSON vs. Other Formats

| Format | Pros | Cons |
|--------|------|------|
| **JSON** | Simple, widely supported, type-safe | No comments, limited data types |
| **YAML** | More human-readable, supports comments | Complex parsing, slower |
| **TOML** | Clean syntax, comments | Less widely adopted |
| **XML** | Schema validation, namespaces | Verbose, complex |

### When to Use JSON

**Use JSON for:**
- Configuration files
- Data interchange between systems
- Web APIs
- Simple data storage

**Avoid JSON for:**
- Large numerical datasets (use binary formats like HDF5)
- Complex document structures (consider YAML)
- Performance-critical serialization (use binary)

### Best Practices

1. **Validate early**: Parse and validate at startup
2. **Provide defaults**: Use sensible default values
3. **Clear error messages**: Tell users exactly what's wrong
4. **Document format**: Provide example configuration files
5. **Version your schema**: Include a version number in your config
