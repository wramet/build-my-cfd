## Part 1: Core Theory - From Continuous to Discrete

### 1.1 Why Face Value Interpolation is Needed in FVM

In the Finite Volume Method (FVM), we store field values at **cell centers** but fluxes occur through **faces**. This creates a fundamental problem: how do we calculate the value of a field at a face when we only know values at cell centers?

Consider the convection-diffusion equation:

$$
\frac{\partial \phi}{\partial t} + \nabla \cdot (\mathbf{U} \phi) = \nabla \cdot (D \nabla \phi)
$$

Integrating over a control volume and applying Gauss divergence theorem:

$$
\int_V \frac{\partial \phi}{\partial t} dV + \oint_{\partial V} \phi \mathbf{U} \cdot d\mathbf{S} = \oint_{\partial V} D \nabla \phi \cdot d\mathbf{S}
$$

The surface integrals require values of $\phi$ at faces. This is where **face value interpolation schemes** come in.

### 1.2 Central Differencing Scheme

The **central differencing** scheme (called `linear` in OpenFOAM) interpolates the face value as a weighted average of the owner and neighbor cell values:

$$
\phi_f = w \phi_P + (1 - w) \phi_N
$$

**Derivation from geometric interpolation:**

Consider two cell centers $P$ and $N$ with a face $f$ between them:

```
    P --------f--------- N
    |<-d_Pf->|<-d_fN->|
    |<------ d_PN ------>|
```

The geometric weight is:

$$
w = \frac{d_{fN}}{d_{PN}} = \frac{|\mathbf{x}_f - \mathbf{x}_N|}{|\mathbf{x}_N - \mathbf{x}_P|}
$$

For a uniform mesh with the face exactly midway, $w = 0.5$.

**⭐ Verified Implementation in OpenFOAM:**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/schemes/linear/linear.H`
> **Lines:** 50-97

```cpp
template<class Type>
class linear
:
    public surfaceInterpolationScheme<Type>
{
public:
    //- Return the interpolation weighting factors
    tmp<surfaceScalarField> weights
    (
        const VolField<Type>&
    ) const
    {
        return this->mesh().surfaceInterpolation::weights();
    }
};
```

The `linear` scheme returns geometric mesh weights calculated by `surfaceInterpolation::weights()`, which computes the distance-based weights for all faces.

### 1.3 First-Order Upwind Scheme

The **upwind scheme** selects the face value based on the flow direction:

$$
\phi_f = 
\begin{cases}
\phi_P & \text{if } \mathbf{U}_f \cdot \mathbf{S}_f > 0 \quad \text{(flow out of owner)} \\
\phi_N & \text{if } \mathbf{U}_f \cdot \mathbf{S}_f < 0 \quad \text{(flow into owner)}
\end{cases}
$$

**Physical intuition:** Information travels with the flow. The upwind cell value is "carried" to the face.

**⭐ Verified Implementation in OpenFOAM:**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/upwind/upwind.H`
> **Lines:** 101-128

```cpp
template<class Type>
class upwind
:
    public limitedSurfaceInterpolationScheme<Type>
{
public:
    //- Return the interpolation limiter
    virtual tmp<surfaceScalarField> limiter
    (
        const VolField<Type>&
    ) const
    {
        return surfaceScalarField::New
        (
            "upwindLimiter",
            this->mesh(),
            dimensionedScalar(dimless, 0)  // Returns 0 - no limiter
        );
    }

    //- Return the interpolation weighting factors
    tmp<surfaceScalarField> weights() const
    {
        return pos0(this->faceFlux_);  // 1 for positive flux, 0 for negative
    }
};
```

The `pos0(faceFlux_)` function returns 1.0 when the flux is positive (outward from owner) and 0.0 when negative.

### 1.4 The Stability-Accuracy Trade-off

| Scheme | Order | Stability | Numerical Diffusion | Oscillations |
|--------|-------|-----------|---------------------|--------------|
| Upwind | 1st   | Always stable | High (false diffusion) | None (bounded) |
| Central | 2nd   | Conditionally stable | None | Possible (unbounded) |
| TVD | 2nd (varies) | Stable | Controlled | Suppressed by limiter |

**Key insight:** There's no perfect scheme. The choice depends on:
- Peclet number (ratio of convection to diffusion)
- Importance of boundedness (e.g., VOF requires $\alpha \in [0,1]$)
- Acceptable numerical diffusion

## Part 2: Physical Challenge - Convection-Dominated Problems

### 2.1 Peclet Number Analysis

The **Peclet number** measures the relative importance of convection vs diffusion:

$$
Pe = \frac{\text{convective transport}}{\text{diffusive transport}} = \frac{\rho u L}{D}
$$

Where:
- $\rho$ = density
- $u$ = velocity
- $L$ = characteristic length
- $D$ = diffusivity

**Interpretation:**
- $Pe \ll 1$: Diffusion-dominated (central differencing works well)
- $Pe \gg 1$: Convection-dominated (upwind or TVD needed)

### 2.2 Why Central Differencing Fails at High Pe

For pure convection ($D \to 0$, $Pe \to \infty$), central differencing produces **dispersive errors** that create non-physical oscillations near sharp gradients.

**Stability criterion for central differencing:**

$$
Pe \leq 2
$$

Beyond this limit, the discretized equation loses diagonal dominance and the solution becomes unstable.

### 2.3 Numerical Diffusion in Upwind Schemes

Upwind schemes are stable but introduce **numerical diffusion** (false diffusion):

- Artificially smears sharp gradients
- Depends on mesh alignment with flow
- Reduces with mesh refinement

For the R410A evaporator, this means:
- Liquid-vapor interface may appear blurred
- Temperature gradients are artificially spread
- Mass transfer rates may be underestimated

### 2.4 The Need for TVD Schemes

**TVD (Total Variation Diminishing)** schemes combine:
- Second-order accuracy in smooth regions
- Boundedness near discontinuities
- No non-physical oscillations

**Key principle:** The total variation of the solution should not increase:

$$
TV(\phi^{n+1}) \leq TV(\phi^n)
$$

where:

$$
TV(\phi) = \sum_i |\phi_{i+1} - \phi_i|
$$

### 3.2 Base Class: surfaceInterpolationScheme

**⭐ Abstract Base Class:**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/surfaceInterpolationScheme/surfaceInterpolationScheme.H`
> **Lines:** 55-142

```cpp
template<class Type>
class surfaceInterpolationScheme
:
    public tmp<surfaceInterpolationScheme<Type>>::refCount
{
    // Private Data
        const fvMesh& mesh_;

public:
    //- Runtime type information
    TypeName("surfaceInterpolationScheme");

    //- Return mesh reference
    const fvMesh& mesh() const { return mesh_; }

    //- Return the interpolation weighting factors (PURE VIRTUAL)
    virtual tmp<surfaceScalarField> weights
    (
        const VolField<Type>&
    ) const = 0;

    //- Return true if this scheme uses an explicit correction
    virtual bool corrected() const { return false; }

    //- Return the face-interpolate of the given cell field
    virtual tmp<SurfaceField<Type>>
    interpolate(const VolField<Type>&) const;
};
```

**Key design patterns:**
1. Template-based for generic field types
2. Pure virtual `weights()` method enforces implementation
3. Runtime type information via `TypeName` macro
4. `tmp` smart pointer for automatic memory management

### 3.3 The Limited Scheme Framework

**⭐ TVD/NVD Framework:**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/LimitedScheme/LimitedScheme.H`
> **Lines:** 63-142

```cpp
template<class Type, class Limiter, template<class> class LimitFunc>
class LimitedScheme
:
    public limitedSurfaceInterpolationScheme<Type>,
    public Limiter
{
public:
    //- Runtime type information
    TypeName("LimitedScheme");

    typedef Limiter LimiterType;

    // Constructors
    LimitedScheme
    (
        const fvMesh& mesh,
        const surfaceScalarField& faceFlux,
        const Limiter& weight
    )
    :
        limitedSurfaceInterpolationScheme<Type>(mesh, faceFlux),
        Limiter(weight)
    {}

    //- Return the interpolation weighting factors
    virtual tmp<surfaceScalarField> limiter
    (
        const VolField<Type>&
    ) const;
};
```

**How it works:**
1. Receives a `Limiter` class (e.g., `vanLeerLimiter`)
2. Calculates the limiter field for all faces
3. Blends high-order and upwind weights using the limiter

### 3.4 Limiter Functions: The vanLeer Implementation

**⭐ vanLeer Limiter:**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/vanLeer/vanLeer.H`
> **Lines:** 53-82

```cpp
template<class LimiterFunc>
class vanLeerLimiter
:
    public LimiterFunc
{
public:
    scalar limiter
    (
        const scalar,
        const scalar faceFlux,
        const typename LimiterFunc::phiType& phiP,
        const typename LimiterFunc::phiType& phiN,
        const typename LimiterFunc::gradPhiType& gradcP,
        const typename LimiterFunc::gradPhiType& gradcN,
        const vector& d
    ) const
    {
        scalar r = LimiterFunc::r
        (
            faceFlux, phiP, phiN, gradcP, gradcN, d
        );

        return (r + mag(r))/(1 + mag(r));
    }
};
```

**The vanLeer formula:**

$$
\psi(r) = \frac{r + |r|}{1 + |r|}
$$

**⚠️ Critical observation:** The implementation uses `mag(r)` (OpenFOAM's absolute value function), NOT plain `r`. This is essential for the limiter to handle negative ratios correctly.

**Behavior of vanLeer limiter:**

| r (ratio) | $\psi(r)$ | Behavior |
|-----------|-----------|----------|
| r < 0 | 0 | Completely upwind (extreme gradient) |
| r = 0 | 0 | Upwind |
| r = 0.5 | 0.33 | Compressive |
| r = 1 | 1 | Second-order central |
| r = 2 | 1 | Second-order central |
| r > 2 | approaches 1 | Approaches second-order |

### 3.5 Common TVD Limiters

| Limiter | Formula | Characteristic | Use Case |
|---------|---------|----------------|----------|
| upwind | $\psi = 0$ | First-order | Very stable, very diffusive |
| minmod | $\psi = \max(\min(r,1), 0)$ | Most diffusive TVD | Default choice |
| vanLeer | $\psi = \frac{r + |r|}{1 + |r|}$ | Smooth, compressive | General purpose |
| SUPERBEE | $\psi = \max(0, \min(2r, 1), \min(r, 2))$ | Most compressive | Sharp interfaces |
| Gamma | $\psi = \max(0, \min(\gamma r, 1))$ | Adjustable | User-controlled |

## Part 4: Quality Assurance - Scheme Selection

### 4.1 Scheme Selection Guidelines

**For R410A two-phase flow in evaporator:**

| Variable | Recommended Scheme | Reason |
|----------|-------------------|---------|
| Velocity (`U`) | `linearUpwind V` | Second-order, bounded |
| Pressure (`p`) | `linear` | Smooth field |
| Temperature (`T`) | `linearUpwind grad(T)` | Preserve gradients |
| VOF (`alpha`) | `vanLeer` or `MULES` | Bounded, sharp interface |
| Turbulence (k, ε, ω) | `upwind` | Stability critical |

### 4.2 Verification Exercises

**Exercise 1: Peclet Number Calculation**

Given R410A liquid properties at 1 MPa:
- Density: $\rho = 1000$ kg/m³
- Velocity: $u = 0.1$ m/s
- Cell size: $L = 1$ mm
- Kinematic viscosity: $\nu = 1.2 \times 10^{-7}$ m²/s

Calculate Peclet number:

$$
Pe = \frac{u L}{\nu} = \frac{0.1 \times 0.001}{1.2 \times 10^{-7}} \approx 833
$$

**Conclusion:** Convection-dominated. Use upwind or TVD scheme.

**Exercise 2: Code Tracing**

Task: Trace how `vanLeer` scheme calculates a face value.

1. User specifies `div(phi, alpha) Gauss vanLeer;` in `fvSchemes`
2. OpenFOAM constructs `vanLeer<limitFuncs::magSqr>` via `makeLimitedSurfaceInterpolationScheme`
3. For each face:
   - Calculate ratio $r$ from gradients
   - Compute limiter $\psi(r) = \frac{r + |r|}{1 + |r|}$
   - Blend: $\phi_f = \phi_f^{upwind} + \psi(r)(\phi_f^{CD} - \phi_f^{upwind})$

### 4.3 Connection to VOF Two-Phase Flow

For the volume fraction equation:

$$
\frac{\partial \alpha}{\partial t} + \nabla \cdot (\mathbf{U} \alpha) + \nabla \cdot (\mathbf{U}_r \alpha(1-\alpha)) = 0
$$

**Scheme requirements:**
1. **Boundedness:** $\alpha$ must stay in $[0, 1]$
2. **Sharp interface:** Minimize numerical diffusion
3. **Compressive:** Preferably sharpen the interface

**MULES** (Multidimensional Universal Limiter with Explicit Solution) is the default OpenFOAM solver for VOF, which uses special limiters to ensure boundedness.

## Part 5: R410A Application

### 5.1 Why Scheme Selection Matters for the Evaporator

In R410A evaporator simulation:

1. **Phase change accuracy:** The liquid-vapor interface location determines heat transfer rate
2. **Mass conservation:** Numerical diffusion artificially spreads phase, affecting mass balance
3. **Temperature gradients:** Evaporation temperature depends on local quality

**Wrong scheme choice leads to:**
- Over-prediction of evaporator length (interface smearing)
- Incorrect heat transfer coefficient
- Unphysical mass imbalance

### 5.2 R410A Property Considerations

| Property | Liquid Phase | Vapor Phase | Impact on Discretization |
|----------|--------------|-------------|--------------------------|
| Density | ~1000 kg/m³ | ~50 kg/m³ | Large ratio → stiff equations |
| Viscosity | ~1.2×10⁻⁷ m²/s | ~7×10⁻⁷ m²/s | Affects Pe calculation |
| Surface tension | 0.007 N/m | - | Interface curvature force |

### 5.3 Implementation Preview

For R410A evaporator, `fvSchemes` configuration:

```cpp

{
    default         Euler;
}

gradSchemes
{
    default         Gauss linear;
    grad(alpha)     Gauss linear;
}

divSchemes
{
    default         none;
    div(phi,U)      Gauss linearUpwind grad(U);
    div(phi,alpha)  Gauss vanLeer  // TVD for sharp interface
    div(phi,k)      Gauss upwind;
    div(phi,epsilon) Gauss upwind;
    div((nuEff*dev2(T(grad(U))))) Gauss linear;
}

laplacianSchemes
{
    default         Gauss linear corrected;
}
```

### 5.4 Verification for R410A

**Validation steps:**
1. Check boundedness: $\min(\alpha) \geq 0$, $\max(\alpha) \leq 1$
2. Monitor mass conservation: $\int_V \alpha dV$ should only change at boundaries
3. Compare interface sharpness across different schemes
4. Verify heat transfer against analytical solution

## Summary

### Key Takeaways

1. **Face interpolation** converts cell-centered values to face values for flux calculation
2. **Central differencing** is second-order but conditionally stable
3. **Upwind** is unconditionally stable but introduces numerical diffusion
4. **TVD schemes** combine accuracy and boundedness using limiters
5. **vanLeer** formula: $\psi(r) = \frac{r + |r|}{1 + |r|}$ uses `mag(r)` function
6. For **R410A VOF**, use `vanLeer` or `MULES` for sharp, bounded interfaces

### Class Hierarchy Summary

```
surfaceInterpolationScheme<Type> (abstract base)
├── linear<Type>                    // Central differencing
└── limitedSurfaceInterpolationScheme<Type> (abstract)
    ├── upwind<Type>                // First-order upwind
    └── LimitedScheme<Type, Limiter, LimitFunc>  // TVD framework
        └── Uses limiters like:
            ├── vanLeerLimiter
            ├── MinmodLimiter
            └── SUPERBEELimiter
```
