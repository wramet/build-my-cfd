# 1. Section 1: Theory (ทฤษฎี)

## 1.1 The Interpolation Problem (ปัญหาการประมาณค่าที่ผิว)

### 1.1.1 Why We Need Interpolation Schemes

In Day 02, we established that the Finite Volume Method (FVM) requires computing fluxes through cell faces. For the convection term:

$$
\nabla \cdot (\rho \mathbf{U} \phi) \approx \sum_f (\rho \mathbf{U} \phi)_f \cdot \mathbf{S}_f
$$

The **fundamental problem**: Our field variables ($\phi$, $\mathbf{U}$, $T$, $\alpha$, etc.) are stored at **cell centers** $\mathbf{C}_P$, but fluxes require values at **face centers** $\mathbf{C}_f$.

**Visual illustration:**

```
        Cell P        Face f        Cell N
      ┌────────┐     ┌─────┐     ┌────────┐
      │        │     │     │     │        │
      │  φ_P   │────▶│ ?   │────▶│  φ_N   │
      │ (known)│     │     │     │ (known)│
      │        │     └─────┘     │        │
      └────────┐                 └────────┘
              │                         ^
              └─────────────────────────┘
                    Need φ_f here!
```

**Mathematical formulation:**

Given cell-centered values $\phi_P$ and $\phi_N$ at distance $\Delta x$ apart, we need to estimate $\phi_f$ at the face located at distance $\alpha \Delta x$ from $P$ (where $\alpha$ is the interpolation factor, typically 0.5 for uniform grids):

$$
\phi_f = w_P \phi_P + w_N \phi_N + \text{higher-order terms}
$$

where $w_P + w_N = 1$. The **interpolation scheme** determines these weights.

### 1.1.2 Key Properties to Evaluate

When selecting an interpolation scheme, we evaluate:

| **Property** | **Definition** | **Why It Matters** |
|:---|:---|:---|
| **Order of Accuracy** | Leading truncation error term $O(\Delta x^n)$ | Determines convergence rate as grid refines |
| **Boundedness** | $\min(\phi_P, \phi_N) \leq \phi_f \leq \max(\phi_P, \phi_N)$ | Critical for VOF (0 ≤ α ≤ 1) |
| **Transportiveness** | Scheme respects flow direction (upwind bias) | Affects stability for convection-dominated flows |
| **Numerical Diffusion** | Artificial smearing of sharp gradients | Causes interface spreading in VOF |
| **Phase Error** | Lag between numerical and physical wave speed | Causes dispersion errors |
| **Computational Cost** | CPU time per face evaluation | Important for large 3D simulations |

**The fundamental trade-off:**

$$
\text{Accuracy} \times \text{Stability} \times \text{Cost} = \text{Constant}
$$

You can optimize at most two of these three factors simultaneously.

## 1.2 Basic Schemes (โครงร่างพื้นฐาน)

### 1.2.1 Central Differencing Scheme (CDS) / Linear Scheme

**Mathematical formulation:**

For a uniform grid with face centered between cells:

$$
\boxed{\phi_f = \frac{1}{2}(\phi_P + \phi_N)}
$$

**Taylor series derivation:**

Expand $\phi_P$ and $\phi_N$ about the face location $f$:

$$
\phi_P = \phi_f - \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f + \frac{(\Delta x/2)^2}{2!}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f - \cdots
$$

$$
\phi_N = \phi_f + \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f + \frac{(\Delta x/2)^2}{2!}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f + \cdots
$$

Adding these equations:

$$
\phi_P + \phi_N = 2\phi_f + \frac{(\Delta x)^2}{4}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f + O(\Delta x^4)
$$

Therefore:

$$
\phi_f = \frac{\phi_P + \phi_N}{2} - \frac{(\Delta x)^2}{8}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f + O(\Delta x^4)
$$

**Truncation error:** $O(\Delta x^2)$ → **Second-order accurate**

**Characteristics:**

| **Aspect** | **Evaluation** |
|:---|:---|
| Accuracy | Second-order on uniform grids |
| Boundedness | **NOT bounded** - can produce $\phi_f \notin [\phi_P, \phi_N]$ |
| Stability | **Unconditional instability** for Pe > 2 |
| Numerical diffusion | Zero (no artificial diffusion) |
| Phase error | Significant (dispersive) |
| Cost | Very low (simple arithmetic) |

**Péclet number limitation:**

The Péclet number compares convection to diffusion:

$$
\text{Pe} = \frac{|\mathbf{U}| \Delta x}{\alpha} = \frac{\text{Convection}}{\text{Diffusion}}
$$

where $\alpha$ is thermal diffusivity (or mass diffusivity).

**Stability criterion:** CDS is stable only if Pe < 2. For typical CFD problems with Pe >> 1, CDS will **oscillate** and may diverge.

**When to use CDS:**
- Diffusive fluxes (e.g., `div((nuEff*dev2(T(grad(U))))))`)
- Laminar flows with low velocity
- When solution is smooth (no sharp gradients)
- Grid-independent studies (second-order convergence)

### 1.2.2 First-Order Upwind Scheme (UDS)

**Mathematical formulation:**

$$
\boxed{\phi_f = \begin{cases}
\phi_P & \text{if } F_f > 0 \text{ (flow from P to N)} \\
\phi_N & \text{if } F_f < 0 \text{ (flow from N to P)}
\end{cases}}
$$

where $F_f = (\rho \mathbf{U})_f \cdot \mathbf{S}_f$ is the mass flux through the face.

**Compact notation:**

$$
\phi_f = \max(F_f, 0) \frac{\phi_P}{F_f} + \min(F_f, 0) \frac{\phi_N}{F_f}
$$

**Taylor series derivation:**

Assume flow from P to N ($F_f > 0$). Using only upwind value $\phi_P$:

$$
\phi_P = \phi_f - \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f + \frac{(\Delta x/2)^2}{2!}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f - \cdots
$$

Therefore:

$$
\phi_f = \phi_P + \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f - \frac{(\Delta x)^2}{8}\left(\frac{\partial^2 \phi}{\partial x^2}\right)_f + \cdots
$$

**Truncation error:** $O(\Delta x)$ → **First-order accurate**

**Characteristics:**

| **Aspect** | **Evaluation** |
|:---|:---|
| Accuracy | First-order (slow convergence) |
| Boundedness | **Guaranteed bounded** - $\phi_f \in [\phi_P, \phi_N]$ |
| Stability | **Unconditionally stable** for any Pe |
| Numerical diffusion | **Severe false diffusion** |
| Phase error | Minimal (transportive) |
| Cost | Very low (simple conditional) |

**Numerical diffusion analysis:**

The truncation error can be rewritten as a diffusion term:

$$
\text{Error} \approx \frac{|\mathbf{U}| \Delta x}{2} \frac{\partial^2 \phi}{\partial x^2}
$$

This is equivalent to adding an **artificial diffusivity**:

$$
\alpha_{\text{artificial}} = \frac{|\mathbf{U}| \Delta x}{2}
$$

**Effective Péclet number:**

$$
\text{Pe}_{\text{eff}} = \frac{|\mathbf{U}| \Delta x}{\alpha + \alpha_{\text{artificial}}} = \frac{\text{Pe}}{1 + \text{Pe}/2}
$$

For Pe → ∞ (pure convection), Pe_eff → 2, which explains stability.

**When to use UDS:**
- Initial runs to get a stable solution
- High Reynolds/Péclet number flows
- Turbulence quantities (k, ε, ω)
- First-order validation studies
- When robustness is critical

### 1.2.3 Second-Order Upwind (QUICK / LUDS)

**QUICK: Quadratic Upwind Interpolation for Convective Kinematics**

Developed by Leonard (1979), QUICK uses three points to achieve third-order accuracy on uniform grids.

**Mathematical formulation:**

For flow from P to N, using two upstream points (P and U) and one downstream (N):

$$
\phi_f = \frac{3}{4}\phi_P + \frac{3}{8}\phi_N - \frac{1}{8}\phi_U
$$

**Geometric interpretation:**

```
    Cell U        Cell P        Face f        Cell N
  ┌────────┐    ┌────────┐    ┌─────┐    ┌────────┐
  │        │    │        │    │     │    │        │
  │  φ_U   │───▶│  φ_P   │───▶│  ?  │───▶│  φ_N   │
  │ (-3Δx) │    │ (0)    │    │(Δx/2)│    │ (Δx)   │
  │        │    │        │    │     │    │        │
  └────────┘    └────────┘    └─────┘    └────────┘
       │             │             │           │
       └─────────────┴─────────────┴───────────┘
              Parabolic fit through 3 points
```

**Taylor series derivation:**

Fit a parabola through $\phi_U$, $\phi_P$, $\phi_N$ and evaluate at $\Delta x/2$:

$$
\phi_f = \frac{3}{4}\phi_P + \frac{3}{8}\phi_N - \frac{1}{8}\phi_U + O(\Delta x^3)
$$

**Truncation error:** $O(\Delta x^3)$ → **Third-order accurate** on uniform grids

**Characteristics:**

| **Aspect** | **Evaluation** |
|:---|:---|
| Accuracy | Third-order on uniform grids |
| Boundedness | **NOT guaranteed** - can overshoot |
| Stability | Conditionally stable (CFL < 8/3) |
| Numerical diffusion | Low (much less than UDS) |
| Phase error | Moderate |
| Cost | Medium (requires neighbor's neighbor) |

**⭐ VERIFIED: OpenFOAM Implementation**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/QUICK/QUICK.H`

OpenFOAM implements QUICK as a TVD scheme with appropriate limiting to maintain boundedness.

**When to use QUICK:**
- Single-phase convection-dominated flows
- When second-order accuracy is insufficient
- Structured meshes with uniform spacing
- Research applications requiring high accuracy
- NOT recommended for VOF (unbounded)

## 1.3 High-Resolution Schemes: TVD & NVD (โครงร่างความละเอียดสูง)

### 1.3.1 Total Variation Diminishing (TVD)

**Motivation:** Can we have **second-order accuracy** AND **boundedness**? TVD schemes achieve this through **nonlinear flux limiters**.

**Total variation definition:**

For a discrete field $\phi$ on a 1D grid:

$$
\text{TV}(\phi) = \sum_i |\phi_{i+1} - \phi_i|
$$

**TVD condition:** A scheme is TVD if:

$$
\text{TV}(\phi^{n+1}) \leq \text{TV}(\phi^n)
$$

This guarantees:
- No new extrema are created
- Monotonicity is preserved
- Overshoots/undershoots are eliminated

**Flux limiter formulation:**

The TVD scheme blends between first-order upwind (UDS) and second-order linear (CDS):

$$
\phi_f = \phi_f^{\text{UDS}} + \psi(r) \left[ \phi_f^{\text{CDS}} - \phi_f^{\text{UDS}} \right]
$$

where:
- $\psi(r)$ is the **flux limiter function**
- $r$ is the **gradient ratio** (smoothness indicator)

**Gradient ratio $r$:**

$$
r = \frac{\phi_P - \phi_U}{\phi_N - \phi_P} = \frac{\text{upwind gradient}}{\text{local gradient}}
$$

**Physical interpretation of $r$:**

| **Value of r** | **Interpretation** | **Desired ψ(r)** |
|:---|:---|:---|
| $r < 0$ | Extremum nearby (should not upwind) | $\psi = 0$ (use UDS) |
| $r \to 0$ | Sharp gradient approaching | $\psi \to 0$ (use UDS) |
| $r = 1$ | Smooth region (linear variation) | $\psi = 1$ (use CDS) |
| $r > 1$ | Gradual variation | $\psi \to 1$ (use CDS) |

**Sweby's TVD region:**

For a scheme to be TVD, the limiter must satisfy:

$$
0 \leq \psi(r) \leq \min(2r, 2)
$$

**Common TVD limiters:**

| **Limiter** | **Formula: ψ(r)** | **Characteristics** |
|:---|:---|:---|
| **Minmod** | $\max(0, \min(1, r))$ | Most diffusive TVD limiter |
| **Superbee** | $\max(0, \min(2r, 1), \min(r, 2))$ | Most compressive TVD limiter |
| **van Leer** | $\frac{r + |r|}{1 + |r|}$ | Good balance, smooth |
| **van Albada** | $\frac{r + r^2}{1 + r^2}$ | Differentiable, smooth |
| **Ospre** | $\frac{r(r + 1)}{r^2 + r + 1}$ | Optimized for accuracy |

**⭐ VERIFIED: van Leer Limiter Formula**

(

(

### 1.3.2 Normalized Variable Diagram (NVD)

The NVD framework, developed by Leonard (1991) and extended by Darwish (2004), provides an alternative approach to TVD.

**Normalized variable definition:**

$$
\tilde{\phi} = \frac{\phi - \phi_U}{\phi_D - \phi_U}
$$

where:
- $\phi_U$ = upstream value (donor cell)
- $\phi_D$ = downstream value (acceptor cell)

**Physical bounds:** $\tilde{\phi} \in [0, 1]$ for bounded schemes

**Convection Boundedness Criterion (CBC):**

A scheme satisfies CBC if it lies within the NVD region:

$$
\tilde{\phi}_f = \begin{cases}
\tilde{\phi}_C & \text{if } \tilde{\phi}_C < 0 \text{ or } \tilde{\phi}_C > 1 \\
\tilde{\phi}_C & \text{for } 0 \leq \tilde{\phi}_C \leq 1 \text{ and } |\nabla \tilde{\phi}| \text{ small} \\
f(\tilde{\phi}_C) & \text{otherwise}
\end{cases}
$$

**NVD framework schemes:**

| **Scheme** | **NVD Formulation** | **Characteristics** |
|:---|:---|:---|
| **Upwind** | $\tilde{\phi}_f = \tilde{\phi}_C$ | Bounded, diffusive |
| **CDS** | $\tilde{\phi}_f = \frac{1}{2}(\tilde{\phi}_C + \tilde{\phi}_D)$ | Unbounded |
| **SMART** | Piecewise linear in NVD space | Bounded, compressive |
| **STOIC** | Smooth, differentiable | Bounded, accurate |

### 1.3.3 Application to Two-Phase Flow (VOF)

**Why boundedness is CRITICAL for VOF:**

The volume fraction $\alpha$ represents the fraction of cell volume occupied by liquid:
- $\alpha = 0$: Pure vapor
- $\alpha = 1$: Pure liquid
- $\alpha \in (0, 1)$: Interface cell

**Physical requirement:** $\alpha$ **must** satisfy:

$$
0 \leq \alpha \leq 1
$$

**Consequences of unbounded schemes:**

```cpp
// If CDS produces α_f = -0.05 (unphysical!)
// Then:
α_P_new = α_P - (flux_out - flux_in) * dt / V_P
        = 0.5 - (positive_flux + 0.05) * dt / V_P
        = 0.5 - ... > 0.5  // Mass conservation violated
```

**MULES: Multidimensional Universal Limiter for Explicit Solution**

MULES is OpenFOAM's algorithm for bounded transport of scalar fields:

1. **Predictor step:** Compute tentative fluxes using selected scheme
2. **Limiter step:** Apply flux limiters to ensure boundedness
3. **Corrector step:** Update solution with limited fluxes

**⭐ Key insight:** Even with TVD schemes, MULES adds an additional layer of protection for extreme cases (e.g., very high density ratios).

**Interface sharpness vs numerical diffusion:**

| **Scheme** | **Interface Width (cells)** | **Mass Conservation** | **Stability** |
|:---|:---|:---|:---|
| Upwind | 5-10 (very smeared) | Excellent | Excellent |
| van Leer | 2-3 (sharp) | Excellent | Excellent |
| SuperBee | 1-2 (very sharp) | Good | Good |
| CDS | Oscillates (unbounded) | Poor | Poor |

**Compressive schemes:**

For VOF, specialized compressive schemes add an extra term to sharpen the interface:

$$
\phi_f^{\text{compressive}} = \phi_f^{\text{TVD}} + c_{\text{comp}} \nabla \alpha \cdot \mathbf{n}_f
$$

where $c_{\text{comp}}$ is the compression factor (typically 1-2).

    "limitedSurfaceInterpolationScheme~Type~" --|> "surfaceInterpolationScheme~Type~"
    upwind~Type~ --> "limitedSurfaceInterpolationScheme~Type~"
    linear~Type~ --> "surfaceInterpolationScheme~Type~"
    "vanLeer~Type~" --> "limitedSurfaceInterpolationScheme~Type~"
    "SuperBee~Type~" --> "limitedSurfaceInterpolationScheme~Type~"

**Key observations:**
1. `upwind` inherits from `limitedSurfaceInterpolationScheme`, NOT directly from `surfaceInterpolationScheme`
2. `linear` (CDS) inherits directly from `surfaceInterpolationScheme` (no limiter)
3. TVD schemes (`vanLeer`, `SuperBee`) use `limitedSurfaceInterpolationScheme` as base
4. All schemes are template classes `Type` to handle scalar, vector, tensor fields

### 2.2.3 TVD Schemes (vanLeer, SuperBee, etc.)

**⭐ VERIFIED: van Leer Implementation**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/vanLeer/vanLeer.H`
> **Lines:** 53-82

```cpp
template<class LimiterFunc>
class vanLeerLimiter
:
    public LimiterFunc
{

public:

    vanLeerLimiter(Istream&)
    {}

    scalar limiter
    (
        const scalar,          // cdWeight (unused)
        const scalar faceFlux, // Direction
        const typename LimiterFunc::phiType& phiP,
        const typename LimiterFunc::phiType& phiN,
        const typename LimiterFunc::gradPhiType& gradcP,
        const typename LimiterFunc::gradPhiType& gradcN,
        const vector& d        // Distance vector
    ) const
    {
        // ⭐ Compute gradient ratio r
        scalar r = LimiterFunc::r
        (
            faceFlux, phiP, phiN, gradcP, gradcN, d
        );

        // ⭐ van Leer limiter formula
        return (r + mag(r))/(1 + mag(r));
    }
};
```

**Key implementation details:**
1. **`LimiterFunc::r(...)`**: Computes gradient ratio from:
   - Face flux (direction)
   - Cell values (phiP, phiN)
   - Cell gradients (gradcP, gradcN)
   - Distance vector (d)
2. **`mag(r)`**: OpenFOAM's `mag()` = absolute value for scalars
3. **Smooth limiter**: van Leer is differentiable, good for convergence

**⭐ VERIFIED: SuperBee Implementation**

> **File:** `openfoam_temp/src/finiteVolume/interpolation/surfaceInterpolation/limitedSchemes/SuperBee/SuperBee.H`
> **Lines:** 53-82

```cpp
template<class LimiterFunc>
class SuperBeeLimiter
:
    public LimiterFunc
{

public:

    scalar limiter
    (
        const scalar cdWeight,
        const scalar faceFlux,
        const typename LimiterFunc::phiType& phiP,
        const typename LimiterFunc::phiType& phiN,
        const typename LimiterFunc::gradPhiType& gradcP,
        const typename LimiterFunc::gradPhiType& gradcN,
        const vector& d
    ) const
    {
        scalar r = LimiterFunc::r
        (
            faceFlux, phiP, phiN, gradcP, gradcN, d
        );

        // ⭐ SuperBee limiter formula
        return max(max(min(2*r, 1), min(r, 2)), 0);
    }
};
```

**Key implementation details:**
1. **Compressive**: SuperBee is the most compressive TVD limiter
2. **Piecewise formula**: Implements `max(min(2r, 1), min(r, 2), 0)`
3. **Sharp interfaces**: Excellent for VOF interface sharpening

### 3.1.3 Example: Custom Compressive Scheme for VOF

**Goal:** Create a scheme that sharpens the VOF interface

```cpp
template<class Type>
class interfaceCompression
:
    public limitedSurfaceInterpolationScheme<Type>
{

    // Compression factor (user-adjustable)
    scalar compressionFactor_;


public:

    TypeName("interfaceCompression");

    // Constructor with parameter
    interfaceCompression
    (
        const fvMesh& mesh,
        const surfaceScalarField& faceFlux,
        Istream& is
    )
    :
        limitedSurfaceInterpolationScheme<Type>(mesh, faceFlux),
        compressionFactor_(readScalar(is))  // ⭐ Read from dict
    {}

    // Limiter function
    virtual tmp<surfaceScalarField> limiter
    (
        const VolField<Type>& vf
    ) const
    {
        const surfaceScalarField& phi = this->faceFlux_;

        // Get mesh geometry
        const fvMesh& mesh = this->mesh();
        const surfaceVectorField& Sf = mesh.Sf();
        const surfaceScalarField& magSf = mesh.magSf();

        // Compute face-normal gradients
        tmp<VolField<typename LimiterFunc::gradPhiType>> tgradcP
        (
            gradcScheme<typename LimiterFunc::gradPhiType>::New
            (
                mesh,
                phi.name()
            )().grad(vf)
        );
        // ... (complete gradient computation) ...

        // ⭐ Compute compression limiter
        scalarField limiterFunc(mesh.nInternalFaces());

        forAll(limiterFunc, i)
        {
            scalar r = computeR(i, phi, vf, gradcP, gradcN);

            // Base TVD limiter (e.g., van Leer)
            scalar psi = (r + mag(r))/(1 + mag(r));

            // ⭐ Add compression term
            vector n = Sf[i] / magSf[i];  // Face normal
            scalar gradNormal = (gradcP[i] + gradcN[i]) & n;

            limiterFunc[i] = psi + compressionFactor_ * gradNormal;
        }

        // Create limiter field
        tmp<surfaceScalarField> tLimiter
        (
            new surfaceScalarField
            (
                IOobject("interfaceCompressionLimiter", mesh),
                mesh,
                dimensionedScalar(dimless, 0),
                calculatedFvPatchScalarField::typeName
            )
        );

        tLimiter.ref().primitiveFieldRef() = limiterFunc;
        return tLimiter;
    }
};
```

# 6. Section 6: Concept Checks (การทดสอบแนวคิด)

## 6.1 Self-Assessment Questions (คำถามประเมินตนเอง)

### Q1: Why does First-Order Upwind introduce numerical diffusion?

<details>
<summary>Show Answer</summary>

**Answer:**

First-order upwind introduces numerical diffusion due to **Taylor series truncation error**.

**Mathematical derivation:**

For flow from P to N, upwind uses only φ_P:

$$
\phi_f = \phi_P
$$

Expanding φ_P about the face:

$$
\phi_P = \phi_f - \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f + O(\Delta x^2)
$$

Solving for φ_f:

$$
\phi_f = \phi_P + \frac{\Delta x}{2}\left(\frac{\partial \phi}{\partial x}\right)_f + O(\Delta x^2)
$$

The truncation error can be rewritten as a diffusion term:

$$
\text{Error} \approx \frac{|\mathbf{U}| \Delta x}{2} \frac{\partial^2 \phi}{\partial x^2}
$$

This is equivalent to adding an **artificial diffusivity**:

$$
\boxed{\alpha_{\text{artificial}} = \frac{|\mathbf{U}| \Delta x}{2}}
$$

**Key implications:**

1. **Interface smearing in VOF:** Sharp gradients become spread over 5-10 cells
2. **False diffusion:** Depends on flow direction (vector quantity)
3. **Grid dependence:** Artificial diffusivity increases with grid spacing Δx
4. **Stability mechanism:** This artificial diffusion is why upwind is unconditionally stable

**Mitigation strategies:**
- Use TVD schemes (vanLeer) for interface regions
- Refine mesh locally near interfaces
- Use higher-order schemes (QUICK, linearUpwind) in smooth regions

</details>

### Q2: What is the TVD condition and why is it important for VOF?

<details>
<summary>Show Answer</summary>

**Answer:**

**TVD Definition:**

A numerical scheme is **Total Variation Diminishing** if:

$$
\boxed{\text{TV}(\phi^{n+1}) \leq \text{TV}(\phi^n)}
$$

where Total Variation is defined as:

$$
\text{TV}(\phi) = \sum_i |\phi_{i+1} - \phi_i|
$$

**Physical interpretation:**

TVD schemes guarantee that **no new extrema are created** during the solution process. This means:
- Overshoots (φ > 1) are prevented
- Undershoots (φ < 0) are prevented
- Monotonicity is preserved

**Why CRITICAL for VOF:**

The volume fraction α has **strict physical bounds**:

$$
0 \leq \alpha \leq 1
$$

| **Scheme Type** | **TVD Property** | **Result for VOF** |
|:---|:---|:---|
| CDS (Linear) | Not TVD | ❌ Can produce α < 0 or α > 1 |
| Upwind | TVD | ✅ Always bounded, but very diffusive |
| vanLeer | TVD | ✅ Bounded + sharper interface |
| SuperBee | TVD | ✅ Bounded + very sharp interface |

**Consequences of violating TVD:**

```cpp
// If α_f = -0.05 (unphysical from CDS)
// Mass conservation is violated:
Δα_P = (flux_out - flux_in) * dt / V_P
     = (positive_flux - negative_flux) * dt / V_P
     // Negative flux was actually inflow with wrong sign!
```

**TVD implementation via flux limiters:**

$$
\phi_f = \phi_f^{\text{UDS}} + \psi(r) \left[ \phi_f^{\text{CDS}} - \phi_f^{\text{UDS}} \right]
$$

where the limiter ψ(r) ensures TVD property:

$$
0 \leq \psi(r) \leq \min(2r, 2)
$$

**MULES:**

OpenFOAM's MULES (Multidimensional Universal Limiter for Explicit Solution) adds an additional layer of boundedness protection on top of TVD schemes, ensuring α stays within [0, 1] even for extreme density ratios.

</details>

### Q3: Compare Central Differencing vs Upwind for high Péclet numbers

<details>
<summary>Show Answer</summary>

**Answer:**

**Péclet number definition:**

$$
\text{Pe} = \frac{|\mathbf{U}| \Delta x}{\alpha} = \frac{\text{Convection}}{\text{Diffusion}}
$$

**Comparison table:**

| **Aspect** | **Central Differencing (CDS)** | **Upwind (UDS)** |
|:---|:---|:---|
| **Order** | Second-order $O(\Delta x^2)$ | First-order $O(\Delta x)$ |
| **Stability limit** | Unstable for Pe > 2 | Unconditionally stable |
| **Accuracy (Pe < 2)** | Excellent (2nd order) | Poor (diffusive) |
| **Accuracy (Pe >> 1)** | ❌ Oscillations | ✅ Stable but smeared |
| **Boundedness** | ❌ Not bounded | ✅ Guaranteed bounded |
| **Phase error** | High (dispersive) | Low |
| **Numerical diffusion** | Zero | $\frac{|\mathbf{U}|\Delta x}{2}$ |

**Stability analysis:**

**CDS discretization of 1D convection-diffusion:**

$$
\underbrace{-\text{Pe}}_{a_W} \phi_{i-1} + \underbrace{2}_{a_P} \phi_i + \underbrace{-\text{Pe}}_{a_E} \phi_{i+1} = 0
$$

For the matrix to be diagonally dominant (required for stability):

$$
|a_P| \geq |a_W| + |a_E| \implies 2 \geq 2\text{Pe} \implies \boxed{\text{Pe} \leq 1}
$$

**Practical interpretation:**

```
Pe = 0.1 (diffusion-dominated)
├─ CDS: ✅ Accurate, stable
└─ UDS: ⚠️  Overly diffusive, but stable

Pe = 10 (convection-dominated)
├─ CDS: ❌ Oscillates, may diverge
└─ UDS: ✅ Stable, but smeared

Pe = 1000 (highly convective)
├─ CDS: ❌❌ Diverges
└─ UDS: ✅✅ Stable (recommended)
```

**Recommended approach:**

For high Pe flows:
1. **Initial solution:** Use upwind for stability
2. **Refined solution:** Use TVD (vanLeer) for accuracy + boundedness
3. **Never use CDS** for pure convection (Pe → ∞)

**For R410A evaporation:**
- Pe ~ 1000 in liquid inlet
- Pe ~ 100 in vapor outlet
- Use upwind or TVD throughout
- CDS only for diffusive fluxes (viscous stress, heat conduction)

</details>

### Q4: Explain the role of flux limiter ψ(r) in TVD schemes

<details>
<summary>Show Answer</summary>

**Answer:**

**Flux limiter function ψ(r)** is the heart of TVD schemes. It controls the blending between first-order upwind (UDS) and second-order linear (CDS).

**Mathematical formulation:**

$$
\phi_f = \phi_f^{\text{UDS}} + \psi(r) \left[ \phi_f^{\text{CDS}} - \phi_f^{\text{UDS}} \right]
$$

where:
- ψ(r) = 0 → Pure upwind (first-order, bounded)
- ψ(r) = 1 → Pure linear (second-order, unbounded)
- 0 < ψ(r) < 1 → Blended TVD scheme

**Gradient ratio r:**

$$
r = \frac{\phi_P - \phi_U}{\phi_N - \phi_P} = \frac{\text{upwind gradient}}{\text{local gradient}}
$$

**Physical interpretation of r:**

| **Value** | **Local behavior** | **Desired ψ(r)** |
|:---|:---|:---|
| r < 0 | **Extremum nearby** (should not upwind) | ψ = 0 (use UDS) |
| r → 0 | **Sharp gradient** approaching | ψ → 0 (use UDS) |
| r = 1 | **Smooth region** (linear) | ψ = 1 (use CDS) |
| r → 2 | **Gradual variation** | ψ → 1 (use CDS) |
| r > 2 | **Very smooth** | ψ = 1 (use CDS) |

**Common limiters:**

| **Limiter** | **Formula: ψ(r)** | **Characteristics** |
|:---|:---|:---|
| Minmod | $\max(0, \min(1, r))$ | Most diffusive TVD |
| van Leer | $\frac{r + |r|}{1 + |r|}$ | Good balance, smooth |
| SuperBee | $\max(0, \min(2r, 1), \min(r, 2))$ | Most compressive TVD |
| van Albada | $\frac{r + r^2}{1 + r^2}$ | Differentiable |
| Ospre | $\frac{r(r+1)}{r^2+r+1}$ | Optimized accuracy |

**Sweby's TVD region:**

For TVD property, the limiter must satisfy:

$$
\boxed{0 \leq \psi(r) \leq \min(2r, 2)}
$$

**Visual representation:**

```
ψ(r)
1 │                    ┌───── van Leer
  │                ┌───┘
  │            ┌───┘
  │        ┌───┘
  │    ┌───┘
  │┌───┘                SuperBee (more aggressive)
0└┴──────────────────────────── r
  0    0.5    1    1.5    2

TVD Region: shaded area below ψ = min(2r, 2)
```

**Impact on VOF interface:**

| **Limiter** | **Interface Width** | **Sharpness** | **Stability** |
|:---|:---|:---|:---|
| Minmod | 3-4 cells | Diffused | Excellent |
| van Leer | 2-3 cells | ⭐ Good | Excellent |
| SuperBee | 1-2 cells | Very sharp | Good |

**⭐ VERIFIED: van Leer in OpenFOAM:**

> **File:** `vanLeer.H:80` → `return (r + mag(r))/(1 + mag(r));`

**⭐ VERIFIED: SuperBee in OpenFOAM:**

> **File:** `SuperBee.H:80` → `return max(max(min(2*r, 1), min(r, 2)), 0);`

## 6.3 Summary and Key Takeaways (สรุปและจุดสำคัญ)

### 📋 Essential Concepts

1. **Interpolation schemes estimate face values** from cell-centered data
   - Required by FVM for flux computation
   - Balance between accuracy, stability, and cost

2. **Trade-off triangle:** Accuracy ↔ Stability ↔ Cost
   - Can optimize at most two simultaneously
   - TVD schemes provide best balance for VOF

3. **TVD/NVD schemes provide boundedness**
   - Critical for VOF (0 ≤ α ≤ 1)
   - Flux limiter ψ(r) controls blending
   - vanLeer is the recommended default

4. **Upwind for stability, TVD for accuracy** in two-phase flows
   - Initial runs: upwind
   - Production runs: vanLeer or SuperBee
   - Never use CDS for volume fraction

5. **Never use pure CDS for volume fraction**
   - Causes unbounded α values
   - Leads to mass conservation errors
   - Always use TVD/NVD or upwind for VOF
