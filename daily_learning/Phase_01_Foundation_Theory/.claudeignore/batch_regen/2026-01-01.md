## Section 1: กฎการอนุรักษ์ (The Conservation Laws)

### 1.1 แนวคิด Control Volume (The Control Volume Concept)

ใน CFD เราไม่ได้ติดตามโมเลกุลทุกตัว แต่เรากำหนดปริมาตรคงที่ในอวกาศ (Eulerian approach) แล้วดูสมดุลของสิ่งที่ไหลเข้าและออก

**Visual Concept: Eulerian Control Volume**

```text
          Flux In (U_in)
             |
      +------+------+
      |             |
      |   Control   | ---> Flux Out (U_out)
      |   Volume    |
      |    (Vol)    |
      +------+------+
             |
       Source Term (S)
```
*Balance:* อัตราการเปลี่ยนแปลง = Flux เข้า - Flux ออก + แหล่งกำเนิด (Source)

### 1.2 Conservation of Mass (สมการความต่อเนื่อง)

สำหรับของไหลที่อัดตัวได้ (Compressible):
$$
\frac{\partial \rho}{\partial t} + \nabla \cdot (\rho \mathbf{U}) = 0
$$

**Variable Mapping (การจับคู่ตัวแปร):**
*   $\rho$ $\rightarrow$ `rho` (volScalarField)
*   $\mathbf{U}$ $\rightarrow$ `U` (volVectorField)
*   $\partial/\partial t$ $\rightarrow$ `fvm::ddt(rho)`
*   $\nabla \cdot (\rho \mathbf{U})$ $\rightarrow$ `fvm::div(phi, rho)` (โดยที่ `phi` คือ Face Flux)

**สำหรับ Incompressible Flow ($\rho = const$):**
$$
\nabla \cdot \mathbf{U} = 0
$$
นี่คือ **Kinematic Constraint** (ข้อจำกัดทางจลนศาสตร์) บนสนามความเร็ว

### 1.3 Conservation of Momentum (สมการ Navier-Stokes)

กฎข้อที่สองของนิวตัน ($F=ma$) ที่ประยุกต์ใช้กับของไหล:

$$
\frac{\partial (\rho \mathbf{U})}{\partial t} + \nabla \cdot (\rho \mathbf{U} \mathbf{U}) = -\nabla p + \nabla \cdot (\mu \nabla \mathbf{U}) + \rho \mathbf{g}
$$

**Term-by-Term Analysis & Mapping:**

| Term | Physics (ความหมาย) | OpenFOAM Code |
| :--- | :--- | :--- |
| $\frac{\partial (\rho \mathbf{U})}{\partial t}$ | **Unsteady:** ความเฉื่อยของของไหล | `fvm::ddt(rho, U)` |
| $\nabla \cdot (\rho \mathbf{U} \mathbf{U})$ | **Convection:** การพาโดยความเร็ว | `fvm::div(phi, U)` |
| $\nabla \cdot (\mu \nabla \mathbf{U})$ | **Diffusion:** แรงเสียดทานความหนืด | `fvm::laplacian(mu, U)` |
| $-\nabla p$ | **Pressure Gradient:** ตัวขับเคลื่อนการไหล | `-fvc::grad(p)` |
| $\rho \mathbf{g}$ | **Gravity:** แรงโน้มถ่วง (Body force) | `rho * g` |

> [!NOTE] **Why fvm vs fvc?**
> *   `fvm` (Finite Volume Method): คืนค่า **Matrix** (Implicit, เพื่อนำไปแก้สมการ)
> *   `fvc` (Finite Volume Calculus): คืนค่า **Field** (Explicit, คำนวณค่าได้เลยจากข้อมูลที่มีอยู่)
> *   โดยปกติ Pressure Gradient มักถูกปฏิบัติแบบ Explicit ใน PISO/SIMPLE loops จึงใช้ `fvc`

## Section 2: Phase Change Extensions

สำหรับ **Evaporator Project** ของเรา เราต้องจัดการกับของเหลว (Liquid) และไอ (Vapor) เราจะใช้วิธี **Volume of Fluid (VOF)**

### 2.1 สมการ VOF (The VOF Equation)

เราจะติดตามค่า Liquid Volume Fraction $\alpha$ (alpha):
*   $\alpha = 1$: ของเหลว 100% (Liquid)
*   $\alpha = 0$: ไอ 100% (Vapor)
*   $0 < \alpha < 1$: พื้นผิวรอยต่อ (Interface)

สมการ Transport สำหรับ $\alpha$:
$$
\frac{\partial \alpha}{\partial t} + \nabla \cdot (\mathbf{U} \alpha) + \nabla \cdot (\mathbf{U}_c \alpha (1-\alpha)) = \frac{\dot{m}}{\rho}
$$

*   **Compression Term:** $\nabla \cdot (\mathbf{U}_c \alpha (1-\alpha))$ เป็นเทอมเทียม (Artificial) เพื่อบีบให้ Interface คมชัด
*   **Source Term:** $\dot{m}$ คือการถ่ายเทมวล (Mass Transfer) จากการระเหย/ควบแน่น

**Variable Mapping (การจับคู่ตัวแปร):**
*   $\alpha$ $\rightarrow$ `alpha1` หรือ `alpha.water`
*   $\dot{m}$ $\rightarrow$ `mDot` (Custom field ที่เราต้องสร้าง)

**Visual Concept: VOF Interface**

```text
      Values of alpha:
      0.0   0.0   0.1   0.9   1.0   1.0
      [V] - [V] - [I] - [I] - [L] - [L]
                   ^     ^
               Interface Region
```

### 2.2 คุณสมบัติของของไหลผสม (Mixture Properties)

โดยปกติ เราจะแก้สมการ Momentum/Energy เพียงชุดเดียวสำหรับ "Mixture" ค่าคุณสมบัติจะเป็นค่าเฉลี่ยถ่วงน้ำหนัก:

$$
\rho_{mix} = \alpha \rho_l + (1 - \alpha) \rho_v
$$
$$
\mu_{mix} = \alpha \mu_l + (1 - \alpha) \mu_v
$$

**Code Implementation:**
```cpp
volScalarField rho = alpha1*rho1 + (1.0 - alpha1)*rho2;
```

## Section 3: โครงสร้าง Case OpenFOAM (OpenFOAM Case Structure)

การจำลอง OpenFOAM จะมีโครงสร้าง Directory ที่ตายตัว

**Visual Concept: Case Directory**

```text
    Case_Name/
    ├── 0/                  <-- Initial Conditions (t=0)
    │   ├── U
    │   ├── p
    │   └── alpha.water
    ├── constant/           <-- Mesh & Physics Properties
    │   ├── polyMesh/       <-- The Grid (Points, Faces, etc.)
    │   ├── transportProperties
    │   └── g
    └── system/             <-- Solver Settings
        ├── controlDict     <-- Time step, write interval
        ├── fvSchemes       <-- Discretization schemes (Upwind, Linear)
        └── fvSolution      <-- Linear Solver settings (PCG, PBiCG)
```

### 3.1 `system/controlDict`

สมองของ Simulation ควบคุมเรื่องเวลา (Time Stepping)

```cpp
application     interFlow;
startFrom       startTime;
startTime       0;
stopAt          endTime;
endTime         10;
deltaT          0.001; // ระวังเรื่อง CFL!
writeControl    timeStep;
writeInterval   100;
adjustTimeStep  yes;   // Adaptive time stepping
maxCo           1.0;
```

### 3.2 `system/fvSchemes`

ที่ที่คุณจับคู่ Operator ทางคณิตศาสตร์กับ Numerical Methods

```cpp
ddtSchemes
{
    default         Euler;          // Implicit First Order
}

gradSchemes
{
    default         Gauss linear;   // Standard Central Difference
}

divSchemes
{
    div(rho*phi,U)  Gauss upwind;   // Robust สำหรับ Momentum
    div(phi,alpha)  Gauss vanLeer;  // TVD สำหรับ Interface
}
```

## Section 4: Concept Checks (ตรวจสอบความเข้าใจ)

**Q1: มิติ (Dimension) ของ $\nabla p$ คืออะไร?**
> [!SUCCESS]- เฉลย (Answer)
> Pressure $p$ ใน Incompressible solvers มักจะเป็น Kinematic Pressure ($P/\rho$) ซึ่งมีหน่วย $m^2/s^2$ ดังนั้น $\nabla p$ คือความเร่ง ($m/s^2$) 
> ส่วนใน Compressible solvers, $p$ คือ Pascals ($kg/m\cdot s^2$) ดังนั้น $\nabla p$ คือ Force/Volume ($N/m^3$)

**Q2: ทำไมเทอม Compression $\alpha(1-\alpha)$ ถึงทำงานเฉพาะที่ Interface?**
> [!SUCCESS]- เฉลย (Answer)
> ถ้า $\alpha=0$ (ไอล้วน), $\alpha(1-\alpha) = 0$
> ถ้า $\alpha=1$ (ของเหลวล้วน), $\alpha(1-\alpha) = 0$
> มันจะมีค่าไม่เป็นศูนย์ (Non-zero) **เฉพาะเมื่อ** $0 < \alpha < 1$ หรือที่ Interface เท่านั้น นี่เป็นกลไกที่ช่วยให้เราบีบอัด Interface ให้คมชัดโดยไม่ไปรบกวนของไหลหลักทั้งสองฝั่ง

## Section 5: Deeper Dive - Tensors & Notation

เพื่ออ่าน Source Code ของ OpenFOAM ให้คล่องแคล่ว คุณต้องเข้าใจ **Tensor Notation**

### 5.1 Index Notation (Einstein Summation)

แทนที่จะเขียน Vector $\mathbf{U}$ เราเขียน $U_i$ (โดยที่ $i=1,2,3$ หรือ $x,y,z$)

*   **Vector:** $U_i$
*   **Gradient:** $\frac{\partial \phi}{\partial x_i} = \nabla \phi$
*   **Divergence:** $\frac{\partial U_i}{\partial x_i} = \nabla \cdot \mathbf{U}$ (ผลรวมเมื่อ Index ซ้ำกัน)
*   **Laplacian:** $\frac{\partial^2 \phi}{\partial x_i \partial x_i} = \nabla^2 \phi$

**OpenFOAM Tensor Classes:**
*   `scalar`: Rank 0 ($s$)
*   `vector`: Rank 1 ($v_i$)
*   `tensor`: Rank 2 ($T_{ij}$) (3x3 Matrix, ใช้สำหรับ Stress Tensor $\tau$)

**Visual Concept: Tensor Rank**

```text
    Scalar (s)      Vector (v)      Tensor (T)
      [ ]           [ ] [ ] [ ]     [ ] [ ] [ ]
                                     [ ] [ ] [ ]
                                     [ ] [ ] [ ]
    Temperature     Velocity        Shear Stress
```

### 5.2 การอ่านไฟล์ `createFields.H`

Solver ทุกตัวจะเริ่มจากการอ่าน Field จาก Directory `0/` ซึ่งเกิดขึ้นในไฟล์ `createFields.H`

```cpp
// createFields.H
Info<< "Reading field p\n" << endl;
volScalarField p
(
    IOobject
    (
        "p",                // ชื่อไฟล์บน Disk
        runTime.timeName(), // Directory (เช่น "0" หรือ "0.001")
        mesh,               // Mesh registry
        IOobject::MUST_READ,// Error ถ้าหาไฟล์ไม่เจอ
        IOobject::AUTO_WRITE // เขียนไฟล์เมื่อถึงเวลา (runTime.write())
    ),
    mesh
);

Info<< "Reading field U\n" << endl;
volVectorField U
(
    IOobject
    (
        "U",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);
```

> [!TIP] **IOobject Flags ที่ควรรู้**
> - `MUST_READ`: Solver จะ Crash ถ้าหาไฟล์ไม่เจอ (ใช้สำหรับ Primary fields เช่น $p, U$)
> - `READ_IF_PRESENT`: ดีสำหรับ Optional fields หรือการ Restart
> - `NO_READ`: ใช้สำหรับ Fields ที่คำนวณภายในโค้ด (Temporary fields)
