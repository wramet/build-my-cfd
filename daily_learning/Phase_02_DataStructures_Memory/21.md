# Day 21: Flat Arrays & Offsets — Compact Cell-to-Face Adjacency

## Part 1: Pattern Identification

### The Problem: `std::vector<std::vector<int>>` is Memory-Inefficient

In computational meshing and graph connectivity, a common requirement is storing adjacency lists where each entity (like a mesh cell) has a variable number of connected neighbors (like faces). The naive C++ approach uses a nested structure:

```cpp
std::vector<std::vector<int>> cellFaces;
```

While convenient, this structure suffers from significant overhead:
1. **Multiple Allocations**: Each inner `vector` performs its own heap allocation, causing memory fragmentation.
2. **Pointer Overhead**: Each `vector` stores three pointers (begin, end, capacity) - typically 24 bytes on 64-bit systems.
3. **Poor Spatial Locality**: The face indices for a single cell are contiguous, but faces for different cells are scattered in memory, hurting cache performance during traversal.
4. **Memory Overhead**: For small lists (common in meshes), the fixed overhead dominates.

### Historical Context: OpenFOAM's `CompactListList<T>`

OpenFOAM, a leading open-source CFD toolbox, historically faced this problem with large meshes (millions of cells). Their solution was `CompactListList<T>` - a specialized container storing all data in a single flat array with a separate offsets array. This design:
- Reduced memory fragmentation
- Improved cache coherence
- Enabled efficient serialization
- Minimized allocation overhead

### Modern Solution: Flat `std::vector<T>` + Offsets

The core insight is decoupling the *storage* from the *logical structure*:
- **Values vector**: Single contiguous array storing all face indices consecutively
- **Offsets vector**: Stores starting positions in the values array for each cell
- **Lengths vector** (optional): Can store counts, but lengths can be derived from offsets

```
Cell 0 faces: [10, 15, 23]  // 3 faces
Cell 1 faces: [5, 8]        // 2 faces
Cell 2 faces: [12, 18, 19, 21]  // 4 faces

Flat storage:
Values:  [10, 15, 23,  5,  8,  12, 18, 19, 21]
Offsets: [0,           3,      5,           9]
         ^ Cell 0      ^ Cell 1 ^ Cell 2     ^ End sentinel
```

## Part 2: Theory — Compact Layout

### Jagged Arrays in Flat Storage

A jagged array (ragged array) has rows of varying lengths. The compact representation flattens all elements into one array:

```
Logical structure:
Row 0: [a00, a01]
Row 1: [a10, a11, a12]
Row 2: [a13]

Flat storage:
Values:   [a00, a01, a10, a11, a12, a13]
Offsets:  [0, 2, 5, 6]  // Start index of each row
```

### Offsets Array: Stores Start Positions

The offsets array has `n+1` elements for `n` rows. The last element is the total number of values, serving as a sentinel:

```cpp
// For row i:
int start = offsets[i];
int end = offsets[i + 1];  // Exclusive end
int length = end - start;
```

This design enables:
- **O(1) random access** to row boundaries
- **No redundant length storage** (except as derived value)
- **Natural empty row representation** (start == end)

### Memory Footprint Comparison

Let's compare for `n` cells with average `k` faces each:

**Nested vectors:**
```
Memory = n * sizeof(std::vector<int>) + total_faces * sizeof(int)
       = n * 24 + total_faces * 4  // Typical 64-bit, 4-byte ints
```

**Compact representation:**
```
Memory = (n + 1) * sizeof(int) + total_faces * sizeof(int)
       = (n + 1) * 4 + total_faces * 4
```

**Savings:**
```
Overhead reduction = 24n - 4(n + 1)
                   = 20n - 4 bytes
```

For 1 million cells: ~20MB saved in overhead alone!

### Zero-Copy Slicing with `std::span`

C++17/20's `std::span` provides a view into contiguous memory without ownership:

```cpp
std::span<const int> getFaces(int cellId) const {
    int start = offsets[cellId];
    int end = offsets[cellId + 1];
    return std::span<const int>(values.data() + start, end - start);
}
```

Benefits:
- **Zero allocation**: Returns a lightweight view
- **Bounds-safe**: Knows its size
- **Iterable**: Works with range-based for loops
- **Composable**: Can be passed to algorithms

## Part 3: C++ Mechanics

### Building `cell_faces(cellId)` Returning `std::span<int>`

The key interface method:

```cpp
class CellFaceAdjacency {
    std::vector<int> values;      // All face indices
    std::vector<int> offsets;     // Start positions (n+1 elements)

public:
    std::span<int> getFaces(int cellId) {
        int start = offsets[cellId];
        int end = offsets[cellId + 1];
        return std::span<int>(values.data() + start, end - start);
    }

    std::span<const int> getFaces(int cellId) const {
        // Const version for read-only access
        int start = offsets[cellId];
        int end = offsets[cellId + 1];
        return std::span<const int>(values.data() + start, end - start);
    }
};
```

### Calculating Memory Usage

```cpp
size_t memoryBytes() const {
    return values.capacity() * sizeof(int) +
           offsets.capacity() * sizeof(int);
}

size_t memoryBytesNested(const std::vector<std::vector<int>>& nested) {
    size_t total = nested.capacity() * sizeof(std::vector<int>);
    for (const auto& vec : nested) {
        total += vec.capacity() * sizeof(int);
    }
    return total;
}
```

### Access Patterns: Sequential Iteration

**Nested vectors** (poor locality):
```cpp
for (const auto& faces : cellFaces) {  // Jump between allocations
    for (int faceId : faces) {         // Contiguous within cell
        process(faceId);
    }
}
```

**Compact storage** (better locality):
```cpp
for (int cellId = 0; cellId < nCells; ++cellId) {
    auto faces = getFaces(cellId);  // Computes start/end
    for (int faceId : faces) {      // All data contiguous
        process(faceId);
    }
}
```

The compact version has better spatial locality because all face indices are stored contiguously, improving cache utilization.

## Part 4: Implementation

### `CellFaceAdjacency` Class with Flat Storage

```cpp
#include <vector>
#include <span>
#include <cassert>
#include <algorithm>
#include <numeric>
#include <iostream>
#include <chrono>
#include <random>

class CellFaceAdjacency {
private:
    std::vector<int> values_;      // Flattened face indices
    std::vector<int> offsets_;     // offsets_[i] = start index for cell i

public:
    // Default constructor
    CellFaceAdjacency() = default;

    // Construct from nested vectors
    explicit CellFaceAdjacency(const std::vector<std::vector<int>>& nested) {
        buildFromNested(nested);
    }

    // Build from nested vectors
    void buildFromNested(const std::vector<std::vector<int>>& nested) {
        int nCells = static_cast<int>(nested.size());
        offsets_.resize(nCells + 1);

        // Compute offsets as prefix sum of lengths
        offsets_[0] = 0;
        for (int i = 0; i < nCells; ++i) {
            offsets_[i + 1] = offsets_[i] + static_cast<int>(nested[i].size());
        }

        // Flatten values
        int totalFaces = offsets_.back();
        values_.resize(totalFaces);

        int pos = 0;
        for (const auto& faces : nested) {
            std::copy(faces.begin(), faces.end(), values_.begin() + pos);
            pos += static_cast<int>(faces.size());
        }
    }

    // Access faces for a cell
    std::span<int> getFaces(int cellId) {
        assert(cellId >= 0 && cellId + 1 < static_cast<int>(offsets_.size()));
        int start = offsets_[cellId];
        int end = offsets_[cellId + 1];
        return std::span<int>(values_.data() + start, end - start);
    }

    std::span<const int> getFaces(int cellId) const {
        assert(cellId >= 0 && cellId + 1 < static_cast<int>(offsets_.size()));
        int start = offsets_[cellId];
        int end = offsets_[cellId + 1];
        return std::span<const int>(values_.data() + start, end - start);
    }

    // Number of cells
    int numCells() const { return static_cast<int>(offsets_.size()) - 1; }

    // Total number of face entries
    int totalFaceEntries() const { return static_cast<int>(values_.size()); }

    // Get number of faces for a cell
    int numFaces(int cellId) const {
        assert(cellId >= 0 && cellId + 1 < static_cast<int>(offsets_.size()));
        return offsets_[cellId + 1] - offsets_[cellId];
    }

    // Memory usage in bytes
    size_t memoryBytes() const {
        return values_.capacity() * sizeof(int) +
               offsets_.capacity() * sizeof(int);
    }
};

// Calculate memory for nested vector
size_t memoryBytesNested(const std::vector<std::vector<int>>& nested) {
    size_t total = nested.capacity() * sizeof(std::vector<int>);
    for (const auto& vec : nested) {
        total += vec.capacity() * sizeof(int);
    }
    return total;
}
```

### Memory Comparison Benchmark

```cpp
void runMemoryComparison() {
    std::cout << "=== Memory Usage Comparison ===\n";

    // Generate test mesh connectivity
    const int nCells = 1000000;  // 1 million cells
    const int avgFacesPerCell = 5;  // Average faces per cell
    const int totalFaces = nCells * avgFacesPerCell;

    // Create nested structure
    std::vector<std::vector<int>> nested;
    nested.reserve(nCells);

    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_int_distribution<> faceDist(3, 7);  // 3-7 faces per cell

    // Generate random connectivity
    int faceCounter = 0;
    for (int i = 0; i < nCells; ++i) {
        int nFaces = faceDist(gen);
        std::vector<int> faces(nFaces);
        for (int j = 0; j < nFaces; ++j) {
            faces[j] = faceCounter++;
        }
        nested.push_back(std::move(faces));
    }

    // Measure nested memory
    size_t nestedMemory = memoryBytesNested(nested);

    // Create compact version
    CellFaceAdjacency compact(nested);
    size_t compactMemory = compact.memoryBytes();

    // Calculate savings
    double savingsMB = (nestedMemory - compactMemory) / (1024.0 * 1024.0);
    double percent = 100.0 * (nestedMemory - compactMemory) / nestedMemory;

    std::cout << "Nested vectors: " << nestedMemory / (1024.0 * 1024.0)
              << " MB\n";
    std::cout << "Compact storage: " << compactMemory / (1024.0 * 1024.0)
              << " MB\n";
    std::cout << "Savings: " << savingsMB << " MB (" << percent << "%)\n";
    std::cout << "Cells: " << nCells
              << ", Total face entries: " << compact.totalFaceEntries() << "\n";
}
```

```cpp
void runAccessTimeComparison() {

    const int nCells = 500000;
    const int iterations = 100;

    // Create test data
    std::vector<std::vector<int>> nested;
    nested.reserve(nCells);

    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_int_distribution<> faceDist(4, 8);

    for (int i = 0; i < nCells; ++i) {
        int nFaces = faceDist(gen);
        std::vector<int> faces(nFaces);
        for (int j = 0; j < nFaces; ++j) {
            faces[j] = i * 10 + j;  // Some pattern
        }
        nested.push_back(std::move(faces));
    }

    CellFaceAdjacency compact(nested);

    // Benchmark nested access
    auto start = std::chrono::high_resolution_clock::now();
    long long nestedSum = 0;

    for (int iter = 0; iter < iterations; ++iter) {
        for (int cellId = 0; cellId < nCells; ++cellId) {
            for (int faceId : nested[cellId]) {
                nestedSum += faceId;
            }
        }
    }

    auto mid = std::chrono::high_resolution_clock::now();

    // Benchmark compact access
    long long compactSum = 0;

    for (int iter = 0; iter < iterations; ++iter) {
        for (int cellId = 0; cellId < nCells; ++cellId) {
            auto faces = compact.getFaces(cellId);
            for (int faceId : faces) {
                compactSum += faceId;
            }
        }
    }

    auto end = std::chrono::high_resolution_clock::now();

    auto nestedTime = std::chrono::duration_cast<std::chrono::milliseconds>(mid - start);
    auto compactTime = std::chrono::duration_cast<std::chrono::milliseconds>(end - mid);

    std::cout << "Nested access time: " << nestedTime.count() << " ms\n";
    std::cout << "Compact access time: " << compactTime.count() << " ms\n";
    std::cout << "Speedup: " << (double)nestedTime.count() / compactTime.count()
              << "x\n";
    std::cout << "Checksum (should match): nested=" << nestedSum
              << ", compact=" << compactSum << "\n";
}
```

## Part 5: Trade-offs

### When to Use Compact vs Nested

| Scenario | Recommended | Reason |
|----------|-------------|--------|
| Large datasets (>100K elements) | Compact | Memory savings significant |
| Frequent insertions/deletions | Nested | Easier to modify structure |
| Random access needed | Both work, compact slightly faster | Better cache locality |
| Serialize to disk/network | Compact | Single memcpy |
| Memory-constrained | Compact | Minimal overhead |

### Cache Behavior Differences

**Nested vectors:**
- Each cell's faces are contiguous
- Different cells' faces are scattered
- Cache misses between cells

**Compact storage:**
- All faces stored contiguously
- Sequential access hits cache repeatedly
- Better prefetcher effectiveness

### Real CFD Mesh Example

For a 3D hex-dominant mesh:
- Interior cells: 6 faces
- Boundary cells: 5 faces
- Edge cells: 4 faces
- Corner cells: 3 faces

**Memory for 10 million cells:**
- Nested: ~240 MB overhead + face data
- Compact: ~40 MB overhead + face data
- **Savings: ~200 MB**

**Deliverable:** A `CellFaceAdjacency` class storing connectivity as flat arrays. Memory and access-time comparison vs nested `vector<vector>`.

### Implementation: Complete `CellFaceAdjacency`

```cpp
#include <vector>
#include <span>
#include <numeric>
#include <cassert>

class CellFaceAdjacency {
    std::vector<int>    faces_;    // flat face indices
    std::vector<int>    offsets_;  // offsets_[c] = start of cell c's faces
public:
    // Build from a ragged array of face lists
    explicit CellFaceAdjacency(std::vector<std::vector<int>> cell_faces) {
        offsets_.resize(cell_faces.size() + 1);
        offsets_[0] = 0;
        for (size_t c = 0; c < cell_faces.size(); ++c)
            offsets_[c+1] = offsets_[c] + (int)cell_faces[c].size();
        faces_.resize(offsets_.back());
        for (size_t c = 0; c < cell_faces.size(); ++c)
            std::copy(cell_faces[c].begin(), cell_faces[c].end(),
                      faces_.begin() + offsets_[c]);
    }

    // Zero-copy view of cell c's faces (Day 20 pattern)
    std::span<const int> faces(int c) const {
        return {faces_.data() + offsets_[c],
                static_cast<size_t>(offsets_[c+1] - offsets_[c])};
    }

    size_t num_cells() const { return offsets_.size() - 1; }
    size_t total_faces() const { return faces_.size(); }
    size_t memory_bytes() const {
        return faces_.capacity() * sizeof(int)
             + offsets_.capacity() * sizeof(int);
    }
};
```

> **Connection to Day 20:** The `faces(int c)` method returns `std::span<const int>` — a zero-copy view into the flat array, exactly the pattern from Day 20. No allocation, no copying, just a pointer + length.

### Quick Reference — Offset Array Arithmetic

| Operation | Formula | Cost |
|-----------|---------|------|
| Start index for cell `c` | `offsets[c]` | O(1) |
| End index for cell `c` | `offsets[c+1]` | O(1) |
| Face count for cell `c` | `offsets[c+1] - offsets[c]` | O(1) |
| Total face count | `offsets.back()` | O(1) |
| Insert new cell (amortized) | `push_back` to both arrays | O(1) amortized |

This is the same offset array pattern used in CSR (Compressed Sparse Row) matrices for the row pointer array — a universal technique in sparse data structures.

The pattern also appears in OpenFOAM's `labelListList` serialization and in GPU graphics APIs (Vulkan draw indirect commands use identical offset structures for variable-length draw calls).
