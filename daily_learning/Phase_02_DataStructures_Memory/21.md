# Day 21: DynamicList & CompactListList - Variable-Size Containers

## Introduction to Variable-Size Containers

In CFD computations, we frequently deal with variable-sized data structures:
- Cell-to-face connectivity (cells have different numbers of faces)
- Face-to-cell connectivity (internal faces have 2 cells, boundary faces have 1)
- Dynamic mesh adaptation (adding/removing cells during simulation)

OpenFOAM provides two specialized containers for these needs:
- `DynamicList<T>` - Growable array with capacity management
- `CompactListList<T>` - Flat array with offsets (similar to CSR format)

These containers address the limitations of `std::vector` in CFD contexts by providing:
1. **Memory efficiency**: Pre-allocation for known growth patterns
2. **View semantics**: Lightweight access to sub-lists
3. **Cache-friendly layout**: Contiguous memory for performance

## DynamicList<T> - Growable Container with Capacity (DynamicList<T> - ตัวเก็บข้อมูลที่ขยายได้)

⭐ ⭐ ### Design Philosophy (หลักการออกแบบ) (หลักการออกแบบ)

`DynamicList<T>` extends `List<T>` with automatic capacity growth while maintaining view semantics. It's designed for scenarios where we know the maximum size but want efficient growth.

```cpp
// Example usage in CFD context

cellFaces.reserve(mesh.nCells() * 8);  // Estimate: ~8 faces per cell
```

```cpp
// DynamicList<T> implementation

template<class T>
class DynamicList : public List<T>
{
    label capacityIncrement_;  // Growth step size

public:
    // Constructors: DynamicList(size, value, "name")
    DynamicList() : List<T>(), capacityIncrement_(100) {}

    explicit DynamicList(const label size)
    : List<T>(size), capacityIncrement_(100) {}

    DynamicList(const label size, const T& val)
    : List<T>(size, val), capacityIncrement_(100) {}

    DynamicList(const label size, const T& val, const label capacityIncrement)
    : List<T>(size, val), capacityIncrement_(capacityIncrement) {}

    // Set capacity increment
    void setCapacityIncrement(const label delta) {
        capacityIncrement_ = delta;
    }

    // Append with automatic growth
    void append(const T& val) {
        if (this->size() == this->capacity()) {
            this->reserve(this->capacity() + capacityIncrement_);
        }
        List<T>::push_back(val);
    }

    void append(T&& val) {
        if (this->size() == this->capacity()) {
            this->reserve(this->capacity() + capacityIncrement_);
        }
        List<T>::push_back(std::move(val));
    }

    // Append multiple values
    void append(const UList<T>& values) {
        const label required = this->size() + values.size();
        if (required > this->capacity()) {
            this->reserve(required);
        }
        for (label i = 0; i < values.size(); ++i) {
            List<T>::push_back(values[i]);
        }
    }

    // Append with growth control
    void append(const UList<T>& values, const label minGrow) {
        const label required = this->size() + values.size();
        if (required > this->capacity()) {
            this->reserve(Foam::max(required, this->capacity() + minGrow));
        }
        for (label i = 0; i < values.size(); ++i) {
            List<T>::push_back(values[i]);
        }
    }

    // shrink_to_fit() - reduce capacity to size
    void shrink() {
        if (this->size() < this->capacity()) {
            this->setSize(this->size());  // reallocates to exact size
        }
    }

    // Optimized clear() without deallocating
    void clear() {
        this->setSize(0);
    }

    // Safe erase with bounds checking
    void erase(const label pos, const label len = 1) {
        if (pos + len > this->size()) {
            throw std::out_of_range("erase position out of range");
        }
        List<T>::erase(pos, len);
    }
};
```

⭐ ### Mathematical Analysis of Growth Strategy (การวิเคราะห์กลยุทธ์การเติบโต)

The growth strategy follows a mathematical pattern:

Let $n_0$ be the initial capacity, $\delta$ be the increment, and $k$ be the number of appends.

The capacity after $k$ appensions is:

$$
c_k = n_0 + \delta \cdot \lceil \frac{k}{\delta} \rceil
$$

The amortized time complexity for $k$ appensions is:

$$
T_{total} = O(n_0 + k)
$$

Thus, amortized cost per operation is $O(1)$.

## CompactListList<T> - CSR-like Container (CompactListList<T> - เหมือน CSR)

⭐ ⭐ ### Design Philosophy (หลักการออกแบบ) (หลักการออกแบบ)

`CompactListList<T>` implements a compressed sparse row (CSR) pattern where:
- All data is stored in a single contiguous array
- Offsets array marks the start of each sub-list
- Provides O(1) access to sub-lists with O(1) offset lookup

This is ideal for CFD connectivity data:
- Cell-to-face mapping
- Face-to-cell mapping
- Matrix storage for linear systems

⭐ ### Mathematical Foundation (พื้นฐานทางคณิตศาสตร์)

Given $n$ cells with varying numbers of faces per cell:

- $\mathbf{F}$ = Flat array of all face labels
- $\mathbf{O}$ = Offset array where $O_i$ = start index in $\mathbf{F}$ for cell $i$
- $O_n$ = total number of faces (size of $\mathbf{F}$)

For cell $i$, faces are $\mathbf{F}[O_i..O_{i+1}-1]$.

```cpp
// File: /Users/woramet/Documents/Build My CFD/.claude/worktrees/upbeat-banach/daily_learning/Phase_02_DataStructures_Memory/21.md
// CompactListList<T> implementation

template<class T>
class CompactListList
{
    // Flat storage for all elements
    List<T> list_;

    // Offsets for each sub-list
    List<label> offsets_;

public:
    // Constructors: CompactListList(size, value, "name")
    CompactListList() : offsets_(1, 0) {}  // Single offset at 0

    explicit CompactListList(const label size)
    : offsets_(size + 1, 0) {}  // Size+1 offsets

    // Add elements to a sub-list
    void append(const label listI, const T& val) {
        if (listI < 0 || listI >= offsets_.size() - 1) {
            throw std::out_of_range("list index out of range");
        }
        list_.append(val);
        // Update all subsequent offsets
        for (label i = listI + 1; i < offsets_.size(); ++i) {
            offsets_[i]++;
        }
    }

    // Get sub-list as UList
    UList<T> operator[](const label listI) const {
        if (listI < 0 || listI >= offsets_.size() - 1) {
            throw std::out_of_range("list index out of range");
        }
        label start = offsets_[listI];
        label size = offsets_[listI + 1] - start;
        return UList<T>(list_.begin() + start, size);
    }

    // Get size of a sub-list
    label size(const label listI) const {
        if (listI < 0 || listI >= offsets_.size() - 1) {
            throw std::out_of_range("list index out of range");
        }
        return offsets_[listI + 1] - offsets_[listI];
    }

    // Total number of elements
    label size() const { return list_.size(); }

    // Number of sub-lists
    label nLists() const { return offsets_.size() - 1; }

    // Resize (add empty sub-lists)
    void setSize(const label newSize) {
        if (newSize > offsets_.size() - 1) {
            offsets_.setSize(newSize + 1);
            // Fill new offsets with current total size
            for (label i = offsets_.size() - 2; i >= offsets_.oldSize() - 1; --i) {
                offsets_[i] = list_.size();
            }
        }
    }

    // Optimized construction from connectivity
    void buildFromConnectivity(const UList< List<T> >& connectivity) {
        // First pass: count total elements
        label totalElements = 0;
        for (label i = 0; i < connectivity.size(); ++i) {
            totalElements += connectivity[i].size();
        }

        // Set sizes
        list_.setSize(totalElements);
        offsets_.setSize(connectivity.size() + 1);

        // Second pass: fill data
        label currentPos = 0;
        for (label i = 0; i < connectivity.size(); ++i) {
            offsets_[i] = currentPos;
            for (label j = 0; j < connectivity[i].size(); ++j) {
                list_[currentPos++] = connectivity[i][j];
            }
        }
        offsets_[connectivity.size()] = currentPos;
    }
};
```

⭐ ### Performance Analysis (การวิเคราะห์ประสิทธิภาพ)

#### Access Pattern
- Sub-list access: $O(1)$ - Two offset lookups
- Element access: $O(1)$ - Direct memory access
- Space overhead: $O(m)$ for $m$ sub-lists

#### Cache Performance
The flat layout provides excellent cache locality:

$$
\text{Cache utilization} = \frac{\text{total data size}}{\text{data size + offset size}}
$$

For large connectivity data, this approaches 100%.

## Comparison with Other Containers

| Feature | DynamicList<T> | CompactListList<T> | std::vector<std::vector<T>> |
|---------|----------------|-------------------|----------------------------|
| Memory layout | Contiguous | Single contiguous | Multiple arrays |
| Access time | O(1) | O(1) + offset | O(1) |
| Sub-list access | No direct support | O(1) | O(1) |
| Growth overhead | Amortized O(1) | O(n) for append | Amortized O(1) |
| Cache efficiency | Excellent | Excellent | Poor (pointer chasing) |

```cpp
// Compact CSR storage for CFD
template<class T>
class CFD_CompactListList
{
private:
    List<T> flatData_;
    List<label> rowPtr_;

public:
    CFD_CompactListList() : rowPtr_(1, 0) {}

    // Build from coordinate format
    void buildFromCOO(const std::vector<std::tuple<label, label, T>>& coo) {
        // Count non-zeros per row
        label maxRow = 0;
        for (const auto& [row, col, val] : coo) {
            maxRow = Foam::max(maxRow, row);
        }

        // Initialize row pointers
        rowPtr_.setSize(maxRow + 2, 0);  // +2 for safety

        // Count non-zeros per row
        for (const auto& [row, col, val] : coo) {
            rowPtr_[row + 1]++;
        }

        // Convert to cumulative sum
        for (label i = 1; i < rowPtr_.size(); ++i) {
            rowPtr_[i] += rowPtr_[i - 1];
        }

        // Allocate and fill data
        flatData_.setSize(rowPtr_.back());
        auto tempCounts = rowPtr_;  // Copy for counting

        for (const auto& [row, col, val] : coo) {
            label pos = tempCounts[row]++;
            flatData_[pos] = val;
        }
    }

    // Matrix-vector product
    void matrixVectorMultiply(const UList<T>& x, List<T>& y) const {
        y.setSize(rowPtr_.size() - 1, 0);

        for (label row = 0; row < y.size(); ++row) {
            for (label i = rowPtr_[row]; i < rowPtr_[row + 1]; ++i) {
                y[row] += flatData_[i] * x[i];
            }
        }
    }

    // Access row
    UList<T> operator[](const label row) const {
        if (row < 0 || row >= rowPtr_.size() - 1) {
            throw std::out_of_range("row index out of range");
        }
        label start = rowPtr_[row];
        label size = rowPtr_[row + 1] - start;
        return UList<T>(flatData_.begin() + start, size);
    }

    // Sparsity pattern
    std::vector<label> getSparsityPattern() const {
        std::vector<label> pattern;
        for (label i = 0; i < rowPtr_.size() - 1; ++i) {
            pattern.push_back(rowPtr_[i + 1] - rowPtr_[i]);
        }
        return pattern;
    }
};
```

## Mathematical Analysis of Container Performance

### Memory Usage Analysis

Let $n$ be the number of cells, $m$ be the average faces per cell, and $s$ be the size of a face label.

#### DynamicList<T>
- Memory = $n \cdot m \cdot s$
- Overhead = $O(1)$ per element
- Total = $O(n \cdot m \cdot s)$

#### CompactListList<T>
- Memory = $n \cdot m \cdot s + (n + 1) \cdot \text{sizeof(label)}$
- Overhead = $O(n)$ offsets
- Total = $O(n \cdot m \cdot s)$ (same order)

#### std::vector<std::vector<T>>
- Memory = $n \cdot m \cdot s + n \cdot \text{sizeof(T*)}$
- Overhead = $n$ pointers + dynamic allocation per vector
- Total = $O(n \cdot m \cdot s)$ but with higher constant factors

### Cache Performance

The cache miss rate for different access patterns:

#### Linear Access
$$
P_{\text{miss}} = \frac{\text{cache line size}}{\text{array size} \cdot \text{element size}}
$$

#### Pointer Chasing (std::vector<std::vector<T>>)
$$
P_{\text{miss}} \approx 1 - \frac{1}{1 + \frac{\text{cache line size}}{\text{element size}}}
$$

For typical cache line size (64B) and element size (8B for label):
- Linear: ~12.5% miss rate
- Pointer chasing: ~87.5% miss rate

```cpp
template<class T>
class AdaptiveDynamicList : public DynamicList<T>
{
    double growthFactor_;
    label minIncrement_;

public:
    AdaptiveDynamicList()
    : DynamicList<T>(), growthFactor_(1.5), minIncrement_(100) {}

    void setGrowthStrategy(double factor, label minIncrement) {
        growthFactor_ = factor;
        minIncrement_ = minIncrement;
    }

    void appendWithGrowth(const T& val) {
        if (this->size() == this->capacity()) {
            label newCapacity = this->capacity() * growthFactor_;
            if (newCapacity - this->capacity() < minIncrement_) {
                newCapacity = this->capacity() + minIncrement_;
            }
            this->reserve(newCapacity);
        }
        List<T>::push_back(val);
    }
};
```

```cpp
        if (isFaceInCell(connectivity, cellI, faceI)) {
            return cellI;
        }
    }
    return -1;  // Not found
}

std::vector<label> findCellsSharingFace(
    const CFD_CompactListList<label>& connectivity,
    const label faceI
) {
    std::vector<label> cells;
    for (label cellI = 0; cellI < connectivity.nLists(); ++cellI) {
        if (isFaceInCell(connectivity, cellI, faceI)) {
            cells.push_back(cellI);
        }
    }
    return cells;
}
```
