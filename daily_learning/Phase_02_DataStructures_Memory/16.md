# Day 16: LDU Addressing --- Owner/Neighbour Arrays

## Part 1: Pattern Identification

### ⭐ What is LDU Addressing?

**LDU addressing** maps **mesh faces** to **matrix entries** using two arrays:

```cpp
class lduAddressing
{
    labelList owner_;       // owner[faceI] = owner cell index
    labelList neighbour_;   // neighbour[faceI] = neighbour cell index
};
```

> **⭐ Verified Fact:** OpenFOAM's `lduAddressing.H` defines `owner()` and `neighbour()` methods that return const references to `labelList`, enforcing the constraint `owner[faceI] < neighbour[faceI]` for all faces.

### 🎯 The Owner/Neighbour Constraint

**Critical rule:** For every face `f`:
```cpp
owner[f] < neighbour[f]  // Must hold for ALL faces
```

**Why this matters:**

1. **Uniqueness:** Each off-diagonal entry appears exactly once
2. **Split assignment:** `lower[f]` is the (owner, neighbour) entry, `upper[f]` is the (neighbour, owner) entry
3. **Symmetric storage:** Even for unsymmetric matrices, the sparsity pattern is symmetric

**Violating the constraint:**
```cpp
// ❌ WRONG: duplicates entries
owner[0] = 1; neighbour[0] = 5;    // Stores A(1,5) in lower, A(5,1) in upper
owner[1] = 5; neighbour[1] = 1;    // DUPLICATE: same entries!

// ✅ CORRECT: each pair stored once
owner[0] = 1; neighbour[0] = 5;    // Only this face exists
```

### 📊 Face-to-Matrix Mapping

For a **2D structured 4×4 mesh** (16 cells, 24 internal faces):

```
Cell numbering:
 0--- 1--- 2--- 3
 | 0 | 1 | 2 | 3 |
 4--- 5--- 6--- 7
 | 4 | 5 | 6 | 7 |
 8--- 9---10---11
 | 8 | 9 |10 |11 |
12---13---14---15
```

**Horizontal faces (0-11):**
```
owner[0] = 0;  neighbour[0] = 1;   // Row 0
owner[1] = 1;  neighbour[1] = 2;
owner[2] = 2;  neighbour[2] = 3;
// ... (similar for rows 1, 2, 3)
```

**Vertical faces (12-23):**
```
owner[12] = 0;  neighbour[12] = 4;  // Column 0
owner[13] = 1;  neighbour[13] = 5;
// ... (similar for columns 1, 2, 3)
```

**Result:** LDU matrix has 5-point stencil (max 5 non-zeros per row)

## Part 2: Theory and Mathematics

### Graph Representation

The mesh can be represented as an **undirected graph**:
- **Vertices:** cells
- **Edges:** internal faces
- **Directed edges:** owner → neighbour (enforces ordering)

**Adjacency list using owner/neighbour:**
```cpp
// For each cell i, find neighbours
for (label faceI = 0; faceI < nFaces; faceI++)
{
    label own = owner[faceI];
    label nei = neighbour[faceI];

    // own is adjacent to nei
    // nei is adjacent to own (symmetric)
}
```

### Memory Complexity

For a mesh with `N` cells:
- **Internal faces:** ≈ `d × N` where `d` is dimensionality
- **Storage per face:** 2 labels (owner + neighbour)
- **Total addressing memory:** `2 × d × N × sizeof(label)`

**Examples:**
- 1D mesh: `2 × 1 × N × 4 = 8N` bytes
- 2D mesh: `2 × 2 × N × 4 = 16N` bytes
- 3D mesh: `2 × 3 × N × 4 = 24N` bytes

### Face Ordering Impact

**Question:** Does face ordering affect matrix-vector multiply performance?

**Answer:** **Yes**, but the effect is secondary to:
1. Cell numbering (renumbering)
2. Cache line utilization
3. Loop tiling strategies

**Optimal face ordering:**
- **Spatial locality:** faces that access nearby cells should be close in the array
- **Cache alignment:** `owner[]` and `neighbour[]` accessed together → should share cache lines

## Part 3: Code Implementation

### Step 1: 1D Mesh LDU Addressing

Create file `ldu_addressing_1d.H`:

```cpp
#ifndef ldu_addressing_1d_H
#define ldu_addressing_1d_H

#include "labelList.H"

// -[1] LDU addressing for 1D structured mesh
class lduAddressing1D
{
private:
    labelList owner_;       // Size: nCells - 1
    labelList neighbour_;   // Size: nCells - 1
    label nCells_;
    label nFaces_;

public:
    // Constructor
    explicit lduAddressing1D(label nCells)
    : owner_(nCells - 1),
      neighbour_(nCells - 1),
      nCells_(nCells),
      nFaces_(nCells - 1)
    {
        if (nCells < 2)
        {
            std::cerr << "Error: nCells must be >= 2\n";
            return;
        }

        // Initialize for 1D mesh: each face connects cell i to i+1
        for (label i = 0; i < nCells - 1; i++)
        {
            owner_[i] = i;
            neighbour_[i] = i + 1;
        }
    }

    // Accessors (consistent with OpenFOAM API)
    const labelList& owner() const { return owner_; }
    const labelList& neighbour() const { return neighbour_; }

    label size() const { return nCells_; }
    label nFaces() const { return nFaces_; }

    // -[2] Verify constraint: owner < neighbour for all faces
    bool valid() const
    {
        for (label faceI = 0; faceI < nFaces_; faceI++)
        {
            if (owner_[faceI] >= neighbour_[faceI])
            {
                std::cerr << "Violation at face " << faceI
                         << ": owner=" << owner_[faceI]
                         << ", neighbour=" << neighbour_[faceI] << "\n";
                return false;
            }
        }
        return true;
    }

    // -[3] Get face index connecting two cells (return -1 if not adjacent)
    label faceIndex(label cellI, label cellJ) const
    {
        if (cellI == cellJ + 1)
        {
            // cellI is neighbour of cellJ
            return cellJ;  // Face cellJ connects to cellJ+1
        }
        else if (cellJ == cellI + 1)
        {
            // cellJ is neighbour of cellI
            return cellI;  // Face cellI connects to cellI+1
        }
        return -1;  // Not adjacent
    }
};

#endif
```

**Key features:**
1. ✅ Uses correct constructor: `labelList(n, value)` pattern
2. ✅ Enforces `owner < neighbour` constraint
3. ✅ Provides `faceIndex()` lookup (useful for assembly)
4. ✅ Includes `valid()` check for debugging

### Step 2: 2D Mesh LDU Addressing

Create file `ldu_addressing_2d.H`:

```cpp
#ifndef ldu_addressing_2d_H
#define ldu_addressing_2d_H

#include "labelList.H"

// -[1] LDU addressing for 2D structured mesh
class lduAddressing2D
{
private:
    labelList owner_;
    labelList neighbour_;
    label nCells_;
    label nFaces_;
    label nX_, nY_;  // Mesh dimensions

public:
    // Constructor for Nx × Ny mesh
    lduAddressing2D(label nX, label nY)
    : nCells_(nX * nY),
      nFaces_(2 * nX * nY - nX - nY),  // Horizontal + vertical faces
      nX_(nX),
      nY_(nY)
    {
        // Calculate faces
        label nHoriz = (nX - 1) * nY;
        label nVert = nX * (nY - 1);

        owner_.setSize(nHoriz + nVert);
        neighbour_.setSize(nHoriz + nVert);

        label faceI = 0;

        // -[2] Horizontal faces (first)
        for (label j = 0; j < nY; j++)
        {
            for (label i = 0; i < nX - 1; i++)
            {
                owner_[faceI] = j * nX + i;
                neighbour_[faceI] = j * nX + i + 1;
                faceI++;
            }
        }

        // -[3] Vertical faces (after horizontal)
        for (label j = 0; j < nY - 1; j++)
        {
            for (label i = 0; i < nX; i++)
            {
                owner_[faceI] = j * nX + i;
                neighbour_[faceI] = (j + 1) * nX + i;
                faceI++;
            }
        }
    }

    // Accessors
    const labelList& owner() const { return owner_; }
    const labelList& neighbour() const { return neighbour_; }

    label size() const { return nCells_; }
    label nFaces() const { return nFaces_; }
    label nX() const { return nX_; }
    label nY() const { return nY_; }

    // -[4] Convert (i, j) cell index to global cell index
    label cellIndex(label i, label j) const
    {
        return j * nX_ + i;
    }

    // -[5] Get cell coordinates from global index
    void cellCoords(label cellI, label& i, label& j) const
    {
        j = cellI / nX_;
        i = cellI % nX_;
    }

    // Verify constraint
    bool valid() const
    {
        for (label faceI = 0; faceI < nFaces_; faceI++)
        {
            if (owner_[faceI] >= neighbour_[faceI])
                return false;
        }
        return true;
    }
};

#endif
```

**Verification checks:**
- ✅ Correct face count: `(nX - 1) * nY + nX * (nY - 1)`
- ✅ Horizontal faces ordered first (consistent with OpenFOAM)
- ✅ Enforces `owner < neighbour` constraint
- ✅ Provides utility methods for coordinate conversion

### Exercise 1: Face Index Lookup

Implement `faceIndex()` for 2D mesh to find the face connecting two adjacent cells.

**Solution:**

```cpp
label lduAddressing2D::faceIndex(label cellI, label cellJ) const
{
    label i1, j1, i2, j2;
    cellCoords(cellI, i1, j1);
    cellCoords(cellJ, i2, j2);

    label nHoriz = (nX_ - 1) * nY_;

    // Horizontal face?
    if (j1 == j2 && std::abs(i1 - i2) == 1)
    {
        label i = std::min(i1, i2);
        return j1 * (nX_ - 1) + i;  // Horizontal face index
    }

    // Vertical face?
    if (i1 == i2 && std::abs(j1 - j2) == 1)
    {
        label j = std::min(j1, j2);
        return nHoriz + j * nX_ + i1;  // Vertical face index
    }

    return -1;  // Not adjacent
}
```

```cpp
label nNeighbours(label cellI) const
{
    label count = 0;

    // Count as owner
    for (label faceI = 0; faceI < nFaces_; faceI++)
    {
        if (owner_[faceI] == cellI)
            count++;
    }

    // Count as neighbour
    for (label faceI = 0; faceI < nFaces_; faceI++)
    {
        if (neighbour_[faceI] == cellI)
            count++;
    }

    return count;
}
```

### Exercise 4: Bandwidth Calculation

Calculate the **matrix bandwidth** for the LDU matrix structure.

**Definition:** Bandwidth = $\max |i - j|$ for all non-zero $A_{ij}$.

**Solution:**

```cpp
label bandwidth() const
{
    label bw = 0;
    for (label faceI = 0; faceI < nFaces_; faceI++)
    {
        label diff = neighbour_[faceI] - owner_[faceI];
        bw = std::max(bw, diff);
    }
    return bw;
}
```

For 2D `nX × nY` mesh: bandwidth = `nX` (max index difference across rows)

## Part 5: Summary

### Key Takeaways

1. **Owner/neighbour constraint** is fundamental:
   - `owner[f] < neighbour[f]` for all faces
   - Ensures unique storage of off-diagonal entries
   - Enables symmetric storage even for unsymmetric matrices

2. **Memory efficiency:**
   - 2 labels per face (8 bytes per face for 32-bit labels)
   - Scales linearly with mesh size: $O(d × N)$

3. **Face ordering affects performance:**
   - Spatial locality improves cache utilization
   - OpenFOAM orders: horizontal → vertical → spanwise faces
   - Optimal ordering depends on access pattern

4. **Utility methods are essential:**
   - `faceIndex(cellI, cellJ)` for assembly
   - `cellCoords(i, j)` for coordinate conversion
   - `valid()` for debugging
