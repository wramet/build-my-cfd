# Day 20: OpenFOAM List<T> - Internals vs std::vector

## Introduction to OpenFOAM List Architecture

⭐ OpenFOAM's `List<T>` container represents one of the most fundamental data structures in the codebase. Unlike `std::vector`, `List<T>` follows a base class pattern with `UList<T>` serving as a non-owning view, enabling efficient memory management and flexible semantics.

```cpp
// Example of List<T> usage in OpenFOAM
List<scalar> fieldValues(100, 0.0);  // Size 100, initialized to 0.0
List<label> cellIndices(mesh.nCells());

// View semantics - UList<T> doesn't own memory
UList<scalar> view = fieldValues;  // No copy, just a view
```

The design reflects OpenFOAM's philosophy of separating ownership from access patterns, crucial for CFD computations where we frequently need non-owning views of data.

## UList<T> - The Non-Owning Foundation (UList<T> - ฐานที่ไม่ถือครอง)

### Design Philosophy

⭐ `UList<T>` serves as the immutable foundation of OpenFOAM's container hierarchy. It provides a view into memory without owning it, making it ideal for:

- Function parameters that shouldn't take ownership
- Temporary views into larger containers
- Sharing data between containers without copying

```cpp
// File: /Users/woramet/Documents/Build My CFD/.claude/worktrees/upbeat-banach/daily_learning/Phase_02_DataStructures_Memory/20.md
// From OpenFOAM source: OpenFOAM/src/OpenFOAM/containers/Lists/UList/UList.H

template<class T>
class UList
{
    // Pointer to the beginning of the data
    T* ptr_;

    // Size of the list
    label size_;

public:
    // Constructor: List(size, value, "name")
    UList(T* p, const label size) : ptr_(p), size_(size) {}

    // Element access with bounds checking
    inline T& operator[](const label i) {
        if (i >= size_) {
            FatalErrorIn("UList<T>::operator[]")
                << "Index " << i << " out of range 0.." << size_-1
                << abort(FatalError);
        }
        return ptr_[i];
    }

    // Const element access
    inline const T& operator[](const label i) const {
        if (i >= size_) {
            FatalErrorIn("UList<T>::operator[]")
                << "Index " << i << " out of range 0.." << size_-1
                << abort(FatalError);
        }
        return ptr_[i];
    }

    // Size operations
    inline label size() const { return size_; }
    inline bool empty() const { return size_ == 0; }
    inline T* begin() { return ptr_; }
    inline const T* begin() const { return ptr_; }
    inline T* end() { return ptr_ + size_; }
    inline const T* end() const { return ptr_ + size_; }
};
```

### Mathematical Foundation of UList Semantics

The mathematical model behind `UList<T>` can be represented as:

Given a list $\mathbf{L} = [l_0, l_1, ..., l_{n-1}]$ where $n$ is the size, we have:

$$
l_i = \mathbf{L}[i] \quad \text{for} \quad 0 \leq i < n
$$

The view property means that multiple `UList<T>` objects can reference the same underlying memory:

$$
\mathbf{L}_1 = \text{view}(\mathbf{M}), \mathbf{L}_2 = \text{view}(\mathbf{M}) \implies \mathbf{L}_1[i] = \mathbf{L}_2[i] = \mathbf{M}[i]
$$

This is particularly important in CFD where we often have:
- Multiple views of the same field data
- Temporary views during computation
- Views that span across different data structures

### Performance Characteristics (คุณสมบัติด้านประสิทธิภาพ)

The non-owning nature provides several performance benefits:

1. **Constant time copy**: `UList<T>` copying is $O(1)$ since it only copies pointers
2. **No allocation overhead**: No memory management overhead
3. **Flexible lifetimes**: Can outsource lifetime management to owner

However, this comes with responsibilities:
- The referenced memory must remain valid
- No automatic bounds checking in performance-critical code
- Manual memory management required for owned data

## List<T> - The Owning Container (List<T> - ตัวเก็บข้อมูลที่ถือครอง)

### Memory Management Strategy

`List<T>` extends `UList<T>` to provide ownership semantics. It manages the underlying memory while maintaining the interface of `UList<T>`.

```cpp
// File: /Users/woramet/Documents/Build My CFD/.claude/worktrees/upbeat-banach/daily_learning/Phase_02_DataStructures_Memory/20.md
// Simplified List<T> implementation

template<class T>
class List : public UList<T>
{
    // Storage management
    T* ptr_;
    label capacity_;
    label size_;

public:
    // Constructor: List(size, value, "name")
    List(const label size, const T& val = T())
    : UList<T>(nullptr, size),
      ptr_(new T[size]),
      capacity_(size),
      size_(size)
    {
        for (label i = 0; i < size; ++i) {
            ptr_[i] = val;
        }
    }

    // Destructor
    ~List() { delete[] ptr_; }

    // Copy constructor (deep copy)
    List(const List& other)
    : UList<T>(nullptr, other.size_),
      ptr_(new T[other.size_]),
      capacity_(other.size_),
      size_(other.size_)
    {
        for (label i = 0; i < size_; ++i) {
            ptr_[i] = other.ptr_[i];
        }
    }

    // Move constructor
    List(List&& other) noexcept
    : UList<T>(other.ptr_, other.size_),
      ptr_(other.ptr_),
      capacity_(other.capacity_),
      size_(other.size_)
    {
        other.ptr_ = nullptr;
        other.capacity_ = 0;
        other.size_ = 0;
    }

    // Assignment operator
    List& operator=(const List& other) {
        if (this != &other) {
            delete[] ptr_;
            ptr_ = new T[other.size_];
            capacity_ = other.size_;
            size_ = other.size_;
            for (label i = 0; i < size_; ++i) {
                ptr_[i] = other.ptr_[i];
            }
        }
        return *this;
    }
};
```

### setSize() vs resize() - Critical Differences (setSize() ต่างจาก resize())

OpenFOAM's `List<T>` provides different methods for size management with distinct semantics:

```cpp
// File: /Users/woramet/Documents/Build My CFD/.claude/worktrees/upbeat-banach/daily_learning/Phase_02_DataStructures_Memory/20.md
// setSize() vs resize() implementation

template<class T>
class List : public UList<T>
{
    // ... existing members ...

    // setSize() - Changes size, destroys elements if shrinking
    void setSize(const label newSize) {
        if (newSize > capacity_) {
            // Need reallocation
            T* newPtr = new T[newSize];
            for (label i = 0; i < size_; ++i) {
                newPtr[i] = std::move(ptr_[i]);
            }
            for (label i = size_; i < newSize; ++i) {
                newPtr[i] = T();  // Default construct
            }
            delete[] ptr_;
            ptr_ = newPtr;
            capacity_ = newSize;
        } else if (newSize < size_) {
            // Destroy elements
            for (label i = newSize; i < size_; ++i) {
                ptr_[i].~T();
            }
        }
        size_ = newSize;
    }

    // resize() - Changes size, preserves existing elements
    void resize(const label newSize, const T& val = T()) {
        if (newSize > capacity_) {
            // Need reallocation
            T* newPtr = new T[newSize];
            for (label i = 0; i < size_; ++i) {
                newPtr[i] = ptr_[i];
            }
            for (label i = size_; i < newSize; ++i) {
                newPtr[i] = val;
            }
            delete[] ptr_;
            ptr_ = newPtr;
            capacity_ = newSize;
        } else if (newSize > size_) {
            // Expand with given value
            for (label i = size_; i < newSize; ++i) {
                ptr_[i] = val;
            }
        }
        size_ = newSize;
    }
};
```

The key difference is:
- `setSize(newSize)`: Destroys elements when shrinking, may lose data
- `resize(newSize, val)`: Preserves existing data, fills new elements with `val`

### Memory Allocation Strategy (กลยุทธ์การจองหน่วยความจำ)

OpenFOAM's `List<T>` uses a simple allocation strategy:

```cpp
template<class T>
class List : public UList<T>
{
    // ... existing members ...

    // Reallocate with new capacity
    void reallocate(const label newCapacity) {
        T* newPtr = new T[newCapacity];

        // Move existing elements
        const label moveSize = Foam::min(size_, newCapacity);
        for (label i = 0; i < moveSize; ++i) {
            newPtr[i] = std::move(ptr_[i]);

        // Destroy remaining elements if shrinking
        if (newCapacity < size_) {
            for (label i = newCapacity; i < size_; ++i) {
                ptr_[i].~T();
            }
        }

        delete[] ptr_;
        ptr_ = newPtr;
        capacity_ = newCapacity;
    }
};
```

## Comparison with std::vector (เปรียบเทียบกับ std::vector)

### Interface Comparison (เปรียบเทียบ Interface)

| Feature | OpenFOAM List<T> | std::vector<T> |
|---------|-----------------|---------------|
| Base class | UList<T> (non-owning) | None |
| Copy semantics | Deep copy by default | Deep copy by default |
| Move semantics | Move constructor/assignment | Move constructor/assignment |
| Bounds checking | FatalError in debug | std::out_of_range exception |
| Resize semantics | setSize/resize methods | resize method only |
| Memory growth | Manual capacity management | Exponential growth |

### Creating Views (สร้าง View)

View semantics are crucial for performance in CFD computations:

```cpp
// View semantics implementation

template<class T>
class SubList : public UList<T>
{
    const T* basePtr_;  // Original list start

public:
    SubList(UList<T> list, label start, label len)
    : UList<T>(list.begin() + start, len),
      basePtr_(list.begin())
    {}

    // Get position in original list
    label baseIndex(const label i) const {
        return (begin() + i) - basePtr_;
    }
};

// Example usage
void createSubListExample() {
    List<scalar> mainList(100, 0.0);

    // Create view - no copying
    SubList<scalar> subView(mainList, 10, 20);

    // Modify through view affects original
    subView[5] = 1.5;  // mainList[15] = 1.5

    // Views can be passed around cheaply
    processSubList(subView);
}

void processSubList(const UList<scalar>& view) {
    // Works on any UList, including views
    for (label i = 0; i < view.size(); ++i) {
        view[i] *= 2.0;
    }
}
```

## Complete Implementation of List<T>/UList<T> (การเขียนโค้ดเต็มรูปแบบ List<T>/UList<T>)

Here's a complete, compilable implementation following OpenFOAM patterns:

```cpp
// File: /Users/woramet/Documents/Build My CFD/.claude/worktrees/upbeat-banach/daily_learning/Phase_02_DataStructures_Memory/20.md
// Complete UList<T>/List<T> implementation

#ifndef UList_H
#define UList_H

#include <cstddef>
#include <algorithm>
#include <stdexcept>

// Forward declaration
template<class T>
class List;

// Non-owning base class
template<class T>
class UList
{
    T* ptr_;
    label size_;

public:
    // Constructor: List(size, value, "name")
    UList(T* p, const label size) : ptr_(p), size_(size) {}

    // Destructor - virtual for polymorphism
    virtual ~UList() = default;

    // Element access with bounds checking
    T& operator[](const label i) {
        if (i >= size_) {
            throw std::out_of_range("UList index out of range");
        }
        return ptr_[i];
    }

    const T& operator[](const label i) const {
        if (i >= size_) {
            throw std::out_of_range("UList index out of range");
        }
        return ptr_[i];
    }

    // At() with bounds checking (alternative)
    T& at(const label i) {
        return (*this)[i];
    }

    const T& at(const label i) const {
        return (*this)[i];
    }

    // Front and back elements
    T& front() { return ptr_[0]; }
    const T& front() const { return ptr_[0]; }
    T& back() { return ptr_[size_ - 1]; }
    const T& back() const { return ptr_[size_ - 1]; }

    // Size operations
    label size() const { return size_; }
    bool empty() const { return size_ == 0; }
    label capacity() const { return size_; }  // Same as size for UList

    // Iterator support
    T* begin() { return ptr_; }
    const T* begin() const { return ptr_; }
    T* end() { return ptr_ + size_; }
    const T* end() const { return ptr_ + size_; }

    // Const iterators
    const T* cbegin() const { return ptr_; }
    const T* cend() const { return ptr_ + size_; }

    // Raw pointer access
    T* data() { return ptr_; }
    const T* data() const { return ptr_; }

    // Comparison
    bool operator==(const UList& other) const {
        if (size_ != other.size_) return false;
        return std::equal(ptr_, ptr_ + size_, other.ptr_);
    }

    bool operator!=(const UList& other) const {
        return !(*this == other);
    }
};

// Owning container
template<class T>
class List : public UList<T>
{
    T* ptr_;
    label capacity_;

public:
    // Constructors: List(size, value, "name")
    List() : UList<T>(nullptr, 0), ptr_(nullptr), capacity_(0) {}

    explicit List(const label size)
    : UList<T>(nullptr, size),
      ptr_(size > 0 ? new T[size] : nullptr),
      capacity_(size)
    {
        // Default construct elements
        for (label i = 0; i < size; ++i) {
            ptr_[i] = T();
        }
    }

    List(const label size, const T& val)
    : UList<T>(nullptr, size),
      ptr_(size > 0 ? new T[size] : nullptr),
      capacity_(size)
    {
        // Construct with given value
        for (label i = 0; i < size; ++i) {
            ptr_[i] = val;
        }
    }

    // Destructor
    ~List() { delete[] ptr_; }

    // Copy constructor
    List(const List& other)
    : UList<T>(nullptr, other.size()),
      ptr_(other.size() > 0 ? new T[other.size()] : nullptr),
      capacity_(other.size())
    {
        for (label i = 0; i < size(); ++i) {
            ptr_[i] = other.ptr_[i];
        }
    }

    // Move constructor
    List(List&& other) noexcept
    : UList<T>(other.ptr_, other.size()),
      ptr_(other.ptr_),
      capacity_(other.capacity_)
    {
        other.ptr_ = nullptr;
        other.capacity_ = 0;
    }

    // Assignment operators
    List& operator=(const List& other) {
        if (this != &other) {
            delete[] ptr_;

            ptr_ = other.size() > 0 ? new T[other.size()] : nullptr;
            capacity_ = other.size();
            this->size_ = other.size();

            for (label i = 0; i < size(); ++i) {
                ptr_[i] = other.ptr_[i];
            }
        }
        return *this;
    }

    List& operator=(List&& other) noexcept {
        if (this != &other) {
            delete[] ptr_;

            ptr_ = other.ptr_;
            capacity_ = other.capacity_;
            this->size_ = other.size_;

            other.ptr_ = nullptr;
            other.capacity_ = 0;
        }
        return *this;
    }

    // Size management
    void setSize(const label newSize) {
        if (newSize > capacity_) {
            reallocate(newSize);
        } else if (newSize < size()) {
            // Destroy elements
            for (label i = newSize; i < size(); ++i) {
                ptr_[i].~T();
            }
        }
        this->size_ = newSize;
    }

    void resize(const label newSize, const T& val = T()) {
        if (newSize > capacity_) {
            reallocate(newSize);
        } else if (newSize > size()) {
            // Fill new elements
            for (label i = size(); i < newSize; ++i) {
                ptr_[i] = val;
            }
        }
        this->size_ = newSize;
    }

    // Capacity management
    void reserve(const label newCapacity) {
        if (newCapacity > capacity_) {
            reallocate(newCapacity);
        }
    }

    label capacity() const { return capacity_; }

    // Element modification
    void push_back(const T& val) {
        if (size() == capacity_) {
            reallocate(capacity_ == 0 ? 1 : capacity_ * 2);
        }
        ptr_[size_] = val;
        this->size_++;
    }

    void push_back(T&& val) {
        if (size() == capacity_) {
            reallocate(capacity_ == 0 ? 1 : capacity_ * 2);
        }
        ptr_[size_] = std::move(val);
        this->size_++;
    }

    void pop_back() {
        if (!empty()) {
            back().~T();
            this->size_--;
        }
    }

    // Clear elements (but keep capacity)
    void clear() {
        for (label i = 0; i < size(); ++i) {
            ptr_[i].~T();
        }
        this->size_ = 0;
    }

    // Erase elements
    void erase(const label pos, const label len = 1) {
        if (pos + len > size()) {
            throw std::out_of_range("erase position out of range");
        }

        // Move elements to fill gap
        for (label i = pos; i < size() - len; ++i) {
            ptr_[i] = std::move(ptr_[i + len]);
        }

        // Destroy moved elements
        for (label i = size() - len; i < size(); ++i) {
            ptr_[i].~T();
        }

        this->size_ -= len;
    }

    // Insert elements
    void insert(const label pos, const T& val) {
        if (pos > size()) {
            throw std::out_of_range("insert position out of range");
        }

        if (size() == capacity_) {
            reallocate(capacity_ == 0 ? 1 : capacity_ * 2);
        }

        // Shift elements right
        for (label i = size(); i > pos; --i) {
            ptr_[i] = std::move(ptr_[i - 1]);
        }

        // Insert new element
        ptr_[pos] = val;
        this->size_++;
    }

private:
    // Reallocation helper
    void reallocate(const label newCapacity) {
        T* newPtr = new T[newCapacity];

        // Move existing elements
        const label moveSize = Foam::min(size(), newCapacity);
        for (label i = 0; i < moveSize; ++i) {
            newPtr[i] = std::move(ptr_[i]);
        }

        // Destroy remaining elements if shrinking
        if (newCapacity < size()) {
            for (label i = newCapacity; i < size(); ++i) {
                ptr_[i].~T();
            }
        }

        delete[] ptr_;
        ptr_ = newPtr;
        capacity_ = newCapacity;
    }
};
```

## Mathematical Analysis of List Operations (การวิเคราะห์ทางคณิตศาสตร์)

### Time Complexity Analysis

Let's analyze the time complexity of various List operations:

#### Access Operations
- `operator[](i)`: $O(1)$ - Direct memory access
- `at(i)`: $O(1)$ - With bounds check
- `front()/back()`: $O(1)$ - Constant time access

#### Size Modification Operations
- `setSize(n)`:
  - If $n \leq capacity$: $O(n)$ (element destruction if shrinking)
  - If $n > capacity$: $O(n)$ (reallocation + copy)

- `resize(n, val)`:
  - If $n \leq capacity$: $O(1)$ (fill only if expanding)
  - If $n > capacity$: $O(n)$ (reallocation + copy + fill)

- `push_back()`: Amortized $O(1)$ due to exponential growth

#### Memory Allocation Pattern
The growth pattern follows:

$$
capacity_n = \max(initialSize, 2^{n-1} \cdot initialSize)
$$

Where $n$ is the number of reallocations.

### Space Complexity Analysis

- **Storage**: $O(n)$ for $n$ elements
- **View overhead**: $O(1)$ additional space for `UList<T>`
- **Overhead per element**: sizeof(T) + padding (minimal)

### Cache Performance Analysis

The contiguous memory layout provides excellent cache performance:

$$
\text{Cache hit ratio} \approx 1 - \frac{\text{cache line size}}{\text{array size} \cdot \text{element size}}
$$

For large arrays, this approaches 100% utilization.

```cpp
class TracingAllocator {
    size_t totalAllocated = 0;

public:
    template<typename T>
    T* allocate(size_t n) {
        totalAllocated += n * sizeof(T);
        return static_cast<T*>(::operator new(n * sizeof(T)));

    void deallocate(T* p, size_t n) {
```
