# Day 24: Memory Pools - SubList Zero-Copy Views

## Introduction to SubList Zero-Copy Semantics

SubList in OpenFOAM represents a revolutionary approach to memory efficiency through zero-copy views into existing data structures. Unlike traditional containers that allocate new memory for slices or subsets, SubList provides a lightweight view that references memory from an existing UList without any data duplication.

```cpp
// SubList is defined as a zero-copy view
template<class T>
class SubList
:
    public UList<T>
{
    // Implementation...
};
```

### Mathematical Foundation of Zero-Copy Views

The zero-copy paradigm eliminates the $O(n)$ memory allocation and copying overhead traditionally associated with subarray operations:

$$
\text{Traditional approach cost: } O(n) \text{ (allocation)} + O(n) \text{ (copying)} = O(n)
$$

$$
\text{SubList approach cost: } O(1) \text{ (view creation)} = O(1)
$$

Where $n$ is the number of elements in the subarray.

## SubList API and Zero-Copy Semantics

### SubList Constructors

SubList provides multiple constructors for creating zero-copy views:

```cpp
// Constructor 1: View from start with subSize
template<class T>
inline SubList<T>::SubList
(
    const UList<T>& list,
    const label subSize
)
:
    UList<T>(list.v_, subSize)
{
    #ifdef FULLDEBUG
    list.checkSize(subSize);
    #endif
}

// Constructor 2: View with start index and size
template<class T>
inline SubList<T>::SubList
(
    const UList<T>& list,
    const label subSize,
    const label startIndex
)
:
    UList<T>(&(list.v_[startIndex]), subSize)
{
    #ifdef FULLDEBUG

    // Artificially allow the start of a zero-sized subList to be
    // one past the end of the original list.
    if (subSize)
    {
        list.checkStart(startIndex);
        list.checkSize(startIndex + subSize);
    }
    else
    {
        // Start index needs to fall between 0 and size.  One position
        // behind the last element is allowed
        list.checkSize(startIndex);
    }
    #endif
}
```

**Key insights:**
- ⭐ SubList derives from UList, not List, to avoid memory allocation
- ⭐ SubList only stores a pointer and size, not the actual data
- ⭐ No memory is allocated during SubList construction

### Zero-Copy Assignment Operations

SubList assignment operations maintain the zero-copy semantics through careful memory management:

```cpp
template<class T>
inline void SubList<T>::operator=(const SubList<T>& sl)
{
    UList<T>::deepCopy(sl);
}

template<class T>
inline void SubList<T>::operator=(const UList<T>& l)
{
    UList<T>::deepCopy(l);
}
```

**Critical understanding:**
- Assignment uses `deepCopy()` which copies data, not pointers
- The SubList object itself remains a zero-copy view
- Only the data is copied, not the container overhead

## Memory Pool Theory: Object Lifetime and Reuse Patterns

### Memory Pool Architecture

Memory pools provide a structured approach to object reuse, eliminating allocation/deallocation overhead:

$$
\text{Pool allocation cost: } O(1) \text{ (with pre-allocation)}
$$

$$
\text{Traditional allocation cost: } O(\log n) \text{ to } O(n) \text{ (fragmentation)}
$$

### Object Lifetime Management

Memory pools implement sophisticated lifetime management patterns:

```cpp
class MemoryPool
{
private:
    // Pool of pre-allocated objects
    DynamicList<T*> freeObjects_;

    // Tracking for active objects
    HashSet<T*> activeObjects_;

public:
    // Get object from pool or create new
    T* acquire()
    {
        if (freeObjects_.empty())
        {
            return new T();  // Expand pool if needed
        }

        T* obj = freeObjects_.last();
        freeObjects_.remove();
        activeObjects_.insert(obj);
        return obj;
    }

    // Return object to pool for reuse
    void release(T* obj)
    {
        if (activeObjects_.found(obj))
        {
            activeObjects_.erase(obj);
            freeObjects_.append(obj);
            // Optional: Reset object state
            resetObject(obj);
        }
    }

    // Batch operations for efficiency
    void acquireBatch(DynamicList<T*>& results, label count)
    {
        results.resize(count);
        for (label i = 0; i < count; i++)
        {
            results[i] = acquire();
        }
    }
};
```

### Reference Counting and Pool Integration

Reference counting ensures safe object reuse:

```cpp
class PooledObject
{
private:
    static MemoryPool pool_;
    label* refCount_;

public:
    static PooledObject* create()
    {
        PooledObject* obj = pool_.acquire();
        obj->refCount_ = new label(1);
        return obj;
    }

    void ref()
    {
        (*refCount_)++;
    }

    void unref()
    {
        (*refCount_)--;
        if (*refCount_ == 0)
        {
            delete refCount_;
            pool_.release(this);
        }
    }
};
```

## Practical Applications: Patch Field Slicing and Boundary Face Views

### Patch Field Slicing for Boundary Conditions

```cpp
class BoundaryConditionProcessor
{
private:
    volScalarField T_;
    volVectorField U_;
    volScalarField p_;

public:
    // Extract zero-copy views for boundary processing
    void applyBoundaryConditions()
    {
        // Zero-copy view of temperature boundary field
        const SubList<scalar> Tb = T_.boundaryField()[0].size();

        // Zero-copy view of velocity boundary field
        const SubList<vector> Ub = U_.boundaryField()[0].size();

        // Zero-copy view of pressure boundary field
        const SubList<scalar> pb = p_.boundaryField()[0].size();

        // Process using zero-copy operations
        forAll(Tb, faceI)
        {
            // Fixed temperature boundary condition
            if (faceI < Tb.size() / 2)
            {
                Tb[faceI] = 300.0;  // Direct access - no copy
            }

            // Fixed velocity boundary condition
            vector fixedU(0.1, 0.0, 0.0);
            Ub[faceI] = fixedU;  // Direct access - no copy

            // Zero gradient pressure
            pb[faceI] = pb[faceI];  // No-op, but shows zero-copy access
        }
    }

    // Advanced boundary condition mixing
    void mixedBoundaryCondition(
        const word& patchName,
        const scalar& fixedValue,
        const scalar& gradientValue
    )
    {
        const label patchIndex = T_.boundaryField().findIndex(patchName);
        if (patchIndex == -1) return;

        const SubList<scalar> patchValues = T_.boundaryField()[patchIndex].size();

        // Apply mixed boundary condition
        forAll(patchValues, i)
        {
            patchValues[i] = fixedValue + gradientValue * i;
        }
    }
};
```

### Memory Pool Integration for Field Views

```cpp
class FieldViewPool
{
private:
    // Memory pool for field views
    MemoryPool<SubList<scalar>> scalarViewPool_;
    MemoryPool<SubList<vector>> vectorViewPool_;
    MemoryPool<SubList<tensor>> tensorViewPool_;

    // Active views tracking
    HashSet<void*> activeViews_;

public:
    // Acquire zero-copy view from pool
    SubList<scalar> acquireScalarView(const UList<scalar>& data, label size, label start = 0)
    {
        SubList<scalar> view = scalarViewPool_.acquire();
        view = SubList<scalar>(data, size, start);
        activeViews_.insert(&view);
        return view;
    }

    // Release view back to pool
    void releaseScalarView(SubList<scalar>& view)
    {
        activeViews_.erase(&view);
        scalarViewPool_.release(view);
    }

    // Batch field processing with pooled views
    void processFieldsBatch(
        const List<volScalarField>& scalarFields,
        const List<volVectorField>& vectorFields
    )
    {
        DynamicList<SubList<scalar>> scalarViews;
        DynamicList<SubList<vector>> vectorViews;

        // Acquire views for processing
        forAll(scalarFields, i)
        {
            forAll(scalarFields[i].boundaryField(), patchI)
            {
                SubList<scalar> view = acquireScalarView(
                    scalarFields[i].boundaryField()[patchI],
                    scalarFields[i].boundaryField()[patchI].size()
                );
                scalarViews.append(view);
            }
        }

        forAll(vectorFields, i)
        {
            forAll(vectorFields[i].boundaryField(), patchI)
            {
                SubList<vector> view = acquireVectorView(
                    vectorFields[i].boundaryField()[patchI],
                    vectorFields[i].boundaryField()[patchI].size()
                );
                vectorViews.append(view);
            }
        }

        // Process all fields
        processFieldBatch(scalarViews, vectorViews);

        // Release all views back to pool
        forAll(scalarViews, i)
        {
            releaseScalarView(scalarViews[i]);
        }

        forAll(vectorViews, i)
        {
            releaseVectorView(vectorViews[i]);
        }
    }
};
```

### Exercise 2: Implement a Memory Pool for Field Views

Create a memory pool optimized for field view reuse:

```cpp
template<class FieldType>
class FieldViewPool
{
private:
    // Memory pool for field views
    DynamicList<SubList<typename FieldType::valueType>> pool_;

    // Active views tracking
    HashSet<void*> activeViews_;

    // Statistics
    label totalAcquires_;
    label totalReleases_;
    label poolExpansions_;

public:
    FieldViewPool()
    :
        totalAcquires_(0),
        totalReleases_(0),
        poolExpansions_(0)
    {}

    // Acquire a view for a field
    SubList<typename FieldType::valueType> acquire(
        const FieldType& field,
        label subSize,
        label startIndex = 0
    )
    {
        totalAcquires_++;

        // Create new view if pool is empty
        if (pool_.empty())
        {
            poolExpansions_++;
            return SubList<typename FieldType::valueType>(
                field, subSize, startIndex
            );
        }

        // Reuse existing view
        SubList<typename FieldType::valueType> view = pool_.last();
        pool_.remove();
        activeViews_.insert(&view);

        // Update view parameters
        view = SubList<typename FieldType::valueType>(
            field, subSize, startIndex
        );

        return view;
    }

    // Release a view back to pool
    void release(SubList<typename FieldType::valueType>& view)
    {
        if (activeViews_.found(&view))
        {
            activeViews_.erase(&view);
            pool_.append(view);
            totalReleases_++;
        }
    }

    // Batch acquisition
    void acquireBatch(
        const FieldType& field,
        const List<SubListParams>& params,
        List<SubList<typename FieldType::valueType>>& results
    )
    {
        results.resize(params.size());

        forAll(params, i)
        {
            results[i] = acquire(field, params[i].size, params[i].start);
        }
    }

    // Batch release
    void releaseBatch(
        List<SubList<typename FieldType::valueType>>& views
    )
    {
        forAll(views, i)
        {
            release(views[i]);
        }
    }

    // Get statistics
    void printStats() const
    {
        Info << "FieldViewPool Statistics:" << endl;
        Info << "  Pool size: " << pool_.size() << endl;
        Info << "  Active views: " << activeViews_.size() << endl;
        Info << "  Total acquires: " << totalAcquires_ << endl;
        Info << "  Total releases: " << totalReleases_ << endl;
        Info << "  Pool expansions: " << poolExpansions_ << endl;
        Info << "  Efficiency: " <<
            (totalReleases_ > 0 ?
             static_cast<scalar>(totalReleases_) / totalAcquires_ * 100 : 0)
            << "%" << endl;
    }

    // Clear pool
    void clear()
    {
        pool_.clear();
        activeViews_.clear();
    }
```

```cpp
class ZeroCopyMatrix
{
private:
    DynamicList<T> data_;
    label rows_;
    label cols_;

public:
    ZeroCopyMatrix(label rows, label cols)
    :
        data_(rows * cols),
        rows_(rows),
        cols_(cols)
    {}

    // Zero-copy row access
    SubList<T> row(label rowIndex)
    {
        return SubList<T>(data_, cols_, rowIndex * cols_);
    }

    // Zero-copy column access
    SubList<T> column(label colIndex)
    {
        DynamicList<T> columnData;

        // Create a virtual column view
        for (label i = 0; i < rows_; i++)
        {
            columnData.append(data_[i * cols_ + colIndex]);
        }

        return SubList<T>(columnData, rows_);
    }

    // Matrix multiplication using zero-copy views
    ZeroCopyMatrix<T> multiply(const ZeroCopyMatrix<T>& other) const
    {
        ZeroCopyMatrix<T> result(rows_, other.cols_);

        for (label i = 0; i < rows_; i++)
        {
            SubList<T> row_i = row(i);
            SubList<T> result_row = result.row(i);

            for (label j = 0; j < other.cols_; j++)
            {
                SubList<T> col_j = other.column(j);

                // Dot product using zero-copy views
                scalar dot = 0;
                forAll(row_i, k)
                {
                    dot += row_i[k] * col_j[k];
                }

                result_row[j] = dot;
            }
        }

        return result;
    }

    // Transpose using zero-copy operations
    ZeroCopyMatrix<T> transpose() const
    {
        ZeroCopyMatrix<T> result(cols_, rows_);

        for (label i = 0; i < rows_; i++)
        {
            SubList<T> row_i = row(i);
            SubList<T> result_col = result.column(i);

            // Copy using zero-copy operations
            forAll(row_i, j)
            {
                result_col[j] = row_i[j];
            }
        }
```

```cpp
        {
            label cellI = nearestCells[i];
            scalar weight = calculateWeight(targetPoint, cellI);

            result += weight * field[cellI];
            totalWeight += weight;

            scalar dist = mag(cellCentres[i] - point);
            if (dist < minDist)
            {
                minDist = dist;
                result.clear();
                result.append(i);
            }
            else if (dist == minDist)
            {
                result.append(i);
            }
```

```cpp
class MemoryUsageProfiler
{
private:

    // Profile allocation operation
    void profileAllocation(
        const word& operationName,
        label size,
        bool isCopy = false
    )
    {
        AllocationRecord record;
        record.operationName = operationName;
        record.size = size;
        record.timestamp = global::timer.elapsedClockTime();
        record.isCopy = isCopy;
        record.isZeroCopy = !isCopy;

        records_.append(record);

        if (isCopy)
        {
            totalCopies_++;
            totalMemoryCopied_ += size;
        }
        else
        {
            totalZeroCopies_++;
            totalMemoryZeroCopied_ += sizeof(SubList<typename BaseType>);  // Header only
        }

        totalAllocations_++;
    }

    {

        Info << "Total copies: " << totalCopies_ << endl;
        Info << "Total zero-copies: " << totalZeroCopies_ << endl;
        Info << "Memory copied: " << totalMemoryCopied_ << " bytes" << endl;
        Info << "Memory zero-copied: " << totalMemoryZeroCopied_ << " bytes" << endl;

        if (totalMemoryCopied_ > 0)
        {
            Info << "Memory savings: "
                 << (totalMemoryCopied_ - totalMemoryZeroCopied_)
                 << " bytes ("
                 << (1 - totalMemoryZeroCopied_ / totalMemoryCopied_) * 100
                 << "%)" << endl;
        }

        // Analyze operation patterns
        analyzeOperationPatterns();
    }

    // Analyze operation patterns
    void analyzeOperationPatterns()
    {
        Info << "\n=== Operation Pattern Analysis ===" << endl;

        // Group operations by name
        HashTable<label> operationCounts;
        HashTable<scalar> operationMemory;

        forAll(records_, i)
        {
            const AllocationRecord& record = records_[i];

            if (!operationCounts.found(record.operationName))
            {
                operationCounts.insert(record.operationName, 0);
                operationMemory.insert(record.operationName, 0);
            }

            operationCounts[record.operationName]++;
            operationMemory[record.operationName] += record.size;
        }

        // Report by operation
        forAllIter(HashTable<label>, operationCounts, it)
        {
            const word& opName = it.key();
            label count = it();
            scalar memory = operationMemory[opName];

            Info << opName << ": " << count << " operations, "
                 << memory << " bytes" << endl;
        }
    }

    void compareStrategies()
    {

        scalar zeroCopyMemory = 0;

        {
            const AllocationRecord& record = records_[i];

            if (record.isCopy)
            {
                traditionalMemory += record.size;
            }
            else
            {
                zeroCopyMemory += sizeof(SubList<typename BaseType>);
            }
        }

        Info << "Traditional approach: " << traditionalMemory << " bytes" << endl;
        Info << "Zero-copy approach: " << zeroCopyMemory << " bytes" << endl;
        Info << "Savings: " << (traditionalMemory - zeroCopyMemory) << " bytes" << endl;

        if (traditionalMemory > 0)
        {
            Info << "Percentage savings: "
                 << (1 - zeroCopyMemory / traditionalMemory) * 100 << "%" << endl;
        }
    }
```

## Summary and Best Practices

### Key Concepts Covered

1. **SubList Zero-Copy Semantics**
   - SubList provides O(1) memory views without data copying
   - Inheritance from UList enables efficient memory access
   - No allocation during construction or view operations

2. **Memory Pool Architecture**
   - Pre-allocated object pools eliminate allocation overhead
   - Reference counting ensures safe object reuse
   - Batch operations optimize pool management

3. **Performance Benefits**
   - Eliminated memory allocation costs
   - Reduced memory fragmentation
   - Superior cache locality
   - Linear vs constant time complexity improvement

### Best Practices for Zero-Copy Views

1. **Use SubList for Field Operations**
   ```cpp
   // Good: Zero-copy view
   SubList<scalar> patchValues = T_.boundaryField()[0];

   // Bad: Unnecessary copying
   Field<scalar> patchValues = T_.boundaryField()[0];
   ```

2. **Combine Memory Pool with Views**
   ```cpp
   // Good: Pooled view reuse
   SubList<scalar> view = viewPool.acquire(field, size);
   // ... use view ...
   viewPool.release(view);

   // Bad: Separate allocations
   SubList<scalar> view(field, size);
   // ... use view ...
   // No reuse mechanism
   ```

3. **Monitor Memory Usage**
   ```cpp
   // Good: Profile memory usage
   MemoryUsageProfiler profiler;
   profiler.profileAllocation("Field Processing", size, false);
   profiler.generateReport();

   // Bad: Blind allocation
   SubList<scalar> view(field, size);
   ```

4. **Batch Operations for Efficiency**
   ```cpp
   // Good: Batch processing
   List<SubListParams> params;
   // ... populate params ...
   List<SubList<scalar>> views;
   viewPool.acquireBatch(field, params, views);

   // Bad: Individual operations
   for (label i = 0; i < n; i++)
   {
       SubList<scalar> view(field, size);
   }
   ```

### Performance Optimization Guidelines

| Pattern | Traditional | Zero-Copy | Performance Gain |
|---------|-------------|-----------|------------------|
| Field Slicing | O(n) allocation + O(n) copy | O(1) view creation | O(n) |
| Boundary Processing | Multiple allocations | Single view | O(k) where k is number of patches |
| Matrix Operations | Full copies | Row/column views | O(n²) to O(n) |
| Memory Fragmentation | High | None | Eliminated |
