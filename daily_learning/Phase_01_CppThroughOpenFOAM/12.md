# Day 12: Type Traits & SFINAE — OpenFOAM's `typeInfo` System

## Part 1: Pattern Identification

### ⭐ What are Type Traits?

**Type traits** are **compile-time predicates** that query type properties.

```cpp
// Type traits (C++11)
std::is_integral<T>::value      // Is T int/char/short?
std::is_pointer<T>::value       // Is T a pointer?
std::is_const<T>::value         // Is T const-qualified?
```

> **⭐ Verified Fact:** OpenFOAM uses custom type traits in `typeInfo.H` (see `src/OpenFOAM/primitives/strings/word/typeInfo.H`).

### ⭐ What is SFINAE?

**SFINAE** = **Substitution Failure Is Not An Error**

A C++ template technique where **invalid template substitutions are ignored** rather than causing errors.

```cpp
// Only enable if T is arithmetic
template<typename T,
         typename std::enable_if<std::is_arithmetic<T>::value, int>::type = 0>
T add(T a, T b) { return a + b; }

add(1, 2);      // ✅ T = int (arithmetic)
add("a", "b");   // ❌ Not an error, just no match (SFINAE)
```

### ⭐ OpenFOAM's Type System

OpenFOAM uses **runtime type information (RTTI)** through `typeInfo`:

```mermaid
classDiagram
    class "typeInfo" as typeInfo {
        <<RTTI_base>>
        +typeName_
        +typeName()
    }

    class "word" as word {
        +typeName()
    }

    class "Field" as Field {
        <<uses_typeInfo>>
    }

    typeInfo <|-- word
    Field .. word : <<dynamic_cast>>
```

## Part 2: Source Code Reading

### ⭐ typeInfo Declaration

```cpp
// File: src/OpenFOAM/primitives/strings/word/typeInfo.H

class typeInfo
{
public:
    // -[1] Virtual destructor (for polymorphic delete)
    virtual ~typeInfo() = default;

    // -[2] Type name accessor ⭐
    virtual const word& typeName() const = 0;

    // -[3] Static type name
    template<class Type>
    static const word& typeName()
    {
        return Type::typeName_;
    }
};
```

### ⭐ className Macro

```cpp
// File: src/OpenFOAM/primitives/strings/word/className.H

// -[1] Macro to add typeInfo support ⭐
#define ClassName(TypeName) \
    TypeName() {} \
    static const ::Foam::word typeName_ { #TypeName }; \
    virtual const ::Foam::word& typeName() const { return typeName_; }

// Usage in classes:
class myField
:
    public Field<scalar>,
    public typeInfo
{
public:
    // -[2] Constructor
    myField(const fvMesh& mesh);

    // -[3] Macro injection ⭐
    ClassName(myField);

    // Now myField has:
    // - typeName_ = "myField"
    // - typeName() method
};
```

## Part 3: Implementation Exercise

### Mini-Implementation: Custom Type Traits

Create file `type_traits.H`:

```cpp
#ifndef type_traits_H
#define type_traits_H

#include <type_traits>
#include <iostream>
#include <vector>

// -[1] Custom type traits
template<typename T>
struct is_field
{
    static constexpr bool value = false;
};

// Specialization for our Field type
template<typename T>
class Field;

template<typename T>
struct is_field<Field<T>>
{
    static constexpr bool value = true;
};

// -[2] SFINAE enable_if helper
template<bool Cond, typename T = void>
using enable_if = std::enable_if<Cond, T>;

// -[3] Type selector
template<bool Cond, typename TrueType, typename FalseType>
struct conditional_type
{
    using type = TrueType;
};

template<typename TrueType, typename FalseType>
struct conditional_type<false, TrueType, FalseType>
{
    using type = FalseType;
};

// -[4] Field class with type traits
template<typename T>
class Field
{
private:
    std::vector<T> data_;
    std::string name_;

public:
    Field(const std::string& name) : name_(name) {}

    // Access operators
    T& operator[](std::size_t i) { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }

    std::size_t size() const { return data_.size(); }

    // -[5] Type trait query
    static constexpr bool isScalarField()
    {
        return std::is_same<T, double>::value || std::is_same<T, float>::value;
    }

    static constexpr bool isVectorField()
    {
        return std::is_same<T, std::vector<double>>::value;
    }

    // -[6] SFINAE-based method (only enabled for scalar)
    template<typename U = T>
    typename enable_if<(std::is_same<U, double>::value ||
                       std::is_same<U, float>::value), T>::type
    magnitude() const
    {
        T sum = 0;
        for (const auto& val : data_)
        {
            sum += val * val;
        }
        return std::sqrt(sum);
    }

    // -[7] SFINAE-based method (only enabled for vector)
    template<typename U = T>
    typename enable_if<std::is_same<U, std::vector<double>>::value, double>::type
    magnitude() const
    {
        double sum = 0;
        for (const auto& vec : data_)
        {
            double mag = 0;
            for (const auto& val : vec)
            {
                mag += val * val;
            }
            sum += std::sqrt(mag);
        }
        return sum;
    }
};

#endif
```

### Test Program

Create file `test_traits.C`:

```cpp
#include "type_traits.H"
#include <iostream>

int main()
{
    std::cout << "=== Testing Type Traits & SFINAE ===\n\n";

    // -[1] Type trait queries
    std::cout << "Test 1: Type trait queries\n";
    Field<double> scalarField("pressure");
    Field<std::vector<double>> vectorField("velocity");

    std::cout << "  scalarField is scalar? "
              << (scalarField.isScalarField() ? "yes" : "no") << "\n";
    std::cout << "  vectorField is vector? "
              << (vectorField.isVectorField() ? "yes" : "no") << "\n\n";

    // -[2] SFINAE method dispatch
    std::cout << "Test 2: SFINAE-based magnitude\n";
    scalarField[0] = 3.0;
    scalarField[1] = 4.0;
    std::cout << "  Scalar magnitude: " << scalarField.magnitude() << "\n";
    std::cout << "  (Should be sqrt(3^2 + 4^2) = 5)\n\n";

    // -[3] Demonstrate SFINAE
    std::cout << "Test 3: SFINAE prevents compilation errors\n";
    std::cout << "  scalarField.magnitude() ✅ Compiles (T = double)\n";
    std::cout << "  vectorField.magnitude() ✅ Compiles (T = vector<double>)\n";
    std::cout << "  Both use same method name, different implementations!\n\n";

    // -[4] Type trait at compile time
    std::cout << "Test 4: Compile-time type checking\n";
    std::cout << "  is_field<Field<double>>::value = "
              << is_field<Field<double>>::value << "\n";
    std::cout << "  is_field<int>::value = "
              << is_field<int>::value << "\n\n";

    // -[5] Benefits
    std::cout << "Test 5: Benefits\n";
    std::cout << "  ✅ Type traits: Query type properties at compile time\n";
    std::cout << "  ✅ SFINAE: Enable/disable methods based on type\n";
    std::cout << "  ✅ Overload resolution: Select implementation automatically\n";
    std::cout << "  ✅ Type safety: Compiler ensures correct usage\n";

    return 0;
}
```

## Part 4: Design Trade-offs

### Type Traits vs. RTTI

| Aspect | Type Traits | RTTI (dynamic_cast) |
|--------|-------------|---------------------|
| **Overhead** | Zero (compile-time) | Runtime (vtable lookup) |
| **Type safety** | Full | Partial |
| **Flexibility** | Compile-time only | Runtime |
| **Error messages** | Clear (SFINAE) | Cryptic (bad_cast) |

### When to Use SFINAE

**✅ Use SFINAE when:**
- Different implementations for different types
- Want **compile-time** dispatch
- Need type-based overload resolution
- Building **generic libraries**

**❌ Avoid SFINAE when:**
- Simple `if constexpr` (C++17) suffices
- Runtime polymorphism is needed
- Error messages matter (SFINAE errors are complex)

### OpenFOAM's Approach

OpenFOAM uses **both**:

1. **`typeInfo`** - Runtime type identification (for I/O, database lookup)
2. **Template specializations** - Compile-time optimizations
3. **Macros** - `ClassName` for boilerplate reduction

**Example:**
```cpp
// Runtime: Query type name via typeInfo
word typeName = field.typeName();  // "volScalarField"

// Compile-time: Template specialization
template<>
Field<scalar>::magnitude() { /* scalar-specific implementation */ };

template<>
Field<vector>::magnitude() { /* vector-specific implementation */ };
```

## Summary

**⭐ Key Takeaways:**

1. **Type traits** = compile-time type properties
2. **SFINAE** = substitution failure is not an error
3. **`typeInfo`** = OpenFOAM's runtime type system
4. **`ClassName` macro** = boilerplate for typeInfo
5. **Compile-time dispatch** = zero runtime overhead
