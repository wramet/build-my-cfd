# Day 11: Iterators & Range-Based Patterns — The `forAll` Macro

## Part 1: Pattern Identification

### ⭐ What are Iterators?

**Iterators** are **generalized pointers** that traverse container elements.

```cpp
// Old-style index loop
for (label i = 0; i < field.size(); i++)
{
    field[i] = ...;
}

// Iterator-based (pre-C++11)
for (Field<scalar>::iterator it = field.begin(); it != field.end(); ++it)
{
    *it = ...;
}

// Range-based for (C++11)
for (auto& val : field)
{
    val = ...;
}
```

> **⭐ Verified Fact:** OpenFOAM uses the `forAll` macro defined in `UList.H` (see `src/OpenFOAM/containers/Lists/UList/UList.H`).

### ⭐ Why `forAll` Matters

OpenFOAM's `forAll` macro provides:
1. **Type-independent iteration** - Works for any container
2. **Boundary-aware** - Separate macros for internal vs. boundary
3. **Performance** - Optimized for OpenFOAM's container types

### ⭐ OpenFOAM's Iteration Macros

| Macro | Purpose | Example |
|-------|---------|---------|
| `forAll(field, i)` | Iterate all elements | `forAll(p, i) { p[i] = ...; }` |
| `forAll(faceI, i)` | Iterate faces | Boundary iteration |
| `forAllIter(container, iter)` | Iterator-based | `forAllIter(patch, iter)` |

```cpp
#define forAll(list, i) \

// forAll(p, cellI)
// {
//     p[cellI] = ...;
```

### ⭐ Range-Based Iteration

```cpp
// File: src/OpenFOAM/primitives/strings/word/wordI.H

// Modern C++11 range-based for
template<typename T>
class UList
{
public:
    T* begin() { return v_; }
    const T* begin() const { return v_; }

    T* end() { return v_ + size_; }
    const T* end() const { return v_ + size_; }
};

// Usage:
for (auto& val : U)
{
    val = ...;
}
```

### ⭐ Iterator Types

```cpp
// File: src/OpenFOAM/containers/Lists/UList/UList.H

template<class T>
class UList
{
public:
    // -[1] Iterator typedefs
    typedef T* iterator;
    typedef const T* const_iterator;

    // -[2] Iterator access
    iterator begin() { return v_; }
    iterator end() { return v_ + size_; }

    const_iterator begin() const { return v_; }
    const_iterator end() const { return v_ + size_; }

    // -[3] Random access (UList is contiguous)
    T& operator[](const label i) { return v_[i]; }
    const T& operator[](const label i) const { return v_[i]; }
};
```

## Part 3: Implementation Exercise

### Mini-Implementation: Iterator Support

Create file `iterator_support.H`:

```cpp
#ifndef iterator_support_H
#define iterator_support_H

#include <vector>
#include <iostream>
#include <stddef.h>

// -[1] Macro-based iteration (OpenFOAM style)
#define forAll(list, i) \
    for (std::size_t i = 0; i < (list).size(); i++)

#define forAllConst(list, i) \
    for (std::size_t i = 0; i < (list).size(); i++)

// -[2] Field with iterator support
template<typename T>
class Field
{
private:
    std::vector<T> data_;
    std::string name_;

public:
    Field(const std::string& name = "field") : name_(name) {}
    Field(std::size_t n, const T& val, const std::string& name = "field")
    : data_(n, val), name_(name) {}

    // -[3] Iterator support (C++11)
    typedef typename std::vector<T>::iterator iterator;
    typedef typename std::vector<T>::const_iterator const_iterator;

    iterator begin() { return data_.begin(); }
    iterator end() { return data_.end(); }

    const_iterator begin() const { return data_.begin(); }
    const_iterator end() const { return data_.end(); }

    // -[4] Random access
    T& operator[](std::size_t i) { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }

    std::size_t size() const { return data_.size(); }
    const std::string& name() const { return name_; }
};

// -[6] Patch (boundary) iterator
template<typename T>
class Patch
{
private:
    std::vector<T> faceValues_;
    std::string name_;

public:
    Patch(const std::string& name, std::size_t nFaces)
    : faceValues_(nFaces), name_(name) {}

    typedef typename std::vector<T>::iterator iterator;
    typedef typename std::vector<T>::const_iterator const_iterator;

    iterator begin() { return faceValues_.begin(); }
    iterator end() { return faceValues_.end(); }

    const_iterator begin() const { return faceValues_.begin(); }
    const_iterator end() const { return faceValues_.end(); }

    T& operator[](std::size_t i) { return faceValues_[i]; }
    const T& operator[](std::size_t i) const { return faceValues_[i]; }

    std::size_t size() const { return faceValues_.size(); }
    const std::string& name() const { return name_; }
};

#endif
```

### Test Program

Create file `test_iterators.C`:

```cpp
#include "iterator_support.H"
#include <iostream>

int main()
{
    std::cout << "=== Testing Iterator Patterns ===\n\n";

    Field<double> p(10, 101325.0, "pressure");

    // -[1] Macro-based iteration (OpenFOAM style)
    std::cout << "Test 1: forAll(p, i) macro\n";
    forAll(p, i)
    {
        p[i] += i * 10.0;
    }
    std::cout << "Result: " << p[0] << ", " << p[1] << ", ..., " << p[9] << "\n\n";

    // -[2] Iterator-based (pre-C++11)
    std::cout << "Test 2: Iterator-based loop\n";
    for (Field<double>::iterator it = p.begin(); it != p.end(); ++it)
    {
        *it *= 1.1;
    }
    std::cout << "After scaling: " << p[0] << "\n\n";

    // -[3] Range-based for (C++11)
    std::cout << "Test 3: Range-based for\n";
    for (auto& val : p)
    {
        val += 5.0;
    }
    std::cout << "After modification: " << p[0] << "\n\n";

    // -[4] Const iteration
    std::cout << "Test 4: Const iteration (read-only)\n";
    const Field<double>& cp = p;
    double sum = 0.0;
    for (const auto& val : cp)
    {
        sum += val;
    }
    std::cout << "Sum of all values: " << sum << "\n\n";

    // -[5] Patch iteration
    std::cout << "Test 5: Boundary patch iteration\n";
    Patch<double> inletPatch("inlet", 5);
    forAll(inletPatch, faceI)
    {
        inletPatch[faceI] = 101000.0;
    }
    std::cout << "Inlet patch values: ";
    for (const auto& val : inletPatch)
    {
        std::cout << val << " ";
    }
    std::cout << "\n\n";

    // -[6] Comparison
    std::cout << "Test 6: Iteration Method Comparison\n";
    std::cout << "  Method         | Speed   | Flexibility\n";
    std::cout << "  -------------- | ------- | ------------\n";
    std::cout << "  forAll macro   | Fast    | Type-specific\n";
    std::cout << "  Iterator       | Medium  | Type-generic\n";
    std::cout << "  Range-based    | Fast    | Clean syntax\n";
    std::cout << "\n  ✅ Range-based for: Best of both worlds!\n";

    return 0;
}
```

## Part 4: Design Trade-offs

### Iteration Method Comparison

| Method | Pros | Cons | Best For |
|--------|------|------|----------|
| **Index-based** | Simple, random access | Only for indexable containers | Arrays, vectors |
| **Iterator** | Generic, STL-compatible | Verbose syntax | Generic algorithms |
| **Range-based for** | Clean, fast | Requires begin()/end() | Most iteration |
| **forAll macro** | OpenFOAM-idiom | Macro expansion | OpenFOAM code |

### When to Use Each

**✅ Index-based (`for` loop):**
- Need index value in loop
- Random access required
- Performance-critical inner loops

**✅ Iterator (`begin()/end()`):**
- Writing generic algorithms
- Working with STL algorithms
- Need iterator category (forward, bidirectional, etc.)

**✅ Range-based for (`for x : y`):**
- Simple sequential traversal
- Read-only or modify-all operations
- Modern C++ style

**✅ `forAll` macro:**
- OpenFOAM compatibility
- Consistent with codebase style
- When you need the index

### OpenFOAM's Strategy

OpenFOAM uses **all four** depending on context:

1. **`forAll`** - Most common (idiomatic OpenFOAM)
2. **Index-based** - When need cell/face index
3. **Iterator** - Generic algorithms (STL integration)
4. **Range-based** - Newer code, simpler loops
