# Day 07: RAII and Resource Management — `IOobject` Lifecycle

## Part 1: Pattern Identification

### ⭐ What is RAII?

**RAII** = **Resource Acquisition Is Initialization**

A C++ idiom where **resource allocation is done during object construction**, and **deallocation during destruction**.

```cpp
class FileHandle {
    FILE* f_;
public:
    FileHandle(const char* name) { f_ = fopen(name, "r"); }  // Acquire
    ~FileHandle() { if (f_) fclose(f_); }                    // Release
};
```

> **⭐ Verified Fact:** OpenFOAM's `IOobject` class uses RAII for file I/O and object registration (see `src/OpenFOAM/db/IOobject/IOobject.H`).

### ⭐ Why RAII Matters

**Without RAII:**
```cpp
// ❌ Manual cleanup
volScalarField* p = new volScalarField(mesh, "p");
p->write();
delete p;  // Easy to forget!

// ❌ Exception unsafe
volScalarField* p = new volScalarField(mesh, "p");
someFunctionThatThrows();
delete p;  // Never reached! Memory leak!
```

**With RAII:**
```cpp
// ✅ Automatic cleanup
volScalarField p(mesh, "p");  // Constructor registers object
p.write();
// Destructor automatically deregisters/deletes
// Even if exception thrown!
```

```cpp
// File: src/OpenFOAM/db/IOobject/IOobject.H

class IOobject
{
private:
    // -[1] Object metadata
    word name_;           // Object name (e.g., "p", "U")
    word instance_;       // Instance number
    const objectRegistry& db_;  // Reference to database

    // -[2] I/O parameters
    fileName caseName_;   // Case directory path
    readOption readOpt_;  // MUST_READ, READ_IF_PRESENT, NO_READ
    writeOption writeOpt_; // AUTO_WRITE, NO_WRITE

public:
    // -[3] Constructor (RAII: check-in) ⭐
    IOobject
    (
        const word& name,
        const fileName& instance,
        const fileName& local,
        const objectRegistry& db,
        readOption r = NO_READ,
        writeOption w = NO_WRITE
    )
    :
        name_(name),
        instance_(instance),
        db_(db),
        readOpt_(r),
        writeOpt_(w)
    {
        // Checked into database
    }

    // -[4] Destructor (RAII: check-out) ⭐
    virtual ~IOobject()
    {
        // Automatically checked out of database
    }

    // -[5] Accessors
    const word& name() const { return name_; }
    const objectRegistry& db() const { return db_; }

    // -[6] File I/O
    fileName path() const;  // Full path to object file
    bool headerOk();        // Check file header validity
};
```

### ⭐ regIOobject (Read/Write IOobject)

```cpp
// File: src/OpenFOAM/db/regIOobject/regIOobject.H

class regIOobject
:
    public IOobject
{
private:
    bool upToDate_;  // Dirty bit for write optimization

public:
    // -[1] Constructor
    regIOobject(const IOobject& io)
    :
        IOobject(io),
        upToDate_(true)
    {}

    // -[2] Read from disk ⭐
    bool read()
    {
        if (readOpt_ == IOobject::MUST_READ)
        {
            IFstream is(path());
            return readData(is);  // Virtual call
        }
        return true;
    }

    // -[3] Write to disk ⭐
    bool write() const
    {
        if (writeOpt_ == IOobject::AUTO_WRITE && upToDate_)
        {
            OFstream os(path());
            return writeData(os);  // Virtual call
        }
        return true;
    }

    // -[4] Mark as modified
    void setUpToDate(bool flag) { upToDate_ = flag; }
};
```

### ⭐ Object Lifecycle

```cpp
// In OpenFOAM solver (e.g., simpleFoam)

// 1. Constructor (RAII: check-in)
volScalarField p
(
    IOobject
    (
        "p",              // Name
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    mesh
);
// At this point:
// - "p" is checked into mesh's object registry
// - File "0/p" is read (if exists)
// - Database now tracks this object

// 2. Use object
p = 101325;  // Set pressure

// 3. Write (manual or automatic)
p.write();  // Writes to "0/p"

// 4. Destructor (automatic at scope end)
// - "p" is checked out of database
// - Resources released
```

## Part 3: Implementation Exercise

### Mini-Implementation: RAII Object Registry

Create file `raii_registry.H`:

```cpp
#ifndef raii_registry_H
#define raii_registry_H

#include <string>
#include <unordered_map>
#include <iostream>
#include <stdexcept>

// -[1] Global object registry
class ObjectRegistry
{
private:
    std::unordered_map<std::string, void*> objects_;
    static ObjectRegistry* instance_;

    ObjectRegistry() = default;

public:
    static ObjectRegistry& instance()
    {
        if (!instance_)
        {
            instance_ = new ObjectRegistry();
        }
        return *instance_;
    }

    void registerObject(const std::string& name, void* obj)
    {
        if (objects_.find(name) != objects_.end())
        {
            throw std::runtime_error("Object '" + name + "' already registered!");
        }
        objects_[name] = obj;
        std::cout << "  [Registry] Checked in: " << name << "\n";
    }

    void unregisterObject(const std::string& name)
    {
        objects_.erase(name);
        std::cout << "  [Registry] Checked out: " << name << "\n";
    }

    void* lookup(const std::string& name) const
    {
        auto it = objects_.find(name);
        if (it == objects_.end())
        {
            throw std::runtime_error("Object '" + name + "' not found!");
        }
        return it->second;
    }

    bool exists(const std::string& name) const
    {
        return objects_.find(name) != objects_.end();
    }
};

ObjectRegistry* ObjectRegistry::instance_ = nullptr;

// -[2] RAII-managed object base class
class ManagedObject
{
protected:
    std::string name_;

    ManagedObject(const std::string& name) : name_(name)
    {
        ObjectRegistry::instance().registerObject(name_, this);
    }

public:
    virtual ~ManagedObject()
    {
        ObjectRegistry::instance().unregisterObject(name_);
    }

    const std::string& name() const { return name_; }
};

// -[3] Concrete managed field
template<typename T>
class Field : public ManagedObject
{
    std::vector<T> data_;

public:
    Field(const std::string& name, std::size_t size)
    :
        ManagedObject(name),
        data_(size)
    {
        std::cout << "    [Field] Constructed: " << name << " with " << size << " elements\n";
    }

    ~Field()
    {
        std::cout << "    [Field] Destroyed: " << name_ << "\n";
    }

    std::size_t size() const { return data_.size(); }

    T& operator[](std::size_t i) { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }
};

#endif
```

### Test Program

Create file `test_raii.C`:

```cpp
#include "raii_registry.H"
#include <iostream>

void simulateException()
{
    std::cout << "  Simulating exception...\n";
    throw std::runtime_error("Something went wrong!");
}

int main()
{
    std::cout << "=== Testing RAII Resource Management ===\n\n";

    // -[1] Normal scope
    std::cout << "Test 1: Normal lifecycle\n";
    {
        Field<double> p("pressure", 100);
        p[0] = 101325.0;
        p[99] = 101450.0;

        std::cout << "  Object exists in registry: "
                  << (ObjectRegistry::instance().exists("pressure") ? "yes" : "no") << "\n";
    }
    std::cout << "  After scope ends:\n";
    std::cout << "  Object exists in registry: "
              << (ObjectRegistry::instance().exists("pressure") ? "yes" : "no") << "\n\n";

    // -[2] Exception safety
    std::cout << "Test 2: Exception safety\n";
    try
    {
        Field<double> T("temperature", 50);
        T[0] = 300.0;

        std::cout << "  About to throw exception...\n";
        simulateException();
    }
    catch (const std::exception& e)
    {
        std::cout << "  Exception caught: " << e.what() << "\n";
    }
    std::cout << "  Object exists after exception: "
              << (ObjectRegistry::instance().exists("temperature") ? "yes" : "no") << "\n\n";

    // -[3] RAII benefits
    std::cout << "Test 3: RAII Benefits\n";
    std::cout << "  ✅ Automatic cleanup (no manual delete)\n";
    std::cout << "  ✅ Exception-safe (cleanup even on throw)\n";
    std::cout << "  ✅ Prevents resource leaks\n";
    std::cout << "  ✅ Deterministic destruction (C++ guarantee)\n";

    return 0;
}
```

## Part 4: Design Trade-offs

### RAII vs. Manual Management

| Aspect | RAII | Manual |
|--------|------|--------|
| **Cleanup** | Automatic (destructor) | Manual (delete/close) |
| **Exception safety** | ✅ Guaranteed | ❌ Leaks on throw |
| **Code clarity** | ✅ Clear ownership | ❌ Scattered cleanup |
| **Performance** | Zero overhead | Same (if done correctly) |

### When RAII Doesn't Work

**❌ RAII not suitable for:**
- Resources without clear ownership (global singletons)
- Asymmetric release (acquire once, release many times)
- Performance-critical sections (needs profiling)

**✅ RAII perfect for:**
- File handles
- Memory allocation
- Mutex locks
- Database connections
- OpenGL contexts

### OpenFOAM's Use of RAII

**All major OpenFOAM classes use RAII:**
- `IOobject` - File I/O and object registration
- `fvMesh` - Mesh loading and geometry
- `Time` - Time directory management
- `GeometricField` - Field I/O and database tracking

## Summary

**⭐ Key Takeaways:**

1. **RAII** = Resource Acquisition Is Initialization
2. **Constructor acquires**, **destructor releases**
3. **`IOobject`** uses RAII for object database check-in/check-out
4. **Exception-safe** - cleanup guaranteed even on throw
5. **`regIOobject`** adds read/write I/O on top of RAII
