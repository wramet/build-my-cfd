# Day 10: Operator Overloading — `Field<T>` Operator Design

## Part 1: Pattern Identification

### ⭐ What is Operator Overloading?

**Operator overloading** allows **custom types to work with C++ operators** (`+`, `-`, `*`, `/`, `[]`, etc.).

```cpp
Field<scalar> p(100);
Field<scalar> T(100);

// ✅ Looks like scalar addition
Field<scalar> sum = p + T;

// ❌ But actually calls:
Field<scalar> operator+(const Field<scalar>& a, const Field<scalar>& b);
```

> **⭐ Verified Fact:** OpenFOAM overloads operators in `FieldFunctionsM.H` (see `src/OpenFOAM/fields/Fields/Field/FieldFunctionsM.H`).

### ⭐ Why Operator Overloading Matters

**Without overloading:**
```cpp
// ❌ Verbose, unintuitive
Field<scalar> result = p.add(T);  // Not mathematical looking
p = p.add(T.multiply(2.0));
```

**With overloading:**
```cpp
// ✅ Clean, mathematical
Field<scalar> result = p + T;
p = p + T * 2.0;
```

### ⭐ OpenFOAM's Operator Categories

| Category | Operators | Example |
|----------|-----------|---------|
| **Compound assignment** | `+=`, `-=`, `*=`, `/=` | `p += T;` |
| **Arithmetic** | `+`, `-`, `*`, `/` | `p + T` |
| **Comparison** | `==`, `!=`, `<`, `>` | `p == T` |
| **Access** | `[]`, `()` | `p[i]` |
| **Stream** | `<<`, `>>` | `os << p;` |

```cpp
template<class Type>
void Field<Type>::operator+=(const Field<Type>& f)
{
    // -[1] Size check
    if (this->size() != f.size())
    {
        FatalErrorInFunction
            << "Field sizes do not match: " << this->size() << " != " << f.size()
            << abort(FatalError);
    }

    // -[2] Element-wise addition
    for (label i = 0; i < this->size(); i++)
    {
        this->v_[i] += f.v_[i];  // ⭐ Inlined operation
    }
}

template<class Type>
void Field<Type>::operator*=(const scalar& s)
{
    for (label i = 0; i < this->size(); i++)
    {
        this->v_[i] *= s;  // ⭐ Scalar multiplication
    }
}
```

### ⭐ Arithmetic Operators

```cpp
// File: src/OpenFOAM/fields/Fields/Field/FieldFunctions.C

template<class Type>
tmp<Field<Type>> operator+(const Field<Type>& a, const Field<Type>& b)
{
    // -[1] Create tmp (temporary field)
    tmp<Field<Type>> tRes(new Field<Type>(a.size()));
    Field<Type>& res = tRes.ref();

    // -[2] Element-wise operation
    for (label i = 0; i < a.size(); i++)
    {
        res[i] = a[i] + b[i];
    }

    // -[3] Return by value (moved via tmp)
    return tRes;
}
```

### ⭐ Comparison Operators

```cpp
// File: src/OpenFOAM/fields/Fields/Field/FieldFunctions.C

template<class Type>
bool operator==(const Field<Type>& a, const Field<Type>& b)
{
    if (a.size() != b.size())
    {
        return false;
    }

    for (label i = 0; i < a.size(); i++)
    {
        if (a[i] != b[i])
        {
            return false;
        }
    }

    return true;
}
```

## Part 3: Implementation Exercise

### Mini-Implementation: Operator Overloaded Field

Create file `operator_overload.H`:

```cpp
#ifndef operator_overload_H
#define operator_overload_H

#include <vector>
#include <iostream>
#include <stdexcept>

template<typename T>
class Field
{
private:
    std::vector<T> data_;
    std::string name_;

public:
    Field(const std::string& name = "field") : name_(name) {}
    Field(std::size_t n, const T& val, const std::string& name = "field")
    : data_(n, val), name_(name) {}

    // -[1] Access operators
    T& operator[](std::size_t i) { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }

    std::size_t size() const { return data_.size(); }
    const std::string& name() const { return name_; }

    // -[2] Compound assignment: +=
    Field& operator+=(const Field& other)
    {
        if (size() != other.size())
        {
            throw std::runtime_error("Field size mismatch in += (" +
                name_ + " vs " + other.name_ + ")");
        }
        for (std::size_t i = 0; i < size(); ++i)
        {
            data_[i] += other.data_[i];
        }
        return *this;
    }

    // -[3] Compound assignment: *=
    Field& operator*=(const T& scalar)
    {
        for (std::size_t i = 0; i < size(); ++i)
        {
            data_[i] *= scalar;
        }
        return *this;
    }

    // -[4] Friend arithmetic operators
    friend Field operator+(const Field& a, const Field& b)
    {
        if (a.size() != b.size())
        {
            throw std::runtime_error("Field size mismatch in +");
        }
        Field result(a.name_ + "+" + b.name_);
        result.data_.reserve(a.size());
        for (std::size_t i = 0; i < a.size(); ++i)
        {
            result.data_.push_back(a.data_[i] + b.data_[i]);
        }
        return result;
    }

    friend Field operator*(const Field& f, const T& scalar)
    {
        Field result(f.name_ + "_scaled");
        result.data_.reserve(f.size());
        for (const auto& val : f.data_)
        {
            result.data_.push_back(val * scalar);
        }
        return result;
    }

    // -[5] Comparison operators
    friend bool operator==(const Field& a, const Field& b)
    {
        return a.data_ == b.data_;
    }

    friend bool operator!=(const Field& a, const Field& b)
    {
        return !(a == b);
    }

    // -[6] Stream operators
    friend std::ostream& operator<<(std::ostream& os, const Field& f)
    {
        os << "Field '" << f.name_ << "' [" << f.size() << "]: ";
        for (std::size_t i = 0; i < std::min(std::size_t(5), f.size()); ++i)
        {
            os << f.data_[i] << " ";
        }
        if (f.size() > 5) os << "...";
        return os;
    }
};

#endif
```

## Part 4: Design Trade-offs

### Operator Overloading Guidelines

**✅ DO:**
- Overload operators **consistently** with their mathematical meaning
- Implement `+=` when implementing `+` (compound assignment pattern)
- Return **appropriate types** (by value for arithmetic, by reference for access)
- Check **preconditions** (size matching, type compatibility)

**❌ DON'T:**
- Overload operators with **unintuitive meanings** (`operator+` that does subtraction)
- Overload `&&`, `||`, `,` (can break short-circuit evaluation)
- Return **pointers/references** to temporaries

### Performance Considerations

**Operator+ creates a new object:**
```cpp
result = a + b + c;  // Creates 2 temporaries
```

**Optimization: Use compound assignment**
```cpp
result = a;
result += b;
result += c;  // No temporaries!
```

**OpenFOAM uses `tmp<>`** to mitigate this cost:
```cpp
tmp<Field> operator+(const Field& a, const Field& b);
// Returns tmp (moved), not copied
```

## Summary

**⭐ Key Takeaways:**

1. **Operator overloading** = custom types work with C++ operators
2. **Consistent semantics** = `+` means addition, `*` means multiplication
3. **Compound assignment** = more efficient than binary operators
4. **OpenFOAM overloads** all field operators
5. **Type-safe** mathematical expressions
