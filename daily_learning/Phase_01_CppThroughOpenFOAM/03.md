# Day 03: Class Templates Deep Dive — `GeometricField<Type, PatchField, GeoMesh>`

## Part 1: Pattern Identification

### ⭐ What Makes a Complex Template Class?

On Day 01, we learned about single-parameter templates (`Field<Type>`). Day 02 introduced multiple template parameters. Today, we dive **deep into `GeometricField`** - OpenFOAM's most complex template class with **three template parameters** and sophisticated responsibilities.

> **⭐ Verified Fact:** `GeometricField` is declared in `src/OpenFOAM/fields/GeometricFields/GeometricField/GeometricField.H` and is the cornerstone of OpenFOAM's field system.

**Why `GeometricField` is Complex:**

1. **Multiple template parameters** - Each with different roles
2. **Boundary condition management** - Dynamic per-patch policies
3. **Time level storage** - Old time fields for temporal schemes
4. **Geometric context** - Knows its mesh, dimensions, and position

### ⭐ The Three Template Parameters

Let's revisit with deeper understanding:

```cpp
template<class Type, template<class> class PatchField, class GeoMesh>
class GeometricField;
```

| Parameter | Role | Example | What It Controls |
|-----------|------|---------|------------------|
| `Type` | **Data** | `scalar`, `vector` | What's stored in each cell |
| `PatchField` | **Boundary Policy** | `fvPatchField` | How boundaries behave |
| `GeoMesh` | **Geometry** | `volMesh`, `pointMesh` | Where data lives (cell, face, point) |

> **⭐ Key Insight:** These three parameters are **orthogonal** - you can mix and match them independently.

```

    ↓

    ↓

    ↓
GeometricField<Type, PatchField, GeoMesh> (adds boundaries + time levels)
```

## Part 2: Source Code Reading

### ⭐ GeometricField Class Declaration (Full)

```cpp
// File: src/OpenFOAM/fields/GeometricFields/GeometricField/GeometricField.H

template<class Type, template<class> class PatchField, class GeoMesh>
class GeometricField
:
    public DimensionedField<Type, GeoMesh>
{
    // -[1] Private data members
    Field<PatchField<Type>> boundaryField_;  // Boundary conditions ⭐
    GeoField* field0Ptr_;                     // Old time level
    GeoField* prevIterPtr_;                  // Previous iteration

    mutable GeoField* fieldPrevIterPtr_;     // Cache pointer

    // -[2] Constructors
    GeometricField(const IOobject&, const GeoMesh&);
    GeometricField(const GeometricField&);
    GeometricField(const GeometricField&, const IOobject&);

    // -[3] Boundary field access
    const Field<PatchField<Type>>& boundaryField() const;
    Field<PatchField<Type>>& boundaryField();

    // -[4] Time level management
    const GeometricField& oldTime() const;
    GeometricField& oldTime();

    const GeometricField& prevIter() const;
    GeometricField& prevIter();

    // -[5] Member operators
    void operator=(const GeometricField&);
    void operator=(const Type&);

    void operator+=(const GeometricField&);
    void operator-=(const GeometricField&);

    // -[6] Geometric fields (from parent)
    const GeoMesh& mesh() const;  // Inherited from DimensionedField
};
```

### ⭐ Key Design Patterns

**1. Boundary Field Storage:**

```cpp
Field<PatchField<Type>> boundaryField_;
```

**Breaking this down:**
- `PatchField<Type>` is the boundary condition type (e.g., `fvPatchField<scalar>`)
- `Field<PatchField<Type>>` is a dynamic array of patches
- Each patch has its **own boundary condition instance**

**Example for a 4-patch mesh:**
```cpp
volScalarField p(mesh);

// boundaryField_ contains:
// boundaryField_[0] → fvPatchField<scalar> (inlet)
// boundaryField_[1] → fvPatchField<scalar> (outlet)
// boundaryField_[2] → fvPatchField<scalar> (walls)
// boundaryField_[3] → fvPatchField<scalar> (symmetry)
```

**2. Time Level Storage (for transient solvers):**

```cpp
GeoField* field0Ptr_;  // Old time value (t - Δt)
```

**How it works:**

```cpp
// In transient solver (e.g., pimpleFoam)
while (runTime.loop())
{
    // Store current as old time
    p.storePrevIter();  // Copies p → p.prevIter()
    p.oldTime();        // Copies p → p.oldTime()

    // Solve (modifies p)
    solve(pEqn);

    // Write both current and old time
    p.write();
    p.oldTime().write();
}
```

**Time levels in a typical transient simulation:**
```
t = 0.0    → p.oldTime() = initial, p = initial
t = 0.001  → p.oldTime() = t=0.0, p = t=0.001
t = 0.002  → p.oldTime() = t=0.001, p = t=0.002
```

### ⭐ Template Instantiation in Practice

When OpenFOAM compiles a solver using multiple field types:

```cpp
// In simpleFoam.C (steady-state)
volScalarField p(mesh);
volVectorField U(mesh);

// Compiler generates TWO versions of GeometricField:
GeometricField<scalar, fvPatchField, volMesh>  // For p
GeometricField<vector, fvPatchField, volMesh>  // For U
```

**Each instantiation has:**
- Separate `boundaryField_` (different BC types)
- Separate `field0Ptr_` (if transient)
- Separate method implementations (all compiled independently)

### ⭐ Boundary Field Access Pattern

```cpp
// OpenFOAM code pattern for accessing boundary values
volScalarField p(mesh);

// Access internal field (cell values)
forAll(p, cellI)
{
    p[cellI] = ...;  // Cell-centered value
}

// Access boundary field
const volScalarField::Boundary& bf = p.boundaryField();
forAll(bf, patchI)
{
    const fvPatchScalarField& patch = bf[patchI];

    // Access patch values
    forAll(patch, faceI)
    {
        patch[faceI] = ...;  // Face value on this patch
    }

    // Check boundary condition type
    if (isA<fixedValueFvPatchScalarField>(patch))
    {
        // Cast to specific BC type
        const fixedValueFvPatchScalarField& fixedPatch =
            dynamicCast<const fixedValueFvPatchScalarField&>(patch);

        scalarField values = fixedPatch.value();
        // Do something with fixed values
    }
}
```

> **⭐ Critical:** The `boundaryField()` returns a reference to `Field<PatchField<Type>>`, which is an array of boundary conditions.

## Part 3: Implementation Exercise

### Mini-Implementation: Advanced `GeometricField` with Time Levels

Create file `advanced_geometric_field.H`:

```cpp
#ifndef advanced_geometric_field_H
#define advanced_geometric_field_H

#include "geometric_field.H"  // From Day 02
#include <memory>

template<typename Type, template<typename> class BC, typename Mesh>
class AdvancedGeometricField : public GeometricField<Type, BC, Mesh>
{
private:
    std::unique_ptr<AdvancedGeometricField<Type, BC, Mesh>> field0Ptr_;
    std::unique_ptr<AdvancedGeometricField<Type, BC, Mesh>> prevIterPtr_;
    label timeIndex_;

public:
    // Constructor
    AdvancedGeometricField(const std::string& name, std::size_t nCells, std::size_t nPatches)
        : GeometricField<Type, BC, Mesh>(name, nCells, nPatches)
        , timeIndex_(0)
    {}

    // Time level management
    void storePrevIter()
    {
        if (!prevIterPtr_)
        {
            prevIterPtr_ = std::make_unique<AdvancedGeometricField>(
                this->name() + "_prevIter", this->size(), 4
            );
        }
        *prevIterPtr_ = *this;
    }

    void storeOldTime()
    {
        if (!field0Ptr_)
        {
            field0Ptr_ = std::make_unique<AdvancedGeometricField>(
                this->name() + "_oldTime", this->size(), 4
            );
        }
        *field0Ptr_ = *this;
        timeIndex_++;
    }

    const AdvancedGeometricField& oldTime() const
    {
        if (!field0Ptr_)
        {
            std::cerr << "Error: oldTime not stored!\n";
            return *this;
        }
        return *field0Ptr_;
    }

    const AdvancedGeometricField& prevIter() const
    {
        if (!prevIterPtr_)
        {
            std::cerr << "Error: prevIter not stored!\n";
            return *this;
        }
        return *prevIterPtr_;
    }

    label timeIndex() const { return timeIndex_; }

    // Relaxation (for under-relaxation in iterative solvers)
    void relax(const Type alpha)
    {
        if (prevIterPtr_)
        {
            for (std::size_t i = 0; i < this->size(); ++i)
            {
                (*this)[i] = alpha * (*this)[i] + (1.0 - alpha) * (*prevIterPtr_)[i];
            }
        }
    }
};

typedef AdvancedGeometricField<double, ZeroGradientBC, VolMesh> AdvVolScalarField;

#endif
```

### Test Program

Create file `test_advanced_field.C`:

```cpp
#include "advanced_geometric_field.H"
#include <iostream>

int main()
{
    std::cout << "=== Testing Advanced GeometricField ===\n\n";

    AdvVolScalarField T("T", 100, 4);

    // Initialize temperature field
    for (std::size_t i = 0; i < 50; ++i) T[i] = 300.0;
    for (std::size_t i = 50; i < 100; ++i) T[i] = 350.0;

    std::cout << "Initial T[0] = " << T[0] << ", T[99] = " << T[99] << "\n";

    // Store previous iteration
    T.storePrevIter();
    std::cout << "After storePrevIter, prevIter[0] = " << T.prevIter()[0] << "\n";

    // Modify current field
    for (std::size_t i = 0; i < 100; ++i) T[i] += 10.0;
    std::cout << "After modification, T[0] = " << T[0] << "\n";
    std::cout << "prevIter still has: " << T.prevIter()[0] << "\n";

    // Test relaxation
    std::cout << "\nTesting under-relaxation:\n";
    std::cout << "Before relax: T[0] = " << T[0] << ", prevIter[0] = " << T.prevIter()[0] << "\n";
    T.relax(0.7);  // alpha = 0.7
    std::cout << "After relax(0.7): T[0] = " << T[0] << "\n";
    std::cout << "  (Should be 0.7*310 + 0.3*310 = " << 0.7*310.0 + 0.3*310.0 << ")\n";

    // Test old time
    std::cout << "\nTesting time levels:\n";
    std::cout << "Current time index: " << T.timeIndex() << "\n";
    T.storeOldTime();
    std::cout << "After storeOldTime, time index: " << T.timeIndex() << "\n";
    std::cout << "oldTime[0] = " << T.oldTime()[0] << "\n";

    // Modify and store again
    for (std::size_t i = 0; i < 100; ++i) T[i] += 5.0;
    T.storeOldTime();
    std::cout << "After second storeOldTime, time index: " << T.timeIndex() << "\n";
    std::cout << "oldTime[0] = " << T.oldTime()[0] << "\n";

    return 0;
}
```

## Part 4: Design Trade-offs & Testing

### Memory vs. Computation Trade-off

**Storing old time fields:**
- ✅ **Enables transient schemes** (backward, Crank-Nicolson)
- ❌ **Doubles memory usage** per field

**Alternative:**
```cpp
// Re-compute instead of storing (saves memory, costs CPU)
Type oldTimeValue = integrate(p, time - dt);  // Expensive
```

OpenFOAM chooses **storage** because:
- Memory is cheap relative to CPU time
- Transient simulations need frequent old time access

### Field Relaxation Pattern

**Under-relaxation equation:**
$$
p^{new} = \alpha p^{*} + (1 - \alpha) p^{old}
$$

Where:
- $p^{*}$ is the newly computed value
- $p^{old}$ is the previous iteration
- $\alpha \in (0, 1]$ is the relaxation factor

**In OpenFOAM solvers:**
```cpp
// In SIMPLE algorithm
p.relax(0.3);  // Strong under-relaxation for pressure
U.relax(0.7);  // Moderate relaxation for velocity
```

### Template Specialization Benefits

**Without `GeometricField` specialization:**
```cpp
// Would need separate classes:
class VolScalarField { /* ... */ };
class VolVectorField { /* ... */ };
class SurfaceScalarField { /* ... */ };
// 15+ classes for all combinations!
```

**With `GeometricField`:**
```cpp
// Single class handles all:
typedef GeometricField<scalar, fvPatchField, volMesh> volScalarField;
typedef GeometricField<vector, fvPatchField, volMesh> volVectorField;
typedef GeometricField<scalar, fvsPatchField, surfaceMesh> surfaceScalarField;
// ... infinite combinations from ONE class!
```

## Summary

**⭐ Key Takeaways:**

1. **`GeometricField`** has 3 template parameters for data, boundaries, and geometry
2. **Inheritance chain:** `List` → `Field` → `DimensionedField` → `GeometricField`
3. **Boundary field** stores per-patch boundary conditions
4. **Time levels** enable transient simulations
5. **Template combination** enables infinite field types from one class
