## 📚 **แหล่งข้อมูลเพิ่มเติมและการอ่านเพิ่มเติม**

### **การอ้างอิงซอร์สโค้ด OpenFOAM**

#### การนำไปใช้งานโมเดลความหนืดหลัก
สถาปัตยกรรมโมเดลความหนืดพื้นฐานถูกนำไปใช้งานใน `src/physicalProperties/viscosityModels/` ซึ่งมีคลาสพื้นฐานและการนำไปใช้งานจริงสำหรับโมเดลรีโอโลยีต่างๆ ไดเรกทอรีนี้สาธิต:

- **กลไกการเลือกเวลาทำงาน**: การนำไปใช้งาน `runTimeSelection` ใน `src/OpenFOAM/db/runTimeSelection/` ให้ระบบการลงทะเบียนอัตโนมัติที่เปิดใช้งานการเลือกโมเดลโพลิมอร์ฟิกของ OpenFOAM ผ่านรายการพจนานุกรม
- **การออกแบบตามเทมเพลต**: คลาสพื้นฐานใช้ CRTP (Curiously Recurring Template Pattern) เพื่อให้โพลิมอร์ฟิกตอนคอมไพล์ในขณะที่ยังคงความยืดหยุ่นเวลาทำงาน

#### ตัวอย่าง Solver ที่ใช้งานจริง
ไดเรกทอรี `applications/solvers/incompressible/icoFoam/` ให้ตัวอย่างมาตรฐานของวิธีการผสานรวมโมเดลความหนืดเหล่านี้กับ CFD solvers:

```cpp
// ตัวอย่างจาก icoFoam solver ที่แสดงการใช้งานโมเดล
volScalarField nu
(
    IOobject
    (
        "nu",
        runTime.timeName(),
        mesh,
        IOobject::MUST_READ,
        IOobject::AUTO_WRITE
    ),
    transportProperties.lookup("nu")
);
```

### **รูปแบบการออกแบบพื้นฐานใน OpenFOAM**

#### รูปแบบ Factory Method
ระบบการเลือกเวลาทำงานคือการนำไปใช้งานที่ซับซ้อนของ **รูปแบบ Factory Method** ตามที่อธิบายโดย Gamma และคณะใน *Design Patterns: Elements of Reusable Object-Oriented Software* รูปแบบนี้เปิดใช้งาน:

- **การสร้างโมเดลแบบไดนามิก**: สามารถเพิ่มโมเดลความหนืดใหม่โดยไม่ต้องแก้ไขโค้ด solver ที่มีอยู่
- **การสร้างอินสแตนซ์ตามพจนานุกรม**: โมเดลถูกเลือกผ่านรายการพจนานุกรมตามข้อความแทนการกำหนดค่าที่คอมไพล์

```cpp
// การใช้งานมาโครการเลือกเวลาทำงาน
addToRunTimeSelectionTable
(
    viscosityModel,
    powerLawViscosity,
    dictionary
);
```

#### รูปแบบ Template Method
เฟรมเวิร์กอัลกอริทึมเชิงตัวเลขของ OpenFOAM ใช้ **รูปแบบ Template Method** ซึ่งคลาสพื้นฐานกำหนดโครงร่างของอัลกอริทึมการคำนวณในขณะที่อนุญาตให้คลาสที่สืบทอดมาสามารถนำไปใช้งานพฤติกรรมรีโอโลยีเฉพาะ:

```cpp
// ตัวอย่างวิธีเทมเพลตในคลาสความหนืดพื้นฐาน
virtual tmp<volScalarField> nu(const volScalarField& gammaDot) const = 0;
```

#### รูปแบบ Strategy
โมเดลความหนืดที่สามารถสลับที่ได้เป็นการนำไปใช้งาน **รูปแบบ Strategy** ซึ่งโมเดลรีโอโลยีต่างๆ สามารถสลับที่ได้ในเวลาทำงานตามลักษณะทางกายภาพของของไหลที่กำลังจำลอง

### **ทฤษฎีพลศาสตร์ของไหลเชิงคำนวณ**

#### พื้นฐานการจำลองรีโอโลยี
สำหรับความเข้าใจอย่างครอบคลุมเกี่ยวกับพฤติกรรมของไหลที่ไม่ใช่นิวตัน:

- **Bird, Stewart, Lightfoot - *Transport Phenomena***: ให้การอนุพัทธ์พื้นฐานของโมเดล Power-law และอภิปรายฐานทางกายภาพสำหรับสมการรัฐต่างๆ
- **Chhabra, Richardson - *Non-Newtonian Flow and Applied Rheology***: นำเสนอความครอบคลุมของโมเดลรีโอโลยีที่ใช้งานจริงและการประยุกต์ใช้ใน CFD อุตสาหกรรม

#### พื้นฐานทางคณิตศาสตร์
โมเดล Power-law แทนความสัมพันธ์รัฐพื้นฐาน:

$$\tau = K \dot{\gamma}^n$$

โดยที่:
- $\tau$ คือความเครียดเฉือน
- $K$ คือดัชนีความสม่ำเสมอ
- $\dot{\gamma}$ คืออัตราการเฉือน
- $n$ คือดัชนีพฤติกรรมการไหล

### **แนวทางปฏิบัติที่ดีที่สุดในการเขียนโปรแกรม C++**

#### เทมเพลตเมตาโปรแกรมมิ่ง
สำหรับการใช้เทมเพลตอย่างมีประสิทธิภาพในการคำนวณเชิงตัวเลข:

- **Meyers, *Effective Modern C***: ให้คำแนะนำอย่างครอบคลุมเกี่ยวกับเทคนิคเทมเพลตเมตาโปรแกรมมิ่งที่จำเป็นสำหรับแอปพลิเคชัน CFD ประสิทธิภาพสูง
- กลยุทธ์การเชี่ยวชาญเทมเพลตสำหรับฟิลด์ประเภทต่างๆ (สเกลาร์, เวกเตอร์, เทนเซอร์)

#### หลักการออกแบบเชิงวัตถุ
สำหรับลำดับชั้นการสืบทอดที่แข็งแกร่งและการออกแบบคลาส:

- **Sutter, Alexandrescu, *C++ Coding Standards***: ให้คำแนะนำที่เชื่อถือได้เกี่ยวกับการออกแบบลำดับชั้นคลาสที่บำรุงรักษาได้และการหลีกเลี่ยงกับดักการสืบทอดทั่วไป
- รูปแบบ RAII (Resource Acquisition Is Initialization) สำหรับการจัดการหน่วยความจำในการคำนวณเชิงตัวเลข

#### แนวทางการเพิ่มประสิทธิภาพประสิทธิภาพ
- **หลักการค่าใช้จ่ายเป็นศูนย์**: การแยกความหมายเทมเพลตไม่ควรมีค่าใช้จ่ายประสิทธิภาพรันไทม์
- **โครงสร้างข้อมูลที่เป็นมิตรกับแคช**: พิจารณารูปแบบการเข้าถึงหน่วยความจำเมื่อออกแบบการดำเนินการฟิลด์
- **การคำนวณตอนคอมไพล์**: ใช้ `constexpr` และเทมเพลตเมตาโปรแกรมมิ่งสำหรับการเพิ่มประสิทธิภาพตอนคอมไพล์

### **เอกสารและทรัพยากร OpenFOAM**

#### เอกสารทางการ
- **คู่มือผู้ใช้ OpenFOAM**: เอกสารครอบคลุมสำหรับการใช้งาน solver และการตั้งค่ากรณีศึกษา
- **คู่มือโปรแกรมเมอร์ OpenFOAM**: คำอธิบายโดยละเอียดเกี่ยวกับสถาปัตยกรรมโค้ดเบสและกลไกการขยาย
- **เอกสาร API Doxygen**: เอกสารโค้ดที่สร้างขึ้นโดยอัตโนมัติสำหรับคลาสและเมธอดทั้งหมด

#### ทรัพยากรชุมชน
- **ฟอรัม OpenFOAM Foundation**: การสนับสนุนและการอภิปรายของชุมชน
- **ฟอรัม CFD Online**: ชุมชน CFD ที่กว้างขึ้นพร้อมส่วน OpenFOAM ที่อุทิศให้
- **ที่เก็บ GitHub**: การเข้าถึงซอร์สโค้ดและการติดตามปัญหาสำหรับรายงานข้อผิดพลาดและคำขอคุณสมบัติ

#### ทรัพยากรทางการศึกษา
- **Journal of Computational Physics**: สิ่งพิมพ์ชั้นนำสำหรับการพัฒนาอัลกอริทึม CFD
- **International Journal for Numerical Methods in Fluids**: มุ่งเน้นการประยุกต์ใช้พลศาสตร์ของไหลเชิงคำนวณ
- **AIAA Journal**: การประยุกต์ใช้ทางการบินและอวกาศและการศึกษาการตรวจสอบความถูกต้องโดยใช้ OpenFOAM

### **แหล่งข้อมูลเฉพาะทาง**

#### การจำลองการไหลหลายเฟส
- **Ishii, Hibiki - *Thermo-Fluid Dynamics of Two-Phase Flow***: ทฤษฎีพื้นฐานสำหรับ CFD หลายเฟส
- **Prosperetti, Tryggvason - *Computational Methods for Multiphase Flow***: วิธีการเชิงตัวเลขขั้นสูงสำหรับการไหลกระจาย

#### การจำลองความปั่นป่วน
- **Wilcox - *Turbulence Modeling for CFD***: ความครอบคลุมของโมเดลความปั่นป่วนและการนำไปใช้งาน
- **Pope - *Turbulent Flows***: พื้นฐานทฤษฎีสำหรับการจำลองการไหลแบบปั่นป่วน

#### วิธีการเชิงตัวเลข
- **Ferziger, Perić - *Computational Methods for Fluid Dynamics***: ข้อมูลอ้างอิงมาตรฐานสำหรับวิธีการปริมาตรจำกัด
- **Patankar - *Numerical Heat Transfer and Fluid Flow***: ข้อมูลอ้างอิงคลาสสิกสำหรับอัลกอริทึมการผสมความดัน-ความเร็ว

แหล่งข้อมูลเหล่านี้ให้รากฐานที่ครอบคลุมสำหรับการเข้าใจทั้งฐานทฤษฎีและการนำไปใช้งานจริงของการจำลอง CFD ใน OpenFOAM โดยเฉพาะสำหรับการประยุกต์ใช้การไหลที่ไม่ใช่นิวตันที่ต้องการโมเดลความหนืดแบบกำหนดเอง
