# การจัดการผนัง (Wall Treatment) และ Wall Functions

ความถูกต้องของการจำลองการไหลที่ติดผนัง (Wall-bounded flows) ขึ้นอยู่กับการแก้ปัญหาในชั้น Boundary Layer ซึ่งมีความชันของความเร็วสูงมาก

---

## 📐 1. โครงสร้างชั้นขอบเขต (Boundary Layer Structure) 

พฤติกรรมการไหลใกล้ผนังถูกแบ่งตามค่าระยะห่างไร้มิติ **$y^+$**:
$$y^+ = \frac{y u_	au}{\nu}$$

| ช่วง $y^+$ | บริเวณ | พฤติกรรม |
|------------|----------|-----------|
| **$y^+ < 5$** | **Viscous Sublayer** | แรงหนืดครอบงำ, โปรไฟล์ความเร็วเป็นเส้นตรง ($u^+ = y^+$) |
| **$5 < y^+ < 30$** | **Buffer Layer** | บริเวณเปลี่ยนผ่านที่ซับซ้อน |
| **$30 < y^+ < 300$** | **Log-Law Region** | ความปั่นป่วนครอบงำ, โปรไฟล์เป็นลอการิทึม |

---

## 🛠️ 2. แนวทางการจัดการผนังใน OpenFOAM

OpenFOAM มีสองแนวทางหลักในการจัดการ Mesh ใกล้ผนัง:

### 2.1 Wall-Resolved (Low-Re Approach)
- **เป้าหมาย**: แก้สมการถึงผนังโดยตรง
- **ความละเอียด**: ต้องมี $y^+ ≈ 1$ (มีเซลล์อย่างน้อย 10-15 ชั้นในชั้นขอบเขต)
- **ข้อดี**: แม่นยำสูงสุดสำหรับการแยกตัวของไหลและความร้อน
- **ข้อเสีย**: ต้นทุนการคำนวณสูงมาก

### 2.2 Wall Functions (High-Re Approach)
- **เป้าหมาย**: ใช้สูตรทางคณิตศาสตร์ประมาณพฤติกรรมในชั้นขอบเขตแทนการสร้าง Mesh ละเอียด
- **ความละเอียด**: รักษาค่า $y^+$ ให้อยู่ในช่วง **30-300**
- **ข้อดี**: ประหยัดทรัพยากร เหมาะสำหรับงานอุตสาหกรรมขนาดใหญ่

## 💻 3. การนำไปใช้งาน (OpenFOAM Boundary Conditions)

ตัวอย่างการตั้งค่าในไฟล์ `0/nut`, `0/k`, และ `0/epsilon`:

**สำหรับความหนืดไหลวน (`0/nut`):**
```cpp
wall
{
    type            nutkWallFunction; // อิงตามค่า k ที่ผนัง
    value           uniform 0;
}
```

**สำหรับอัตราการสลายตัว (`0/epsilon`):**
```cpp
wall
{
    type            epsilonWallFunction;
    value           uniform 0.01;
}
```

## 🔍 4. การตรวจสอบคุณภาพ (Verification)

หลังจากรันการจำลอง ควรตรวจสอบค่า $y^+$ เพื่อยืนยันว่า Mesh สอดคล้องกับแนวทางที่เลือก:
```bash
# รันคำสั่งใน terminal
postProcess -func yPlus
```
ค่าที่ได้จะถูกเขียนลงในโฟลเดอร์เวลา (เช่น `100/yPlus`) เพื่อให้นำไปแสดงผลใน ParaView
